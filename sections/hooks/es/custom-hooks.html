Custom Hooks: Tus Superpoderes en React

🎯 ¿Qué es un Custom Hook?

Un custom hook es una función de JavaScript que usa los hooks integrados de React para resolver un problema específico.

Es una forma de extraer y reutilizar lógica con estado en varios componentes.

🚨 El Problema: Código Repetido

Déjame mostrarte el problema que resuelven los custom hooks:

Sin custom hooks:

// Componente 1
function LoginPage() {
  const user = React.useContext(AuthContext)

  if (!user) return <p>No has iniciado sesión</p>
  return <p>Bienvenido, {user.name}!</p>
}

// Componente 2
function Dashboard() {
  const user = React.useContext(AuthContext)

  if (!user) return <p>No has iniciado sesión</p>
  return <p>Hola, {user.name}!</p>
}

// Componente 3
function Settings() {
  const user = React.useContext(AuthContext)

  if (!user) return <p>No has iniciado sesión</p>
  return <p>Configuración para {user.name}</p>
}

¿Cuál es el problema?

  • ❌ Repetimos `React.useContext(AuthContext)` en todas partes
  • ❌ Repetimos la misma validación 3 veces
  • ❌ Si la lógica cambia, hay que actualizar 3 lugares
  • ❌ Más difícil de testear
  • ❌ Más propenso a errores

💡 La Solución: Custom Hooks

Creamos un custom hook que encapsula esta lógica:

function useAuth() {
  const user = React.useContext(AuthContext)
  return user
}

¡Eso es! Ahora lo usamos en todas partes:

// Componente 1
function LoginPage() {
  const user = useAuth()

  if (!user) return <p>No has iniciado sesión</p>
  return <p>Bienvenido, {user.name}!</p>
}

// Componente 2
function Dashboard() {
  const user = useAuth()

  if (!user) return <p>No has iniciado sesión</p>
  return <p>Hola, {user.name}!</p>
}

// Componente 3
function Settings() {
  const user = useAuth()

  if (!user) return <p>No has iniciado sesión</p>
  return <p>Configuración para {user.name}</p>
}

Beneficios:

  • ✅ Una sola línea: `const user = useAuth()`
  • ✅ Componentes más limpios
  • ✅ Lógica en UN lugar (fácil de cambiar)
  • ✅ Reutilizable en todos los componentes
  • ✅ Fácil de testear

🔥 Convención de Nombres en Custom Hooks

Los custom hooks deben empezar con "use":

✅ useAuth()
✅ useTheme()
✅ useCart()
✅ useFetch()
✅ useForm()
✅ useCounter()

❌ getAuth() (no empieza con use)
❌ authHook() (no empieza con use)

Esta convención de nombres le dice a React que esta función usa hooks.

📋 El Patrón Profesional

Regla importante:

Cada vez que creás un Context, siempre creá un custom hook para acceder a él.

Este patrón previene errores y hace tu código más limpio:

// Crear Context
const AuthContext = React.createContext(null)

// Crear Provider
export function AuthProvider({ children }) {
  const [user, setUser] = React.useState(null)

  return (
    <AuthContext.Provider value={{ user, setUser }}>
      {children}
    </AuthContext.Provider>
  )
}

// Crear Custom Hook
export function useAuth() {
  const context = React.useContext(AuthContext)

  if (!context) {
    throw new Error('useAuth debe usarse dentro de AuthProvider')
  }

  return context
}

🎁 Custom Hooks Comunes en la Naturaleza

1. useAuth() - Obtener autenticación del usuario

const user = useAuth()

if (!user) return <p>No has iniciado sesión</p>
return <p>Bienvenido, {user.name}!</p>

2. useTheme() - Obtener el tema actual

const { theme, toggleTheme } = useTheme()

return (
  <div style={{ background: theme === 'dark' ? '#000' : '#fff' }}>
    <button onClick={toggleTheme}>Cambiar Tema</button>
  </div>
)

3. useCart() - Acceder al carrito de compras

const { items, total, addItem, removeItem } = useCart()

return (
  <div>
    <p>Total: ${total}</p>
    {items.map(item => (
      <p key={item.id}>{item.name} - ${item.price}</p>
    ))}
  </div>
)

4. useFetch() - Obtener datos de una API

const { data, loading, error } = useFetch('/api/posts')

if (loading) return <p>Cargando...</p>
if (error) return <p>Error: {error}</p>
return <div>{data.map(post => ...)}</div>

const { values, handleChange, handleSubmit } = useForm({
  name: '',
  email: ''

return (

    <input
      name="name"
      value={values.name}
      onChange={handleChange}

)

🎯 Cuándo Crear un Custom Hook

Creá un custom hook cuando:

  • ✅ Repetís la misma lógica de hooks en varios componentes
  • ✅ Querés extraer el acceso a Context en una función
  • ✅ Combinás múltiples hooks y querés reutilizar esa combinación
  • ✅ Querés esconder lógica compleja detrás de una función simple
  • ✅ Querés compartir lógica con estado sin pasar props

NO creés un custom hook cuando:

  • ❌ Solo lo usás en un componente (mantén la simplicidad)
  • ❌ Es una sola línea simple (como `const user = useContext(AuthContext)`)
  • ❌ La lógica es específica de un componente

🚀 Beneficios de los Custom Hooks

1. Abstracción

Escondé lógica compleja detrás de un nombre simple:

// Lógica compleja escondida
const { user, logout } = useAuth()

// En lugar de
const context = useContext(AuthContext)
const user = context?.user
const logout = () => {
  // ... lógica compleja de logout ...
}

2. Mensajes de Error Claros

Tu hook puede lanzar errores significativos:

export function useAuth() {
  const context = React.useContext(AuthContext)

  if (!context) {
    throw new Error('useAuth debe usarse dentro de <AuthProvider>')
  }

  return context
}

Esto previene errores confusos como "context is undefined".

3. Fácil de Cambiar la Implementación

Si la lógica de autenticación cambia, actualizá en UN lugar:

// Antes: usando Context
export function useAuth() {
  return React.useContext(AuthContext)
}

// Después: cambiar a Redux sin modificar código de componentes
export function useAuth() {
  return useSelector(state => state.auth)
}

4. Más Fácil de Testear

Testeá el hook aislado de los componentes:

// Podés mockear y testear useAuth() por separado
const mockUser = { id: 1, name: 'Juan' }

// En tu test
const { result } = renderHook(() => useAuth())

🔗 Conexión con Context

Los custom hooks y Context van juntos:

Context (createContext, Provider)
  ↓
  Proporciona datos globalmente
  ↓
Custom Hook (useAuth)
  ↓
  Acceder a datos desde Context
  ↓
Componentes
  ↓
  Usar el custom hook

Context es el almacenamiento.

Los custom hooks son la capa de acceso.

⚙️ Reglas de los Custom Hooks

Regla 1: El Nombre Debe Empezar con "use"

✅ useAuth()
❌ getAuth()

Regla 2: Solo Podés Llamar a Hooks en el Nivel Superior

// ✅ Correcto
function useAuth() {
  const context = React.useContext(AuthContext)
  return context
}

// ❌ Incorrecto - hook dentro de if
function useAuth() {
  if (something) {
    const context = React.useContext(AuthContext)  // ¡MAL!
  }
  return context
}

Regla 3: Los Custom Hooks Pueden Llamar a Otros Hooks

// ¡Esto está bien!
function useUser() {
  const auth = useAuth()           // Llamar custom hook
  const theme = useTheme()          // Llamar otro custom hook
  const [loading, setLoading] = React.useState(false)  // Llamar hook integrado

  return { auth, theme, loading }
}

📝 Resumen

  • Los custom hooks extraen y reutilizan lógica con estado
  • Son funciones de JavaScript que usan los hooks de React
  • Deben empezar con "use"
  • Siempre creá un custom hook para cada Context
  • Hacen el código más limpio, reutilizable y fácil de testear
  • Esconden lógica compleja detrás de funciones simples

Ejemplos Completos de Código

Ejemplo 1: Hook useAuth Simple


// Paso 1: Crear Context
const AuthContext = React.createContext(null)

// Paso 2: Crear Provider
function AuthProvider({ children }) {
  const [user, setUser] = React.useState(null)

  const login = (userData) => setUser(userData)
  const logout = () => setUser(null)

  return (
    <AuthContext.Provider value={{ user, login, logout }}>
      {children}
    </AuthContext.Provider>
  )
}

// Paso 3: Crear Custom Hook
function useAuth() {
  const context = React.useContext(AuthContext)

  if (!context) {
    throw new Error('useAuth debe usarse dentro de AuthProvider')
  }

  return context
}

// Paso 4: Usar el hook en componentes
function LoginPage() {
  const { user, login, logout } = useAuth()

  if (!user) {
    return (
      <button onClick={() => login({ id: 1, name: 'Juan' })}>
        Iniciar sesión
      </button>
    )
  }

  return (
    <div>
      <p>Bienvenido, {user.name}!</p>
      <button onClick={logout}>Cerrar sesión</button>
    </div>
  )
}

Ejemplo 2: Hook useTheme


// Crear Context
const ThemeContext = React.createContext(null)

// Crear Provider
function ThemeProvider({ children }) {
  const [theme, setTheme] = React.useState('light')

  const toggleTheme = () => {
    setTheme(prev => prev === 'light' ? 'dark' : 'light')
  }

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}

// Crear Custom Hook
function useTheme() {
  const context = React.useContext(ThemeContext)

  if (!context) {
    throw new Error('useTheme debe usarse dentro de ThemeProvider')
  }

  return context
}

// Usar en componentes
function App() {
  const { theme, toggleTheme } = useTheme()

  const bgColor = theme === 'light' ? '#fff' : '#222'
  const textColor = theme === 'light' ? '#000' : '#fff'

  return (
    <div style={{ background: bgColor, color: textColor, padding: '20px' }}>
      <p>Tema actual: {theme}</p>
      <button onClick={toggleTheme}>Cambiar Tema</button>
    </div>
  )
}

Ejemplo 3: Hook useForm


function useForm(initialValues, onSubmit) {
  const [values, setValues] = React.useState(initialValues)

  const handleChange = (e) => {
    const { name, value } = e.target
    setValues(prev => ({ ...prev, [name]: value }))
  }

  const handleSubmit = (e) => {
    e.preventDefault()
    onSubmit(values)
  }

  return { values, handleChange, handleSubmit }
}

// Usar en un componente
function LoginForm() {
  const handleSubmit = (values) => {
    console.log('Formulario enviado:', values)
  }

  const { values, handleChange, handleSubmit: onSubmit } = useForm(
    { email: '', password: '' },
    handleSubmit
  )

  return (
    <form onSubmit={onSubmit}>
      <input
        name="email"
        type="email"
        value={values.email}
        onChange={handleChange}
        placeholder="Email"
      />
      <input
        name="password"
        type="password"
        value={values.password}
        onChange={handleChange}
        placeholder="Contraseña"
      />
      <button type="submit">Iniciar sesión</button>
    </form>
  )
}

Ejemplo 4: Hook useCounter (Combinando Múltiples Hooks)


function useCounter(initialValue = 0) {
  const [count, setCount] = React.useState(initialValue)

  const increment = () => setCount(prev => prev + 1)
  const decrement = () => setCount(prev => prev - 1)
  const reset = () => setCount(initialValue)

  return { count, increment, decrement, reset }
}

// Usar en componentes
function Counter() {
  const { count, increment, decrement, reset } = useCounter(0)

  return (
    <div>
      <p>Cuenta: {count}</p>
      <button onClick={increment}>+</button>
      <button onClick={decrement}>-</button>
      <button onClick={reset}>Resetear</button>
    </div>
  )
}

// Puedo reutilizarlo en cualquier lado
function ScoreBoard() {
  const { count: score, increment } = useCounter(0)

  return (
    <div>
      <p>Puntuación: {score}</p>
      <button onClick={increment}>Agregar Punto</button>
    </div>
  )
}

Ejemplo 5: Hook useLocalStorage


function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = React.useState(() => {
    try {
      const item = window.localStorage.getItem(key)
      return item ? JSON.parse(item) : initialValue
    } catch (error) {
      console.error(error)
      return initialValue
    }
  })

  const setValue = (value) => {
    try {
      const valueToStore = value instanceof Function ? value(storedValue) : value
      setStoredValue(valueToStore)
      window.localStorage.setItem(key, JSON.stringify(valueToStore))
    } catch (error) {
      console.error(error)
    }
  }

  return [storedValue, setValue]
}

// Usar en componentes
function UserPreferences() {
  const [preferences, setPreferences] = useLocalStorage('userPrefs', {
    theme: 'light',
    fontSize: 16
  })

  const updateTheme = (newTheme) => {
    setPreferences(prev => ({ ...prev, theme: newTheme }))
  }

  return (
    <div>
      <p>Tema: {preferences.theme}</p>
      <button onClick={() => updateTheme('dark')}>
        Establecer Tema Oscuro
      </button>
    </div>
  )
}

Ejemplo 6: Hook useFetch


function useFetch(url) {
  const [data, setData] = React.useState(null)
  const [loading, setLoading] = React.useState(true)
  const [error, setError] = React.useState(null)

  React.useEffect(() => {
    let isMounted = true

    const fetchData = async () => {
      try {
        setLoading(true)
        const response = await fetch(url)
        if (!response.ok) throw new Error('Error al obtener datos')
        const result = await response.json()
        if (isMounted) {
          setData(result)
          setError(null)
        }
      } catch (err) {
        if (isMounted) {
          setError(err.message)
          setData(null)
        }
      } finally {
        if (isMounted) setLoading(false)
      }
    }

    fetchData()

    return () => {
      isMounted = false
    }
  }, [url])

  return { data, loading, error }
}

// Usar en componentes
function UsersList() {
  const { data: users, loading, error } = useFetch('/api/users')

  if (loading) return <p>Cargando usuarios...</p>
  if (error) return <p>Error: {error}</p>

  return (
    <ul>
      {users?.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  )
}

Analogía Visual: Construyendo una Recepción (Custom Hooks)

Escenario: Una Empresa en Crecimiento

Imaginá una empresa con muchos empleados que necesitan acceder a la oficina del CEO.

❌ El Problema: Acceso Directo (Sin Custom Hook)

Empleado 1    Empleado 2    Empleado 3
    ↓              ↓              ↓
    Camina a oficina del CEO
    ↓              ↓              ↓
    Habla directamente con CEO
    ↓              ↓              ↓
    Obtiene información
    ↓              ↓              ↓
    Regresa a su escritorio

Problemas:

  • ❌ Todos caminan a la oficina del CEO (repetición)
  • ❌ El CEO se interrumpe constantemente
  • ❌ Si la oficina se mueve, todos los empleados se pierden
  • ❌ No hay verificación de quién debería estar ahí
  • ❌ Ineficiente y caótico

✅ La Solución: Escritorio de Recepción (Custom Hook)

Empleado 1    Empleado 2    Empleado 3
    ↓              ↓              ↓
    Le pregunta al recepcionista
    ↓              ↓              ↓
    Recepción valida y proporciona info
    ↓              ↓              ↓
    Regresa con información
    ↓              ↓              ↓
    De vuelta al trabajo inmediatamente

Beneficios:

  • ✅ Un único punto de acceso (un escritorio)
  • ✅ El CEO no se interrumpe (acceso centralizado)
  • ✅ Si la oficina se mueve, solo cambia el escritorio
  • ✅ Recepción valida a los empleados (control de acceso)
  • ✅ Organizado y eficiente

Mapeo al Código React

Oficina del CEO     = AuthContext (datos globales)
Escritorio Recepción = useAuth() hook (punto de acceso)
Empleados           = Componentes React
Preguntar Recepción = Llamar useAuth() en componentes

En Términos de Código

Sin Custom Hook (Acceso Directo - Ineficiente)

// Componente 1: Empleado pregunta directamente al CEO
function Dashboard() {
  const authContext = React.useContext(AuthContext)  // Camina al CEO
  const user = authContext?.user
  return <p>Bienvenido {user?.name}</p>
}

// Componente 2: Empleado pregunta directamente al CEO
function Settings() {
  const authContext = React.useContext(AuthContext)  // Camina al CEO
  const user = authContext?.user
  return <p>Configuración para {user?.name}</p>
}

// Componente 3: Empleado pregunta directamente al CEO
function Profile() {
  const authContext = React.useContext(AuthContext)  // Camina al CEO
  const user = authContext?.user
  return <p>Perfil de {user?.name}</p>
}

Con Custom Hook (Escritorio de Recepción - Eficiente)

// Escritorio de Recepción (Custom Hook)
function useAuth() {
  const authContext = React.useContext(AuthContext)

  if (!authContext) {
    throw new Error('useAuth debe usarse dentro de AuthProvider')
  }

  return authContext  // Devuelve info validada
}

// Componente 1: Empleado pregunta a recepción
function Dashboard() {
  const { user } = useAuth()  // Visita rápida a recepción
  return <p>Bienvenido {user?.name}</p>
}

// Componente 2: Empleado pregunta a recepción
function Settings() {
  const { user } = useAuth()  // Visita rápida a recepción
  return <p>Configuración para {user?.name}</p>
}

// Componente 3: Empleado pregunta a recepción
function Profile() {
  const { user } = useAuth()  // Visita rápida a recepción
  return <p>Perfil de {user?.name}</p>
}

Ejemplo de Empresa Real

Escenario: HR Necesita Actualizar la Ubicación del CEO

Sin custom hook:

HR debe enviar 50 emails a todos los empleados:
"El CEO ahora está en Piso 10, no en Piso 5"

❌ Propenso a errores (empleados podrían perder el email)
❌ Lleva tiempo comunicarse
❌ Algunos empleados siguen yendo al lugar antiguo

Con custom hook:

HR actualiza el procedimiento en recepción:
"Dirigir a empleados al Piso 10"

✅ Un cambio en un único lugar
✅ Todos los empleados automáticamente van al piso correcto
✅ No necesitan notificaciones individuales

En código React:

// ANTES: AuthContext almacenado en Piso 5
export const AuthContext = React.createContext(...)

// DESPUÉS: AuthContext almacenado en Piso 10 (en archivo diferente)
export const AuthContext = React.createContext(...)

// ¡EL CÓDIGO DE COMPONENTES NO CAMBIA!
function Dashboard() {
  const user = useAuth()  // Siempre funciona
  return <p>Bienvenido {user?.name}</p>
}

Por Qué Esto Importa

Fácil de mantener:

Si cambiás cómo funciona la autenticación:
- Actualiza useAuth() una vez
- Los 100 componentes automáticamente usan la nueva lógica
- Ningún código de componente cambia

Manejo de errores:

// Recepción verifica que el empleado esté autorizado
function useAuth() {
  const context = React.useContext(AuthContext)

  if (!context) {
    // LANZA ERROR si el empleado no está autorizado
    throw new Error('useAuth debe usarse dentro de AuthProvider')
  }

  return context
}

Previene errores:

// Sin custom hook: podrías olvidar de validar el contexto
const context = React.useContext(AuthContext)
const user = context?.user  // Podría ser undefined!

// Con custom hook: garantizado tener contexto
const { user } = useAuth()  // Siempre válido o lanza error
