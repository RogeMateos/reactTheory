Hook use() de React 19: Leer Context y Promesas

React 19 introdujo el hook use(), una evolución simple que cambia cómo leemos Context y Promesas en componentes.

No es un reemplazo. Es una evolución.

🏢 Analogía: La Recepción de una Oficina

Imaginate una oficina:

Recepción (Provider)
       ↓
      💬 "Acá está la información compartida"
       ↓
Empleados (Componentes)

La forma antigua (useContext)

La recepcionista te dice: "Buscate vos mismo la información de mi escritorio."

const theme = useContext(ThemeContext)

La forma nueva (use)

La recepcionista te dice: "Te doy la información directamente."

const theme = use(ThemeContext)

Cambia una palabra. Eso es todo.

1️⃣ ¿Qué es use()?

use() es un hook que lee valores de Context O Promesas.

Pensalo como el lector universal:

use() puede leer:
  ✅ Valores de Context
  ✅ Promesas (con Suspense)
  ❌ No useRef
  ❌ No useState
  ❌ No otros hooks

Está diseñado específicamente para:

  • Leer estado compartido (Context)
  • Leer datos asincronos (Promesas)

2️⃣ La Diferencia Mínima: useContext vs use()

useContext (React 16.8+)

import { useContext } from 'react'

function Card() {
  const theme = useContext(ThemeContext)

  return <div style={{ color: theme.color }}>...</div>
}

use() (React 19+)

import { use } from 'react'

function Card() {
  const theme = use(ThemeContext)

  return <div style={{ color: theme.color }}>...</div>
}

El cambio: Un nombre de función. Literalmente eso.

Pero la gran diferencia:

🔥 La Gran Diferencia: Promesas

use() puede leer Promesas. useContext NO puede.

Con Promesas (SOLO con use())

import { use } from 'react'

function UserProfile() {
  const userPromise = fetchUser(userId)
  const user = use(userPromise)  // ✅ Funciona!

  return <h1>{user.name}</h1>
}

useContext NO puede hacer esto:

function UserProfile() {
  const userPromise = fetchUser(userId)
  const user = useContext(userPromise)  // ❌ Error!
}

📚 Tres Cosas que use() Puede Leer

1. Context (Caso Simple)

// Crear context
const ThemeContext = React.createContext('light')

// En un componente
const theme = use(ThemeContext)

2. Promesas (Caso Asincrónico)

// Crear una promesa
const userPromise = fetch('/api/user').then(r => r.json())

// En un componente (necesita Suspense)
const user = use(userPromise)

3. Leer Condicionalmente (La Parte Inteligente)

function Card({ userId }) {
  let user = null

  // ✅ PODÉS usar en condicionales
  if (userId) {
    user = use(fetchUser(userId))
  }

  return <div>{user ? user.name : 'Sin usuario'}</div>
}

Esto es diferente de useState/useRef.

Esos NO pueden ser condicionales. use() SÍ.

🎬 Cómo Funciona use() con Promesas

1. Componente se renderiza
   ↓
2. use(promise) se llama
   ↓
3. Promesa aún NO está resuelta
   ↓
4. React lanza la promesa (Suspense la atrapa)
   ↓
5. React espera la promesa
   ↓
6. Promesa se resuelve
   ↓
7. Componente se re-renderiza con datos

Esto requiere Suspense:

<Suspense fallback={<Loading />}>
  <UserProfile />
</Suspense>

🏗️ Analogía Real: Máquina Expendedora vs Bibliotecaria

useContext: Máquina Expendedora

Interactuás con una máquina:

1. Insertás dinero
2. Máquina te da gaseosa
3. Sin esperar
4. Respuesta instantánea
5. Interacción limitada

Directo, rápido, simple. Funciona solo para valores almacenados.

use(): Bibliotecaria Inteligente

Le pedís a la bibliotecaria:

1. Pedís un libro
2. Bibliotecaria no lo tiene aún
3. Bibliotecaria va a buscarlo
4. Vos esperás (Suspense muestra carga)
5. Bibliotecaria vuelve con el libro
6. Vos obtenés el libro

Flexible, puede traer información, maneja la espera.

🧠 Cuándo Usar use() vs useContext()

Situación useContext() use()
Leer context en React 18 ✅ Única opción ❌ No disponible
Leer context en React 19 ✅ Sigue funcionando ✅ También funciona
Leer una Promesa ❌ No puede ✅ ¡Sí!
Leer context condicionalmente ❌ Regla de hooks ✅ ¡Sí!
Server Components ❌ No soportado ✅ Soportado

💡 La Regla Clave: use() es Más Flexible

useContext() siempre está en el nivel superior. Nunca puede ser condicional.

use() puede ser condicional porque funciona diferente internamente.

// ❌ Esto rompe las reglas de hooks
function Profile() {
  if (someCondition) {
    const theme = useContext(ThemeContext)
  }
}

// ✅ Esto funciona bien con use()
function Profile() {
  if (someCondition) {
    const theme = use(ThemeContext)
  }
}

🚀 Novedad en React 19: Cliente vs Servidor

useContext: Solo Client Components

use(): Client Components AND Server Components

// Server Component
export async function UserCard() {
  // ❌ useContext no funciona acá
  // ✅ Pero use(promise) funciona!
  const user = use(fetchUser())
  return <h1>{user.name}</h1>
}

🎯 Regla de Decisión: Checklist Simple

¿Estás en React 19?

  • ¿Proyecto nuevo? → Usá use()
  • ¿Proyecto existente? → Usá useContext() (no lo rompas)

¿Necesitás leer una Promesa?

  • ¿Sí? → Debes usar use()
  • ¿No? → Cualquiera funciona bien

¿El context es condicional?

  • ¿Sí? → Debes usar use()
  • ¿No? → Cualquiera funciona bien

❌ Qué use() NO Puede Hacer

use() NO puede leer:
  ❌ useState
  ❌ useRef
  ❌ useEffect
  ❌ useReducer
  ❌ useCallback
  ❌ useMemo
  ❌ Valores regulares

use() SOLO lee:

✅ Context
✅ Promesas

📖 La Verdad Importante

use() NO es un reemplazo de useContext().
Es una evolución que agrega nuevas capacidades.

Pensalo así:

useContext() resuelve: "¿Cómo leo context?"
use() resuelve: "¿Cómo leo context O promesas?"

useContext sigue ahí. Sigue funcionando. Algunos equipos no migrarán todo.

use() simplemente te da MÁS opciones cuando las necesitás.

🧠 La Perspectiva del Senior

Un developer senior diría: 

"Para nuevos proyectos en React 19, usá use().
Es más simple, más flexible.

¿Para proyectos existentes con useContext?
Dejalo así. No refactorices.
use() es para CÓDIGO NUEVO donde suma valor."

El objetivo es claridad, no moda.

🎓 Qué Debes Recordar

  • use() lee Context o Promesas
  • useContext() lee solo Context
  • Cambia una palabra en el código
  • La gran diferencia: use() puede leer Promesas
  • use() puede ser condicional (a diferencia de useState/useRef)
  • Las Promesas necesitan Suspense
  • use() es para React 19+
  • No refactorices código viejo de useContext
  • Esto es una evolución, no una revolución

Ejemplos de Código

Ejemplo 1: Context Simple con use()


// 1. Crear context
const ThemeContext = React.createContext('light')

// 2. Provider en el padre
function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Card />
    </ThemeContext.Provider>
  )
}

// 3. Leer con use()
function Card() {
  const theme = use(ThemeContext)

  return (
    <div style={{
      background: theme === 'dark' ? '#333' : '#fff'
    }}>
      Tema actual: {theme}
    </div>
  )
}

Ejemplo 2: Promesa con use() (Server Component)


async function fetchUser(id) {
  const response = await fetch(`/api/users/${id}`)
  return response.json()
}

// Client Component
import { Suspense } from 'react'
import { use } from 'react'

function UserCard({ userId }) {
  const userPromise = fetchUser(userId)
  const user = use(userPromise)

  return (
    <div>
      <h1>{user.name}</h1>
      <p>Email: {user.email}</p>
    </div>
  )
}

// Envolver con Suspense
export default function App({ userId }) {
  return (
    <Suspense fallback={<p>Cargando usuario...</p>}>
      <UserCard userId={userId} />
    </Suspense>
  )
}

Ejemplo 3: Context Condicional con use()


const NotificationContext = React.createContext(null)

function NotificationBadge({ showNotifications }) {
  // ✅ Esto está permitido con use()
  let notifications = null
  if (showNotifications) {
    notifications = use(NotificationContext)
  }

  return (
    <div>
      {notifications ? (
        <span>Tenés {notifications.count} notificaciones</span>
      ) : (
        <span>Notificaciones deshabilitadas</span>
      )}
    </div>
  )
}

Ejemplo 4: Antes vs Después (useContext a use)


// ANTES: useContext
function LanguageSwitcher() {
  const { language, setLanguage } = useContext(LanguageContext)

  return (
    <div>
      Actual: {language}
      <button onClick={() => setLanguage('es')}>
        Español
      </button>
    </div>
  )
}

// DESPUÉS: use()
function LanguageSwitcher() {
  const { language, setLanguage } = use(LanguageContext)

  return (
    <div>
      Actual: {language}
      <button onClick={() => setLanguage('es')}>
        Español
      </button>
    </div>
  )
}

Ejemplo 5: Obtención Compleja de Datos


// Crear un context para theme
const ThemeContext = React.createContext('light')

// Obtener usuario con lógica consciente del tema
async function fetchUserWithTheme(userId, theme) {
  const response = await fetch(
    `/api/users/${userId}?theme=${theme}`
  )
  return response.json()
}

// Componente que necesita context y promesa
function UserProfile({ userId }) {
  const theme = use(ThemeContext)
  const userPromise = fetchUserWithTheme(userId, theme)
  const user = use(userPromise)

  return (
    <div style={{
      color: theme === 'dark' ? '#fff' : '#000'
    }}>
      <h1>{user.name}</h1>
      <p>{user.bio}</p>
    </div>
  )
}

// Envolver con Suspense
export default function App({ userId }) {
  return (
    <ThemeContext.Provider value="dark">
      <Suspense fallback={<p>Cargando...</p>}>
        <UserProfile userId={userId} />
      </Suspense>
    </ThemeContext.Provider>
  )
}

Ejemplo 6: Múltiples Contexts con use()


const ThemeContext = React.createContext()
const UserContext = React.createContext()

function Dashboard() {
  const theme = use(ThemeContext)
  const user = use(UserContext)

  return (
    <div style={{
      background: theme === 'dark' ? '#333' : '#fff'
    }}>
      <h1>¡Bienvenido, {user.name}!</h1>
      <p>Tema: {theme}</p>
    </div>
  )
}

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <UserContext.Provider value={{ name: 'Alice' }}>
        <Dashboard />
      </UserContext.Provider>
    </ThemeContext.Provider>
  )
}

Guía de Migración: useContext a use()

¿Deberías Migrar?

Respuesta rápida: Si funciona, no lo toques.

Migra solo si:

  • Estás iniciando un nuevo proyecto en React 19
  • Necesitás leer Promesas
  • Necesitás leer context condicionalmente
  • Tu equipo está de acuerdo con el cambio

No migres si:

  • Tu proyecto usa React 18 o anterior
  • Tu equipo no entiende la diferencia
  • No hay un beneficio claro
  • Tenés codebases grandes con useContext

Paso 1: Identificar Uso de useContext

Encontrá todos los useContext:

grep -r "useContext" src/

Paso 2: Reemplazar Imports


// ANTES
import { useContext } from 'react'

// DESPUÉS
import { use } from 'react'

Paso 3: Reemplazar Llamadas de Función


// ANTES
const theme = useContext(ThemeContext)

// DESPUÉS
const theme = use(ThemeContext)

Paso 4: Testear Todo

Después de cambiar:

  • Renderizar componentes
  • Cambiar valores del context
  • Verificar que la UI se actualiza
  • Revisar la consola por errores

Ejemplo Completo: Antes y Después

Antes (useContext)


import { useContext } from 'react'

const ThemeContext = React.createContext('light')

function Button() {
  const theme = useContext(ThemeContext)

  return (
    <button style={{
      background: theme === 'dark' ? '#333' : '#fff',
      color: theme === 'dark' ? '#fff' : '#000'
    }}>
      Hacé clic
    </button>
  )
}

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Button />
    </ThemeContext.Provider>
  )
}

Después (use)


import { use } from 'react'

const ThemeContext = React.createContext('light')

function Button() {
  const theme = use(ThemeContext)

  return (
    <button style={{
      background: theme === 'dark' ? '#333' : '#fff',
      color: theme === 'dark' ? '#fff' : '#000'
    }}>
      Hacé clic
    </button>
  )
}

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Button />
    </ThemeContext.Provider>
  )
}

El único cambio: import y nombre de función.

Checklist de Migración

✅ Todos los useContext encontrados
✅ Imports actualizados a use
✅ Llamadas de función actualizadas
✅ Componentes testeados
✅ Sin errores en consola
✅ Acuerdo del equipo

Trampas Comunes Durante la Migración

Trampa 1: Olvidar Actualizar el Import


// ❌ Romperá
import { useContext } from 'react'
const value = use(MyContext)

// ✅ Correcto
import { use } from 'react'
const value = use(MyContext)

Trampa 2: Usar use() en Class Components


// ❌ No funcionará
class MyComponent extends React.Component {
  render() {
    const value = use(ThemeContext)
    return ...
  }
}

// ✅ Correcto - usar function components
function MyComponent() {
  const value = use(ThemeContext)
  return ...
}

Trampa 3: No Testear Después de Migrar

Siempre testa:

  • Renderizado inicial con context
  • Cambios de valor del context
  • Múltiples consumidores de context
  • Consola del navegador

Cuándo NO Migrar

Razón 1: Codebase en React 18

use() solo funciona en React 19+. Si estás en React 18:

❌ No podés migrar aún
⚠️ Actualiza React primero
✅ Luego migra

Razón 2: Equipo Grande, Sin Consenso

Si el equipo no entiende o no está de acuerdo:

❌ No fuercés el cambio
📚 Educá primero
✅ Luego decidan juntos

Razón 3: Sin Beneficio Claro

Si useContext funciona bien:

❌ No cambies por cambiar
✅ Dejalo así
🚀 Enfocáte en features

Estrategia de Migración: Enfoque Gradual

Opción 1: Solo código nuevo

Usá use() en componentes nuevos. Dejá los antiguos con useContext.

Opción 2: Componente por componente

Migra un componente, testa, commit. Luego el siguiente.

Opción 3: Archivo por archivo

Migra todos los componentes de un archivo juntos.

Opción 4: Migración completa

Migra todo de una vez. Solo si el equipo está seguro.

Consejo Final

La Regla de Oro:
"Si useContext funciona, no lo rompas. Usá use() para CÓDIGO NUEVO."

Preferencia:

  • Features nuevas → usar use()
  • Código viejo → dejá useContext así
  • Refactors grandes → discutí con el equipo primero
