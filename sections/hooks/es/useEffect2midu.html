Hook useEffect() — Parte II

🌊 ¿Qué es un efecto secundario?

El render de un componente debe ser una función pura: misma entrada → misma salida, sin sorpresas. Pero las apps necesitan interactuar con el mundo exterior. useEffect es la vía de escape para esas operaciones. 

Efectos secundarios comunes:
  🌐 Llamadas a APIs
  📝 Cambiar document.title o localStorage
  🔔 Suscribirse a eventos del navegador
  ⏱️ Timers e intervals
  📡 WebSockets

⏰ ¿Cuándo se ejecuta useEffect?

La clave está en el segundo parámetro: el array de dependencias. Hay tres escenarios: 

Sin segundo parámetro  →  en CADA render
[], array vacío        →  solo una vez (al montar)
[dep1, dep2]           →  al montar + cada vez que dep1 o dep2 cambien

Importante: el efecto siempre corre al menos una vez en el primer render, sin importar las dependencias.

🔗 El array de dependencias

Pensá en el array de dependencias como una "lista de vigilancia". React observa esos valores, y si alguno cambia entre renders, vuelve a ejecutar el efecto.

Si la lista está vacía [], React no vigila nada → el efecto corre solo una vez.
Si no hay lista, React lo corre en cada render.

💡 La regla de oro: si usás una variable dentro del efecto, tiene que estar en el array de dependencias. ESLint (con el plugin de React) te avisa si te olvidás alguna.

♾️ El bucle infinito: el error más común

¿Cuándo ocurre? Cuando el efecto modifica una de sus propias dependencias. Esto crea un ciclo: 

1. componente monta → useEffect corre
2. setCount() → estado cambia
3. count cambió → useEffect corre
4. setCount() → estado cambia
5. ... ♾️ infinito

La solución: nunca modificar una dependencia dentro del efecto que la tiene como dependencia.

⚠️ Objetos y arrays como dependencias

Los objetos y arrays se recrean en cada render. React usa === para comparar dependencias: un objeto nuevo nunca es igual al anterior, aunque tenga el mismo contenido: 

// En cada render:
const filters = { tech: 'react' }   ← nuevo objeto, nueva referencia

// React compara: filters anterior === filters nuevo?
// {} === {} → false ❌ → el efecto corre en cada render

// Solución: usar las propiedades primitivas como dependencias
[filters.tech, filters.location]   ← strings, se comparan por valor ✅

1️⃣ Sin dependencias: en cada render

function App() {
  const [count, setCount] = useState(0)

  useEffect(() => {
    console.log('El componente se renderizó')
    // ⚠️ Cuidado: corre en CADA render
  })

  return (
    <div>
      <p>Contador: {count}</p>
      <button onClick={() => setCount(count + 1)}>Incrementar</button>
    </div>
  )
}

2️⃣ Array vacío: solo al montar

function App() {
  const [count, setCount] = useState(0)

  useEffect(() => {
    console.log('Solo se ejecuta una vez al montar')
    // Útil para: fetch inicial, suscripciones, etc.
  }, [])

  return (
    <div>
      <p>Contador: {count}</p>
      <button onClick={() => setCount(count + 1)}>Incrementar</button>
    </div>
  )
}

Ahora aunque hagas clic 100 veces, el efecto solo corre una vez.

3️⃣ Con dependencias: cuando cambia algo específico

function App() {
  const [count, setCount] = useState(0)
  const [name, setName] = useState('Midu')

  useEffect(() => {
    console.log('El contador cambió:', count)
    // Solo corre cuando `count` cambia
    // NO corre cuando `name` cambia
  }, [count])

  return (
    <div>
      <p>Contador: {count}</p>
      <button onClick={() => setCount(count + 1)}>Incrementar</button>
      <input value={name} onChange={(e) => setName(e.target.value)} />
    </div>
  )
}

4️⃣ Ejemplo real: actualizar el título del documento

function App() {
  const [currentPage, setCurrentPage] = useState(1)
  const [textToFilter, setTextToFilter] = useState('')
  const [jobsData] = useState([
    { id: 1, title: 'React Developer' },
    { id: 2, title: 'Vue Developer' },
    { id: 3, title: 'Angular Developer' },
  ])

  const jobsFiltered =
    textToFilter === ''
      ? jobsData
      : jobsData.filter((job) =>
          job.title.toLowerCase().includes(textToFilter.toLowerCase())
        )

  useEffect(() => {
    const jobCount = jobsFiltered.length

    if (textToFilter === '') {
      document.title = `DevJobs - Página ${currentPage}`
    } else {
      document.title = `${jobCount} trabajos de "${textToFilter}" - Pág. ${currentPage}`
    }
  }, [jobsFiltered.length, currentPage, textToFilter])

  return (
    <div>
      <input
        value={textToFilter}
        onChange={(e) => setTextToFilter(e.target.value)}
        placeholder="Filtrar trabajos..."
      />

      <ul>
        {jobsFiltered.map((job) => (
          <li key={job.id}>{job.title}</li>
        ))}
      </ul>

      <button
        onClick={() => setCurrentPage(currentPage - 1)}
        disabled={currentPage === 1}
      >
        Anterior
      </button>
      <span> Página {currentPage} </span>
      <button onClick={() => setCurrentPage(currentPage + 1)}>
        Siguiente
      </button>
    </div>
  )
}

document.title es un efecto secundario: está fuera del componente. Caso de uso perfecto para useEffect.

5️⃣ Múltiples dependencias

function JobListings() {
  const [currentPage, setCurrentPage] = useState(1)
  const [textToFilter, setTextToFilter] = useState('')

  useEffect(() => {
    console.log('Página o filtro cambiaron')
    console.log('Página:', currentPage, '| Filtro:', textToFilter)
    // Corre si CUALQUIERA de las dos dependencias cambia
  }, [currentPage, textToFilter])

  return (
    <div>
      <input
        value={textToFilter}
        onChange={(e) => setTextToFilter(e.target.value)}
        placeholder="Filtrar..."
      />
      <button onClick={() => setCurrentPage(currentPage + 1)}>
        Siguiente página
      </button>
    </div>
  )
}

📊 Tabla resumen de dependencias

Dependencias            ¿Cuándo corre el efecto?
──────────────────────────────────────────────────────
Sin segundo parámetro   En cada render
[]                      Solo una vez (al montar)
[dep1]                  Al montar + cuando dep1 cambie
[dep1, dep2]            Al montar + cuando dep1 o dep2 cambien

🐛 Error 1: bucle infinito

// ❌ BUCLE INFINITO — el efecto modifica su propia dependencia
function App() {
  const [count, setCount] = useState(0)

  useEffect(() => {
    setCount(count + 1)  // ← modifica count
  }, [count])             // ← count es dependencia → loop ♾️

  return <div>{count}</div>
}

// ✅ Solución 1: array vacío si no necesitás que reaccione al cambio
useEffect(() => {
  setCount(1)
}, [])

// ✅ Solución 2: forma funcional de setState (no necesita count como dep)
useEffect(() => {
  setCount((prev) => prev + 1)
}, [])

// ✅ Solución 3: solo leer, no modificar
useEffect(() => {
  document.title = `Contador: ${count}`
}, [count])

🐛 Error 2: dependencia faltante

// ❌ textToFilter se usa pero no está en las dependencias
useEffect(() => {
  console.log('Filtro:', textToFilter)  // usa textToFilter...
}, [])                                  // ...pero no está aquí ❌
// Bug: el efecto solo corre al montar, usa el valor inicial de textToFilter para siempre

// ✅ Incluir todas las variables que se usan
useEffect(() => {
  console.log('Filtro:', textToFilter)
}, [textToFilter])  // ✅

🐛 Error 3: objeto como dependencia

// ❌ El objeto se recrea en cada render → el efecto corre en cada render
const filters = { technology: 'react', location: 'remote' }

useEffect(() => {
  console.log('Filtros:', filters)
}, [filters])  // filters nuevo en cada render → siempre diferente ❌

// ✅ Usar propiedades primitivas como dependencias
useEffect(() => {
  console.log('Filtros:', filters.technology, filters.location)
}, [filters.technology, filters.location])  // strings se comparan por valor ✅

💪 Ejercicio

Implementá los siguientes efectos en un componente de búsqueda:

1. Al montar el componente:
   □ Mostrá en consola "Componente montado"
   □ Cargá datos iniciales (fetch o datos de prueba)

2. Cuando cambie el filtro de texto:
   □ Actualizá document.title con el número de resultados
   □ Reseteá currentPage a 1

3. Cuando cambie la página:
   □ Actualizá document.title con la página actual

4. Verificación:
   □ ¿Hay algún bucle infinito?
   □ ¿Las dependencias incluyen todo lo que se usa?
   □ ¿El título del documento cambia correctamente?
   □ ¿El efecto del mount corre solo una vez?

✅ Resumen

Sin deps         →  corre en cada render (cuidado con rendimiento)
[]               →  corre una vez al montar
[dep1, dep2]     →  corre al montar + cuando dep1 o dep2 cambian
Bucle infinito   →  ocurre cuando el efecto modifica su propia dependencia
Objetos/arrays   →  nunca como dependencias directas, usar sus propiedades
Deps faltantes   →  causan bugs de valores "congelados"
Regla de oro     →  todo lo que usás dentro del efecto va en las dependencias
