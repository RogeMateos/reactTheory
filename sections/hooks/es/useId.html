Hook useId()

🆔 El Problema: IDs Duplicados en HTML

En HTML, los IDs deben ser únicos en toda la página. Son la forma en que un <label> sabe a qué campo pertenece, y en que los lectores de pantalla asocian elementos entre sí.

Cuando escribís IDs a mano en un componente React, todo parece funcionar... hasta que usás ese componente dos veces en la misma página. 

Primer SearchForm:
  <label for="filter-location">Ubicación</label>
  <select id="filter-location">

Segundo SearchForm:
  <label for="filter-location">Ubicación</label>
  <select id="filter-location">   ← ID duplicado ❌

Las consecuencias son sutiles pero reales:

❌ HTML técnicamente inválido
❌ Click en el label puede enfocar el elemento equivocado
❌ Lectores de pantalla se confunden (accesibilidad rota)
❌ Herramientas de testing reportan errores

💡 La Solución: Dejar que React Genere los IDs

useId() es un hook de React 18 que genera un ID único automáticamente cada vez que se crea una nueva instancia del componente.

No tenés que pensar en nombres, no tenés que preocuparte por duplicados. React se encarga de que cada llamado a useId() devuelva un valor único en toda la aplicación. 

Primer SearchForm:
  label for=":r0:"  →  select id=":r0:"   ✅

Segundo SearchForm:
  label for=":r2:"  →  select id=":r2:"   ✅ diferente y único

📋 ¿Qué Genera useId() Exactamente?

El formato que genera React es :r0:, :r1:, :r2:, etc. Los dos puntos son válidos en HTML5 y el número incrementa automáticamente con cada nueva instancia. 

Características del ID generado:

  🔢 Incremental    — :r0:, :r1:, :r2:...
  🔒 Único          — nunca se repite en la misma página
  📌 Estable        — no cambia si el componente se re-renderiza
  🌐 Funciona en SSR — compatible con Server-Side Rendering
  📏 Corto          — no infla el HTML innecesariamente

📝 El Evento Submit y preventDefault()

Antes de hablar solo de IDs, hay algo igual de importante en los formularios: el evento submit.

Cuando un formulario HTML se envía, el navegador por defecto recarga la página. Eso destruye el estado de React. Con e.preventDefault() cancelamos ese comportamiento y manejamos el envío completamente con JavaScript. 

Sin preventDefault():
  1. Usuario hace click en "Buscar"
  2. El navegador recarga la página ❌
  3. Todo el estado de React se pierde

Con preventDefault():
  1. Usuario hace click en "Buscar"
  2. Se ejecuta handleSubmit ✅
  3. React mantiene su estado

🔘 Tipos de Botones en Formularios

Un detalle que suele sorprender: dentro de un <form>, un <button> sin type es automáticamente type="submit". 

<button type="submit">  →  envía el formulario (dispara onSubmit)
<button type="button">  →  no envía nada (solo ejecuta onClick)
<button type="reset">   →  resetea los campos al valor inicial
<button>               →  ⚠️ equivale a type="submit" dentro de un form

Siempre es buena práctica declarar el type explícitamente para que el comportamiento sea predecible.

✅ Formulario con useId() + handleSubmit

import { useId } from 'react'

export function SearchForm() {
  // useId() genera un ID único por cada instancia del componente
  const locationId = useId()
  const experienceId = useId()

  const handleSubmit = (e) => {
    e.preventDefault()        // cancela la recarga de página
    console.log('Formulario enviado')
  }

  return (
    <form className="search-form" onSubmit={handleSubmit}>

      <div className="form-group">
        <label htmlFor={locationId}>Ubicación</label>
        <select id={locationId} name={locationId}>
          <option value="">Ubicación</option>
          <option value="remoto">Remoto</option>
          <option value="cdmx">Ciudad de México</option>
          <option value="guadalajara">Guadalajara</option>
          <option value="barcelona">Barcelona</option>
        </select>
      </div>

      <div className="form-group">
        <label htmlFor={experienceId}>Nivel de experiencia</label>
        <select id={experienceId} name={experienceId}>
          <option value="">Nivel de experiencia</option>
          <option value="junior">Junior</option>
          <option value="mid">Mid-level</option>
          <option value="senior">Senior</option>
        </select>
      </div>

      <button type="submit">Buscar</button>

    </form>
  )
}

🎯 Casos de Uso de useId()

Labels e inputs

function TextField({ label }) {
  const id = useId()
  return (
    <>
      <label htmlFor={id}>{label}</label>
      <input id={id} type="text" />
    </>
  )
}

Atributos ARIA

function Tooltip({ children, tip }) {
  const id = useId()
  return (
    <>
      <div aria-describedby={id}>{children}</div>
      <div role="tooltip" id={id}>{tip}</div>
    </>
  )
}

IDs relacionados con sufijos

function FormField({ label }) {
  const baseId = useId()              // ":r0:"

  const inputId = `${baseId}-input`  // ":r0:-input"
  const helpId  = `${baseId}-help`   // ":r0:-help"

  return (
    <>
      <label htmlFor={inputId}>{label}</label>
      <input id={inputId} aria-describedby={helpId} />
      <small id={helpId}>Texto de ayuda</small>
    </>
  )
}

⚖️ useId() vs Alternativas

// ❌ Math.random() — cambia en cada render, bugs garantizados
const id = `input-${Math.random()}`

// ❌ useState con string — innecesariamente complejo
const [id] = useState('my-static-id')  // y sigue siendo duplicable

// ✅ useId() — simple, único, estable
const id = useId()

📏 Reglas de useId() — Como Todo Hook

useId() sigue las mismas reglas que todos los hooks de React: debe llamarse en el nivel superior del componente. 

// ✅ Nivel superior — correcto
function MyComponent() {
  const id1 = useId()
  const id2 = useId()
  const id3 = useId()
  return <div id={id1}>...</div>
}

// ✅ Concatenar al ID — correcto
const base = useId()
const inputId = `${base}-input`
const errorId = `${base}-error`
// ❌ Dentro de una condición
if (isVisible) {
  const id = useId()  // Error — viola las reglas de hooks
}

// ❌ Dentro de un loop
for (let i = 0; i < 3; i++) {
  const id = useId()  // Error
}

// ❌ Dentro de un callback
const handleClick = () => {
  const id = useId()  // Error
}

// ❌ Como key en .map() — no es para eso
items.map((item) => <div key={useId()}>{item}</div>)
// ✅ Para keys, usá item.id

✅ Resumen de la Clase

🆔 useId()           — genera IDs únicos por instancia de componente
⛔ e.preventDefault() — cancela la recarga de página en submit
🔘 type="submit"      — siempre declarar el type del botón
🔒 IDs estables       — no cambian entre re-renders
🌐 Compatible SSR     — funciona en Server-Side Rendering
🎯 Para labels/ARIA   — accesibilidad correcta sin esfuerzo
📏 Reglas de hooks    — solo en el nivel superior del componente
