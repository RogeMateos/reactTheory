React 19 use() Hook: Reading Context and Promises

React 19 introduced the use() hook, a simple evolution that changes how we read Context and Promises in components.

It's not a replacement. It's an evolution.

🏢 Analogy: The Office Reception

Imagine a company office:

Reception Desk (Provider)
       ↓
      💬 "Here's shared information"
       ↓
Office Employees (Components)

The Old Way (useContext)

The receptionist says: "Go find the information yourself from my desk."

const theme = useContext(ThemeContext)

The New Way (use)

The receptionist says: "I'll give you the information directly."

const theme = use(ThemeContext)

One word changes. That's it.

1️⃣ What is use()?

use() is a hook that reads values from Context OR Promises.

Think of it as the universal reader:

use() can read:
  ✅ Context values
  ✅ Promises (with Suspense)
  ❌ Not useRef
  ❌ Not useState
  ❌ Not other hooks

It's specifically designed for:

  • Reading shared state (Context)
  • Reading async data (Promises)

2️⃣ The Minimal Difference: useContext vs use()

useContext (React 16.8+)

import { useContext } from 'react'

function Card() {
  const theme = useContext(ThemeContext)

  return <div style={{ color: theme.color }}>...</div>
}

use() (React 19+)

import { use } from 'react'

function Card() {
  const theme = use(ThemeContext)

  return <div style={{ color: theme.color }}>...</div>
}

The change: One function name. That's literally it.

But the big difference:

🔥 The Big Difference: Promises

use() can read Promises. useContext cannot.

With Promises (ONLY with use())

import { use } from 'react'

function UserProfile() {
  const userPromise = fetchUser(userId)
  const user = use(userPromise)  // ✅ Works!

  return <h1>{user.name}</h1>
}

useContext cannot do this:

function UserProfile() {
  const userPromise = fetchUser(userId)
  const user = useContext(userPromise)  // ❌ Error!
}

📚 Three Things use() Reads

1. Context (Simple Case)

// Create context
const ThemeContext = React.createContext('light')

// In a component
const theme = use(ThemeContext)

2. Promises (Async Case)

// Create a promise
const userPromise = fetch('/api/user').then(r => r.json())

// In a component (needs Suspense)
const user = use(userPromise)

3. Conditionally Read (The Smart Part)

function Card({ userId }) {
  let user = null

  // ✅ You CAN use in conditionals
  if (userId) {
    user = use(fetchUser(userId))
  }

  return <div>{user ? user.name : 'No user'}</div>
}

This is different from useState/useRef.

Those CANNOT be conditional. use() CAN.

🎬 How use() Works with Promises

1. Component renders
   ↓
2. use(promise) is called
   ↓
3. Promise is NOT resolved yet
   ↓
4. React throws the promise (Suspense catches it)
   ↓
5. React waits for the promise
   ↓
6. Promise resolves
   ↓
7. Component re-renders with data

This requires Suspense:

<Suspense fallback={<Loading />}>
  <UserProfile />
</Suspense>

🏗️ Real-World Analogy: Vending Machine vs Librarian

useContext: Vending Machine

You interact with a machine:

1. You insert money
2. Machine gives you soda
3. No waiting
4. Instant response
5. Limited interaction

Direct, fast, simple. Works only for stored values.

use(): Smart Librarian

You ask a librarian:

1. You ask for a book
2. Librarian doesn't have it yet
3. Librarian goes to get it
4. You wait (Suspense shows loading)
5. Librarian returns with book
6. You get the book

Flexible, can fetch information, handles waiting.

🧠 When to Use use() vs useContext()

Situation Use useContext() Use use()
Reading context in React 18 ✅ Only option ❌ Not available
Reading context in React 19 ✅ Still works ✅ Also works
Reading a Promise ❌ Can't do it ✅ Yes!
Conditionally read context ❌ Rule of hooks ✅ Yes!
Server Components ❌ Not supported ✅ Supported

💡 The Key Rule: use() is More Flexible

useContext() is always at the top level. It must never be conditional.

use() can be conditional because it works differently internally.

// ❌ This breaks the rules of hooks
function Profile() {
  if (someCondition) {
    const theme = useContext(ThemeContext)
  }
}

// ✅ This works fine with use()
function Profile() {
  if (someCondition) {
    const theme = use(ThemeContext)
  }
}

🚀 React 19 New Feature: Client vs Server

useContext: Client Components only

use(): Client Components AND Server Components

// Server Component
export async function UserCard() {
  // ❌ useContext doesn't work here
  // ✅ But use(promise) works here!
  const user = use(fetchUser())
  return <h1>{user.name}</h1>
}

🎯 Decision Rule: Simple Checklist

Are you on React 19?

  • New project? → Use use()
  • Existing project? → Use useContext() (don't break it)

Do you need to read a Promise?

  • Yes? → Must use use()
  • No? → Either works fine

Is the context conditional?

  • Yes? → Must use use()
  • No? → Either works fine

❌ What use() CANNOT Do

use() CANNOT read:
  ❌ useState
  ❌ useRef
  ❌ useEffect
  ❌ useReducer
  ❌ useCallback
  ❌ useMemo
  ❌ Regular values

use() ONLY reads:

✅ Context
✅ Promises

📖 The Important Truth

use() is NOT a replacement for useContext().
It's an evolution that adds new capabilities.

Think of it like this:

useContext() solves: "How do I read context?"
use() solves: "How do I read context OR promises?"

useContext is still there. It still works. Some teams won't migrate everything.

use() just gives you MORE options when you need them.

🧠 The Senior's Perspective

A senior developer would say: 

"For new React 19 projects, use use().
It's simpler, more flexible.

For existing projects with useContext?
Leave it alone. Don't refactor.
use() is for NEW code where it adds value."

The goal is clarity, not fashion.

🎓 What You Must Remember

  • use() reads Context or Promises
  • useContext() reads only Context
  • One word changes in the code
  • The big difference: use() can read Promises
  • use() can be conditional (unlike useState/useRef)
  • Promises need Suspense
  • use() is for React 19+
  • Don't refactor old useContext code
  • This is an evolution, not a revolution

Code Examples

Example 1: Simple Context with use()


// 1. Create context
const ThemeContext = React.createContext('light')

// 2. Provider in parent
function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Card />
    </ThemeContext.Provider>
  )
}

// 3. Read with use()
function Card() {
  const theme = use(ThemeContext)

  return (
    <div style={{
      background: theme === 'dark' ? '#333' : '#fff'
    }}>
      Current theme: {theme}
    </div>
  )
}

Example 2: Promise with use() (Server Component)


async function fetchUser(id) {
  const response = await fetch(`/api/users/${id}`)
  return response.json()
}

// Client Component
import { Suspense } from 'react'
import { use } from 'react'

function UserCard({ userId }) {
  const userPromise = fetchUser(userId)
  const user = use(userPromise)

  return (
    <div>
      <h1>{user.name}</h1>
      <p>Email: {user.email}</p>
    </div>
  )
}

// Wrap with Suspense
export default function App({ userId }) {
  return (
    <Suspense fallback={<p>Loading user...</p>}>
      <UserCard userId={userId} />
    </Suspense>
  )
}

Example 3: Conditional Context with use()


const NotificationContext = React.createContext(null)

function NotificationBadge({ showNotifications }) {
  // ✅ This is allowed with use()
  let notifications = null
  if (showNotifications) {
    notifications = use(NotificationContext)
  }

  return (
    <div>
      {notifications ? (
        <span>You have {notifications.count} notifications</span>
      ) : (
        <span>Notifications disabled</span>
      )}
    </div>
  )
}

Example 4: Before vs After (useContext to use)


// BEFORE: useContext
function LanguageSwitcher() {
  const { language, setLanguage } = useContext(LanguageContext)

  return (
    <div>
      Current: {language}
      <button onClick={() => setLanguage('es')}>
        Español
      </button>
    </div>
  )
}

// AFTER: use()
function LanguageSwitcher() {
  const { language, setLanguage } = use(LanguageContext)

  return (
    <div>
      Current: {language}
      <button onClick={() => setLanguage('es')}>
        Español
      </button>
    </div>
  )
}

Example 5: Complex Data Fetching


// Create a context for theme
const ThemeContext = React.createContext('light')

// Fetch user with theme-aware logic
async function fetchUserWithTheme(userId, theme) {
  const response = await fetch(
    `/api/users/${userId}?theme=${theme}`
  )
  return response.json()
}

// Component that needs both context and promise
function UserProfile({ userId }) {
  const theme = use(ThemeContext)
  const userPromise = fetchUserWithTheme(userId, theme)
  const user = use(userPromise)

  return (
    <div style={{
      color: theme === 'dark' ? '#fff' : '#000'
    }}>
      <h1>{user.name}</h1>
      <p>{user.bio}</p>
    </div>
  )
}

// Wrap with Suspense
export default function App({ userId }) {
  return (
    <ThemeContext.Provider value="dark">
      <Suspense fallback={<p>Loading...</p>}>
        <UserProfile userId={userId} />
      </Suspense>
    </ThemeContext.Provider>
  )
}

Example 6: Multiple Contexts with use()


const ThemeContext = React.createContext()
const UserContext = React.createContext()

function Dashboard() {
  const theme = use(ThemeContext)
  const user = use(UserContext)

  return (
    <div style={{
      background: theme === 'dark' ? '#333' : '#fff'
    }}>
      <h1>Welcome, {user.name}!</h1>
      <p>Theme: {theme}</p>
    </div>
  )
}

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <UserContext.Provider value={{ name: 'Alice' }}>
        <Dashboard />
      </UserContext.Provider>
    </ThemeContext.Provider>
  )
}

Migration Guide: useContext to use()

Should You Migrate?

Quick answer: If it works, don't touch it.

Migrate only if:

  • You're starting a new React 19 project
  • You need to read Promises
  • You need conditional context reading
  • Your team agrees on the change

Don't migrate if:

  • Your project uses React 18 or earlier
  • Your team doesn't understand the difference
  • There's no clear benefit
  • You have large existing codebases with useContext

Step 1: Identify useContext Usage

Find all useContext calls:

grep -r "useContext" src/

Step 2: Replace Imports


// BEFORE
import { useContext } from 'react'

// AFTER
import { use } from 'react'

Step 3: Replace Function Calls


// BEFORE
const theme = useContext(ThemeContext)

// AFTER
const theme = use(ThemeContext)

Step 4: Test Everything

After changing:

  • Render components
  • Change context values
  • Verify UI updates
  • Check console for errors

Full Example: Before & After

Before (useContext)


import { useContext } from 'react'

const ThemeContext = React.createContext('light')

function Button() {
  const theme = useContext(ThemeContext)

  return (
    <button style={{
      background: theme === 'dark' ? '#333' : '#fff',
      color: theme === 'dark' ? '#fff' : '#000'
    }}>
      Click me
    </button>
  )
}

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Button />
    </ThemeContext.Provider>
  )
}

After (use)


import { use } from 'react'

const ThemeContext = React.createContext('light')

function Button() {
  const theme = use(ThemeContext)

  return (
    <button style={{
      background: theme === 'dark' ? '#333' : '#fff',
      color: theme === 'dark' ? '#fff' : '#000'
    }}>
      Click me
    </button>
  )
}

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Button />
    </ThemeContext.Provider>
  )
}

The only change: import and function name.

Migration Checklist

✅ React 19 installed
✅ All useContext found
✅ Imports updated to use
✅ Function calls updated
✅ Components tested
✅ No console errors
✅ Team agreement

Common Pitfalls During Migration

Pitfall 1: Forgetting to Update Import


// ❌ Will break
import { useContext } from 'react'
const value = use(MyContext)

// ✅ Correct
import { use } from 'react'
const value = use(MyContext)

Pitfall 2: Using use() in Class Components


// ❌ Won't work
class MyComponent extends React.Component {
  render() {
    const value = use(ThemeContext)
    return ...
  }
}

// ✅ Correct - use function components
function MyComponent() {
  const value = use(ThemeContext)
  return ...
}

Pitfall 3: Not Testing After Migration

Always test:

  • Initial render with context
  • Context value changes
  • Multiple context consumers
  • Browser console

When NOT to Migrate

Reason 1: React 18 Codebase

use() only works in React 19+. If you're on React 18:

❌ Can't migrate yet
⚠️ Upgrade React first
✅ Then migrate

Reason 2: Large Team, No Consensus

If the team doesn't understand or agree:

❌ Don't force change
📚 Educate first
✅ Then decide together

Reason 3: No Clear Benefit

If useContext works fine:

❌ Don't change for change's sake
✅ Leave it alone
🚀 Focus on features

Migration Strategy: Gradual Approach

Option 1: New code only

Use use() in new components. Leave old components with useContext.

Option 2: Component by component

Migrate one component, test, commit. Then next.

Option 3: File by file

Migrate all components in a file together.

Option 4: Full migration

Migrate everything at once. Only if team is confident.

Final Advice

The Golden Rule:
"If useContext works, don't break it. Use use() for NEW code."

Prefer:

  • New features → use use()
  • Old code → leave useContext alone
  • Big refactors → discuss with team first
