React State with useState: The Component's Memory

Until now, our components could display information and respond to events, but they didn't remember anything.

If you clicked a button, the function executed... but the component didn't change visually.

To solve this, React gives us state (estado).

1️⃣ What is State in React?

State is the memory of a component.

It's where we store information that can change over time.

Examples of things that usually go in state:

  • If a button was clicked
  • The text of an input
  • A counter
  • Data loaded from an API
  • If a modal is open

2️⃣ Analogy: The Light Switch

Imagine a light switch:

off  →  on

When you press the switch:

  • its state changes
  • the light changes automatically

React works the same way.

state changes → React updates the UI

Example:

applied = false
  ↓
user clicks
  ↓
applied = true
  ↓
React updates the button

3️⃣ Problem Without State

Without state, a component doesn't remember anything.

function Button() {
  return <button>Click me</button>
}

This button will always be the same.

It can't remember if you clicked it.

4️⃣ Introducing useState

React has a Hook to handle state:

useState

Simple example:

function Button() {
  const [clicks, setClicks] = React.useState(0)

  return (
    <button onClick={() => setClicks(clicks + 1)}>
      Clicks: {clicks}
    </button>
  )
}

Now the button:

  • remembers the clicks
  • updates the number
  • React re-renders automatically

5️⃣ How useState Works

The syntax:

const [state, setState] = React.useState(initialValue)

Real example:

const [applied, setApplied] = React.useState(false)

Meaning:

React.useState(false)
  ↓
creates state with initial value

applied
↓
current state value

setApplied
↓
function to change the state

6️⃣ Why Do We Use Destructuring?

useState returns an array.

This:

const result = React.useState(false)
const applied = result[0]
const setApplied = result[1]

Simplifies with destructuring:

const [applied, setApplied] = React.useState(false)

Much cleaner.

7️⃣ Real Example: JobCard

Let's improve our job component.

We want the button to:

  • Change to "Applied"
  • Become disabled
  • Change style

Complete Component

function JobCard({ title, company, location, description }) {
  const [applied, setApplied] = React.useState(false)

  const handleApply = () => {
    setApplied(true)
  }

  return (
    <article className="job-listing-card">
      <div>
        <h3>{title}</h3>
        <small>
          {company} | {location}
        </small>
        <p>{description}</p>
      </div>

      <button
        className={applied ? "button-apply-job is-applied" : "button-apply-job"}
        onClick={handleApply}
        disabled={applied}
      >
        {applied ? "Applied!" : "Apply"}
      </button>

    </article>
  )
}

8️⃣ Complete React Flow

When the user clicks, this happens:

Initial state
applied = false
  ↓
User clicks
  ↓
handleApply()
  ↓
setApplied(true)
  ↓
React updates the state
  ↓
React re-renders the component
  ↓
The UI changes automatically

This is declarative React.

You don't manipulate the DOM manually.

9️⃣ Using State in JSX

State controls three things:

  • Style
  • Button state
  • Text
<button
  className={applied ? "button-apply-job is-applied" : "button-apply-job"}
  disabled={applied}
>
  {applied ? "Applied!" : "Apply"}
</button>

If applied === true:

  • text → "Applied!"
  • button → disabled
  • class → is-applied

🔟 The Ternary Operator

The ternary operator is like a short if.

condition ? valueIfTrue : valueIfFalse

Example:

applied ? "Applied!" : "Apply"

Equivalent to:

if (applied) {
  return "Applied!"
} else {
  return "Apply"
}

But JSX only allows expressions, not if statements.

That's why we use ternary operators.

1️⃣1️⃣ Before vs Now

Before (Vanilla JavaScript)

We manipulated the DOM manually.

jobsListingSection.addEventListener('click', function (event) {
  const element = event.target

  if (element.classList.contains('button-apply-job')) {
    element.textContent = 'Applied!'
    element.classList.add('is-applied')
    element.disabled = true
  }
})

Problems:

  • ❌ Manual DOM manipulation
  • ❌ Complex logic
  • ❌ Hard to maintain
  • ❌ Prone to errors

Now (React)

function JobCard() {
  const [applied, setApplied] = React.useState(false)

  return (
    <button
      onClick={() => setApplied(true)}
      disabled={applied}
    >
      {applied ? "Applied!" : "Apply"}
    </button>
  )
}

Advantages:

  • ✅ UI synchronized with state
  • ✅ Declarative code
  • ✅ Easy to test
  • ✅ Each component has its own state

1️⃣2️⃣ Each Component Has Its Own State

Example:

<JobCard title="Frontend Developer" />
<JobCard title="Backend Developer" />

React creates two instances of the component.

  • JobCard 1 → independent state
  • JobCard 2 → independent state

If you apply to the first one:

  • Frontend → applied
  • Backend → not applied

They don't affect each other.

1️⃣3️⃣ Types of State

You can store any type in state.

Boolean

const [visible, setVisible] = React.useState(false)

Number

const [count, setCount] = React.useState(0)

String

const [name, setName] = React.useState("")

Array

const [jobs, setJobs] = React.useState([])

Object

const [user, setUser] = React.useState({
  name: "Miguel",
  age: 25
})

1️⃣4️⃣ Multiple States

A component can have several states.

function JobCard({ title }) {
  const [applied, setApplied] = React.useState(false)
  const [saved, setSaved] = React.useState(false)
  const [likes, setLikes] = React.useState(0)

  return (
    <article>
      <h3>{title}</h3>

      <button onClick={() => setApplied(true)}>
        {applied ? "Applied" : "Apply"}
      </button>

      <button onClick={() => setSaved(!saved)}>
        {saved ? "❤️ Saved" : "🤍 Save"}
      </button>

      <button onClick={() => setLikes(likes + 1)}>
        👍 {likes}
      </button>
    </article>
  )
}

Each state works independently.

1️⃣5️⃣ Update State Based on Previous Value

Sometimes you need to use the previous value.

Safe way:

setCount(prevCount => prevCount + 1)

Example:

function Counter() {
  const [count, setCount] = React.useState(0)

  const increment = () => {
    setCount(prev => prev + 1)
  }

  return (
    <button onClick={increment}>
      {count}
    </button>
  )
}

This prevents errors when React batches updates.

1️⃣6️⃣ Rules of useState

1. Only at the Top Level

❌ Incorrect:

if (something) {
  const [state, setState] = React.useState(false)
}

2. Only in React Components

❌ Incorrect:

function myFunction() {
  const [state, setState] = React.useState(0)
}

3. Never Modify State Directly

❌ Incorrect:

applied = true

✅ Correct:

setApplied(true)

1️⃣7️⃣ Key Idea for Interviews

In interviews, React experts expect to hear:

"React UI is a function of state."
UI = f(state)

It means:

state changes
  ↓
React re-renders
  ↓
UI updates

📋 What You Must Remember

  • State = component's memory
  • When state changes:
    • React re-renders the component
    • The UI automatically updates
  • Key rules:
    • ✔ Use useState
    • ✔ Never modify state directly
    • ✔ Use setState
    • ✔ Each component has its own state

1️⃣9️⃣ The Fundamental React Equation

UI = f(state)

What this means:

  • The UI is a function of state
  • When state changes, UI changes automatically
  • No manual DOM manipulation needed

The Flow:

state changes
  ↓
React re-renders component
  ↓
new UI appears automatically

2️⃣0️⃣ Mental Model: What Should Be State?

Rule of thumb: If it changes over time, it's probably state.

Examples of State:

  • ✅ Counter (increments)
  • ✅ Input value (user types)
  • ✅ Modal visibility (open/closed)
  • ✅ User logged in (true/false)
  • ✅ Like count (increases)
  • ✅ Toggle switches

Examples of Props (NOT State):

  • ❌ Button label (fixed)
  • ❌ Component colors (fixed)
  • ❌ Initial data (static)

Example 4: Form with Multiple States


function SimpleForm() {
  const [name, setName] = React.useState("")
  const [email, setEmail] = React.useState("")
  const [submitted, setSubmitted] = React.useState(false)

  const handleSubmit = (event) => {
    event.preventDefault()
    setSubmitted(true)
  }

  if (submitted) {
    return (
      <div>
        <h2>Thank you, {name}!</h2>
        <p>We'll contact you at {email}</p>
        <button onClick={() => setSubmitted(false)}>
          Reset
        </button>
      </div>
    )
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Your name"
      />
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="Your email"
      />
      <button type="submit">Submit</button>
    </form>
  )
}

Example 5: Like Button


function LikeButton() {
  const [likes, setLikes] = React.useState(0)
  const [liked, setLiked] = React.useState(false)

  const handleLike = () => {
    if (!liked) {
      setLikes(likes + 1)
    } else {
      setLikes(likes - 1)
    }
    setLiked(!liked)
  }

  return (
    <button onClick={handleLike}>
      {liked ? "❤️" : "🤍"} {likes}
    </button>
  )
}

Example 6: List with Add/Remove


function TodoList() {
  const [todos, setTodos] = React.useState([])
  const [input, setInput] = React.useState("")

  const addTodo = () => {
    if (input.trim()) {
      setTodos([...todos, input])
      setInput("")
    }
  }

  const removeTodo = (index) => {
    setTodos(todos.filter((_, i) => i !== index))
  }

  return (
    <div>
      <input
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Add a todo"
      />
      <button onClick={addTodo}>Add</button>

      <ul>
        {todos.map((todo, index) => (
          <li key={index}>
            {todo}
            <button onClick={() => removeTodo(index)}>
              Delete
            </button>
          </li>
        ))}
      </ul>
    </div>
  )
}

Exercise 5: Multiple State Updates

Challenge: Create a component with multiple independent state values.

Solution:


function UserProfile() {
  const [name, setName] = React.useState("")
  const [age, setAge] = React.useState("")
  const [submitted, setSubmitted] = React.useState(false)

  const handleSubmit = () => {
    if (name && age) {
      setSubmitted(true)
    }
  }

  if (submitted) {
    return (
      <div>
        <h2>Profile: {name}, Age {age}</h2>
        <button onClick={() => setSubmitted(false)}>
          Edit
        </button>
      </div>
    )
  }

  return (
    <div>
      <input
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Name"
      />
      <input
        value={age}
        onChange={(e) => setAge(e.target.value)}
        placeholder="Age"
      />
      <button onClick={handleSubmit}>Save</button>
    </div>
  )
}
