Custom Hooks II: Creating a Hook for a Form

😤 The problem: a component with too many responsibilities

We have a main component that handles all the search logic, and that makes it too complex. The component mixes:

  • handleSubmit
  • handleChange / handleTextChange
  • Search logic (search)
  • IDs like technology, location, experience, etc.

All together this makes the component hard to read and maintain. Let's separate concerns using a custom hook.

Warning sign: when a component has too many responsibilities, it's a sign we need to refactor.

✨ The solution: useSearchForm

We'll create a custom hook called useSearchForm that encapsulates all the form logic. Unlike hooks such as useRouter (seen earlier), custom hooks can accept parameters like regular functions.

Parameters the hook needs

  • onSearch: function executed when searching
  • onTextFilter: function to filter by text
  • IDs needed for the filters:
    • IDTechnology
    • IDLocation
    • IDExperienceLevel

🎯 Advantages of the refactor

  • ✅ Cleaner code: the component focuses on rendering
  • ✅ Reusable logic: the hook can be used in other components
  • ✅ Easier to test: we can test the hook in isolation
  • ✅ Better organization: clear separation of concerns

Step 2 — Add internal state

The search text lives inside the hook:

function useSearchForm({ onSearch, onTextFilter, IDTechnology, IDLocation, IDExperienceLevel }) {
  const [searchText, setSearchText] = useState('')

  // This state:
  // - holds the text typed by the user
  // - is updated in handleChange
  // - is returned to the component to display in the input
}

Step 3 — Implement the handlers

function useSearchForm({ onSearch, onTextFilter, IDTechnology, IDLocation, IDExperienceLevel }) {
  const [searchText, setSearchText] = useState('')

  const handleSubmit = (event) => {
    event.preventDefault()

    const formData = new FormData(event.target)

    const filters = {
      text: formData.get('search'),
      technology: formData.get('technology'),
      location: formData.get('location'),
      experience: formData.get('experience'),
    }

    onSearch(filters)
  }

  const handleChange = (event) => {
    const text = event.target.value
    setSearchText(text)
    onTextFilter(text)
  }

  // Return what the component needs
  return {
    searchText,
    handleSubmit,
    handleChange,
  }
}

Step 4 — Use the hook in the component

The component gets much simpler:

function SearchComponent() {
  const { searchText, handleSubmit, handleChange } = useSearchForm({
    onSearch: performSearch,
    onTextFilter: filterByText,
    IDTechnology: 'tech',
    IDLocation: 'loc',
    IDExperienceLevel: 'exp',
  })

  return (
    <form onSubmit={handleSubmit}>
      <input value={searchText} onChange={handleChange} name="search" />
      {/* Rest of the form */}
    </form>
  )
}

Result: the component no longer implements the logic, it just consumes the functions and values from the hook.

📏 Important custom hook rules

1. Call only at the top level

Custom hooks must be called only at the top level of a component or another custom hook.

❌ NOT inside:

  • Callbacks (handleClick, handleSubmit, etc.)
  • Conditionals (if)
  • Loops (for, while)
  • useEffect or other hooks
// ❌ BAD - inside a callback
function Component() {
  const handleClick = () => {
    const { data } = useSearchForm() // ERROR
  }
}

// ❌ BAD - inside a conditional
function Component() {
  if (condition) {
    const { data } = useSearchForm() // ERROR
  }
}

// ✅ GOOD - top level
function Component() {
  const { searchText, handleSubmit } = useSearchForm()

  const handleClick = () => {
    // Using the hook values here is fine
    console.log(searchText)
  }
}

2. Must start with the use prefix

Like useState, useEffect, etc. This is not just a convention: React relies on that name to enforce internal rules. 

// ✅ GOOD
function useSearchForm() {}
function useUserData() {}
function useAuthentication() {}

// ❌ BAD
function searchFormHook() {}  // Doesn't start with "use"
function getSearchForm() {}   // Not a hook

3. Only usable in components or custom hooks

// ❌ BAD - regular function
function calculateTotal() {
  const { data } = useData() // ERROR
  return data.total
}

// ✅ GOOD - component
function Component() {
  const { data } = useData()
  return <div>{data.total}</div>
}

// ✅ GOOD - another custom hook
function useCalculateTotal() {
  const { data } = useData()
  return data.total
}

🔑 Conclusion

Extracting logic to custom hooks is an essential practice in modern React:

  • The component focuses on rendering
  • Logic lives in a reusable place
  • Development becomes more scalable

This technique keeps your code ✅ clean, ✅ modular, ✅ easy to test, and ✅ maintainable long-term.
