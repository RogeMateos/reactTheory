Custom Hooks: Build Your Own React Superpowers

🎯 What Is a Custom Hook?

A custom hook is a JavaScript function that uses React's built-in hooks to solve a specific problem.

It's a way to extract and reuse stateful logic across components.

🚨 The Problem: Code Repetition

Let me show you the problem custom hooks solve:

Without custom hooks:

// Component 1
function LoginPage() {
  const user = React.useContext(AuthContext)

  if (!user) return <p>Not logged in</p>
  return <p>Welcome, {user.name}!</p>
}

// Component 2
function Dashboard() {
  const user = React.useContext(AuthContext)

  if (!user) return <p>Not logged in</p>
  return <p>Hello, {user.name}!</p>
}

// Component 3
function Settings() {
  const user = React.useContext(AuthContext)

  if (!user) return <p>Not logged in</p>
  return <p>Settings for {user.name}</p>
}

What's the problem?

  • ❌ We're repeating `React.useContext(AuthContext)` everywhere
  • ❌ We're repeating the same check 3 times
  • ❌ If the logic changes, we have to update 3 places
  • ❌ Harder to test
  • ❌ More error-prone

💡 The Solution: Custom Hooks

Create a custom hook that wraps this logic:

function useAuth() {
  const user = React.useContext(AuthContext)
  return user
}

That's it! Now use it everywhere:

// Component 1
function LoginPage() {
  const user = useAuth()

  if (!user) return <p>Not logged in</p>
  return <p>Welcome, {user.name}!</p>
}

// Component 2
function Dashboard() {
  const user = useAuth()

  if (!user) return <p>Not logged in</p>
  return <p>Hello, {user.name}!</p>
}

// Component 3
function Settings() {
  const user = useAuth()

  if (!user) return <p>Not logged in</p>
  return <p>Settings for {user.name}</p>
}

Benefits:

  • ✅ Single line: `const user = useAuth()`
  • ✅ Cleaner components
  • ✅ Logic in ONE place (easy to change)
  • ✅ Reusable across all components
  • ✅ Easy to test

🔥 Custom Hook Naming Convention

Custom hooks must start with "use":

✅ useAuth()
✅ useTheme()
✅ useCart()
✅ useFetch()
✅ useForm()
✅ useCounter()

❌ getAuth() (doesn't start with use)
❌ authHook() (doesn't start with use)

This naming convention tells React that this function uses hooks.

📋 The Professional Pattern

Important rule:

Whenever you create a Context, always create a custom hook to access it.

This pattern prevents mistakes and makes your code cleaner:

// Create Context
const AuthContext = React.createContext(null)

// Create Provider
export function AuthProvider({ children }) {
  const [user, setUser] = React.useState(null)

  return (
    <AuthContext.Provider value={{ user, setUser }}>
      {children}
    </AuthContext.Provider>
  )
}

// Create Custom Hook
export function useAuth() {
  const context = React.useContext(AuthContext)

  if (!context) {
    throw new Error('useAuth must be used inside AuthProvider')
  }

  return context
}

🎯 When to Create a Custom Hook

Create a custom hook when:

  • ✅ You repeat the same hook logic in multiple components
  • ✅ You want to extract Context access into a function
  • ✅ You combine multiple hooks and want to reuse that combination
  • ✅ You want to hide complex logic behind a simple function
  • ✅ You want to share stateful logic without components

Don't create a custom hook when:

  • ❌ It's only used in one component (keep it simple)
  • ❌ It's a simple one-liner (like `const user = useContext(AuthContext)`)
  • ❌ The logic is specific to a single component

🚀 Benefits of Custom Hooks

1. Abstraction

Hide complex logic behind a simple name:

// Complex logic hidden
const { user, logout } = useAuth()

// Instead of
const context = useContext(AuthContext)
const user = context?.user
const logout = () => {
  // ... complex logout logic ...
}

2. Clear Error Messages

Your hook can throw meaningful errors:

export function useAuth() {
  const context = React.useContext(AuthContext)

  if (!context) {
    throw new Error('useAuth must be used inside <AuthProvider>')
  }

  return context
}

This prevents confusing "context is undefined" errors.

3. Easy to Change Implementation

If auth logic changes, update it in ONE place:

// Before: using Context
export function useAuth() {
  return React.useContext(AuthContext)
}

// Later: switch to Redux without changing component code
export function useAuth() {
  return useSelector(state => state.auth)
}

4. Easier Testing

Test the hook in isolation from components:

// You can mock and test useAuth() separately
const mockUser = { id: 1, name: 'John' }

// In your test
const { result } = renderHook(() => useAuth())

🔗 Connection to Context

Custom hooks and Context go together:

Context (createContext, Provider)
  ↓
  Provides data globally
  ↓
Custom Hook (useAuth)
  ↓
  Access data from Context
  ↓
Components
  ↓
  Use the custom hook

Context is the storage.

Custom hooks are the access layer.

⚙️ Rules of Custom Hooks

Rule 1: Name Must Start with "use"

✅ useAuth()
❌ getAuth()

Rule 2: Can Only Call Hooks at Top Level

// ✅ Correct
function useAuth() {
  const context = React.useContext(AuthContext)
  return context
}

// ❌ Incorrect - hook inside if
function useAuth() {
  if (something) {
    const context = React.useContext(AuthContext)  // BAD!
  }
  return context
}

Rule 3: Custom Hooks Can Call Other Hooks

// This is fine!
function useUser() {
  const auth = useAuth()           // Call custom hook
  const theme = useTheme()          // Call another custom hook
  const [loading, setLoading] = React.useState(false)  // Call built-in hook

  return { auth, theme, loading }
}

📝 Summary

  • Custom hooks extract and reuse stateful logic
  • They're JavaScript functions that use React hooks
  • They must start with "use"
  • Always create a custom hook for each Context
  • They make code cleaner, reusable, and testable
  • They hide complex logic behind simple functions

Complete Code Examples

Example 1: Simple useAuth Hook


// Step 1: Create Context
const AuthContext = React.createContext(null)

// Step 2: Create Provider
function AuthProvider({ children }) {
  const [user, setUser] = React.useState(null)

  const login = (userData) => setUser(userData)
  const logout = () => setUser(null)

  return (
    <AuthContext.Provider value={{ user, login, logout }}>
      {children}
    </AuthContext.Provider>
  )
}

// Step 3: Create Custom Hook
function useAuth() {
  const context = React.useContext(AuthContext)

  if (!context) {
    throw new Error('useAuth must be used inside AuthProvider')
  }

  return context
}

// Step 4: Use the hook in components
function LoginPage() {
  const { user, login, logout } = useAuth()

  if (!user) {
    return (
      <button onClick={() => login({ id: 1, name: 'John' })}>
        Login
      </button>
    )
  }

  return (
    <div>
      <p>Welcome, {user.name}!</p>
      <button onClick={logout}>Logout</button>
    </div>
  )
}

Example 2: useTheme Hook


// Create Context
const ThemeContext = React.createContext(null)

// Create Provider
function ThemeProvider({ children }) {
  const [theme, setTheme] = React.useState('light')

  const toggleTheme = () => {
    setTheme(prev => prev === 'light' ? 'dark' : 'light')
  }

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}

// Create Custom Hook
function useTheme() {
  const context = React.useContext(ThemeContext)

  if (!context) {
    throw new Error('useTheme must be used inside ThemeProvider')
  }

  return context
}

// Use in components
function App() {
  const { theme, toggleTheme } = useTheme()

  const bgColor = theme === 'light' ? '#fff' : '#222'
  const textColor = theme === 'light' ? '#000' : '#fff'

  return (
    <div style={{ background: bgColor, color: textColor, padding: '20px' }}>
      <p>Current theme: {theme}</p>
      <button onClick={toggleTheme}>Toggle Theme</button>
    </div>
  )
}

Example 3: useForm Hook


function useForm(initialValues, onSubmit) {
  const [values, setValues] = React.useState(initialValues)

  const handleChange = (e) => {
    const { name, value } = e.target
    setValues(prev => ({ ...prev, [name]: value }))
  }

  const handleSubmit = (e) => {
    e.preventDefault()
    onSubmit(values)
  }

  return { values, handleChange, handleSubmit }
}

// Use in a component
function LoginForm() {
  const handleSubmit = (values) => {
    console.log('Form submitted:', values)
  }

  const { values, handleChange, handleSubmit: onSubmit } = useForm(
    { email: '', password: '' },
    handleSubmit
  )

  return (
    <form onSubmit={onSubmit}>
      <input
        name="email"
        type="email"
        value={values.email}
        onChange={handleChange}
        placeholder="Email"
      />
      <input
        name="password"
        type="password"
        value={values.password}
        onChange={handleChange}
        placeholder="Password"
      />
      <button type="submit">Login</button>
    </form>
  )
}

Example 4: useCounter Hook (Combining Multiple Hooks)


function useCounter(initialValue = 0) {
  const [count, setCount] = React.useState(initialValue)

  const increment = () => setCount(prev => prev + 1)
  const decrement = () => setCount(prev => prev - 1)
  const reset = () => setCount(initialValue)

  return { count, increment, decrement, reset }
}

// Use in components
function Counter() {
  const { count, increment, decrement, reset } = useCounter(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={increment}>+</button>
      <button onClick={decrement}>-</button>
      <button onClick={reset}>Reset</button>
    </div>
  )
}

// Can reuse anywhere
function ScoreBoard() {
  const { count: score, increment } = useCounter(0)

  return (
    <div>
      <p>Score: {score}</p>
      <button onClick={increment}>Add Point</button>
    </div>
  )
}

Example 5: useLocalStorage Hook


function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = React.useState(() => {
    try {
      const item = window.localStorage.getItem(key)
      return item ? JSON.parse(item) : initialValue
    } catch (error) {
      console.error(error)
      return initialValue
    }
  })

  const setValue = (value) => {
    try {
      const valueToStore = value instanceof Function ? value(storedValue) : value
      setStoredValue(valueToStore)
      window.localStorage.setItem(key, JSON.stringify(valueToStore))
    } catch (error) {
      console.error(error)
    }
  }

  return [storedValue, setValue]
}

// Use in components
function UserPreferences() {
  const [preferences, setPreferences] = useLocalStorage('userPrefs', {
    theme: 'light',
    fontSize: 16
  })

  const updateTheme = (newTheme) => {
    setPreferences(prev => ({ ...prev, theme: newTheme }))
  }

  return (
    <div>
      <p>Theme: {preferences.theme}</p>
      <button onClick={() => updateTheme('dark')}>
        Set Dark Theme
      </button>
    </div>
  )
}

Example 6: useFetch Hook


function useFetch(url) {
  const [data, setData] = React.useState(null)
  const [loading, setLoading] = React.useState(true)
  const [error, setError] = React.useState(null)

  React.useEffect(() => {
    let isMounted = true

    const fetchData = async () => {
      try {
        setLoading(true)
        const response = await fetch(url)
        if (!response.ok) throw new Error('Failed to fetch')
        const result = await response.json()
        if (isMounted) {
          setData(result)
          setError(null)
        }
      } catch (err) {
        if (isMounted) {
          setError(err.message)
          setData(null)
        }
      } finally {
        if (isMounted) setLoading(false)
      }
    }

    fetchData()

    return () => {
      isMounted = false
    }
  }, [url])

  return { data, loading, error }
}

// Use in components
function UsersList() {
  const { data: users, loading, error } = useFetch('/api/users')

  if (loading) return <p>Loading users...</p>
  if (error) return <p>Error: {error}</p>

  return (
    <ul>
      {users?.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  )
}

Visual Analogy: Building Reception (Custom Hooks)

Scenario: A Growing Company

Imagine a company with many employees needing to access the CEO's office.

❌ The Problem: Direct Access (No Custom Hook)

Employee 1    Employee 2    Employee 3
    ↓              ↓              ↓
    Walks to CEO office
    ↓              ↓              ↓
    Talks directly to CEO
    ↓              ↓              ↓
    Gets information
    ↓              ↓              ↓
    Returns to desk

Problems:

  • ❌ Everyone walks to the CEO office (repetition)
  • ❌ CEO is interrupted constantly
  • ❌ If office location changes, all employees get lost
  • ❌ No verification that employees should be there
  • ❌ Inefficient and chaotic

✅ The Solution: Reception Desk (Custom Hook)

Employee 1    Employee 2    Employee 3
    ↓              ↓              ↓
    Asks receptionist
    ↓              ↓              ↓
    Reception validates and provides info
    ↓              ↓              ↓
    Returns with information
    ↓              ↓              ↓
    Back to work immediately

Benefits:

  • ✅ Single point of access (one reception desk)
  • ✅ CEO not interrupted (centralized access)
  • ✅ If office moves, only reception desk changes
  • ✅ Reception validates employees (access control)
  • ✅ Organized and efficient

Mapping to React Code

CEO Office        = AuthContext (global data)
Reception Desk    = useAuth() hook (access point)
Employees         = React components
Asking Reception  = Calling useAuth() in components

Why This Matters

Easy to maintain:

If you change how auth works:
- Update useAuth() once
- All 100 components automatically use new logic
- No component code changes

Error handling:

// Reception desk checks ID
function useAuth() {
  const context = React.useContext(AuthContext)

  if (!context) {
    // THROW ERROR if employee isn't authorized
    throw new Error('useAuth must be inside AuthProvider')
  }

  return context
}

Prevents mistakes:

// Without custom hook: might forget to check context

// With custom hook: guaranteed to have context
const { user } = useAuth()  // Always valid or throws error
