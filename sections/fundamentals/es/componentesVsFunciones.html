Componentes vs Funciones Normales: Entender la Diferencia

Cuando empezamos con React es muy común pensar:

"Si los componentes son funciones… ¿entonces cualquier función es un componente?"

La respuesta es no.

Aunque los componentes son funciones de JavaScript, no todas las funciones son componentes.

Entender esta diferencia es clave para escribir React correctamente.

1️⃣ ¿Qué es un componente en React?

Un componente de React es una función especial que React usa para renderizar interfaz (UI).

Un componente:

  • Retorna algo que React puede mostrar en pantalla
  • Se escribe en PascalCase
  • Se usa como una etiqueta JSX

Ejemplo

function Saludo() {
  return <h1>¡Hola desde React!</h1>
}

Usar el componente:

<Saludo />

React ejecutará la función y renderizará:

<h1>¡Hola desde React!</h1>

Analogía sencilla: La Impresora

Imagina React como una impresora de interfaz.

Los componentes son como plantillas.

  • Componente → plantilla de UI
  • React → impresora
  • Resultado → HTML renderizado

Ejemplo mental:

<Saludo />
  ↓
function Saludo()
  ↓
<h1>Hola</h1>

Otra Analogía: El Semáforo

Piensa en Componentes y Funciones Normales como un sistema de semáforo:

  • 🟢 Componentes (Luz verde) → Controlan lo que VES en pantalla
    • Renderan UI
    • Están en PascalCase
    • Son la parte "visual"
  • 🟡 Funciones Normales (Luz amarilla) → Ayudan con la lógica DETRÁS de escenas
    • Hacen cálculos
    • Están en camelCase
    • Apoyan a los componentes

El semáforo te dice:

🟢 Verde = "Esto es UI" (Componente)
🟡 Amarillo = "Esto es lógica" (Función Normal)

2️⃣ ¿Qué es una función normal?

Una función normal de JavaScript es simplemente una función que ejecuta lógica.

No está diseñada para renderizar UI.

Ejemplo

function calcularTotal(precio, iva) {
  return precio + precio * iva
}

const total = calcularTotal(100, 0.21)

Esta función:

  • No devuelve JSX
  • No se usa como etiqueta
  • Solo ejecuta lógica

Diferencia clave

Tipo Propósito
Componente Renderizar UI
Función normal Ejecutar lógica

3️⃣ Cómo se invocan

Esta es una de las diferencias más importantes.

Componentes

Se usan como etiquetas JSX.

function Boton() {
  return <button>Haz clic</button>
}

<Boton />

React internamente hace algo parecido a:

Boton()

Pero gestiona el ciclo de renderizado.

Funciones normales

Se llaman con paréntesis.

function saludar(nombre) {
  return `Hola ${nombre}`
}

saludar("Miguel")

4️⃣ Lo que retornan

Componentes

Devuelven cosas que React puede renderizar.

function Titulo() {
  return <h1>Mi título</h1>
}

React lo convierte en HTML.

Funciones normales

Pueden devolver cualquier cosa.

function sumar(a, b) {
  return a + b
}

function obtenerUsuario() {
  return { nombre: "Miguel", edad: 25 }
}

5️⃣ Convención de nombres

React usa una regla muy importante.

Componentes → PascalCase

function MiComponente() {}
function BotonPrimario() {}
function TarjetaProducto() {}

Funciones normales → camelCase

function calcularPrecio() {}
function validarEmail() {}
function obtenerFecha() {}

6️⃣ ¿Por qué PascalCase es obligatorio?

React necesita distinguir entre:

1️⃣ HTML nativo

<div>
<button>
<h1>

2️⃣ Componentes personalizados

<MiBoton>
<App>
<Producto>

Ejemplo

<MiBoton />

React entiende: → Es un componente personalizado

Si usas minúscula:

<miboton />

React piensa: → Es una etiqueta HTML

Y eso no funciona correctamente.

7️⃣ Qué puede renderizar React

Un componente puede devolver varios tipos de valores.

JSX

function App() {
  return <h1>Hola mundo</h1>
}

Strings

function Saludo() {
  return "Hola"
}

Números

function Contador() {
  return 42
}

null o undefined

No renderizan nada.

function Invisible() {
  return null
}

Esto se usa mucho para renderizado condicional.

8️⃣ Mezclando componentes y funciones

En proyectos reales siempre se mezclan.

Las funciones normales ayudan a los componentes.

Ejemplo

function formatearPrecio(precio) {
  return `${precio}€`
}

function Producto({ nombre, precio }) {
  const precioFormateado = formatearPrecio(precio)

  return (
    <div>
      <h3>{nombre}</h3>
      <p>{precioFormateado}</p>
    </div>
  )
}

Aquí tenemos:

  • Producto → componente
  • formatearPrecio → función helper

9️⃣ Cuándo usar cada uno

Usa componentes cuando

Necesitas UI.

Ejemplos:

  • Tarjetas
  • Botones
  • Layouts
  • Listas
  • Formularios
function Tarjeta({ titulo, descripcion }) {
  return (
    <div className="card">
      <h2>{titulo}</h2>
      <p>{descripcion}</p>
    </div>
  )
}

Usa funciones normales cuando

Necesitas lógica.

Ejemplos:

  • Formatear precio
  • Validar email
  • Calcular totales
  • Transformar datos
function validarEmail(email) {
  return email.includes("@")
}

🔟 Errores comunes de principiantes

Error 1: Llamar un componente como función

❌ Incorrecto:

function Saludo() {
  return <h1>Hola</h1>
}

Saludo()

✅ Correcto:

<Saludo />

Error 2: Usar minúscula

❌ Incorrecto:

function saludo() {
  return <h1>Hola</h1>
}

✅ Correcto:

function Saludo() {
  return <h1>Hola</h1>
}

Error 3: Intentar renderizar objetos

❌ Incorrecto:

function Usuario() {
  return { nombre: "Miguel" }
}

React no puede renderizar objetos.

✅ Correcto:

function Usuario() {
  const usuario = { nombre: "Miguel", edad: 25 }
  return (
    <div>
      {usuario.nombre} - {usuario.edad}
    </div>
  )
}

📋 Resumen final

Concepto Componente Función normal
Uso Renderizar UI Ejecutar lógica
Cómo se usa <Componente /> funcion()
Nombre PascalCase camelCase
Retorna JSX o valores renderizables Cualquier dato JS

💡 Idea clave que debes recordar

Componentes → construyen la interfaz
Funciones → ayudan con la lógica

React está construido alrededor de componer componentes pequeños.

Ejemplos de Código Completos

Ejemplo 1: Componente vs Función lado a lado


// COMPONENTE - renderiza UI
function Bienvenida() {
  return <h1>¡Bienvenido!</h1>
}

// FUNCIÓN NORMAL - ejecuta lógica
function calcularDescuento(precio, porcentaje) {
  return precio * (1 - porcentaje / 100)
}

// Uso
export default function App() {
  const precioConDescuento = calcularDescuento(100, 20)

  return (
    <div>
      <Bienvenida />
      <p>Precio: ${precioConDescuento}</p>
    </div>
  )
}

Ejemplo 2: Producto con función helper


// Función helper (función normal)
function formatearMoneda(valor) {
  return new Intl.NumberFormat('es-ES', {
    style: 'currency',
    currency: 'EUR'
  }).format(valor)
}

function aplicarIVA(precio, tasaIVA) {
  return precio + (precio * tasaIVA)
}

// Componente
function TarjetaProducto({ nombre, precioBase, tasaIVA }) {
  const precioTotal = aplicarIVA(precioBase, tasaIVA)
  const precioFormateado = formatearMoneda(precioTotal)

  return (
    <div style={{
      border: '1px solid #ddd',
      padding: '16px',
      borderRadius: '8px'
    }}>
      <h3>{nombre}</h3>
      <p style={{ fontSize: '20px', fontWeight: 'bold' }}>
        {precioFormateado}
      </p>
      <button>Añadir al carrito</button>
    </div>
  )
}

// Uso
<TarjetaProducto nombre="Laptop" precioBase={999} tasaIVA={0.21} />

Ejemplo 3: Validación de formulario


// Funciones helper (funciones normales)
function validarEmail(email) {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)
}

function validarContraseña(contraseña) {
  return contraseña.length >= 8
}

// Componente
function FormularioLogin() {
  const [email, setEmail] = React.useState('')
  const [contraseña, setContraseña] = React.useState('')
  const [errores, setErrores] = React.useState({})

  const handleSubmit = (e) => {
    e.preventDefault()
    const nuevosErrores = {}

    if (!validarEmail(email)) {
      nuevosErrores.email = 'Email inválido'
    }

    if (!validarContraseña(contraseña)) {
      nuevosErrores.contraseña = 'La contraseña debe tener al menos 8 caracteres'
    }

    if (Object.keys(nuevosErrores).length > 0) {
      setErrores(nuevosErrores)
    } else {
      console.log('¡Formulario enviado!')
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <input
          type="email"
          value={email}
          onChange={(e) => setEmail(e.target.value)}
          placeholder="Email"
        />
        {errores.email && <p style={{ color: 'red' }}>{errores.email}</p>}
      </div>

      <div>
        <input
          type="password"
          value={contraseña}
          onChange={(e) => setContraseña(e.target.value)}
          placeholder="Contraseña"
        />
        {errores.contraseña && <p style={{ color: 'red' }}>{errores.contraseña}</p>}
      </div>

      <button type="submit">Ingresar</button>
    </form>
  )
}

Ejemplo 4: Transformación de datos


// Funciones helper para transformar datos
function filtrarPorPrecio(productos, precioMaximo) {
  return productos.filter(p => p.precio <= precioMaximo)
}

function ordenarPorNombre(productos) {
  return [...productos].sort((a, b) => a.nombre.localeCompare(b.nombre))
}

// Componente
function ListaProductos({ productos, precioMaximo = Infinity }) {
  const filtrados = filtrarPorPrecio(productos, precioMaximo)
  const ordenados = ordenarPorNombre(filtrados)

  return (
    <div>
      <h2>Productos</h2>
      <ul>
        {ordenados.map(producto => (
          <li key={producto.id}>
            {producto.nombre} - ${producto.precio}
          </li>
        ))}
      </ul>
    </div>
  )
}

// Uso
const productos = [
  { id: 1, nombre: 'Laptop', precio: 1000 },
  { id: 2, nombre: 'Ratón', precio: 25 },
  { id: 3, nombre: 'Teclado', precio: 75 }
]

<ListaProductos productos={productos} precioMaximo={500} />

Ejercicios Prácticos

Ejercicio 1: Identifica componentes vs funciones

Desafío: Mira el siguiente código e identifica cuáles son componentes y cuáles son funciones normales. Explica por qué.


// 1
function calcularEdad(añoNacimiento) {
  return new Date().getFullYear() - añoNacimiento
}

// 2
function PerfilUsuario({ nombre, añoNacimiento }) {
  const edad = calcularEdad(añoNacimiento)
  return (
    <div>
      <h3>{nombre}</h3>
      <p>Edad: {edad}</p>
    </div>
  )
}

// 3
function obtenerInicialeS(nombre) {
  return nombre
    .split(' ')
    .map(palabra => palabra[0])
    .join('')
}

// 4
function Avatar({ iniciales }) {
  return <div className="avatar">{iniciales}</div>
}

Solución:

  • calcularEdad → Función normal (calcula lógica)
  • PerfilUsuario → Componente (renderiza JSX)
  • obtenerInicialeS → Función normal (transforma datos)
  • Avatar → Componente (renderiza JSX)

Ejercicio 2: Crea un componente con funciones helper

Desafío: Crea un componente llamado TarjetaEstudiante que:

  • Reciba props: nombre, calificación (0-100)
  • Use una función helper para convertir la calificación a letra (A, B, C, D, F)
  • Use otra función helper para determinar si el estudiante pasó (calificación >= 60)
  • Muestre la información en una tarjeta estilizada

Solución:


function calificacionALetra(calificación) {
  if (calificación >= 90) return 'A'
  if (calificación >= 80) return 'B'
  if (calificación >= 70) return 'C'
  if (calificación >= 60) return 'D'
  return 'F'
}

function aprobo(calificación) {
  return calificación >= 60
}

function TarjetaEstudiante({ nombre, calificación }) {
  const letra = calificacionALetra(calificación)
  const paso = aprobo(calificación)

  return (
    <div style={{
      border: '1px solid #ddd',
      padding: '16px',
      borderRadius: '8px',
      backgroundColor: paso ? '#e8f5e9' : '#ffebee'
    }}>
      <h3>{nombre}</h3>
      <p>Calificación: {calificación} ({letra})</p>
      <p>Estado: {paso ? '✅ Aprobó' : '❌ Reprobó'}</p>
    </div>
  )
}

// Uso
<TarjetaEstudiante nombre="María" calificación={85} />
<TarjetaEstudiante nombre="Juan" calificación={55} />

Ejercicio 3: Construye una app completa

Desafío: Crea una app con:

  • Una función normal para calcular costo de envío
  • Una función normal para calcular total con impuesto
  • Un componente para mostrar el resumen del pedido

Solución:


// Funciones helper
function calcularEnvio(peso) {
  return peso * 2.5 // €2.50 por kilo
}

function calcularTotal(subtotal, tasaIVA = 0.21) {
  return subtotal + (subtotal * tasaIVA)
}

// Componente
function ResumenPedido({ items, peso }) {
  const subtotal = items.reduce((suma, item) => suma + item.precio, 0)
  const envio = calcularEnvio(peso)
  const total = calcularTotal(subtotal + envio)

  return (
    <div style={{
      border: '1px solid #ddd',
      padding: '20px',
      maxWidth: '400px'
    }}>
      <h2>Resumen del Pedido</h2>

      <h3>Productos:</h3>
      <ul>
        {items.map((item, idx) => (
          <li key={idx}>{item.nombre} - ${item.precio}</li>
        ))}
      </ul>

      <hr />
      <p>Subtotal: ${subtotal.toFixed(2)}</p>
      <p>Envío: ${envio.toFixed(2)}</p>
      <p style={{ fontSize: '18px', fontWeight: 'bold' }}>
        Total: ${total.toFixed(2)}
      </p>
    </div>
  )
}

// Uso
const items = [
  { nombre: 'Laptop', precio: 999 },
  { nombre: 'Ratón', precio: 25 }
]

<ResumenPedido items={items} peso={5} />
