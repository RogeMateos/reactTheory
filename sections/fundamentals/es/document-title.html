Gestión del <title> en React

🐛 El enfoque habitual (y sus problemas)

Es muy común ver código que actualiza el título de la página con useEffect: 

// Funciona, pero no es la forma más limpia
useEffect(() => {
  document.title = 'Mi aplicación React'
}, [])

Manipular el DOM directamente desde un efecto rompe con la naturaleza declarativa de React. Existe una alternativa más elegante.

✅ La solución: usar <title> directamente en JSX

React permite usar etiquetas de SEO como <title> o <meta> directamente en el JSX. No hace falta manipular el DOM manualmente: 

function App() {
  return (
    <>
      <title>Mi aplicación React</title>
      <div>{/* Resto de tu aplicación */}</div>
    </>
  )
}
Ventaja Descripción
✅ Declarativo El título es parte del render, no un efecto secundario
✅ Menos código No necesitás un hook adicional
✅ Más React Todo está en el JSX, donde corresponde
✅ Automático Se actualiza con cada render sin código extra

⚠️ El problema oculto: duplicación del <title>

Al implementar esta solución, si inspeccionás el DOM en DevTools vas a ver dos etiquetas <title> en el <head>.

¿Por qué? Porque:

  • React renderiza su propio <title> desde el componente
  • Pero en el index.html ya existe otro <title> por defecto

Advertencia oficial de React: Solo deberías renderizar un único <title> a la vez. Si más de un componente renderiza un <title> simultáneamente, React coloca todos en el <head> y el comportamiento de los navegadores y motores de búsqueda queda indefinido.

🔧 Solución: eliminar el <title> del index.html

Eliminá la etiqueta <title> del HTML estático y dejá que React controle el título completamente: 

<!-- ❌ Antes (index.html) -->
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Vite + React</title>  <!-- eliminar esta línea -->
</head>

<!-- ✅ Después (index.html) -->
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <!-- sin título aquí — React lo gestiona -->
</head>

🔄 Título dinámico según el estado

La gran ventaja de gestionar el título desde React es que puede depender del estado de la aplicación. El título de la pestaña se actualiza en tiempo real: 

function App() {
  const [jobs, setJobs] = useState([])
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)

  return (
    <>
      <title>
        {loading
          ? 'Cargando...'
          : error
          ? 'Error al cargar'
          : `${jobs.length} empleos encontrados`}
      </title>
      <div>{/* Resto de la aplicación */}</div>
    </>
  )
}

Importante sobre interpolación: Los hijos de <title> deben ser una cadena única de texto. Usá template literals (`${jobs.length} empleos`) para generar una sola cadena. Si usaras {jobs.length} empleos sin backticks, estarías pasando un array de dos elementos (número + texto), lo que causa un error.

📋 Casos de uso del título dinámico

// El título de la pestaña muestra en tiempo real:

"Cargando…"             ← mientras se obtienen los datos
"Error al cargar"       ← si ocurre un problema
"25 empleos encontrados"← cuando los datos llegan
"Sin resultados"        ← si la búsqueda no devuelve nada

🏗️ Ejemplo completo con función getTitle

function JobsApp() {
  const [jobs, setJobs]     = useState([])
  const [loading, setLoading] = useState(true)
  const [error, setError]   = useState(null)

  useEffect(() => {
    fetch('https://jscamp-api.vercel.app/api/jobs')
      .then((res) => res.json())
      .then((data) => {
        setJobs(data.data)
        setLoading(false)
      })
      .catch((err) => {
        setError(err)
        setLoading(false)
      })
  }, [])

  const getTitle = () => {
    if (loading)          return 'Cargando empleos...'
    if (error)            return 'Error al cargar empleos'
    if (jobs.length === 0) return 'No se encontraron empleos'
    return `${jobs.length} empleos disponibles - JSCamp`
  }

  return (
    <>
      <title>{getTitle()}</title>
      <main>
        {loading && <p>Cargando...</p>}
        {error   && <p>Error: {error.message}</p>}
        {!loading && !error && <JobsList jobs={jobs} />}
      </main>
    </>
  )
}

🚀 Ventajas del título dinámico

Ventaja Detalle
Feedback instantáneo El usuario ve el progreso en la pestaña aunque no esté mirando
Mejor UX Las apps modernas funcionan así
SEO amigable Los crawlers ven el título correcto en cada estado
Mantenible Toda la lógica del título vive en el componente, no dispersa

⚠️ Casos donde <title> NO va al <head>

Según la documentación oficial de React, hay dos excepciones donde <title> no se coloca en el <head> del documento:

1. Dentro de un componente <svg>

En este contexto, <title> no representa el título del documento sino una anotación de accesibilidad para ese gráfico SVG: 

// ✅ Correcto — accesibilidad del SVG, no afecta al <head>
<svg>
  <title>Gráfico de ventas</title>
  {/* Contenido del SVG */}
</svg>

2. Con la prop itemProp

Si <title> tiene una prop itemProp, tampoco tiene comportamiento especial. Representa metadatos de schema.org sobre una parte de la página, no el título del documento: 

// ✅ Correcto — metadatos estructurados, no afecta al <head>
<div itemScope itemType="https://schema.org/Movie">
  <title itemProp="name">El Padrino</title>
  {/* Más metadatos estructurados */}
</div>

📚 Librerías para proyectos grandes

En aplicaciones más grandes, podés usar librerías dedicadas. El principio es el mismo: gestionar el SEO de forma declarativa desde React. 

// react-helmet
import { Helmet } from 'react-helmet'

function Page() {
  return (
    <>
      <Helmet>
        <title>Mi página - JSCamp</title>
        <meta name="description" content="..." />
      </Helmet>
      {/* Contenido */}
    </>
  )
}

// Next.js
import Head from 'next/head'

function Page() {
  return (
    <>
      <Head>
        <title>Mi página - JSCamp</title>
      </Head>
      {/* Contenido */}
    </>
  )
}

✅ Buenas prácticas y ❌ qué evitar

✅ Hacer siempre:

  • Usar <title> en JSX en lugar de document.title en un efecto
  • Eliminar el <title> del index.html para evitar duplicados
  • Usar template literals para interpolar valores en el título
  • Renderizar un solo <title> a la vez
  • Reflejar el estado de la app en el título (cargando, error, resultados)

❌ Nunca hacer:

  • Dejar el <title> en el index.html y también renderizar uno desde React
  • Usar document.title directamente cuando podés usar JSX
  • Pasar un array de nodos como hijos de <title> ({count} empleos sin template literal)
  • Renderizar múltiples <title> desde distintos componentes a la vez
