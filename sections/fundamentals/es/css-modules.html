CSS Modules

😰 El Problema: CSS Global se Rompe Solo

Cuando escribís CSS de forma global, todos los estilos viven en el mismo espacio. Eso significa que si dos archivos distintos usan el mismo nombre de clase, uno pisa al otro.

No es que alguien lo haga mal — es que a medida que la app crece, los nombres se repiten. .button, .card, .active, .title... son nombres que cualquier componente querría usar. 

/* styles.css */
.button { background: blue; }

/* otro-componente.css */
.button { background: red; }  ← sobreescribe al anterior ❌

El resultado es CSS impredecible: cambiás un estilo y se rompe algo en otro lugar.

💡 La Solución: Estilos con Ámbito Local

CSS Modules resuelve esto de forma elegante: cada archivo de estilos tiene su propio ámbito local. Las clases que escribís solo existen dentro del componente que las usa — no se filtran al resto de la aplicación.

La idea es simple: en lugar de llamar al archivo Pagination.css, lo llamás Pagination.module.css. Con ese pequeño cambio, React (a través de Vite o Webpack) se encarga de transformar los nombres de clases para que sean únicos a nivel global. 

Pagination.css        → CSS global (todos pueden usarlo)
Pagination.module.css → CSS Module (solo Pagination lo ve)

🔒 Encapsulación: Cada Componente es una Isla

Con CSS Modules podés usar el mismo nombre de clase en distintos componentes sin que se interfieran: 

Button.module.css   → .button  (el Button azul)
Card.module.css     → .button  (el botón rojo de la Card)
Modal.module.css    → .button  (el botón verde del Modal)

Los tres usan .button, pero como cada uno vive en su propio módulo, no se pisan. React los transforma internamente a nombres únicos como Button_button__a1b2c.

🗂️ Convención: Un Módulo por Componente

Los archivos de CSS Modules van junto al componente que los usa, con el mismo nombre: 

src/components/
  Pagination.jsx
  Pagination.module.css   ← mismo nombre, misma carpeta

  JobCard.jsx
  JobCard.module.css

  Button.jsx
  Button.module.css

Esto tiene una ventaja práctica: si borrás el componente, sabés exactamente qué CSS borrar junto con él.

⚖️ ¿Cuándo CSS Modules vs CSS Global?

Usá CSS Modules para:
  ✅ Estilos propios de un componente
  ✅ Componentes reutilizables (Button, Card, Modal)
  ✅ Proyectos medianos o grandes

Usá CSS global para:
  ✅ Reset / Normalize
  ✅ Variables (:root)
  ✅ Estilos de body y html
  ✅ Utilidades genéricas (.container, .text-center)
  ✅ Fuentes y tipografía base

No es una elección excluyente — pueden convivir en el mismo proyecto.

📁 Crear el Archivo .module.css

/* src/components/Pagination.module.css */

.pagination {
  display: flex;
  justify-content: center;
  gap: 0.5rem;
  margin-block: 2rem;

  a {
    display: flex;
    align-items: center;
    justify-content: center;
    width: 2.5rem;
    height: 2.5rem;
    text-decoration: none;
    color: var(--text-muted);
    border-radius: 0.375rem;
    transition: all 0.3s;

    &:hover,
    &:focus {
      background-color: #fff;
    }

    &:active {
      transform: scale(0.9);
    }
  }
}

.isActive {
  background-color: var(--primary-light);
  color: white;
  pointer-events: none;
}

💡 Nota: La anidación CSS (a dentro de .pagination) y el uso de &:hover funcionan con Vite y herramientas modernas. Usá camelCase para los nombres de clases: .isActive en lugar de .is-active.

📥 Importar y Usar en el Componente

// src/components/Pagination.jsx
import styles from './Pagination.module.css'
//      ↑
//  "styles" es un objeto con las clases del archivo

function Pagination({ currentPage = 1, totalPages = 5, onPageChange = () => {} }) {
  const pages = Array.from({ length: totalPages }, (_, i) => i + 1)

  // ...handlers...

  return (
    <nav className={styles.pagination}>
    {/*            ↑ styles.pagination → "Pagination_pagination__a1b2c" */}

      {pages.map((page) => (
        <a
          key={page}
          className={currentPage === page ? styles.isActive : ''}
          {/*                               ↑ solo se aplica si es la página activa */}
          href="#"
          onClick={(e) => handlePageClick(e, page)}
        >
          {page}
        </a>
      ))}

    </nav>
  )
}

🔍 Qué Pasa Por Dentro

Cuando importás un CSS Module, recibís un objeto JavaScript donde cada clave es el nombre de tu clase y el valor es el nombre único generado: 

// Lo que obtenés al importar
styles = {
  pagination: 'Pagination_pagination__a1b2c',
  isActive:   'Pagination_isActive__d3e4f',
}

// Lo que React renderiza en el DOM
<nav class="Pagination_pagination__a1b2c">
  <a class="Pagination_isActive__d3e4f">1</a>
</nav>
El patrón del nombre generado:

[Componente]_[clase]__[hash]

.pagination → Pagination_pagination__a1b2c
.isActive   → Pagination_isActive__d3e4f

Ese hash es lo que garantiza que nunca colisionen con otras clases.

🎨 Múltiples Clases

// Template literal — la forma más común
<a className={`${styles.link} ${styles.active}`}>Link</a>

// Condicional
<a className={currentPage === page ? styles.isActive : ''}>{page}</a>

// Combinar módulo + clase global
<nav className={`${styles.pagination} container`}>
{/*             ↑ CSS Module           ↑ CSS global */}
</nav>

// Array + filter — más legible con muchas condiciones
<a className={[styles.link, isActive && styles.active].filter(Boolean).join(' ')}>
  Link
</a>

🌐 Escapar al Ámbito Global: :global()

Dentro de un CSS Module podés marcar una clase como global cuando necesitás estilar elementos de librerías externas que inyectan su propio HTML: 

/* Pagination.module.css */

.pagination {
  display: flex;  /* local ✅ */
}

:global(.container) {
  max-width: 1200px;  /* global — afecta a toda la app */
}

/* Estilar clases de una librería externa */
:global(.external-tooltip) {
  background: #333;
  color: #fff;
}

🐛 Errores Comunes

1. Olvidar .module en el nombre del archivo

Pagination.css        ❌ CSS global — no tiene ámbito local
Pagination.module.css ✅ CSS Module — ámbito local garantizado

2. Usar guiones en los nombres de clase

/* ❌ Con guiones — acceso incómodo */
.is-active { color: white; }

// En el componente tenés que usar corchetes
<a className={styles['is-active']}>Link</a>

/* ✅ Con camelCase — acceso limpio */
.isActive { color: white; }

<a className={styles.isActive}>Link</a>

3. Intentar usar la clase como string

// ❌ La clase real es "Pagination_pagination__a1b2c", no "pagination"
<nav className="pagination">

// ✅ Siempre a través del objeto styles
<nav className={styles.pagination}>

💪 Ejercicio: Agregar Estilos a JobCard

Creá un CSS Module para el componente JobCard.

Paso 1 — Crear el archivo

src/components/
  JobCard.jsx
  JobCard.module.css   ← crear este archivo

Paso 2 — Escribir los estilos

/* JobCard.module.css */
.card {
  border: 1px solid #e2e8f0;
  border-radius: 0.5rem;
  padding: 1.5rem;
  display: flex;
  flex-direction: column;
  gap: 0.75rem;
}

.title {
  font-size: 1.125rem;
  font-weight: 600;
  color: #1a202c;
}

.company {
  color: #718096;
  font-size: 0.875rem;
}

.badge {
  display: inline-block;
  padding: 0.25rem 0.5rem;
  border-radius: 9999px;
  font-size: 0.75rem;
  font-weight: 500;
  background-color: #ebf8ff;
  color: #2b6cb0;
}

.featured {
  border-left: 4px solid #667eea;
}

Paso 3 — Aplicar en el componente

import styles from './JobCard.module.css'

export function JobCard({ job, isRemote, isFeatured }) {
  return (
    <article className={`${styles.card} ${isFeatured ? styles.featured : ''}`}>
      <h3 className={styles.title}>{job.titulo}</h3>
      <p className={styles.company}>{job.empresa}</p>
      {isRemote && <span className={styles.badge}>🏠 Remoto</span>}
    </article>
  )
}

✅ Resumen de la Clase

📦 CSS Modules      — CSS con ámbito local, sin colisiones
🏷️ .module.css      — la extensión que activa el ámbito local
🔒 Encapsulación    — los estilos no se filtran a otros componentes
📥 import styles    — obtenés un objeto con los nombres de clase únicos
🎨 styles.clase     — así se aplican las clases en el JSX
⚠️ camelCase        — .isActive en lugar de .is-active
🌐 :global()        — para escapar cuando necesitás estilos globales
