Integrando Navegación con Formulario de Búsqueda

🎯 El objetivo

Conectar un formulario de búsqueda con el router de una SPA. Cuando el usuario envía el formulario, la app debe navegar a /search con el texto introducido como query param — sin recargar la página.

😤 El problema inicial

Tenemos un formulario con input y botón de búsqueda en la Home, pero no hace nada. Necesitamos:

  • Capturar el evento submit
  • Extraer el valor del input
  • Construir la URL con query params
  • Navegar usando nuestro router, sin recargar
// Estado actual (no funcional)
function Home() {
  return (
    <form>
      <input type="text" placeholder="Buscar..." />
      <button disabled>🔍</button>
    </form>
  )
}

⚠️ Atributos SVG en JSX — camelCase

En HTML los atributos SVG usan kebab-case, pero en JSX tienen que ir en camelCase: 

// ❌ Incorrecto (HTML)
<path stroke-width="2" stroke-linecap="round" />

// ✅ Correcto (JSX)
<path strokeWidth="2" strokeLinecap="round" />

Cambios a aplicar:

  • stroke-width → strokeWidth
  • stroke-linecap → strokeLinecap
  • stroke-linejoin → strokeLinejoin
  • Quitar el atributo disabled del botón

🛑 ¿Por qué event.preventDefault()?

En una web tradicional, al enviar un formulario el navegador:

  • Recarga la página completa
  • Pierde todo el estado de JavaScript
  • Hace una petición al servidor

En una SPA queremos lo contrario:

  • ✅ Mantener la página sin recargar
  • ✅ Conservar el estado
  • ✅ Manejar la navegación con JavaScript
const handleSearch = (event) => {
  event.preventDefault() // ← Evita el reload
  // Ahora controlamos la navegación nosotros
}

📦 FormData — extraer valores sin estado controlado

La API FormData permite leer los valores de un formulario directamente, sin necesidad de useState para cada campo. 

const handleSearch = (event) => {
  event.preventDefault()

  const formData = new FormData(event.target)
  const searchTerm = formData.get('search')

  console.log('Buscando:', searchTerm)
}

Importante: el input necesita un atributo name para que FormData pueda identificarlo: <input name="search" />

⚖️ FormData vs useState — ¿cuándo cada uno?

Usar FormData Usar useState
Solo necesitás el valor al enviar Necesitás validación en tiempo real
No hace falta validación mientras escribe Querés mostrar el valor en otros componentes
Múltiples campos y querés menos código Necesitás deshabilitar el botón según el input

🔗 encodeURIComponent — caracteres seguros en la URL

Si el usuario escribe "React de Dev", la URL quedaría rota: 

/search?text=React de Dev  ← ¡inválida! los espacios no son válidos

Hay que codificar:

const encodedTerm = encodeURIComponent(searchTerm)
targetUrl += `?text=${encodedTerm}`

Ejemplos:

  • "React de Dev" → /search?text=React%20de%20Dev ✅
  • "C++" → /search?text=C%2B%2B ✅
  • "¿Qué es React?" → /search?text=%C2%BFQu%C3%A9%20es%20React%3F ✅
Carácter Codificado Razón
espacio%20No válido en URL
+%2BSignificado especial
?%3FDelimita query params
&%26Separa query params
=%3DAsigna valores
#%23Indica fragmento

🧩 Reutilizar navigateTo del custom hook useRouter

Si usáramos window.history.pushState directamente:

// ❌ Problema: solo cambia la URL, no renderiza la vista
window.history.pushState({}, '', targetUrl)

Problemas:

  • ❌ La URL cambia, pero la vista no se actualiza
  • ❌ Los listeners del router no se disparan
  • ❌ El estado del componente no cambia
  • ❌ Duplicás lógica del router

La solución: usar el custom hook useRouter que ya tiene navigateTo. El hook se encarga de actualizar la URL y el estado → re-render automático.

Ventaja: separación de responsabilidades. El componente solo se preocupa por su UI; la lógica del router vive en un solo lugar.

Paso 1 — Manejador inicial del submit

function Home() {
  const handleSearch = (event) => {
    event.preventDefault()
    // Lógica acá
  }

  return (
    <form onSubmit={handleSearch}>
      <input type="text" name="search" placeholder="Buscar cursos..." />
      <button type="submit">🔍</button>
    </form>
  )
}

Paso 2 — Extraer valor con FormData

Paso 3 — Construir la URL con encodeURIComponent

const handleSearch = (event) => {
  event.preventDefault()

  const formData = new FormData(event.target)
  const searchTerm = formData.get('search')

  let targetUrl = '/search'

  if (searchTerm) {
    const encodedTerm = encodeURIComponent(searchTerm)
    targetUrl += `?text=${encodedTerm}`
  }

  console.log('Navegar a:', targetUrl)
}

Paso 4 — El custom hook useRouter

// hooks/useRouter.js
import { useState, useEffect } from 'react'

export function useRouter() {
  const [currentPath, setCurrentPath] = useState(window.location.pathname)

  useEffect(() => {
    const handleLocationChange = () => {
      setCurrentPath(window.location.pathname)
    }

    window.addEventListener('popstate', handleLocationChange)
    return () => window.removeEventListener('popstate', handleLocationChange)
  }, [])

  const navigateTo = (path) => {
    window.history.pushState({}, '', path)
    setCurrentPath(path)
  }

  return { currentPath, navigateTo }
}

Paso 5 — Integración completa en Home

import { useRouter } from './hooks/useRouter'

function Home() {
  const { navigateTo } = useRouter()

  const handleSearch = (event) => {
    event.preventDefault()

    const formData = new FormData(event.target)
    const searchTerm = formData.get('search')

    let targetUrl = '/search'

    if (searchTerm) {
      const encodedTerm = encodeURIComponent(searchTerm)
      targetUrl += `?text=${encodedTerm}`
    }

    navigateTo(targetUrl)
  }

  return (
    <div className="home">
      <h1>Encuentra el mejor curso para ti</h1>

      <form onSubmit={handleSearch}>
        <input
          type="text"
          name="search"
          placeholder="Buscar cursos..."
          className="search-input"
        />
        <button type="submit" className="search-button">
          <svg>...</svg>
        </button>
      </form>
    </div>
  )
}

export default Home

🧪 Casos de prueba

Caso 1 — Búsqueda con texto

  1. Escribir "programador" en el input
  2. Presionar Enter o clic en "Buscar"
  3. ✅ Debe navegar a /search?text=programador
  4. ✅ La URL se actualiza sin recargar

Caso 2 — Búsqueda vacía

  1. Dejar el input vacío
  2. Presionar "Buscar"
  3. ✅ Debe navegar a /search (sin query params)

Caso 3 — Búsqueda con espacios / caracteres especiales

  1. Escribir "React de Dev"
  2. Presionar "Buscar"
  3. ✅ Debe navegar a /search?text=React%20de%20Dev

Caso 4 — Botón "Atrás" del navegador

  1. Hacer una búsqueda
  2. Presionar el botón "Atrás"
  3. ✅ El evento popstate se dispara
  4. ✅ La vista vuelve a la Home
