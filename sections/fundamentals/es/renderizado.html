Renderizado y Virtual DOM en React

1️⃣ ¿Qué Significa "Renderizar"?

Renderizar significa que React toma tu JSX y lo convierte en HTML real que aparece en el navegador.

Es como traducir de un idioma a otro:

JSX (código React)
  ↓
React procesa
  ↓
HTML (lo que ves en pantalla)

Ejemplo:

// Tu código JSX
function App() {
  return <h1>Hola Mundo</h1>
}

// React lo convierte en HTML
<h1>Hola Mundo</h1>

2️⃣ ¿Cuándo Renderiza React?

React renderiza (o re-renderiza) tu componente en estas situaciones:

┌─────────────────────────────────────┐
│  SITUACIONES DE RENDERIZADO         │
├─────────────────────────────────────┤
│  1. Montaje inicial (primera vez)   │
│  2. El state cambia                 │
│  3. Las props cambian               │
│  4. El componente padre re-renderiza│
└─────────────────────────────────────┘

🎯 Situación 1: Montaje Inicial

Cuando React renderiza tu componente por primera vez.

const root = ReactDOM.createRoot(document.getElementById('root'))
root.render(<App />)

// React dice: "Ok, primera vez, voy a renderizar App"

🔄 Situación 2: El State Cambia

Esta es la más importante.

function Counter() {
  const [count, setCount] = React.useState(0)

  // Cuando haces click:
  // 1. setCount(1) actualiza el state
  // 2. React detecta el cambio
  // 3. React re-renderiza Counter

  return (
    <button onClick={() => setCount(count + 1)}>
      Clicks: {count}
    </button>
  )
}

Flujo visual del re-renderizado:

Estado inicial: count = 0
       ↓
Usuario hace click
       ↓
setCount(1) se ejecuta
       ↓
🔄 React re-renderiza Counter
       ↓
Botón ahora muestra: "Clicks: 1"

3️⃣ ¿Por Qué el State No Se Pierde?

Muchos desarrolladores se preguntan:

"Si React vuelve a ejecutar la función del componente, ¿por qué el state no se resetea?"

Respuesta: Porque React guarda el state FUERA del componente.

🧠 Modelo Mental

┌─────────────────────────────────────────┐
│  COMPONENTE (tu código)                 │
│  ┌─────────────────────────────────┐    │
│  │ function Counter() {            │    │
│  │   const [count, setCount] = ... │    │
│  │   return <button>...</button>   │    │
│  │ }                               │    │
│  └─────────────────────────────────┘    │
│           ↑                              │
│           │ React conecta                │
│           ↓                              │
│  ┌─────────────────────────────────┐    │
│  │  MEMORIA DE REACT               │    │
│  │  count = 5   ← guardado aquí    │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘

📊 Paso a Paso

1. Primera ejecución
   ├─ useState(0) crea state
   ├─ React guarda: count = 0
   └─ Retorna: [0, setCount]

2. Usuario hace click
   ├─ setCount(1) se ejecuta
   ├─ React actualiza en memoria: count = 1
   └─ React re-ejecuta el componente

3. Segunda ejecución (re-render)
   ├─ useState() NO crea nuevo state
   ├─ React encuentra state existente: count = 1
   └─ Retorna: [1, setCount]

💡 Clave: React mantiene el state entre renders. No se pierde porque no está dentro de tu función, está en la memoria interna de React.

4️⃣ El Ciclo de Vida de un Render

┌──────────────────────────────────────────────────┐
│          CICLO DE VIDA DEL RENDERIZADO           │
├──────────────────────────────────────────────────┤
│                                                  │
│  1. 🎬 TRIGGER (Disparador)                      │
│     ├─ Cambio de state                          │
│     ├─ Cambio de props                          │
│     └─ Montaje inicial                          │
│                                                  │
│            ↓                                     │
│                                                  │
│  2. 🎨 RENDER PHASE (Fase de Renderizado)       │
│     ├─ React ejecuta tu función                 │
│     ├─ Calcula nuevo JSX                        │
│     └─ Crea Virtual DOM                         │
│                                                  │
│            ↓                                     │
│                                                  │
│  3. 🔍 RECONCILIATION (Reconciliación)          │
│     ├─ Compara Virtual DOM anterior vs nuevo    │
│     ├─ Detecta diferencias (diffing)            │
│     └─ Decide qué actualizar                    │
│                                                  │
│            ↓                                     │
│                                                  │
│  4. ⚡ COMMIT PHASE (Fase de Commit)            │
│     ├─ Actualiza el DOM real                    │
│     ├─ Ejecuta efectos (useEffect)              │
│     └─ La UI se actualiza                       │
│                                                  │
└──────────────────────────────────────────────────┘

Ejemplo con Código

function Counter() {
  const [count, setCount] = React.useState(0)

  console.log('🎨 Renderizando... count =', count)

  return (
    <button onClick={() => setCount(count + 1)}>
      Clicks: {count}
    </button>
  )
}

// Salida en consola:
// 🎨 Renderizando... count = 0  (montaje inicial)
// (usuario hace click)
// 🎨 Renderizando... count = 1  (re-render)
// (usuario hace click)
// 🎨 Renderizando... count = 2  (re-render)

5️⃣ Virtual DOM: La Magia Detrás de React

¿Qué es el Virtual DOM?

Es una copia del DOM real que React mantiene en memoria. Es una representación ligera (en JavaScript) de tu interfaz.

🌳 Analogía: El Plano de una Casa

PLANO (Virtual DOM)        CASA REAL (DOM Real)
     📋                           🏠

Barato de modificar      Caro de modificar
Rápido de cambiar        Lento de cambiar
Solo papel               Madera, cemento, etc.

React hace lo mismo:

  • Primero modifica el "plano" (Virtual DOM) → rápido
  • Compara con el plano anterior
  • Solo actualiza lo necesario en la "casa real" (DOM) → eficiente

📊 Comparación Visual

SIN VIRTUAL DOM (Vanilla JS):
┌──────────────────────────────────┐
│  Cambio pequeño en UI            │
│         ↓                        │
│  Re-renderizar TODO el DOM       │
│         ↓                        │
│  💰 Muy costoso                  │
│  🐌 Muy lento                    │
└──────────────────────────────────┘


CON VIRTUAL DOM (React):
┌──────────────────────────────────┐
│  Cambio pequeño en UI            │
│         ↓                        │
│  Actualizar Virtual DOM          │
│         ↓                        │
│  Comparar (diffing)              │
│         ↓                        │
│  Actualizar SOLO lo que cambió   │
│         ↓                        │
│  ⚡ Muy eficiente                │
│  🚀 Muy rápido                   │
└──────────────────────────────────┘

6️⃣ El Algoritmo de Diffing

Diffing = Encontrar las diferencias

React compara el Virtual DOM anterior con el nuevo para saber qué cambió.

🔍 Ejemplo Visual

ANTES (Virtual DOM anterior):
┌─────────────────────────┐
│ <div>                   │
│   <h1>Count: 0</h1>     │
│   <button>+</button>    │
│ </div>                  │
└─────────────────────────┘


DESPUÉS (Virtual DOM nuevo):
┌─────────────────────────┐
│ <div>                   │
│   <h1>Count: 1</h1>  ⭐ │ ← Solo esto cambió
│   <button>+</button>    │
│ </div>                  │
└─────────────────────────┘


REACT DETECTA:
  ✅ <div> → sin cambios
  ⭐ <h1> → cambió el texto (0 → 1)
  ✅ <button> → sin cambios

RESULTADO:
React solo actualiza el texto del <h1>
¡No toca el resto del DOM!

Código con Console.log

function Counter() {
  const [count, setCount] = React.useState(0)

  console.log('🔄 Componente ejecutándose')

  return (
    <div>
      <h1>Count: {count}</h1>
      <button onClick={() => setCount(count + 1)}>+</button>
    </div>
  )
}

// Salida en consola cada vez que haces click:
// 🔄 Componente ejecutándose
//
// Pero en el DOM real, React SOLO actualiza el texto del h1
// No reconstruye todo el componente

7️⃣ Reconciliación: El Proceso Completo

Reconciliación es el proceso que React usa para decidir qué actualizar en el DOM real.

┌───────────────────────────────────────────────────────┐
│               PROCESO DE RECONCILIACIÓN               │
├───────────────────────────────────────────────────────┤
│                                                       │
│  PASO 1: Crear Virtual DOM Nuevo                     │
│  ┌─────────────────────────────────────┐             │
│  │  React ejecuta tu función           │             │
│  │  Genera nuevo árbol JSX             │             │
│  └─────────────────────────────────────┘             │
│                    ↓                                  │
│                                                       │
│  PASO 2: Comparar (Diffing Algorithm)                │
│  ┌─────────────────────────────────────┐             │
│  │  Virtual DOM Anterior  vs  Nuevo    │             │
│  │                                     │             │
│  │  ¿Mismo tipo de elemento?           │             │
│  │  ¿Mismos atributos?                 │             │
│  │  ¿Mismo contenido?                  │             │
│  └─────────────────────────────────────┘             │
│                    ↓                                  │
│                                                       │
│  PASO 3: Generar Lista de Cambios                    │
│  ┌─────────────────────────────────────┐             │
│  │  Actualizar texto del h1            │             │
│  │  Cambiar clase de button            │             │
│  │  Agregar nuevo li                   │             │
│  └─────────────────────────────────────┘             │
│                    ↓                                  │
│                                                       │
│  PASO 4: Aplicar Cambios al DOM Real                 │
│  ┌─────────────────────────────────────┐             │
│  │  Ejecutar cambios mínimos           │             │
│  │  UI se actualiza                    │             │
│  └─────────────────────────────────────┘             │
│                                                       │
└───────────────────────────────────────────────────────┘

Ejemplo Comparativo

// Estado 1: applied = false
<button className="button-apply-job">
  Apply
</button>

// Usuario hace click → applied = true
// React re-renderiza

// Estado 2: applied = true
<button className="button-apply-job is-applied" disabled>
  Applied!
</button>

// React detecta:
// ✅ Elemento: <button> (mismo tipo)
// ⭐ className: cambió
// ⭐ disabled: agregado
// ⭐ texto: cambió

// React aplica SOLO esos 3 cambios al DOM real

8️⃣ Batch Updates: React es Inteligente

React NO re-renderiza inmediatamente después de cada setState. Agrupa (batch) múltiples cambios.

Ejemplo con Múltiples Estados

function Form() {
  const [name, setName] = React.useState('')
  const [email, setEmail] = React.useState('')
  const [age, setAge] = React.useState(0)

  const handleSubmit = () => {
    setName('Miguel')   // 1
    setEmail('m@g.com') // 2
    setAge(25)          // 3

    // React NO re-renderiza 3 veces
    // React agrupa los 3 cambios
    // Re-renderiza UNA SOLA VEZ
  }

  console.log('🎨 Renderizando...')

  return (
    <button onClick={handleSubmit}>Submit</button>
  )
}

🔄 Flujo de Batch Updates

Usuario hace click
       ↓
setName('Miguel')   ─┐
setEmail('m@g.com') ─┼─ React agrupa
setAge(25)          ─┘
       ↓
React re-renderiza UNA vez
       ↓
console.log ejecuta UNA vez

💡 Optimización automática: React agrupa automáticamente múltiples setState en un solo re-render para mejor rendimiento.

9️⃣ ¿Cuándo NO Re-renderiza React?

React es inteligente y NO re-renderiza en estas situaciones:

┌────────────────────────────────────────┐
│  CASOS SIN RE-RENDERIZADO              │
├────────────────────────────────────────┤
│  1. setState con el MISMO valor        │
│  2. Variables locales cambian          │
│  3. Props no cambian                   │
└────────────────────────────────────────┘

Caso 1: setState con Mismo Valor

function Counter() {
  const [count, setCount] = React.useState(0)

  const handleClick = () => {
    setCount(0) // ← Ya es 0
    // React detecta que no cambió
    // NO re-renderiza
  }

  console.log('🎨 Renderizando...')

  return <button onClick={handleClick}>{count}</button>
}

// Salida en consola:
// 🎨 Renderizando... (montaje inicial)
// (usuario hace click)
// (nada en consola - no re-renderiza)

Caso 2: Variables Locales

function Counter() {
  const [count, setCount] = React.useState(0)
  let clicks = 0 // ← Variable local

  const handleClick = () => {
    clicks = clicks + 1 // ❌ Esto NO causa re-render
    console.log('clicks:', clicks) // Verás el cambio
    // Pero la UI NO se actualiza
  }

  return <button onClick={handleClick}>{clicks}</button>
}

// La UI siempre muestra 0
// Porque clicks NO es state
// React no sabe que cambió

⚠️ Regla importante: Solo los cambios en STATE y PROPS causan re-renderizado. Variables locales no.

🔟 Re-renderizado por Props

Cuando un componente padre re-renderiza, sus hijos también re-renderizan (por defecto).

Ejemplo

function Parent() {
  const [count, setCount] = React.useState(0)

  console.log('🎨 Parent renderizando')

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>
        Count: {count}
      </button>
      <Child />
    </div>
  )
}

function Child() {
  console.log('🎨 Child renderizando')
  return <p>Soy el hijo</p>
}

// Salida en consola:
// 🎨 Parent renderizando
// 🎨 Child renderizando
// (usuario hace click)
// 🎨 Parent renderizando  ← Parent re-renderiza
// 🎨 Child renderizando   ← Child también re-renderiza

🔍 Flujo Visual

Parent re-renderiza
       ↓
React ejecuta función Parent()
       ↓
Encuentra <Child /> en el JSX
       ↓
React ejecuta función Child()
       ↓
Child re-renderiza también

Aunque las props de Child no cambien, React re-renderiza por defecto.

button.addEventListener('click', () => {

// ❌ Código imperativo
// ❌ Manipulación manual del DOM
// ❌ Difícil de mantener
// ❌ Propenso a errores

function Counter() {

    <div>
      <h1 style={{ color: count > 5 ? 'red' : 'black' }}>
        {count}
      </h1>

      <button
        onClick={() => setCount(count + 1)}
        disabled={count > 10}
      >
        {count > 10 ? 'Max alcanzado' : 'Incrementar'}
      </button>
    </div>
  )
}

// ✅ Código declarativo
// ✅ React actualiza el DOM automáticamente
// ✅ Fácil de leer y mantener
// ✅ Menos propenso a errores

   ├─ setCount(count + 1)

   ├─ Compara Virtual DOM anterior vs nuevo
   ├─ Algoritmo de diffing

📋 Lo Que Debes Recordar

  • Renderizar = React convierte JSX a HTML real
  • Re-renderizar = React actualiza la UI cuando state/props cambian
  • Virtual DOM = Copia ligera del DOM en memoria
  • Diffing = Comparar Virtual DOM anterior vs nuevo
  • Reconciliación = Decidir qué actualizar en el DOM real
  • Batch Updates = React agrupa múltiples setState en un solo re-render
  • React NO re-renderiza si el state no cambió realmente
  • Variables locales NO causan re-renderizado

1️⃣3️⃣ Pregunta de Entrevista Típica

❓ "¿Por qué React usa Virtual DOM?"

Respuesta correcta:

Porque actualizar el DOM real es costoso y lento. React usa Virtual DOM para:

  • Calcular cambios rápidamente en memoria
  • Comparar versiones anterior y nueva (diffing)
  • Aplicar solo los cambios mínimos necesarios al DOM real
  • Optimizar rendimiento automáticamente

Analogía para la entrevista:

"Es como hacer un borrador en papel antes de construir. Probar cambios en el borrador es rápido y barato. Solo cuando sabes exactamente qué cambiar, modificas la construcción real."

Ejemplo 2: Batch Updates en Acción


function MultipleStates() {
  const [name, setName] = React.useState('')
  const [email, setEmail] = React.useState('')
  const [age, setAge] = React.useState(0)

  // Contador de renders
  const renderCount = React.useRef(0)
  renderCount.current = renderCount.current + 1

  const handleUpdate = () => {
    // Tres cambios de estado
    setName('Miguel')
    setEmail('miguel@example.com')
    setAge(25)

    // React agrupa estos 3 cambios
    // Solo re-renderiza UNA vez
  }

  console.log('🎨 Render #', renderCount.current)

  return (
    <div>
      <p>Renders: {renderCount.current}</p>
      <p>Name: {name}</p>
      <p>Email: {email}</p>
      <p>Age: {age}</p>
      <button onClick={handleUpdate}>
        Actualizar Todo
      </button>
    </div>
  )
}

// Resultado: Solo incrementa en 1 por click
// No en 3, porque React agrupa los cambios

Ejemplo 3: setState con Mismo Valor (No Re-renderiza)


function SameValue() {
  const [count, setCount] = React.useState(0)
  const renderCount = React.useRef(0)

  renderCount.current = renderCount.current + 1

  const handleClick = () => {
    // Establecer el mismo valor
    setCount(0)

    // React detecta que no cambió
    // NO re-renderiza
  }

  console.log('🎨 Render #', renderCount.current)

  return (
    <div>
      <p>Count: {count}</p>
      <p>Renders: {renderCount.current}</p>
      <button onClick={handleClick}>
        setState(0) - No cambia nada
      </button>
      <button onClick={() => setCount(count + 1)}>
        Incrementar
      </button>
    </div>
  )
}

// Haz click en "setState(0)"
// Renders no aumenta porque React es inteligente

Ejemplo 4: Variable Local vs State


function LocalVsState() {
  const [stateCount, setStateCount] = React.useState(0)
  let localCount = 0 // ❌ Variable local

  const incrementLocal = () => {
    localCount = localCount + 1
    console.log('localCount:', localCount)
    // ❌ La UI NO se actualiza
  }

  const incrementState = () => {
    setStateCount(stateCount + 1)
    // ✅ La UI se actualiza
  }

  return (
    <div>
      <p>Variable Local: {localCount}</p>
      <p>State: {stateCount}</p>

      <button onClick={incrementLocal}>
        Incrementar Local (no funciona)
      </button>

      <button onClick={incrementState}>
        Incrementar State (funciona)
      </button>
    </div>
  )
}

// Variable local siempre muestra 0 en la UI
// Aunque en consola ves que aumenta

  // Manualmente actualizar DOM
  counterEl.textContent = count

  if (count >= 10) {
    counterEl.style.color = 'red'
  }

  if (count >= 20) {
    buttonEl.disabled = true
  }
})

// -------------------------

// ✅ REACT (Declarativo)
function Counter() {
  const [count, setCount] = React.useState(0)

  // React actualiza automáticamente
  return (
    <div>
      <h1 style={{ color: count >= 10 ? 'red' : 'black' }}>
        {count}
      </h1>
      <button
        onClick={() => setCount(count + 1)}
        disabled={count >= 20}
      >
        Incrementar
      </button>
    </div>
  )
}

Ejercicio 1: Contador de Renders

Desafío: Crea un componente que muestre cuántas veces se ha renderizado.

Pistas:

  • Usa useRef para mantener un contador que NO cause re-renders
  • Incrementa el contador en cada render
  • Muestra el número en la UI

Solución:


function RenderCounter() {
  const [count, setCount] = React.useState(0)
  const renderCount = React.useRef(0)

  // Se ejecuta en cada render
  renderCount.current = renderCount.current + 1

  return (
    <div>
      <p>Count: {count}</p>
      <p>Componente renderizado {renderCount.current} veces</p>
      <button onClick={() => setCount(count + 1)}>
        Incrementar
      </button>
    </div>
  )
}

Ejercicio 4: Batch Updates

Desafío: Demuestra que React agrupa múltiples setState.

Solución:


function BatchDemo() {
  const [a, setA] = React.useState(0)
  const [b, setB] = React.useState(0)
  const [c, setC] = React.useState(0)

  const renderCount = React.useRef(0)
  renderCount.current = renderCount.current + 1

  const handleBatch = () => {
    setA(a + 1)
    setB(b + 1)
    setC(c + 1)

    // React agrupa estos 3 cambios
    // Render count aumenta solo en 1
  }

  const handleSeparate = () => {
    setA(a + 1)

    setTimeout(() => {
      setB(b + 1)
    }, 100)

    setTimeout(() => {
      setC(c + 1)
    }, 200)

    // Estos NO se agrupan (antes de React 18)
    // Render count aumenta en 3
  }

  return (
    <div>
      <p>Renders: {renderCount.current}</p>
      <p>A: {a}, B: {b}, C: {c}</p>

      <button onClick={handleBatch}>
        Batch Update (1 render)
      </button>

      <button onClick={handleSeparate}>
        Separate Updates (3 renders)
      </button>
    </div>
  )
}

Ejercicio 5: Prevenir Re-renders Innecesarios

Desafío: Usa React.memo para evitar re-renders de un hijo cuando no es necesario.

Solución:


function Parent() {
  const [parentCount, setParentCount] = React.useState(0)
  const [childCount, setChildCount] = React.useState(0)

  console.log('🎨 Parent renderizando')

  return (
    <div>
      <h1>Parent: {parentCount}</h1>
      <button onClick={() => setParentCount(parentCount + 1)}>
        Incrementar Parent
      </button>

      <button onClick={() => setChildCount(childCount + 1)}>
        Incrementar Child
      </button>

      {/* Child solo re-renderiza si childCount cambia */}
      <MemoizedChild count={childCount} />
    </div>
  )
}

// Sin React.memo, Child re-renderiza siempre que Parent lo haga
const MemoizedChild = React.memo(function Child({ count }) {
  console.log('🎨 Child renderizando')

  return <p>Child Count: {count}</p>
})

// Ahora Child SOLO re-renderiza cuando count cambia
// No cuando parentCount cambia

Ejercicio 6: Visualizar el Proceso de Reconciliación

function ReconciliationDemo() {
  const [count, setCount] = React.useState(0)
  const [name, setName] = React.useState('Miguel')

  // Solo count y name cambiarán en el DOM
  // Los elementos h1, h2, button permanecen iguales

  return (
    <div>
      <h1>Título Estático</h1>
      <h2>Count: {count}</h2>  {/* ⭐ Solo esto cambia */}
      <h2>Name: {name}</h2>    {/* ⭐ Solo esto cambia */}

      <button onClick={() => setCount(count + 1)}>
        Incrementar Count
      </button>

      <button onClick={() => setName(name === 'Miguel' ? 'Ana' : 'Miguel')}>
        Cambiar Nombre
      </button>

      <p>React solo actualiza el contenido de los h2</p>
      <p>No reconstruye todo el componente</p>
    </div>
  )
}
