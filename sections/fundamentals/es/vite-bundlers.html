🚀 Vite y Empaquetadores Modernos

🧠 ¿Qué es un Empaquetador (Bundler)?

Un bundler es una herramienta que toma múltiples archivos JavaScript, CSS, imágenes y otros assets, y los combina en unos pocos archivos optimizados para el navegador.

❌ Problemas sin un bundler

  • ❌ 100+ archivos separados - Cada import es una petición HTTP adicional
  • ❌ Lento en desarrollo - El navegador debe descargar y procesar cada archivo
  • ❌ No hay optimización - Código sin minificar, sin tree-shaking
  • ❌ Problemas de orden - Los scripts deben cargarse en el orden correcto
  • ❌ Sin transformaciones modernas - No puedes usar JSX, TypeScript, etc.

📦 Ejemplo de estructura de proyecto

mi-proyecto/
├── src/
│   ├── components/
│   │   ├── Header.jsx       ← 1 archivo
│   │   ├── Footer.jsx       ← 1 archivo
│   │   ├── Button.jsx       ← 1 archivo
│   │   └── Card.jsx         ← 1 archivo
│   ├── utils/
│   │   ├── formatDate.js    ← 1 archivo
│   │   └── api.js           ← 1 archivo
│   ├── styles/
│   │   ├── main.css         ← 1 archivo
│   │   └── variables.css    ← 1 archivo
│   ├── App.jsx
│   └── main.jsx
└── index.html

SIN BUNDLER: 10+ peticiones HTTP separadas
CON BUNDLER: 1-3 archivos optimizados

✅ Solución con bundler

      100+ archivos (src/)
            ⬇️
      [BUNDLER] 🔄
            ⬇️
      3 archivos optimizados (dist/)
      ✅ index.js (minificado)
      ✅ styles.css (comprimido)
      ✅ assets/ (optimizados)

🚀 ¿Qué es Vite?

Vite (palabra francesa que significa "rápido") es una herramienta de desarrollo moderna creada por Evan You (creador de Vue.js) en 2020. Es el reemplazo de Create React App y Webpack para proyectos modernos.

🔥 Características clave

  • ⚡ Inicio instantáneo - El servidor de desarrollo arranca en milisegundos
  • 🔥 HMR ultra rápido - Los cambios se reflejan instantáneamente sin recargar la página
  • 📦 Build optimizado - Usa Rollup para producción (tree-shaking, code splitting)
  • 🔌 Plugins - Ecosistema extensible (React, TypeScript, PWA, etc.)
  • 🎯 Zero config - Funciona out-of-the-box para React, Vue, Svelte

⚔️ Comparación de Bundlers

Bundler Velocidad Configuración Tecnología Año
Webpack 🐢 Lento 😫 Compleja JavaScript 2012
esbuild 🚀 Ultra rápido 😊 Simple Go 2020
Vite ⚡ Instantáneo 😍 Zero config esbuild + Rollup 2020
Turbopack 🚀 Muy rápido 😊 Simple Rust 2022

💡 ¿Por qué Vite para React?

1. ⚡ Configuración instantánea

# Con Create React App (antiguo)
npx create-react-app mi-app
# ⏰ Tarda 2-5 minutos, instala 1000+ dependencias

# Con Vite (moderno)
npm create vite@latest mi-app -- --template react
# ⚡ Tarda 10 segundos, instala ~50 dependencias

2. 🔥 Fast Refresh (HMR)

Cuando editas un componente, Vite actualiza SOLO ese componente sin recargar la página completa. El estado de React se mantiene intacto. 

      Editas Counter.jsx
            ⬇️
      Vite detecta el cambio
            ⬇️
      Envía el nuevo código al navegador
            ⬇️
      React actualiza Counter sin perder el state
      ✅ El contador sigue en "5"

3. 📦 Imports naturales

// Vite soporta todo tipo de imports sin configuración
import React from 'react'
import './App.css'                    // CSS
import logo from './logo.svg'         // Imágenes
import data from './data.json'        // JSON
import Button from './Button.jsx'     // Componentes

4. 🏗️ Build optimizado para producción

npm run build
# Genera carpeta dist/ con:
# ✅ Código minificado
# ✅ Tree-shaking (elimina código no usado)
# ✅ Code splitting (divide en chunks)
# ✅ Assets optimizados

🏗️ ¿Cómo funciona Vite?

🔍 Arquitectura en DESARROLLO

      Navegador solicita: /src/App.jsx
            ⬇️
      Vite intercepta la petición
            ⬇️
      Transforma JSX → JavaScript usando esbuild
            ⬇️
      Envía el archivo transformado al navegador
            ⬇️
      Navegador ejecuta el código

      ⚡ CLAVE: NO bundlea en dev, usa ES Modules nativos

📦 Arquitectura en PRODUCCIÓN

      npm run build
            ⬇️
      Rollup analiza todo el proyecto
            ⬇️
      Tree-shaking (elimina código no usado)
      Code splitting (divide en chunks)
      Minificación (reduce tamaño)
            ⬇️
      Genera carpeta dist/
      ├── assets/
      │   ├── index-abc123.js  (optimizado)
      │   └── index-def456.css (comprimido)
      └── index.html

🔌 ES Modules - La tecnología detrás de Vite

Los ES Modules son el sistema de módulos nativo de JavaScript (introducido en ES6/2015). Vite aprovecha esto para NO bundlear en desarrollo.

Ejemplo de ES Module

<!-- index.html -->
<script type="module" src="/src/main.jsx"></script>

<!-- El navegador entiende "type=module" y maneja los imports -->
// src/main.jsx
import React from 'react'  // El navegador solicita este archivo
import App from './App'     // El navegador solicita este archivo

// Cada import es una petición HTTP separada
// Pero Vite las sirve instantáneamente desde memoria

🧠 Por qué esto es RÁPIDO

  • ✅ No hay bundling en dev - Vite NO combina archivos, solo los transforma
  • ✅ Cache del navegador - Los archivos sin cambios se reutilizan
  • ✅ Transformación bajo demanda - Solo procesa los archivos que realmente usas
  • ✅ esbuild es ultra rápido - Escrito en Go, 10-100x más rápido que Babel

🔌 Plugins y Configuración

vite.config.js - Archivo de configuración

// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],           // Plugin oficial de React
  server: {
    port: 3000,                 // Puerto de desarrollo
    open: true                  // Abre el navegador automáticamente
  },
  build: {
    outDir: 'dist',             // Carpeta de salida
    sourcemap: true             // Genera sourcemaps para debug
  }
})

🔌 Plugins populares

  • @vitejs/plugin-react - Soporte para React con Fast Refresh
  • vite-plugin-pwa - Progressive Web App
  • vite-plugin-checker - TypeScript y ESLint en el servidor de dev
  • vite-plugin-compression - Compresión gzip/brotli

📝 Resumen para entrevistas

  • 🚀 Vite es el bundler moderno para React - Reemplaza CRA y Webpack
  • ⚡ Usa ES Modules en dev - Por eso es instantáneo (no bundlea)
  • 🔥 HMR ultra rápido - Solo actualiza el componente modificado
  • 📦 Usa Rollup en producción - Tree-shaking, code splitting, minificación
  • 🔧 Zero config - Funciona sin configuración para React
  • 🎯 esbuild - Transforma JSX 10-100x más rápido que Babel (escrito en Go)

💡 Tip: En entrevistas, menciona que Vite NO bundlea en desarrollo (usa ES Modules nativos), pero SÍ bundlea en producción con Rollup. Esto demuestra que entiendes la arquitectura.

5️⃣ Fast Refresh en acción

// ANTES de editar (el contador está en 5)
function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  )
}

// DESPUÉS de editar (cambias el texto)
function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Contador: {count}</p>  {/* ← Cambiaste esto */}
      <button onClick={() => setCount(count + 1)}>
        Incrementar  {/* ← Y esto */}
      </button>
    </div>
  )
}

// ✅ El componente se actualiza SIN recargar la página
// ✅ El state sigue siendo 5 (no se pierde)

6️⃣ Configuración personalizada

// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 3000,              // Cambiar puerto
    open: true,              // Abrir navegador automáticamente
    host: true               // Exponer en red local
  },
  resolve: {
    alias: {
      '@': '/src'            // Alias para imports (@/components/Button)
    }
  },
  build: {
    outDir: 'build',         // Cambiar carpeta de salida
    sourcemap: true,         // Generar sourcemaps
    minify: 'esbuild'        // Usar esbuild para minificar
  }
})

7️⃣ Tree Shaking - Eliminación de código no usado

// utils.js - exportamos 3 funciones
export function suma(a, b) { return a + b }
export function resta(a, b) { return a - b }
export function multiplica(a, b) { return a * b }

// App.jsx - solo usamos 1 función
import { suma } from './utils'

function App() {
  const resultado = suma(5, 3)
  return <div>{resultado}</div>
}

// ✅ En producción (npm run build):
// - Solo se incluye suma() en el bundle
// - resta() y multiplica() se eliminan (tree-shaking)

🔟 Importar assets dinámicamente

// Importar imagen dinámicamente
function Avatar({ userId }) {
  // La URL se resuelve correctamente en build
  const avatarUrl = new URL(`./avatars/${userId}.png`, import.meta.url).href

  return <img src={avatarUrl} alt="Avatar" />
}

// Alternativa: usar import estático
import avatar1 from './avatars/1.png'
import avatar2 from './avatars/2.png'

const avatars = { 1: avatar1, 2: avatar2 }

function Avatar({ userId }) {
  return <img src={avatars[userId]} alt="Avatar" />
}

📦 Ejercicio 3: Analizar el bundle de producción

Objetivo:

Ver cómo Vite optimiza tu código para producción.

Pasos:

  1. Ejecuta: npm run build
  2. Observa el output en la terminal (tamaños de archivos)
  3. Abre la carpeta dist/ generada

⚡ Ejercicio 4: Comparar con el enfoque CDN

Objetivo:

Entender por qué Vite es mejor que importar React desde CDN.

Pasos:

  1. Crea un archivo test-cdn.html:
<!DOCTYPE html>
<html>
<head>
  <script crossorigin src="https://unpkg.com/react@18/umd/react.development.js"></script>
  <script crossorigin src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
  <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
</head>
<body>
  <div id="root"></div>
  <script type="text/babel">
    function App() {
      return <h1>Hola desde CDN</h1>
    }
    ReactDOM.render(<App />, document.getElementById('root'))
  </script>
</body>
</html>
  1. Abre el navegador y ve a DevTools → Network
  2. Recarga la página y observa:
    • ❌ 3+ peticiones HTTP para librerías CDN
    • ❌ React en modo desarrollo (sin optimizar, ~1 MB)
    • ❌ Babel transforma JSX en el navegador (lento)

Ahora compara con Vite:

  1. En tu proyecto Vite, ve a Network
  2. Observa:
    • ✅ 1 archivo JS optimizado (~150 KB comprimido)
    • ✅ React en modo producción (minificado)
    • ✅ JSX ya transformado (no se hace en el navegador)

🔌 Ejercicio 6: Configurar un plugin

Objetivo:

Personalizar Vite con un plugin para cambiar el puerto y abrir el navegador automáticamente.

Pasos:

  1. Abre vite.config.js
  2. Modifica la configuración:
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 3000,        // ← Cambiar a puerto 3000
    open: true         // ← Abrir navegador automáticamente
  }
})
  1. Detén el servidor (Ctrl + C)
  2. Reinicia: npm run dev

✅ Verificación:

  • El servidor ahora corre en http://localhost:3000
  • El navegador se abre automáticamente al iniciar
