Tutorial: Componentes en React (Guía clara para entenderlos de verdad)

Antes de React, cuando hacíamos interfaces con JavaScript puro, el código se volvía algo así:

  • HTML + JS mezclado
  • Muchos querySelector
  • Muchos appendChild
  • Código repetido

React resuelve esto con una idea clave:

👉 Componentes

Un componente es una pieza reutilizable de la interfaz.

Piensa en ellos como bloques LEGO para construir interfaces.

1️⃣ ¿Qué es un componente?

Un componente es simplemente una función de JavaScript que devuelve JSX. 

function Saludo() {
  return <h1>¡Hola desde un componente!</h1>
}

Ese JSX React lo convertirá en HTML.

Usar el componente

Una vez creado, se usa como si fuera una etiqueta HTML:

<Saludo />

Esto es una de las ideas más importantes de React.

En vez de escribir HTML repetido:

<h1>Hola</h1>
<h1>Hola</h1>
<h1>Hola</h1>

Puedes usar:

<Saludo />
<Saludo />
<Saludo />

2️⃣ Diferencia entre elemento y componente

Esta pregunta aparece muchísimo en entrevistas React.

Elemento

Un elemento es JSX simple.

<h1>Hola</h1>

React lo renderiza directamente en el DOM.

Componente

Un componente es una función que devuelve elementos.

function Saludar() {
  return <h1>Hola</h1>
}

Luego se usa así:

<Saludar />

🎯 Regla clave para entrevistas:
Un componente es una función que retorna elementos JSX

3️⃣ Tu primer componente completo

Vamos a verlo en un ejemplo real.

HTML base

<div id="root"></div>

React siempre renderiza dentro de un contenedor.

Componente React

function Saludar() {
  return <h1>¡Hola desde un componente!</h1>
}

Renderizarlo

const rootEl = document.querySelector('#root')
const root = ReactDOMClient.createRoot(rootEl)

root.render(<Saludar />)

Qué ocurre internamente

<Saludar />
     ↓
ejecuta la función
     ↓
return <h1>
     ↓
lo convierte en DOM real

4️⃣ Props: pasar datos a los componentes

Los componentes pueden recibir datos.

Es como pasar argumentos a una función.

function Saludar(props) {
  return <h1>¡Hola {props.nombre}!</h1>
}

Uso

<Saludar nombre="Miguel" />

Resultado

Hola Miguel

Analogía simple

Un componente es como una máquina de café.

  • Componente = máquina
  • Props = ingredientes
  • UI = café final

Ejemplo:

<Saludar nombre="Miguel" />
<Saludar nombre="Ana" />

Misma máquina, diferentes ingredientes.

5️⃣ Destructuring de props

En React casi siempre verás esto:

function Saludar({ nombre }) {
  return <h1>Hola {nombre}</h1>
}

Esto es equivalente a:

props.nombre

Pero más limpio.

6️⃣ Múltiples props

Puedes pasar varios datos.

function Saludar({ nombre, edad, ciudad }) {
  return (
    <div>
      <h1>Hola {nombre}</h1>
      <p>Tienes {edad} años y vives en {ciudad}</p>
    </div>
  )
}

Uso

<Saludar nombre="Miguel" edad={25} ciudad="Madrid" />

Regla importante

Strings:

nombre="Miguel"

Valores JavaScript:

edad={25}

Las llaves significan:

👉 "Esto es JavaScript"

7️⃣ Estilos en JSX

En HTML normal haríamos:

<h1 style="color:red; font-size:32px">

Pero en JSX los estilos son objetos JavaScript.

<h1 style={{ color: 'red', fontSize: '32px' }}>

Ejemplo completo

function Saludar({ nombre, color }) {

  const estilos = {
    color: color,
    fontSize: '32px',
    fontWeight: 'bold',
    textAlign: 'center'
  }

  return <h1 style={estilos}>Hola {nombre}</h1>
}

Uso

<Saludar nombre="Miguel" color="blue" />
<Saludar nombre="Ana" color="green" />

8️⃣ CamelCase en estilos

En CSS usamos:

font-size
background-color
text-align

En JSX usamos:

fontSize
backgroundColor
textAlign

Porque JSX es JavaScript, no HTML.

9️⃣ Estilos inline con props

Podemos escribirlo más corto:

function Saludar({ nombre, color }) {
  return (
    <h1 style={{ color, fontSize: '32px' }}>
      Hola {nombre}
    </h1>
  )
}

Esto funciona porque en JavaScript:

color: color

Puede escribirse:

color

🔟 Reutilización de componentes

Aquí está el verdadero poder de React.

<Saludar nombre="Miguel" color="blue" />
<Saludar nombre="Ana" color="green" />
<Saludar nombre="Carlos" color="red" />

Un mismo componente genera interfaces diferentes.

1️⃣1️⃣ Composición de componentes

Los componentes se pueden combinar.

Ejemplo:

function App() {
  return (
    <div>
      <Saludar nombre="Miguel" />
      <Saludar nombre="Ana" />
    </div>
  )
}

Esto es como construir con bloques LEGO.

1️⃣2️⃣ Ventajas de los componentes

  • Reutilización: Escribes una vez, usas muchas.
  • Organización: Cada componente tiene su lógica.
  • Mantenibilidad: Es más fácil cambiar cosas.
  • Escalabilidad: Las apps grandes se construyen con muchos componentes pequeños.

1️⃣3️⃣ Mental model (muy importante)

Piensa en React así:

App
 ├── Header
 ├── Sidebar
 ├── ProductCard
 ├── ProductCard
 ├── ProductCard
 └── Footer

Cada parte de la UI es un componente.

1️⃣4️⃣ Resumen final

Los componentes son:

Funciones
     ↓
Reciben props
     ↓
Retornan JSX
     ↓
React lo convierte en DOM

Lo que debes recordar

  • ✔ Componentes = funciones que devuelven JSX
  • ✔ Props = datos que recibe el componente
  • ✔ JSX = JavaScript + HTML
  • ✔ Estilos = objetos JavaScript
  • ✔ Componentes = reutilización

Ejemplo 5: Componente Reutilizable (bloques LEGO)


function Botón({ texto, color, onClick }) {
  const estilosBotón = {
    padding: '10px 20px',
    backgroundColor: color,
    color: 'white',
    border: 'none',
    borderRadius: '4px',
    cursor: 'pointer',
    fontSize: '16px'
  }

  return (
    <button style={estilosBotón} onClick={onClick}>
      {texto}
    </button>
  )
}

// Uso
function App() {
  return (
    <div>
      <Botón texto="¡Hazme clic!" color="blue" onClick={() => alert('¡Hecho!')} />
      <Botón texto="Guardar" color="green" onClick={() => alert('¡Guardado!')} />
      <Botón texto="Eliminar" color="red" onClick={() => alert('¡Eliminado!')} />
    </div>
  )
}

Ejemplo 7: Componente Complejo con Múltiples Props


function TarjetaProducto({ título, precio, descripción, imagen }) {
  const estilosTarjeta = {
    border: '1px solid #ddd',
    borderRadius: '8px',
    overflow: 'hidden',
    maxWidth: '300px',
    boxShadow: '0 4px 6px rgba(0,0,0,0.1)'
  }

  const estilosImagen = {
    width: '100%',
    height: '200px',
    objectFit: 'cover'
  }

  const estilosContenido = {
    padding: '16px'
  }

  const estilosTítulo = {
    fontSize: '18px',
    fontWeight: 'bold',
    marginBottom: '8px'
  }

  const estilosPrecio = {
    fontSize: '20px',
    color: '#28a745',
    fontWeight: 'bold',
    marginBottom: '8px'
  }

  return (
    <div style={estilosTarjeta}>
      <img src={imagen} alt={título} style={estilosImagen} />
      <div style={estilosContenido}>
        <h3 style={estilosTítulo}>{título}</h3>
        <p style={estilosPrecio}>${precio}</p>
        <p>{descripción}</p>
      </div>
    </div>
  )
}

// Uso
<TarjetaProducto
  título="Laptop"
  precio={999}
  descripción="Potente y portátil"
  imagen="laptop.jpg"
/>

Ejercicios Prácticos

Ejercicio 1: Componente Saludo Simple

Desafío: Crea un componente Saludo que:

  • Acepte una prop llamada hora (mañana, tarde, noche)
  • Retorne diferentes saludos según la hora:
    • Mañana: "¡Buenos días!"
    • Tarde: "¡Buenas tardes!"
    • Noche: "¡Buenas noches!"

Uso esperado:

<Saludo hora="mañana" />  // "¡Buenos días!"

Solución:


function Saludo({ hora }) {
  const saludos = {
    mañana: '¡Buenos días!',
    tarde: '¡Buenas tardes!',
    noche: '¡Buenas noches!'
  }

  return <h1>{saludos[hora]}</h1>
}

Ejercicio 2: Componente Perfil de Usuario

Desafío: Crea un componente PerfilUsuario que:

  • Acepte props: nombre, rol, email, avatar
  • Muestre la información del usuario en una tarjeta estilizada
  • Incluya una imagen de avatar

Uso esperado:


<PerfilUsuario
  nombre="Juan Pérez"
  rol="Desarrollador"
  email="juan@example.com"
  avatar="juan.jpg"
/>

Solución:


function PerfilUsuario({ nombre, rol, email, avatar }) {
  const estilosPerfil = {
    textAlign: 'center',
    padding: '20px',
    border: '1px solid #ddd',
    borderRadius: '8px'
  }

  const estilosAvatar = {
    width: '100px',
    height: '100px',
    borderRadius: '50%',
    objectFit: 'cover',
    marginBottom: '10px'
  }

  return (
    <div style={estilosPerfil}>
      <img src={avatar} alt={nombre} style={estilosAvatar} />
      <h2>{nombre}</h2>
      <p><strong>{rol}</strong></p>
      <p>{email}</p>
    </div>
  )
}

Ejercicio 3: Componente Alerta

Desafío: Crea un componente Alerta que:

  • Acepte props: tipo (éxito, error, advertencia, info), mensaje
  • Muestre diferentes colores según el tipo:
    • Éxito: verde
    • Error: rojo
    • Advertencia: naranja
    • Info: azul
  • Muestre el mensaje

Uso esperado:


<Alerta tipo="éxito" mensaje="¡Datos guardados con éxito!" />
<Alerta tipo="error" mensaje="¡Algo salió mal!" />

Solución:


function Alerta({ tipo, mensaje }) {
  const colores = {
    éxito: '#28a745',
    error: '#dc3545',
    advertencia: '#ffc107',
    info: '#17a2b8'
  }

  const estilosAlerta = {
    padding: '12px 16px',
    borderRadius: '4px',
    backgroundColor: colores[tipo],
    color: tipo === 'advertencia' ? '#000' : '#fff',
    marginBottom: '10px'
  }

  return <div style={estilosAlerta}>{mensaje}</div>
}

Ejercicio 4: Composición Múltiple de Componentes

Desafío: Crea una app completa con:

  • Un componente Encabezado
  • Un componente Tarjeta
  • Un componente App que use ambos

Solución:


function Encabezado() {
  return <header style={{ background: '#333', color: '#fff', padding: '20px' }}>
    <h1>Mi Sitio Web</h1>
  </header>
}

function Tarjeta({ título, contenido }) {
  const estiloTarjeta = {
    border: '1px solid #ddd',
    padding: '16px',
    margin: '10px',
    borderRadius: '8px'
  }

  return (
    <div style={estiloTarjeta}>
      <h3>{título}</h3>
      <p>{contenido}</p>
    </div>
  )
}

function App() {
  return (
    <div>
      <Encabezado />
      <Tarjeta título="Tarjeta 1" contenido="Esta es la primera tarjeta" />
      <Tarjeta título="Tarjeta 2" contenido="Esta es la segunda tarjeta" />
    </div>
  )
}

export default App;

Ejercicio 5: Desafío Interactivo

Desafío: Crea un componente Insignia que:

  • Muestre una insignia con un número
  • Tenga color configurable
  • Pueda ser de diferentes tamaños (pequeña, mediana, grande)

Uso esperado:


<Insignia número={5} color="red" tamaño="pequeña" />
<Insignia número={12} color="blue" tamaño="grande" />

Solución:


function Insignia({ número, color = 'blue', tamaño = 'mediana' }) {
  const tamaños = {
    pequeña: { width: '24px', height: '24px', fontSize: '12px' },
    mediana: { width: '32px', height: '32px', fontSize: '16px' },
    grande: { width: '40px', height: '40px', fontSize: '20px' }
  }

  const estilosInsignia = {
    ...tamaños[tamaño],
    borderRadius: '50%',
    backgroundColor: color,
    color: '#fff',
    display: 'flex',
    alignItems: 'center',
    justifyContent: 'center',
    fontWeight: 'bold'
  }

  return <div style={estilosInsignia}>{número}</div>
}
