Paginación con la API: limit y offset

🔎 El problema inicial

Ya tenemos filtros funcionando con la API, pero la paginación no está integrada. El estado existe en el frontend pero la petición nunca lo usa: 

// Estado actual:
✅ Filtros se envían correctamente a la API
✅ Paginación existe en el frontend
❌ Paginación NO funciona con la API — cada cambio de página no hace una nueva petición

📐 ¿Cómo funcionan las APIs de paginación?

La mayoría de APIs no usan un parámetro page. Usan dos parámetros fundamentales: limit y offset.

Parámetro Descripción Ejemplo
limit Cuántos resultados devolver por página 10
offset Cuántos resultados saltarse desde el inicio 0, 10, 20… 
// Página 1 → ?limit=10&offset=0   → resultados del 0 al 9
// Página 2 → ?limit=10&offset=10  → resultados del 10 al 19
// Página 3 → ?limit=10&offset=20  → resultados del 20 al 29

🧮 La fórmula del offset

Para calcular el offset a partir del número de página:

offset = (currentPage - 1) * RESULTS_PER_PAGE
Página actual Resultados por página Offset Cálculo
1 10 0 (1-1) × 10 = 0
2 10 10 (2-1) × 10 = 10
3 10 20 (3-1) × 10 = 20
5 20 80 (5-1) × 20 = 80

🐛 Error crítico: total de páginas incorrecto

Al calcular el total de páginas, hay un error muy común: 

// ❌ Error: jobs.length es solo la página actual (siempre 10 o menos)
const totalPages = Math.ceil(jobs.length / RESULTS_PER_PAGE)
// → Math.ceil(10 / 10) = 1 página... siempre

// ✅ Correcto: usar el total real que devuelve la API
const totalPages = Math.ceil(total / RESULTS_PER_PAGE)
// → Math.ceil(143 / 10) = 15 páginas

Regla: jobs.length es el número de resultados de la página actual. El total que devuelve la API es el número real de registros en la base de datos. Son dos cosas completamente distintas.

🔄 Reiniciar paginación al cambiar filtros

Si el usuario está en la página 5 de "JavaScript" y cambia a "Python" (que solo tiene 2 páginas), quedarse en la página 5 no tiene sentido. Hay que volver a página 1 al cambiar cualquier filtro: 

// ✅ Segundo useEffect que solo reinicia la página
useEffect(() => {
  setCurrentPage(1)
}, [textToFilter, filters.technology, filters.location, filters.experienceLevel])

🏗️ useEffect con paginación integrada

Agregamos limit, offset y currentPage como dependencia: 

const RESULTS_PER_PAGE = 10

useEffect(() => {
  async function fetchJobs() {
    try {
      setLoading(true)

      const params = new URLSearchParams()

      // Filtros
      if (textToFilter)              params.append('text',       textToFilter)
      if (filters.technology)        params.append('technology', filters.technology)
      if (filters.location)          params.append('type',       filters.location)
      if (filters.experienceLevel)   params.append('level',      filters.experienceLevel)

      // Paginación
      const offset = (currentPage - 1) * RESULTS_PER_PAGE
      params.append('limit',  RESULTS_PER_PAGE)
      params.append('offset', offset)

      const res = await fetch(
        `https://jscamp-api.vercel.app/api/jobs?${params.toString()}`
      )
      const json = await res.json()

      setJobs(json.data)
      setTotal(json.total)
    } catch (error) {
      console.error('Error fetching jobs:', error)
    } finally {
      setLoading(false)
    }
  }

  fetchJobs()
}, [textToFilter, filters.technology, filters.location, filters.experienceLevel, currentPage])
//                                                                                 ↑ clave

🔄 Flujo completo con dos useEffect

function JobSearch() {
  const [jobs, setJobs]           = useState([])
  const [loading, setLoading]     = useState(true)
  const [total, setTotal]         = useState(0)
  const [currentPage, setCurrentPage] = useState(1)
  const [textToFilter, setTextToFilter] = useState('')
  const [filters, setFilters] = useState({
    technology: '',
    location: '',
    experienceLevel: '',
  })

  // Total de páginas — usa el total de la API, no jobs.length
  const totalPages = Math.ceil(total / RESULTS_PER_PAGE)

  // useEffect 1: reiniciar página cuando cambia un filtro
  useEffect(() => {
    setCurrentPage(1)
  }, [textToFilter, filters.technology, filters.location, filters.experienceLevel])

  // useEffect 2: fetch cada vez que cambia página o filtros
  useEffect(() => {
    async function fetchJobs() {
      try {
        setLoading(true)

        const params = new URLSearchParams()
        if (textToFilter)              params.append('text',       textToFilter)
        if (filters.technology)        params.append('technology', filters.technology)
        if (filters.location)          params.append('type',       filters.location)
        if (filters.experienceLevel)   params.append('level',      filters.experienceLevel)

        const offset = (currentPage - 1) * RESULTS_PER_PAGE
        params.append('limit',  RESULTS_PER_PAGE)
        params.append('offset', offset)

        const res = await fetch(
          `https://jscamp-api.vercel.app/api/jobs?${params.toString()}`
        )
        const json = await res.json()

        setJobs(json.data)
        setTotal(json.total)
      } catch (error) {
        console.error('Error fetching jobs:', error)
      } finally {
        setLoading(false)
      }
    }

    fetchJobs()
  }, [textToFilter, filters.technology, filters.location, filters.experienceLevel, currentPage])

  const nextPage = () => { if (currentPage < totalPages) setCurrentPage(p => p + 1) }
  const prevPage = () => { if (currentPage > 1) setCurrentPage(p => p - 1) }

  return (
    <div>
      <SearchForm ... />
      {loading ? (
        <div>Cargando empleos...</div>
      ) : (
        <>
          <JobListing jobs={jobs} />
          <Pagination
            currentPage={currentPage}
            totalPages={totalPages}
            onNext={nextPage}
            onPrev={prevPage}
          />
        </>
      )}
    </div>
  )
}

📋 Flujo completo paso a paso

1. Usuario escribe "JavaScript"
   └─ textToFilter cambia
   └─ useEffect 1 ejecuta: setCurrentPage(1)
   └─ currentPage cambia a 1
   └─ useEffect 2 ejecuta: fetch con página 1 + text=javascript

2. Usuario navega a página 3
   └─ setCurrentPage(3)
   └─ useEffect 2 ejecuta: fetch con página 3 + text=javascript
   └─ URL → ?text=javascript&limit=10&offset=20

3. Usuario cambia filtro a "Python"
   └─ filters.technology cambia
   └─ useEffect 1 ejecuta: setCurrentPage(1)   ← reset
   └─ useEffect 2 ejecuta: fetch con página 1 + technology=python

🧩 Componente Pagination reutilizable

function Pagination({ currentPage, totalPages, onNext, onPrev, onGoToPage }) {
  return (
    <div className="pagination">
      <button onClick={onPrev} disabled={currentPage === 1}>
        Anterior
      </button>

      <span>Página {currentPage} de {totalPages}</span>

      <button onClick={onNext} disabled={currentPage === totalPages}>
        Siguiente
      </button>

      {/* Números de página opcionales */}
      {Array.from({ length: totalPages }, (_, i) => i + 1).map((page) => (
        <button
          key={page}
          onClick={() => onGoToPage(page)}
          className={page === currentPage ? 'active' : ''}
        >
          {page}
        </button>
      ))}
    </div>
  )
}

🔍 Verificando en DevTools → Network

Al cambiar de página deberías ver peticiones como estas:

Página 1 sin filtros:  /api/jobs?limit=10&offset=0
Página 2 sin filtros:  /api/jobs?limit=10&offset=10
Página 3 sin filtros:  /api/jobs?limit=10&offset=20

Con filtros activos:
/api/jobs?text=javascript&technology=react&limit=10&offset=20

Y la respuesta de la API debe incluir:

{
  "total": 143,   // ← el que usamos para calcular totalPages
  "limit": 10,
  "offset": 20,
  "results": 10,
  "data": [ /* empleos de esta página */ ]
}

🐛 Error 1: loop infinito por objeto como dependencia

// ❌ Peligroso — filters es un objeto nuevo en cada render
useEffect(() => {
  // ...
}, [filters])

// ✅ Correcto — valores primitivos
useEffect(() => {
  // ...
}, [filters.technology, filters.location, filters.experienceLevel])

🐛 Error 2: total de páginas calculado con jobs.length

// ❌ jobs.length siempre es el tamaño de la página actual
const totalPages = Math.ceil(jobs.length / RESULTS_PER_PAGE)
// → Math.ceil(10/10) = 1 ... siempre 1 página

// ✅ total viene de la API — es el count real de la base de datos
const totalPages = Math.ceil(total / RESULTS_PER_PAGE)
// → Math.ceil(143/10) = 15 páginas

🐛 Error 3: no reiniciar paginación al filtrar

// ❌ El usuario puede quedar en página 10 cuando Python solo tiene 2
// No hay nada que reinicie currentPage al cambiar filtros

// ✅ useEffect dedicado solo al reset
useEffect(() => {
  setCurrentPage(1)
}, [textToFilter, filters.technology, filters.location, filters.experienceLevel])

🐛 Error 4: falta currentPage en las dependencias del fetch

// ❌ El fetch no se ejecuta al cambiar de página
useEffect(() => {
  fetchJobs()
}, [textToFilter, filters.technology]) // ← falta currentPage

// ✅ currentPage debe estar en las dependencias
useEffect(() => {
  fetchJobs()
}, [textToFilter, filters.technology, filters.location, filters.experienceLevel, currentPage])

  3. Verificá que el total de páginas se actualiza correctamente

✅ Buenas prácticas y ❌ qué evitar

✅ Hacer siempre:

  • Usar total de la API para calcular totalPages, nunca jobs.length
  • Incluir currentPage en las dependencias del useEffect de fetch
  • Tener un useEffect separado que reinicie la página al cambiar filtros
  • Deshabilitar los botones Anterior/Siguiente cuando corresponde (disabled)
  • Verificar en DevTools que el offset cambia correctamente entre páginas

❌ Nunca hacer:

  • Usar jobs.length para calcular el total de páginas
  • Pasar el objeto filters completo como dependencia
  • Omitir currentPage del array de dependencias
  • Olvidar reiniciar la página cuando cambia un filtro

🚀 Próximos pasos
