Estado en React con useState: La Memoria del Componente

Hasta ahora nuestros componentes podían mostrar información y responder a eventos, pero no recordaban nada.

Si hacías clic en un botón, la función se ejecutaba… pero el componente no cambiaba visualmente.

Para solucionar esto, React nos da estado (state).

1️⃣ ¿Qué es el estado en React?

El estado es la memoria de un componente.

Es donde guardamos información que puede cambiar con el tiempo.

Ejemplos de cosas que suelen estar en estado:

  • Si un botón fue clicado
  • El texto de un input
  • Un contador
  • Datos cargados de una API
  • Si un modal está abierto

2️⃣ Analogía: El interruptor de luz

Imagina un interruptor:

apagado  →  encendido

Cuando presionas el interruptor:

  • cambia su estado
  • la luz cambia automáticamente

React funciona igual.

estado cambia → React actualiza la UI

Ejemplo:

aplicado = false
  ↓
usuario hace click
  ↓
aplicado = true
  ↓
React actualiza el botón

3️⃣ Problema sin estado

Sin estado, un componente no recuerda nada.

function Boton() {
  return <button>Haz clic</button>
}

Este botón siempre será igual.

No puede recordar si hiciste clic.

4️⃣ Introduciendo useState

React tiene un Hook para manejar estado:

useState

Ejemplo simple:

function Boton() {
  const [clicks, setClicks] = React.useState(0)

  return (
    <button onClick={() => setClicks(clicks + 1)}>
      Clicks: {clicks}
    </button>
  )
}

Ahora el botón:

  • recuerda los clics
  • actualiza el número
  • React re-renderiza automáticamente

5️⃣ Cómo funciona useState

La sintaxis:

const [estado, setEstado] = React.useState(valorInicial)

Ejemplo real:

const [aplicado, setAplicado] = React.useState(false)

Significado:

React.useState(false)
  ↓
crea estado con valor inicial

aplicado
↓
valor actual del estado

setAplicado
↓
función para cambiar el estado

6️⃣ ¿Por qué usamos destructuring?

useState devuelve un array.

Esto:

const resultado = React.useState(false)
const aplicado = resultado[0]
const setAplicado = resultado[1]

Se simplifica con destructuring:

const [aplicado, setAplicado] = React.useState(false)

Mucho más limpio.

7️⃣ Ejemplo real: JobCard

Vamos a mejorar nuestro componente de empleo.

Queremos que el botón:

  • Cambie a "Aplicado"
  • Se desactive
  • Cambie el estilo

Componente completo

function JobCard({ titulo, empresa, ubicacion, descripcion }) {
  const [aplicado, setAplicado] = React.useState(false)

  const handleAplicar = () => {
    setAplicado(true)
  }

  return (
    <article className="job-listing-card">
      <div>
        <h3>{titulo}</h3>
        <small>
          {empresa} | {ubicacion}
        </small>
        <p>{descripcion}</p>
      </div>

      <button
        className={aplicado ? "button-apply-job is-applied" : "button-apply-job"}
        onClick={handleAplicar}
        disabled={aplicado}
      >
        {aplicado ? "¡Aplicado!" : "Aplicar"}
      </button>

    </article>
  )
}

8️⃣ Flujo completo de React

Cuando el usuario hace clic ocurre esto:

Estado inicial
aplicado = false
  ↓
Usuario hace click
  ↓
handleAplicar()
  ↓
setAplicado(true)
  ↓
React actualiza el estado
  ↓
React re-renderiza el componente
  ↓
La UI cambia automáticamente

Esto es React declarativo.

No manipulas el DOM manualmente.

9️⃣ Usando estado en JSX

El estado controla tres cosas:

  • Estilo
  • Estado del botón
  • Texto
<button
  className={aplicado ? "button-apply-job is-applied" : "button-apply-job"}
  disabled={aplicado}
>
  {aplicado ? "¡Aplicado!" : "Aplicar"}
</button>

Si aplicado === true:

  • texto → "¡Aplicado!"
  • botón → disabled
  • clase → is-applied

🔟 El operador ternario

El operador ternario es como un if corto.

condición ? valorSiTrue : valorSiFalse

Ejemplo:

aplicado ? "¡Aplicado!" : "Aplicar"

Equivalente a:

if (aplicado) {
  return "¡Aplicado!"
} else {
  return "Aplicar"
}

Pero JSX solo permite expresiones, no if.

Por eso usamos ternarios.

1️⃣1️⃣ Antes vs Ahora

Antes (JavaScript vanilla)

Manipulábamos el DOM manualmente.

jobsListingSection.addEventListener('click', function (event) {
  const element = event.target

  if (element.classList.contains('button-apply-job')) {
    element.textContent = '¡Aplicado!'
    element.classList.add('is-applied')
    element.disabled = true
  }
})

Problemas:

  • ❌ Manipulación manual del DOM
  • ❌ Lógica compleja
  • ❌ Difícil de mantener
  • ❌ Propenso a errores

Ahora (React)

function JobCard() {
  const [aplicado, setAplicado] = React.useState(false)

  return (
    <button
      onClick={() => setAplicado(true)}
      disabled={aplicado}
    >
      {aplicado ? "¡Aplicado!" : "Aplicar"}
    </button>
  )
}

Ventajas:

  • ✅ UI sincronizada con estado
  • ✅ Código declarativo
  • ✅ Fácil de testear
  • ✅ Cada componente tiene su propio estado

1️⃣2️⃣ Cada componente tiene su propio estado

Ejemplo:

<JobCard titulo="Desarrollador Frontend" />
<JobCard titulo="Desarrollador Backend" />

React crea dos instancias del componente.

  • JobCard 1 → estado independiente
  • JobCard 2 → estado independiente

Si aplicas al primero:

  • Frontend → aplicado
  • Backend → sin aplicar

No se afectan entre sí.

1️⃣3️⃣ Tipos de estado

Puedes guardar cualquier tipo en estado.

Boolean

const [visible, setVisible] = React.useState(false)

Número

const [count, setCount] = React.useState(0)

String

const [nombre, setNombre] = React.useState("")

Array

const [empleos, setEmpleos] = React.useState([])

Objeto

const [usuario, setUsuario] = React.useState({
  nombre: "Miguel",
  edad: 25
})

1️⃣4️⃣ Múltiples estados

Un componente puede tener varios estados.

function JobCard({ titulo }) {
  const [aplicado, setAplicado] = React.useState(false)
  const [guardado, setGuardado] = React.useState(false)
  const [likes, setLikes] = React.useState(0)

  return (
    <article>
      <h3>{titulo}</h3>

      <button onClick={() => setAplicado(true)}>
        {aplicado ? "Aplicado" : "Aplicar"}
      </button>

      <button onClick={() => setGuardado(!guardado)}>
        {guardado ? "❤️ Guardado" : "🤍 Guardar"}
      </button>

      <button onClick={() => setLikes(likes + 1)}>
        👍 {likes}
      </button>
    </article>
  )
}

Cada estado funciona independientemente.

1️⃣5️⃣ Actualizar estado basado en el anterior

A veces necesitas usar el valor anterior.

Forma segura:

setCount(prevCount => prevCount + 1)

Ejemplo:

function Contador() {
  const [count, setCount] = React.useState(0)

  const incrementar = () => {
    setCount(prev => prev + 1)
  }

  return (
    <button onClick={incrementar}>
      {count}
    </button>
  )
}

Esto evita errores cuando React agrupa actualizaciones.

1️⃣6️⃣ Reglas de useState

1. Solo en el nivel superior

❌ Incorrecto:

if (algo) {
  const [estado, setEstado] = React.useState(false)
}

2. Solo en componentes React

❌ Incorrecto:

function miFuncion() {
  const [estado, setEstado] = React.useState(0)
}

3. Nunca modificar el estado directamente

❌ Incorrecto:

aplicado = true

✅ Correcto:

setAplicado(true)

1️⃣7️⃣ Idea clave para entrevistas

En entrevistas React esperan escuchar esto:

"React UI is a function of state."
UI = f(state)

Significa:

estado cambia
  ↓
React re-renderiza
  ↓
la UI se actualiza

📋 Lo que debes recordar

  • Estado = memoria del componente
  • Cuando el estado cambia:
    • React vuelve a renderizar el componente
    • La UI se actualiza automáticamente
  • Reglas clave:
    • ✔ usar useState
    • ✔ nunca modificar estado directamente
    • ✔ usar setState
    • ✔ cada componente tiene su propio estado

1️⃣8️⃣ Analogías del Mundo Real para Recordar State

🚦 Analogía 1: El Semáforo

Props (Características Fijas)

  • El poste
  • Colores de las luces (rojo, amarillo, verde)
  • Posición
  • Orientación (vertical/horizontal)

State (Condición Actual)

🔴 Rojo
🟡 Amarillo
🟢 Verde

Solo UNA luz está activa a la vez. ¡Esto es exactamente como React state!

const [color, setColor] = React.useState("red")

Los Eventos Cambian el State

pasa tiempo
  ↓
rojo → amarillo → verde → rojo
  ↓
setColor("yellow")

🌡️ Analogía 2: El Termómetro

Analogía perfecta para state numérico.

Props

  • Escala (Celsius o Fahrenheit)
  • Tipo
  • Color

State

const [temperatura, setTemperatura] = React.useState(20)

Cuando Cambia la Temperatura

cambia el clima
  ↓
setTemperatura(25)
  ↓
React re-renderiza
  ↓
la pantalla cambia: 20°C → 25°C

📝 Analogía 3: El Formulario (Ejemplo Más Real)

Props

  • Texto placeholder
  • Label
  • Tipo de input

State

const [nombre, setNombre] = React.useState("")

Los Eventos Actualizan el State

<input
  value={nombre}
  onChange={(e) => setNombre(e.target.value)}
/>

React Renderiza el Cambio

usuario escribe
  ↓
onChange dispara
  ↓
setNombre() actualiza state
  ↓
React re-renderiza
  ↓
pantalla muestra texto nuevo

1️⃣9️⃣ La Ecuación Fundamental de React

UI = f(state)

Qué significa:

  • La UI es una función del state
  • Cuando state cambia, la UI cambia automáticamente
  • No necesitas manipular el DOM manualmente

El Flujo:

state cambia
  ↓
React re-renderiza componente
  ↓
UI nueva aparece automáticamente

2️⃣0️⃣ Modelo Mental: ¿Qué Debería Ser State?

Regla de oro: Si cambia con el tiempo, probablemente es state.

Ejemplos de State:

  • ✅ Contador (incrementa)
  • ✅ Valor de input (usuario escribe)
  • ✅ Visibilidad del modal (abierto/cerrado)
  • ✅ Usuario logeado (verdadero/falso)
  • ✅ Contador de likes (aumenta)
  • ✅ Toggle switches

Ejemplos de Props (NO State):

  • ❌ Etiqueta de botón (fija)
  • ❌ Colores del componente (fijos)
  • ❌ Datos iniciales (estáticos)

Ejemplo 4: Formulario con múltiples estados


function FormularioSimple() {
  const [nombre, setNombre] = React.useState("")
  const [email, setEmail] = React.useState("")
  const [enviado, setEnviado] = React.useState(false)

  const handleSubmit = (event) => {
    event.preventDefault()
    setEnviado(true)
  }

  if (enviado) {
    return (
      <div>
        <h2>¡Gracias, {nombre}!</h2>
        <p>Te contactaremos en {email}</p>
        <button onClick={() => setEnviado(false)}>
          Resetear
        </button>
      </div>
    )
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={nombre}
        onChange={(e) => setNombre(e.target.value)}
        placeholder="Tu nombre"
      />
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="Tu email"
      />
      <button type="submit">Enviar</button>
    </form>
  )
}

Ejemplo 5: Botón de Me gusta


function BotonMeGusta() {
  const [likes, setLikes] = React.useState(0)
  const [liked, setLiked] = React.useState(false)

  const handleLike = () => {
    if (!liked) {
      setLikes(likes + 1)
    } else {
      setLikes(likes - 1)
    }
    setLiked(!liked)
  }

  return (
    <button onClick={handleLike}>
      {liked ? "❤️" : "🤍"} {likes}
    </button>
  )
}

Ejemplo 6: Lista con agregar/eliminar


function ListaTareas() {
  const [tareas, setTareas] = React.useState([])
  const [input, setInput] = React.useState("")

  const agregarTarea = () => {
    if (input.trim()) {
      setTareas([...tareas, input])
      setInput("")
    }
  }

  const eliminarTarea = (index) => {
    setTareas(tareas.filter((_, i) => i !== index))
  }

  return (
    <div>
      <input
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Agregar una tarea"
      />
      <button onClick={agregarTarea}>Agregar</button>

      <ul>
        {tareas.map((tarea, index) => (
          <li key={index}>
            {tarea}
            <button onClick={() => eliminarTarea(index)}>
              Eliminar
            </button>
          </li>
        ))}
      </ul>
    </div>
  )
}

Ejercicios Prácticos

Ejercicio 1: Contador básico

Desafío: Crea un contador que incremente y decremente.

Solución:


function Contador() {
  const [count, setCount] = React.useState(0)

  return (
    <div>
      <p>Contador: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        +
      </button>
      <button onClick={() => setCount(count - 1)}>
        -
      </button>
      <button onClick={() => setCount(0)}>
        Resetear
      </button>
    </div>
  )
}

function Alternador() {

Ejercicio 4: Renderizado condicional

Desafío: Muestra/oculta contenido basado en estado.

Solución:


function Acordeon() {
  const [expandido, setExpandido] = React.useState(false)

  return (
    <div>
      <button onClick={() => setExpandido(!expandido)}>
        {expandido ? "▼" : "▶"} Detalles
      </button>

      {expandido && (

          ¡Este contenido oculto se alterna!
        </div>
      )}
    </div>
  )
}

Ejercicio 5: Actualizar múltiples estados

Desafío: Crea un componente con múltiples valores de estado independientes.

Solución:


function PerfilUsuario() {
  const [nombre, setNombre] = React.useState("")
  const [edad, setEdad] = React.useState("")
  const [enviado, setEnviado] = React.useState(false)

  const handleSubmit = () => {
    if (nombre && edad) {
      setEnviado(true)
    }
  }

  if (enviado) {
    return (
      <div>
        <h2>Perfil: {nombre}, Edad {edad}</h2>
        <button onClick={() => setEnviado(false)}>
          Editar
        </button>
      </div>
    )
  }

  return (
    <div>
      <input
        value={nombre}
        onChange={(e) => setNombre(e.target.value)}
        placeholder="Nombre"
      />
      <input
        value={edad}
        onChange={(e) => setEdad(e.target.value)}
        placeholder="Edad"
      />
      <button onClick={handleSubmit}>Guardar</button>
    </div>
  )
}
