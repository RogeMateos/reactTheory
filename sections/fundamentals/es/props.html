Props — Comunicación entre Componentes

🎁 ¿Qué son las Props?

Props (abreviatura de properties) son argumentos que pasás a un componente React, de la misma forma que pasás argumentos a una función. 

// Función normal
function saludar(nombre) {
  return `Hola ${nombre}`
}
saludar('Miguel')  // "Hola Miguel"

// Componente React — misma idea
function Saludo({ nombre }) {
  return <h1>Hola {nombre}</h1>
}
<Saludo nombre="Miguel" />  // <h1>Hola Miguel</h1>

Las props tienen 4 características clave:

📦 Son objetos con los datos que le pasás al componente
⬇️ Fluyen de padre a hijo (nunca al revés)
🔒 Son inmutables (no se modifican dentro del componente)
🔄 Hacen que los componentes sean reutilizables

⬇️ Flujo Unidireccional

Las props siempre van de padre a hijo. El flujo es unidireccional:

App (padre)
  │
  ├─ pasa: title="Frontend Developer"
  ├─ pasa: company="TechCorp"
  │
  ▼
JobCard (hijo)
  └─ usa: {title} y {company} para renderizar

⚠️ Regla fundamental: Un hijo nunca puede pasar props a su padre directamente. Para eso se usan callbacks, que veremos más adelante.

🔓 Desestructuración de Props

Hay dos formas de recibir props. La segunda es la recomendada:

// ❌ Sin desestructuración — repetitivo y verboso
function JobCard(props) {
  return (
    <article>
      <h3>{props.title}</h3>
      <p>{props.company}</p>
    </article>
  )
}

// ✅ Con desestructuración — limpio y legible
function JobCard({ title, company }) {
  return (
    <article>
      <h3>{title}</h3>
      <p>{company}</p>
    </article>
  )
}

Ventajas de desestructurar:

✅ Código más limpio y legible
✅ Ves inmediatamente qué props usa el componente
✅ Menos repetición de código

🎯 Props por Defecto

Podés asignar un valor por defecto si la prop no se proporciona:

function JobCard({ title, company, location = 'Remoto' }) {
  return (
    <article>
      <h3>{title}</h3>
      <p>{company}</p>
      <p>{location}</p>  {/* "Remoto" si no se pasa */}
    </article>
  )
}

// Sin location → usa "Remoto"
<JobCard title="Frontend Dev" company="TechCorp" />

// Con location → usa ese valor
<JobCard title="Frontend Dev" company="TechCorp" location="Madrid" />

❓ Renderizado Condicional

Podés usar props para mostrar u ocultar contenido. Hay dos formas:

Operador && — muestra si es true, no muestra nada si es false:

{isRemote && <span>🏠 Remoto</span>}
{isFeatured && <span>⭐ Destacado</span>}

⚠️ Trampa clásica con &&: si el valor no es booleano sino un número, React renderiza el número 0 en pantalla.

{results.length && <span>Resultados</span>}
Si results.length es 0, React renderiza el número 0 en la UI.

Ternaria — la opción más segura y expresiva:

{isRemote ? <span>🏠 Remoto</span> : null}
{results.length > 0 ? <span>Resultados</span> : null}

La ternaria funciona siempre correctamente porque el resultado es explícito.

🧩 Fragments — Agrupar sin contaminar el DOM

React tiene una regla: un componente solo puede retornar un único elemento raíz. La solución tradicional es envolver en un <div>, pero eso agrega nodos innecesarios al DOM. 

// ❌ Error — múltiples elementos sin wrapper
function JobActions() {
  return (
    <button>Aplicar</button>
    <button>Guardar</button>
  )
}

// ⚠️ Funciona, pero agrega un <div> innecesario
function JobActions() {
  return (
    <div>
      <button>Aplicar</button>
      <button>Guardar</button>
    </div>
  )
}

// ✅ Fragment — agrupa sin agregar nodos al DOM
function JobActions() {
  return (
    <>
      <button>Aplicar</button>
      <button>Guardar</button>
    </>
  )
}

Regla simple para elegir:

Necesitás estilos o event handler en el wrapper  → <div>
Solo agrupás elementos sin necesitar el wrapper  → <></>

📦 Tipos de Valores que se Pueden Pasar como Props

// String
<JobCard title="Frontend Developer" location="Madrid" />

// Number — siempre entre llaves, nunca entre comillas
<JobCard salary={45000} experience={3} />

// Boolean
<JobCard isRemote={true} isFeatured={false} />
<JobCard isRemote />          {/* shorthand para true */}

// Array
<JobCard tags={['React', 'TypeScript', 'CSS']} />

// Object
<JobCard
  job={{
    title: 'Frontend Developer',
    company: 'TechCorp',
    salary: 45000,
  }}
/>

// Function (callback)
<JobCard onApply={() => console.log('Aplicando...')} />

🏗️ Desestructuración Anidada

Cuando pasás un objeto como prop, podés desestructurar su contenido de dos formas:

// Opción 1: desestructurar dentro del componente (más legible)
function JobCard({ job }) {
  const { title, company, location } = job

  return (
    <article>
      <h3>{title}</h3>
      <p>{company}</p>
      <p>{location}</p>
    </article>
  )
}

// Opción 2: desestructurar en los parámetros (más compacto)
function JobCard({ job: { title, company, location } }) {
  return (
    <article>
      <h3>{title}</h3>
      <p>{company}</p>
      <p>{location}</p>
    </article>
  )
}

💡 Recomendación: La opción 1 es más legible, especialmente cuando el objeto tiene muchas propiedades.

🧩 Composición de Componentes

Las props permiten construir componentes más complejos a partir de componentes simples. Es uno de los patrones más poderosos de React: 

// Componente simple y reutilizable
function Badge({ children, type = 'default' }) {
  return <span className={`badge badge-${type}`}>{children}</span>
}

// Componente que compone Badge
function JobCard({ title, isRemote, isFeatured }) {
  return (
    <article>
      <h3>{title}</h3>
      {isRemote && <Badge type="info">🏠 Remoto</Badge>}
      {isFeatured && <Badge type="success">⭐ Destacado</Badge>}
    </article>
  )
}
Ventajas de la composición:

♻️ Badge es reutilizable en cualquier parte de la app
🧩 Fácil de mantener — cambiar Badge actualiza todo
📖 Código más legible y declarativo

❓ Renderizado Condicional con Fragments

function JobCard({ job, showDetails }) {
  return (
    <article>
      <h3>{job.title}</h3>

      {showDetails && (
        <>
          <p>{job.description}</p>
          <p>{job.requirements}</p>
          <p>{job.benefits}</p>
        </>
      )}
    </article>
  )
}

💪 Ejercicio: JobCard Completa con Props

Ahora integramos todo lo visto. Vas a construir un componente JobCard que recibe props de diferentes tipos y renderiza condicionalmente badges y detalles.

JobCard.jsx

function JobCard({ job, isRemote, isFeatured, isNew }) {
  const { title, company, location, salary, description, tags } = job

  return (
    <article className={`job-card ${isFeatured ? 'featured' : ''}`}>

      <header className="job-card-header">
        <h3 className="job-title">{title}</h3>
        <p className="job-company">{company}</p>

        <div className="badges">
          {isNew && <span className="badge new">🆕 Nuevo</span>}
          {isFeatured && <span className="badge featured">⭐ Destacado</span>}
          {isRemote && <span className="badge remote">🏠 Remoto</span>}
        </div>
      </header>

      <div className="job-card-body">
        <p className="job-location">📍 {location}</p>
        <p className="job-salary">💰 {salary}</p>
        <p className="job-description">{description}</p>
      </div>

      <footer className="job-card-footer">
        <span className="job-tags">{tags.join(', ')}</span>
        <button className="btn-apply">Aplicar</button>
      </footer>

    </article>
  )
}

export default JobCard

✅ Resumen de la Clase

🎁 Props         — argumentos que pasás a los componentes
⬇️ Flujo         — siempre de padre a hijo, nunca al revés
🔓 Desestructurar — { title, company } en los parámetros
🎯 Defaults      — location = 'Remoto' si no se pasa
❓ Condicional   — ternaria es más segura que &&
🧩 Fragments     — <></> para agrupar sin contaminar el DOM
