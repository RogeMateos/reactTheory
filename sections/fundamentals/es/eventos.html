Eventos en React: Cómo hacer que componentes respondan a acciones del usuario

Hasta ahora nuestros componentes de React solo muestran información. Son estáticos.

Pero las aplicaciones reales necesitan interacción:

  • Hacer click en botones
  • Escribir en inputs
  • Enviar formularios
  • Mover el ratón
  • Presionar teclas

Todo eso se maneja con eventos.

En esta clase aprenderás a escuchar eventos en React y ejecutar código cuando ocurren.

1️⃣ ¿Qué son los eventos?

Un evento es una acción que ocurre en el navegador.

Ejemplos comunes:

  • 👆 Click → el usuario hace click
  • ⌨️ Input → el usuario escribe en un campo
  • 📝 Change → cambia el valor de un input
  • 📤 Submit → se envía un formulario
  • 🖱️ Hover → el ratón pasa sobre un elemento

React permite escuchar esos eventos y reaccionar a ellos.

2️⃣ Nuestro ejemplo: botón "Aplicar"

Vamos a actualizar nuestro componente JobCard para que el botón Aplicar responda a clicks.

function JobCard({ titulo, empresa, ubicacion, descripcion }) {

  const handleAplicar = () => {
    console.log('¡Click en el botón!')
    console.log('Aplicando a:', titulo, '-', empresa)

    alert(`Has aplicado a: ${titulo} en ${empresa}`)
  }

  return (
    <article className="job-listing-card">
      <div>
        <h3>{titulo}</h3>
        <small>
          {empresa} | {ubicacion}
        </small>
        <p>{descripcion}</p>
      </div>

      <button
        className="button-apply-job"
        onClick={handleAplicar}
      >
        Aplicar
      </button>

    </article>
  )
}

Ahora el botón ejecuta una función cuando hacemos click.

3️⃣ Función manejadora de eventos

Primero definimos una función que se ejecutará cuando ocurra el evento.

const handleAplicar = () => {
  console.log('¡Click en el botón!')
  alert(`Has aplicado a: ${titulo} en ${empresa}`)
}

A esto se le llama:

  • Event Handler
  • o Event Handler Function

Por convención suelen empezar con handle.

Ejemplos:

  • handleClick
  • handleSubmit
  • handleChange
  • handleAplicar

Esto hace el código más claro.

4️⃣ Conectar la función al evento

Luego conectamos la función al botón.

<button onClick={handleAplicar}>
  Aplicar
</button>

Cuando el usuario hace click:

  • React → detecta el click
  • React → ejecuta handleAplicar()

5️⃣ Sintaxis de eventos en React

Los eventos en React tienen dos reglas importantes.

1. Usan camelCase

❌ Incorrecto:

onclick
onchange

✅ Correcto:

onClick
onChange
onSubmit
onMouseEnter

2. Se pasa la referencia a la función

✅ Correcto:

<button onClick={handleAplicar}>

❌ Incorrecto:

<button onClick={handleAplicar()}>

¿Por qué?

Porque esto ejecutaría la función inmediatamente al renderizar, no cuando hacemos click.

6️⃣ Alternativa: funciones inline

También puedes escribir la función directamente.

<button onClick={() => alert('¡Click!')}>
  Aplicar
</button>

Esto se llama función inline.

¿Cuándo usar cada opción?

Función separada (recomendado)

const handleClick = () => {
  console.log('Click!')
  alert(titulo)
}

<button onClick={handleClick}>Aplicar</button>

Mejor cuando:

  • hay varias líneas
  • reutilizas la función
  • quieres testearla

Función inline

<button onClick={() => alert('¡Click!')}>
  Aplicar
</button>

Mejor cuando:

  • es una sola línea
  • es algo muy simple

7️⃣ Acceso a las props dentro del evento

Los manejadores de eventos pueden usar todas las props del componente.

function JobCard({ titulo, empresa, ubicacion }) {

  const handleAplicar = () => {
    console.log(titulo)
    console.log(empresa)
    console.log(ubicacion)

    alert(`Aplicando a ${titulo}`)
  }

  return (
    <button onClick={handleAplicar}>
      Aplicar
    </button>
  )
}

Esto funciona gracias a un concepto fundamental de JavaScript:

Closures

La función recuerda las variables del contexto donde fue creada.

8️⃣ Comparación con JavaScript Vanilla

Antes (JavaScript tradicional):

const jobsListingSection = document.querySelector('.jobs-listings')

jobsListingSection.addEventListener('click', function (event) {

  const element = event.target

  if (element.classList.contains('button-apply-job')) {

    element.textContent = '¡Aplicado!'
    element.classList.add('is-applied')
    element.disabled = true

  }

})

Problemas:

  • ❌ buscar elementos con querySelector
  • ❌ delegación de eventos
  • ❌ manipular DOM manualmente
  • ❌ código difícil de mantener

En React todo es mucho más simple:

const handleAplicar = () => {
  alert(`Aplicando a ${titulo}`)
}

<button onClick={handleAplicar}>
  Aplicar
</button>

Ventajas:

  • ✅ código más limpio
  • ✅ cada componente gestiona su lógica
  • ✅ sin manipulación manual del DOM
  • ✅ acceso directo a props

9️⃣ Eventos comunes en React

Click

<button onClick={() => console.log('¡Click!')}>
  Click
</button>

Change (inputs)

<input
  onChange={(e) => console.log(e.target.value)}
/>

Submit (formularios)

<form
  onSubmit={(e) => {
    e.preventDefault()
    console.log('Formulario enviado')
  }}
>
  <button type="submit">Enviar</button>
</form>

Hover

<div
  onMouseEnter={() => console.log('Entrando')}
  onMouseLeave={() => console.log('Saliendo')}
>
  Pasa el ratón
</div>

Teclas

<input
  onKeyDown={(e) => console.log(e.key)}
/>

🔟 El objeto evento

Los eventos reciben un objeto event.

const handleClick = (event) => {
  console.log(event)
}

Propiedades útiles:

  • event.target - el elemento que disparó el evento
  • event.currentTarget - el elemento con el listener
  • event.type - tipo de evento
  • event.preventDefault() - prevenir comportamiento por defecto
  • event.stopPropagation() - detener propagación de evento

1️⃣1️⃣ Ejemplo con preventDefault

Cuando enviamos un formulario el navegador recarga la página.

Podemos evitarlo.

function Formulario() {

  const handleSubmit = (event) => {
    event.preventDefault()

    console.log('Formulario enviado')
  }

  return (
    <form onSubmit={handleSubmit}>
      <input type="text" />
      <button type="submit">Enviar</button>
    </form>
  )
}

1️⃣2️⃣ Pasar argumentos a eventos

A veces necesitas pasar datos extra.

Ejemplo:

function JobCard({ id }) {

  const handleAplicar = (jobId) => {
    console.log(jobId)
  }

  return (
    <button onClick={() => handleAplicar(id)}>
      Aplicar
    </button>
  )
}

1️⃣3️⃣ Múltiples eventos en un componente

Un componente puede tener varios manejadores.

function JobCard({ titulo }) {

  const handleAplicar = () => {
    alert(`Aplicando a ${titulo}`)
  }

  const handleGuardar = () => {
    alert(`Guardando ${titulo}`)
  }

  const handleCompartir = () => {
    alert(`Compartiendo ${titulo}`)
  }

  return (
    <article>

      <h3>{titulo}</h3>

      <button onClick={handleAplicar}>
        Aplicar
      </button>

      <button onClick={handleGuardar}>
        Guardar
      </button>

      <button onClick={handleCompartir}>
        Compartir
      </button>

    </article>
  )
}

1️⃣4️⃣ Limitación actual

Cuando hacemos click en Aplicar:

  • ✔ se ejecuta la función
  • ✔ aparece la alerta
  • ✔ vemos mensajes en consola

Pero...

  • ❌ el botón no cambia
  • ❌ no recuerda que aplicaste

¿Por qué?

Porque todavía no usamos estado.

1️⃣5️⃣ Errores típicos que cometen los juniors

Error 1: Ejecutar la función en lugar de pasarla

❌ Incorrecto:

<button onClick={handleClick()}>

✅ Correcto:

<button onClick={handleClick}>

Error 2: Usar onclick en vez de onClick

❌ Incorrecto:

onclick

✅ Correcto:

onClick

Error 3: Olvidar preventDefault en formularios

Esto provoca recarga de página.

Error 4: Usar funciones inline enormes

Esto dificulta el mantenimiento.

Ejemplo 2: Formulario con manejador de envío


function FormularioLogin() {

Ejemplo 4: Múltiples manejadores de eventos


function TarjetaInteractiva() {
  const handleMouseEnter = () => {
    console.log('El ratón entró')
  }

        cursor: 'pointer'
      }}

      ¡Interactúa conmigo!
    </div>
  )
}

Ejemplo 5: Pasar argumentos a manejadores


function ListaItems() {
  const items = ['Item 1', 'Item 2', 'Item 3']

  const handleDelete = (itemId) => {
    console.log(`Eliminando item: ${itemId}`)
    alert(`Eliminado: ${itemId}`)
  }

  return (
    <ul>
      {items.map((item) => (
        <li key={item}>
          {item}
          <button onClick={() => handleDelete(item)}>
            Eliminar
          </button>
        </li>
      ))}
    </ul>
  )
}

Ejemplo 7: Eventos de teclado


function BuscadorTeclado() {

    />
  )
}

Ejercicios Prácticos

Ejercicio 1: Botón contador simple

Desafío: Crea un botón que registre en la consola un contador cuando se hace click.

Solución:


function BotonContador() {
  let count = 0

  const handleClick = () => {
    count++
    console.log('Contador:', count)
  }

  return (
    <button onClick={handleClick}>
      Hazme click
    </button>
  )
}

Ejercicio 2: Validación de formulario

Desafío: Crea un formulario que evite el envío si el email es inválido.

Solución:


function FormularioEmail() {
  const handleSubmit = (event) => {
    event.preventDefault()

    const email = event.target.email.value

    if (!email.includes('@')) {
      alert('¡Email inválido!')
      return
    }

    alert('¡Email válido!')
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="email"
        name="email"
        placeholder="Ingresa tu email"
      />
      <button type="submit">Enviar</button>
    </form>
  )
}

Ejercicio 3: Detección de teclas

Desafío: Crea un input que registre cuando se presionan teclas específicas.

Solución:


function DetectorTeclas() {
  const handleKeyPress = (event) => {
    if (event.key === 'Enter') {
      console.log('Enter fue presionado')
    } else if (event.key === 'Escape') {
      console.log('Escape fue presionado')
    } else {
      console.log('Tecla:', event.key)
    }
  }

  return (
    <input
      type="text"
      onKeyDown={handleKeyPress}
      placeholder="Presiona cualquier tecla"
    />
  )
}

function TarjetaHover() {
