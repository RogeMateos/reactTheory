🔥 Fast Refresh y Actualizaciones de Estado

🧠 ¿Qué es Fast Refresh?

Fast Refresh es una tecnología específica de React que preserva el estado de tus componentes cuando guardas cambios en el código durante el desarrollo.

📖 Ejemplo práctico

Imagina que estás desarrollando un contador:

function Contador() {
  const [count, setCount] = React.useState(0)

  return (
    <div>
      <p>Contador: {count}</p>
      <button onClick={() => setCount(count + 1)}>Incrementar</button>
    </div>
  )
}

❌ Sin Fast Refresh

1. Has clickeado varias veces → Contador: 15
2. Cambias el texto "Incrementar" por "Sumar"
3. Guardas el archivo
4. La página se recarga completamente
5. ❌ El contador vuelve a 0

✅ Con Fast Refresh

1. Has clickeado varias veces → Contador: 15
2. Cambias el texto "Incrementar" por "Sumar"
3. Guardas el archivo
4. El componente se actualiza
5. ✅ El contador sigue en 15

💡 ¡El estado se mantiene! Esto es súper útil cuando estás desarrollando porque no pierdes el estado de tu aplicación cada vez que guardas un cambio.

🧪 Experimenta Fast Refresh tú mismo

Para ver Fast Refresh en acción, sigue estos pasos en un proyecto Vite:

Paso 1: Código inicial (App.jsx)

// src/App.jsx
import { useState } from 'react'

function App() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <h1>Mi Contador</h1>
      <p>Contador: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Incrementar
      </button>
    </div>
  )
}

export default App

Paso 2: Ejecuta el proyecto

npm run dev

Abre http://localhost:5173/ en tu navegador

Paso 3: Incrementa el contador

Haz click en el botón varias veces hasta llegar a 15 

Contador: 15 ← El estado actual

❌ Sin Fast Refresh (recarga manual)

Si recargas la página manualmente (F5 o Cmd+R):

Contador: 0 ← ¡Perdiste el estado!

El estado vuelve a 0 porque la página se recargó.

✅ Con Fast Refresh (editar código)

Paso 4: Edita el código

Cambia esta línea en App.jsx:

❌ Código original ✅ Nuevo código 
<button onClick={() => setCount(count + 1)}>
  Incrementar
</button>
 
<button onClick={() => setCount(count + 1)}>
  Sumar +1
</button>

Paso 5: Guarda el archivo (Ctrl+S / Cmd+S)

¿Qué pasa?

🔥 Fast Refresh detecta el cambio
⚡ Actualiza el componente instantáneamente
✅ El botón ahora dice "Sumar +1"
💾 El contador sigue en 15 ← ¡El estado se preservó!

🎯 Otros cambios que puedes probar

1. Cambiar el título:

<h1>🚀 Contador con Fast Refresh</h1>

// Estado preservado: ✅

2. Cambiar el estilo del párrafo:

  Reset
</button>

// Estado preservado: ✅

⚠️ Cuándo Fast Refresh NO preserva el estado

Fast Refresh pierde el estado en estos casos:

  • ❌ Cambias el nombre del componente (de App a Contador)
  • ❌ Cambias la firma del hook (de useState(0) a useState({count: 0}))
  • ❌ Hay un error de sintaxis en el código
  • ❌ Cambias el tipo de export (de default a named)

En estos casos, Vite hace una recarga completa y el estado vuelve a su valor inicial.

🔄 Fast Refresh vs Hot Module Replacement (HMR)

Aunque parecen lo mismo, hay una diferencia importante:

🔧 Hot Module Replacement (HMR)

  • 🔧 Tecnología genérica para cualquier tipo de código
  • 📦 Funciona con JavaScript, CSS, imágenes, etc.
  • 🔄 Reemplaza módulos sin recargar la página
  • ⚠️ No preserva el estado automáticamente
  • 💻 Usado por Vite, Webpack, Parcel, etc.
HMR: "Cambió el archivo CSS → Actualizo solo el CSS"
HMR: "Cambió el archivo JS → Actualizo ese módulo"

⚛️ Fast Refresh

  • ⚛️ Tecnología específica de React
  • 🎯 Solo funciona con componentes de React
  • 💾 Preserva el estado de los componentes inteligentemente
  • 🔄 Re-renderiza solo los componentes que cambiaron
  • ✨ Construido sobre HMR (lo usa por debajo)
Fast Refresh: "Cambió el componente → Actualizo Y mantengo el estado"

📊 Comparación visual

Característica HMR Fast Refresh
Específico de React ❌ ✅
Preserva estado ⚠️ Manual ✅ Automático
Funciona con CSS/Assets ✅ ❌ Solo componentes
Velocidad ⚡ Rápido ⚡⚡ Más rápido

🎯 En resumen

HMR = Reemplaza código sin recargar la página
Fast Refresh = HMR + Preservar estado de React

💡 Fast Refresh usa HMR como base, pero añade inteligencia específica de React para mantener el estado de tus componentes.

⚠️ El Problema: Actualizar estado con valor directo

Intenta adivinar qué pasa cuando haces click en este botón:

function Contador() {
  const [count, setCount] = React.useState(0)

  const incrementarTresVeces = () => {
    setCount(count + 1)
    setCount(count + 1)
    setCount(count + 1)
  }

  return (
    <div>
      <p>Contador: {count}</p>
      <button onClick={incrementarTresVeces}>+3</button>
    </div>
  )
}

¿El resultado? ❌ Solo suma 1, no 3

🤔 ¿Por qué pasa esto?

Cuando llamas a setCount, React no actualiza el estado inmediatamente. Lo hace de forma asíncrona. 

const incrementarTresVeces = () => {
  // En este momento, count = 0
  setCount(count + 1) // Programa: "poner en 0 + 1 = 1"
  setCount(count + 1) // Programa: "poner en 0 + 1 = 1"
  setCount(count + 1) // Programa: "poner en 0 + 1 = 1"
  // count sigue siendo 0 aquí

  // React aplica las actualizaciones después
  // Como todas dicen "poner en 1", el resultado es 1
}

🔍 Visualización paso a paso

Estado inicial: count = 0

Click en el botón:
  ├─ setCount(0 + 1) → "Pon count en 1"
  ├─ setCount(0 + 1) → "Pon count en 1"
  └─ setCount(0 + 1) → "Pon count en 1"

React procesa:
  "Todas dicen que count debe ser 1"
  → count = 1 ❌

Esperabas: count = 3 ✅

⚠️ El problema: Es como si dijeras tres veces "pon el contador en 1", en lugar de "suma 1 al contador".

✅ La Solución: Usar un callback

La forma correcta es usar una función callback:

function Contador() {
  const [count, setCount] = React.useState(0)

  const incrementarTresVeces = () => {
    setCount((prevCount) => prevCount + 1)
    setCount((prevCount) => prevCount + 1)
    setCount((prevCount) => prevCount + 1)
  }

  return (
    <div>
      <p>Contador: {count}</p>
      <button onClick={incrementarTresVeces}>+3</button>
    </div>
  )
}

Ahora sí funciona: ✅ Suma 3

🎯 ¿Cómo funciona el callback?

setCount((prevCount) => prevCount + 1)

Le dices a React: "Dame el valor más reciente y yo te devuelvo el nuevo valor"

🔍 Visualización paso a paso

Estado inicial: count = 0

Click en el botón:
  ├─ setCount(prev => prev + 1)
  │    └─ prevCount = 0 → devuelve 1
  │
  ├─ setCount(prev => prev + 1)
  │    └─ prevCount = 1 → devuelve 2
  │
  └─ setCount(prev => prev + 1)
       └─ prevCount = 2 → devuelve 3

React procesa:
  0 → 1 → 2 → 3
  → count = 3 ✅

✅ React procesa las actualizaciones en orden, usando siempre el valor más reciente.

📊 Comparación lado a lado

❌ Valor directo (Incorrecto) ✅ Callback (Correcto) 
setCount(count + 1)
setCount(count + 1)
setCount(count + 1)

// Todas usan el mismo count (0)
// Resultado: 1 ❌
 
setCount(prev => prev + 1)
setCount(prev => prev + 1)
setCount(prev => prev + 1)

// Cada una usa el valor actualizado
// Resultado: 3 ✅

🧪 Otros ejemplos comunes

❌ Incorrecto - Alternar booleano:

// Si haces click múltiples veces rápido, puede fallar
setVisible(!visible)

✅ Correcto - Alternar booleano:

// Siempre funciona correctamente
setVisible(prev => !prev)

❌ Incorrecto - Agregar a un array:

// Puede perder items si se actualiza rápido
setItems([...items, nuevoItem])

✅ Correcto - Agregar a un array:

// Garantiza que usas el array más reciente
setItems(prevItems => [...prevItems, nuevoItem])

🎯 ¿Cuándo usar cada forma?

📝 Valor directo (forma simple)

Usa esta forma cuando el nuevo valor es INDEPENDIENTE del valor actual:

// ✅ Bien: el nuevo valor NO depende del anterior
setNombre('Miguel')
setMostrar(true)
setColor('#ff0000')
setUsuario({ id: 1, nombre: 'Ana' })

🔄 Callback (forma con función)

Usa esta forma cuando el nuevo valor DEPENDE del valor anterior:

// ✅ Bien: el nuevo valor depende del anterior
setCount(prev => prev + 1)
setItems(prevItems => [...prevItems, nuevoItem])
setVisible(prev => !prev)
setUsuario(prev => ({ ...prev, edad: prev.edad + 1 }))

🚦 Cuándo usar callback (checklist)

Usa setState(prev => ...) cuando:

  • ⚡ El nuevo valor depende del valor anterior
  • 🔄 Haces múltiples actualizaciones seguidas
  • 🎯 Quieres garantizar que usas el valor más reciente
  • 🐛 Estás experimentando bugs de estado desactualizado

📚 Ejercicios para practicar

Ejercicio 1: ¿Cuál es correcto para este caso?

// Necesitas incrementar un contador cuando el usuario hace click
const handleClick = () => {
  // Opción A
  setCount(count + 1)

  // Opción B
  setCount(prev => prev + 1)
}
Ver respuesta

Respuesta: Opción B es más segura. Si el usuario hace click muy rápido o hay múltiples actualizaciones, la Opción A puede perder algunos clicks. La Opción B garantiza que cada click se cuente correctamente.

Ejercicio 2: ¿Cuál es correcto para este caso?

// Necesitas actualizar el nombre del usuario cuando escribe en un input
const handleChange = (e) => {
  // Opción A
  setNombre(e.target.value)

  // Opción B
  setNombre(prev => e.target.value)
}
Ver respuesta

Respuesta: Opción A está bien. El nuevo valor (e.target.value) es completamente independiente del valor anterior. No necesitas el callback aquí.

Ejercicio 3: ¿Qué resultado tendrá este código?

const [count, setCount] = useState(0)

const handleClick = () => {
  setCount(count + 1)
  setCount(count + 1)
  console.log(count) // ¿Qué se imprime?
}
Ver respuesta

Se imprime: 0
1. El console.log se ejecuta inmediatamente, antes de que React actualice el estado
2. Los dos setCount programan actualizaciones pero no las ejecutan de inmediato
3. Después de que la función termine, React actualiza el estado a 1 (no a 2, por el problema explicado)

🎯 Regla de oro

Si ves el nombre del estado dentro del setState, usa callback:

❌ setCount(count + 1)        → ¡count está aquí!
✅ setCount(prev => prev + 1)  → Mejor con callback

✅ setNombre('Ana')            → No usa el valor anterior, está bien
