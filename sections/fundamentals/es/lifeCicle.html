Virtual DOM y Renderizado en React

Para dominar React, es fundamental entender cómo funciona el Virtual DOM y el proceso de renderizado.

🧱 ¿Qué es el DOM?

Cuando una página web se carga por primera vez, el navegador crea una estructura en memoria llamada DOM (Document Object Model).

  • Es un árbol de nodos
  • Cada nodo representa un elemento HTML
  • Modificar el DOM real es costoso en rendimiento

Visualmente:

document
 └── body
     └── div
         ├── h1
         └── button

⚡ El problema del DOM tradicional

Si cada cambio en la interfaz modificara directamente el DOM:

  • ❌ Sería lento
  • ❌ Provocaría muchos repaints
  • ❌ Mala experiencia en apps grandes

Aquí es donde entra React.

🪞 ¿Qué es el Virtual DOM?

El Virtual DOM es una copia ligera del DOM real, mantenida en memoria por React.

  • 👉 No es el DOM del navegador
  • 👉 Es una representación en JavaScript

React trabaja primero con el Virtual DOM y solo toca el DOM real cuando es necesario.

🔁 Proceso de renderizado en React (paso a paso)

1️⃣ Renderizado inicial

  • React crea el Virtual DOM
  • Se genera el DOM real y se pinta la UI
Virtual DOM  →  DOM Real  →  Pantalla

2️⃣ Cambio de estado (state o props)

Cuando cambia algo en la aplicación:

  • React no toca el DOM real directamente
  • Crea un nuevo Virtual DOM
Virtual DOM (anterior)
Virtual DOM (nuevo)

3️⃣ Diffing (comparación)

React compara ambas versiones:

ANTES: <h1>Hola</h1>
DESPUÉS: <h1>Hola Roge</h1>

👉 Solo detecta qué cambió

4️⃣ Actualización eficiente

  • React actualiza solo ese nodo
  • El resto del DOM queda intacto
  • ✅ Más rápido
  • ✅ Más eficiente
  • ✅ Mejor rendimiento

🧠 ¿Por qué React es tan rápido?

Porque:

  • Minimiza accesos al DOM real
  • Agrupa cambios
  • Actualiza solo lo necesario

Este proceso es lo que se conoce como diffing + reconciliation.

🔥 Tip avanzado

Aunque solemos hablar de "Virtual DOM", internamente React utiliza un sistema de reconciliación basado en Fiber, que permite pausar, priorizar y reanudar renderizados para mejorar la experiencia del usuario.

🎯 Conclusión

React no actualiza la interfaz directamente. Primero piensa, compara y decide.
Solo después modifica el DOM real, y únicamente donde hace falta.

Ejemplos de Código – Virtual DOM y Renderizado en React

1️⃣ Renderizado inicial con createRoot


// index.js
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);

🔹 En este punto:

  • React crea el Virtual DOM inicial
  • Se genera el DOM real
  • La UI se pinta por primera vez en pantalla

3️⃣ Re-render provocado por cambio de estado


// index.js
import { useState, useEffect } from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

const Root = () => {
  const [showApp, setShowApp] = useState(false);

  useEffect(() => {
    const timer = setTimeout(() => {
      setShowApp(true);
    }, 5000);

    return () => clearTimeout(timer);
  }, []);

  return showApp ? <App /> : <div>Cargando...</div>;
};

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<Root />);

🧠 Punto clave para entender React:

  • Cada vez que cambia el state, React vuelve a ejecutar el componente
  • → crea un nuevo Virtual DOM
  • → lo compara con el anterior

4️⃣ ¿Qué pasa internamente? (Virtual DOM)

⚠️ Esto NO es código real de React, es una representación conceptual:


const virtualDOM = {

  • Virtual DOM anterior

🔹 Y solo actualiza las diferencias en el DOM real

React no actualiza el DOM directamente.
Primero crea un nuevo Virtual DOM, lo compara con el anterior y solo aplica los cambios necesarios.
Esto hace que las aplicaciones sean más rápidas, predecibles y fáciles de mantener.

Life Cycle, Virtual DOM, Rendering y DOM real (cómo encaja todo)

En React, cuando un componente "vive" (monta, actualiza, desmonta), pasan dos ideas clave:

  • Render (reconciliación): React ejecuta tu componente para calcular qué UI debería existir (un árbol de elementos).
  • Commit: React aplica los cambios necesarios al DOM real y luego ejecuta effects (useEffect).

⚠️ Importante: render no significa "pintar en el DOM". Render es calcular el resultado. El DOM se toca en la fase de commit.

🟢 Mounting (Montaje)

  • React ejecuta el componente por primera vez.
  • Se genera el árbol "virtual" (representación de la UI).
  • React hace commit: crea nodos/atributos necesarios en el DOM real.
  • Después del commit, corre useEffect(() => { ... }, []) (si existe).

🔄 Updating (Actualización)

Ocurre cuando cambian state, props, o el padre re-renderiza:

  • React vuelve a ejecutar el componente (nuevo árbol).
  • Compara el árbol nuevo con el anterior (reconciliation / diff).
  • En el commit, actualiza solo lo que cambió en el DOM real.
  • Luego React ejecuta effects:
    • Si el effect tiene dependencias y cambiaron, hace:
      • cleanup anterior
      • effect nuevo

🔴 Unmounting (Desmontaje)

  • React elimina el componente del árbol.
  • En el commit, quita sus nodos del DOM real.
  • Ejecuta el cleanup final del effect (si lo hay).

✅ Resultado: React evita tocar el DOM "a mano" continuamente, y por eso es más eficiente y consistente.

Ejemplo práctico (Mount / Update / Unmount) con logs claros

Este ejemplo demuestra:

  • Mount: cuando aparece el VideoItem
  • Update: cuando cambia duration (misma instancia, re-render + effect si depende)
  • Unmount: cuando lo ocultas (cleanup final)

import React, { useState, useEffect } from "react";
import ReactDOM from "react-dom/client";

function VideoItem({ title, duration }) {
  console.log("[Render] VideoItem:", title, duration);

  useEffect(() => {
    console.log("[Effect mount] VideoItem mounted:", title);

    return () => {
      console.log("[Effect cleanup] VideoItem unmounted:", title);
    };
  }, []); // solo mount/unmount

  useEffect(() => {
    console.log("[Effect] duration changed:", duration);

    return () => {
      console.log("[Cleanup] before next duration effect:", duration);
    };
  }, [duration]); // update + cleanup en cada cambio de duration

  return (
    <div className="video-item">
      <h2>{title}</h2>
      <p>Duration: {duration}</p>
    </div>
  );
}

function VideoList() {
  const [visible, setVisible] = useState(true);
  const [duration, setDuration] = useState("2 hours");

  return (
    <div>
      {visible && <VideoItem title="React Course" duration={duration} />}

      <button onClick={() => setVisible((v) => !v)}>
        Toggle component (mount/unmount)
      </button>

      <button onClick={() => setDuration((d) => (d === "2 hours" ? "3 hours" : "2 hours"))}>
        Change duration (update)
      </button>
    </div>
  );
}

const root = ReactDOM.createRoot(document.getElementById("root"));
root.render(<VideoList />);

📊 Qué verás en consola (idea mental rápida)

Al cargar:

[Render] ...
[Effect mount] ...
[Effect] duration changed ... (primera vez también cuenta)

Al cambiar duration:

[Render] ...
[Cleanup] before next duration effect ...
[Effect] duration changed ...

Al ocultar:

[Effect cleanup] VideoItem unmounted ...
