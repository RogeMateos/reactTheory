Arreglando la Paginación: URL como Estado

🧠 ¿Qué problema resuelve?

Antes, la ruta /search era siempre igual sin importar lo que el usuario hacía: 

página 1
página 2
búsqueda React
búsqueda Vue

👉 La URL NO cambiaba.

Entonces:

Situación Resultado
Recargar la página ❌ Perdés todo el estado
Compartir la URL ❌ No funciona
Botón atrás del navegador ❌ Comportamiento raro

✅ Lo que queremos conseguir

Que la URL represente el estado de la app. Por ejemplo: 

/search?page=3&text=react

Esa URL lo dice todo:

Parámetro Significado
page=3 Estás en página 3
text=react Estás buscando "React"

La URL ahora "cuenta la historia" de la aplicación.

🚀 Beneficio REAL

Con este cambio, la app:

✅ Recuerda el estado
✅ Puede recargarse sin perder nada
✅ Puede compartirse con cualquiera
✅ Funciona bien con el historial del navegador
✅ Tiene URLs profesionales

📌 La idea MÁS importante

La URL se convierte en LA FUENTE DE LA VERDAD. No solo el estado de React. La URL ahora también guarda el estado de la aplicación.

🔥 Qué hace URLSearchParams

Ayuda a leer y modificar los parámetros de la URL fácilmente, sin romper los que ya existen. 

const searchParams = new URLSearchParams(window.location.search)

// Si la URL es: /search?text=react
searchParams.set('page', '2')
// Ahora es:    /search?text=react&page=2

👉 Mantiene lo anterior y añade o modifica solo lo necesario.

🎯 Qué consigue buildPageURL

Construir correctamente los links de paginación para que cada botón tenga una URL real: 

// ANTES — todos los botones apuntaban a "#"
<a href="#">1</a>
<a href="#">2</a>
<a href="#">3</a>

// DESPUÉS — cada botón tiene su URL correcta
/search?page=1&text=react
/search?page=2&text=react
/search?page=3&text=react

💻 Implementación

function buildPageURL(search, page) {
  // Tomamos los parámetros actuales de la URL
  const searchParams = new URLSearchParams(search)

  // Modificamos solo el parámetro "page"
  searchParams.set('page', page)

  // Devolvemos la query string completa
  return `?${searchParams.toString()}`
}

// Uso en el componente de paginación:
function Pagination({ search, currentPage, totalPages }) {
  return (
    <nav>
      {Array.from({ length: totalPages }, (_, i) => i + 1).map(page => (
        <a
          key={page}
          href={buildPageURL(search, page)}
          className={page === currentPage ? 'active' : ''}
        >
          {page}
        </a>
      ))}
    </nav>
  )
}

⚠️ Errores comunes

Error Problema Solución
Usar href="#" URL nunca cambia Usar buildPageURL()
Construir URL manualmente Pisás otros parámetros Usar URLSearchParams
Estado en useState solamente No sobrevive recarga Leer desde useSearchParams
No resetear página al buscar Página 5 con nueva búsqueda Al cambiar text, resetear page=1

🎯 Ejercicio

Modificá tu componente de búsqueda para que:

  1. La URL cambie cuando el usuario escribe en el input de búsqueda
  2. La URL cambie cuando el usuario hace click en un número de página
  3. Al recargar la página, se restaure tanto el texto como la página actual
  4. Al cambiar el texto de búsqueda, la página vuelva a 1 automáticamente

🔧 Bloque 1: simplificando buildPageURL

Problema detectado: el código funciona, pero usar window.location.href dentro del componente lo hace menos testeable y más acoplado al navegador.

// ❌ Versión acoplada — toma más de lo necesario
const url = new URL(window.location.href)

// ✅ Versión mejorada — solo trabaja con lo que necesita
const buildPageURL = (pageNumber) => {
  const searchParams = new URLSearchParams(window.location.search)
  searchParams.set('page', String(pageNumber))
  return `${window.location.pathname}?${searchParams.toString()}`
}

La mejora está en usar window.location.search y window.location.pathname en vez de construir un objeto URL completo. Trabajás solo con lo que necesitás.

🔧 Bloque 2: páginas anterior y siguiente

Problema detectado: sin control, podés generar URLs inválidas:

// ❌ Sin límites — genera URLs rotas
previousPage = 0            → /search?page=0
nextPage = totalPages + 1   → /search?page=11

// ✅ Con Math.max / Math.min — siempre válido
const previousPage = Math.max(currentPage - 1, 1)
const nextPage = Math.min(currentPage + 1, totalPages)

Aunque bloquees el click con preventDefault, el href seguía apuntando a una URL incorrecta. Con esto, siempre es válida.

🔧 Bloque 3: legibilidad con isFirstPage / isLastPage

Problema detectado: aria-disabled solo ayuda a accesibilidad, pero no desactiva el enlace. Además, repetir currentPage === 1 en varios lugares ensucia el código.

// ✅ Variables descriptivas para mayor legibilidad
const isFirstPage = currentPage === 1
const isLastPage = currentPage === totalPages

<a
  href={buildPageURL(previousPage)}
  aria-disabled={isFirstPage}
  onClick={(event) => {
    if (isFirstPage) event.preventDefault()
  }}
>
  Anterior
</a>

✅ Versión final mejorada completa

function Pagination({ currentPage, totalPages }) {
  const buildPageURL = (pageNumber) => {
    const searchParams = new URLSearchParams(window.location.search)
    searchParams.set('page', String(pageNumber))
    return `${window.location.pathname}?${searchParams.toString()}`
  }

  const isFirstPage = currentPage === 1
  const isLastPage  = currentPage === totalPages

  const previousPage = Math.max(currentPage - 1, 1)
  const nextPage     = Math.min(currentPage + 1, totalPages)

  return (
    <nav aria-label="Paginación">
      <ul>
        <li>
          <a
            href={buildPageURL(previousPage)}
            aria-disabled={isFirstPage}
            onClick={(event) => { if (isFirstPage) event.preventDefault() }}
          >
            Anterior
          </a>
        </li>

        {Array.from({ length: totalPages }, (_, index) => {
          const page = index + 1
          return (
            <li key={page}>
              <a
                href={buildPageURL(page)}
                aria-current={page === currentPage ? 'page' : undefined}
              >
                {page}
              </a>
            </li>
          )
        })}

        <li>
          <a
            href={buildPageURL(nextPage)}
            aria-disabled={isLastPage}
            onClick={(event) => { if (isLastPage) event.preventDefault() }}
          >
            Siguiente
          </a>
        </li>
      </ul>
    </nav>
  )
}

📌 Resumen de mejoras

Problema Solución
new URL(window.location.href) Usar location.search + location.pathname
Páginas inválidas (0 o totalPages+1) Math.max / Math.min como guardia
Condición repetida en múltiples lugares isFirstPage / isLastPage como variables
aria-disabled no bloquea el click onClick con preventDefault
