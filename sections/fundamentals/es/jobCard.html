Creando JobCard: Nuestro primer componente real

Hasta ahora hemos aprendido:

  • Qué es JSX
  • Qué son los componentes
  • La diferencia entre componentes y funciones normales

Ahora vamos a aplicar todo eso creando un componente real para nuestra aplicación DevJobs.

Este será el primer componente que representa una pieza real de UI.

1️⃣ El objetivo: una tarjeta de empleo

Queremos crear algo como esto:

-------------------------------
Desarrollador Frontend
Tech Solutions | Remoto

Buscamos un desarrollador...
[ Aplicar ]
-------------------------------

Cada tarjeta tendrá datos diferentes, pero la estructura será la misma.

Aquí es donde React brilla.

Creamos un componente reutilizable.

2️⃣ El componente JobCard

Este es nuestro primer componente real.

function JobCard({ titulo, empresa, ubicacion, descripcion }) {
  return (
    <article className="job-listing-card">
      <h3>{titulo}</h3>
      <small>
        {empresa} | {ubicacion}
      </small>
      <p>{descripcion}</p>
      <button className="button-apply-job">
        Aplicar
      </button>
    </article>
  )
}

Este componente:

  • Recibe 4 props
  • Devuelve JSX
  • Representa una tarjeta de empleo

3️⃣ ¿Qué son las props?

Las props son datos que se pasan a un componente.

Ejemplo:

<JobCard
  titulo="Desarrollador Frontend"
  empresa="Tech Solutions"
  ubicacion="Remoto"
  descripcion="Buscamos un desarrollador..."
/>

Dentro del componente podemos usarlas:

{titulo}
{empresa}
{ubicacion}
{descripcion}

React reemplaza esos valores automáticamente.

4️⃣ Destructuring de props

Observa esta línea:

function JobCard({ titulo, empresa, ubicacion, descripcion })

Esto se llama destructuring.

En lugar de hacer esto:

function JobCard(props) {
  return (
    <h3>{props.titulo}</h3>
  )
}

Podemos hacer esto:

function JobCard({ titulo }) {
  return <h3>{titulo}</h3>
}

Ventajas:

  • Código más limpio
  • Más fácil de leer
  • Muy común en React

5️⃣ El componente App

Ahora tenemos un componente principal que usa JobCard.

function App() {
  return (
    <section>
      <h2>Empleos disponibles</h2>
      <div className="jobs-listings">
        <JobCard
          titulo="Desarrollador Frontend"
          empresa="Tech Solutions"
          ubicacion="Remoto"
          descripcion="Buscamos..."
        />
        <JobCard
          titulo="Ingeniero Senior"
          empresa="Data Driven Co."
          ubicacion="Ciudad de México"
          descripcion="Buscamos un ingeniero..."
        />
      </div>
    </section>
  )
}

Aquí vemos composición de componentes.

6️⃣ ¿Qué es composición de componentes?

React funciona como LEGO.

Construyes interfaces usando componentes pequeños.

Ejemplo:

App
 ├── Header
 ├── JobList
 │    ├── JobCard
 │    ├── JobCard
 │    └── JobCard
 └── Footer

Cada componente tiene su propia responsabilidad.

7️⃣ Comparación con JavaScript Vanilla

Antes con JavaScript puro hacíamos esto:

jobs.forEach((job) => {
  const article = document.createElement('article')
  article.innerHTML = `
    <h3>${job.titulo}</h3>
    <small>${job.empresa}</small>
  `
  container.appendChild(article)
})

Problemas:

  • ❌ Mucho código repetido
  • ❌ Manipulación manual del DOM
  • ❌ HTML dentro de strings
  • ❌ Difícil de mantener

8️⃣ Con React

Solo escribimos el componente una vez:

function JobCard({ titulo, empresa, ubicacion, descripcion }) {
  return (
    <article className="job-listing-card">
      <h3>{titulo}</h3>
      <p>{descripcion}</p>
    </article>
  )
}

Y lo usamos tantas veces como queramos.

Ventajas:

  • ✅ Código declarativo
  • ✅ Componentes reutilizables
  • ✅ React gestiona el DOM
  • ✅ Más fácil de mantener

1️⃣1️⃣ Buenas prácticas con componentes

1. Un componente = Una responsabilidad

Correcto:

function JobTitle({ children }) {
  return <h3>{children}</h3>
}

Incorrecto:

function TodoComponent() {
  // hace demasiadas cosas
}

2. Nombres claros

Correcto:

TarjetaPerfilUsuario
ElementoListaEmpleos
MenuNavegacion

Incorrecto:

Item
Cosa
Comp1

3. Props descriptivas

Correcto:

function Boton({ texto, onClick, deshabilitado })

Incorrecto:

function Boton({ t, oc, d })

Ejercicios Prácticos

Ejercicio 1: Crea un componente ProductCard

Desafío: Crea un componente similar a JobCard pero para productos.

Debe mostrar:

  • Nombre del producto
  • Precio
  • Descripción
  • Un botón "Añadir al carrito"

Solución:


function ProductCard({ nombre, precio, descripcion }) {
  return (
    <article className="product-card">
      <h3>{nombre}</h3>
      <p className="price">${precio}</p>
      <p>{descripcion}</p>
      <button className="button-add-cart">
        Añadir al carrito
      </button>
    </article>
  )
}

// Uso
function App() {
  return (
    <div>
      <h2>Nuestros Productos</h2>
      <ProductCard
        nombre="Laptop"
        precio={999}
        descripcion="Laptop de alto rendimiento para desarrolladores"
      />
      <ProductCard
        nombre="Ratón"
        precio={25}
        descripcion="Ratón inalámbrico con conexión 2.4GHz"
      />
    </div>
  )
}

Ejercicio 3: Crea un componente BlogPostCard

Desafío: Crea un componente para mostrar posts de blog.

Debe tener:

  • Título
  • Autor
  • Fecha de publicación
  • Resumen
  • Un botón "Leer más"

Solución:


function BlogPostCard({ titulo, autor, fecha, resumen }) {
  return (
    <article className="blog-post-card">
      <h2>{titulo}</h2>
      <div className="post-meta">
        <small>Por {autor} | {fecha}</small>
      </div>
      <p>{resumen}</p>
      <button className="button-read-more">
        Leer más
      </button>
    </article>
  )
}

// Uso
function BlogApp() {
  return (
    <section>
      <h1>Artículos más recientes</h1>

        titulo="Entendiendo React Hooks"

Ejercicio 4: Crea un componente UserProfileCard

Desafío: Crea una tarjeta de perfil de usuario reutilizable.

Debe mostrar:

  • Foto de perfil
  • Nombre de usuario
  • Biografía
  • Ubicación
  • Un botón "Seguir"

Solución:


function UserProfileCard({ avatar, usuario, bio, ubicacion }) {
  return (
    <article className="user-profile-card">
      <img src={avatar} alt={usuario} className="avatar" />
      <h3>{usuario}</h3>
      <p className="bio">{bio}</p>
      <p className="location">📍 {ubicacion}</p>
      <button className="button-follow">
        Seguir
      </button>
    </article>
  )
}

// Uso
<UserProfileCard
  avatar="https://example.com/avatar1.jpg"
  usuario="Jane Developer"
  bio="Entusiasta de React y desarrolladora full-stack"
  ubicacion="San Francisco, CA"
/>
