Callbacks — Pasar Funciones como Props

🔄 ¿Qué es un Callback?

Un callback es una función que se pasa como argumento a otra función, y que será ejecutada más tarde — de ahí el nombre "call back" (llamar de vuelta).

En React, los callbacks son la forma en que los componentes hijos se comunican de vuelta con su padre: 

El padre define una función    →  handlePageChange
El padre la pasa como prop     →  onPageChange={handlePageChange}
El hijo la recibe              →  function Pagination({ onPageChange })
El hijo la invoca al necesitar →  onPageChange(3)

⬆️ Comunicación Hijo → Padre

Las props fluyen de padre a hijo. Los callbacks invierten esa comunicación: el hijo le avisa al padre que algo ocurrió. 

Sin callback — el hijo no puede comunicarse:

  App (padre)
    │
    └─ currentPage, totalPages ──→ Pagination (hijo)
                                       │
                                     click  ← usuario
                                       │
                                    ¿ahora qué? 🤷
Con callback — el hijo notifica al padre:

  App (padre)
    │
    ├─ currentPage, totalPages ──→ Pagination (hijo)
    │                                  │
    └─ handlePageChange ────────→ onPageChange
                                       │
                                     click ← usuario
                                       │
                                  onPageChange(3)
                                       │
    handlePageChange(3) ←─────────────┘
    └─ actualiza datos en App

📛 Convención on / handle

Props que reciben funciones   →  on + Evento
  onPageChange, onClick, onSubmit, onDelete

Funciones dentro del componente  →  handle + Evento
  handlePageChange, handleClick, handleSubmit
// Padre — define handle..., pasa on...
const handlePageChange = (page) => { ... }
<Pagination onPageChange={handlePageChange} />

// Hijo — recibe on..., define handle... internos
function Pagination({ onPageChange }) {
  const handleNext = () => {
    onPageChange(currentPage + 1)  // invoca el callback del padre
  }
}

⚡ Referencia vs Ejecución Inmediata

Esta es una de las confusiones más comunes en React — prestá mucha atención:

// ✅ Correcto — pasa la REFERENCIA a la función
<button onClick={handleClick}>Click</button>

// ❌ Incorrecto — EJECUTA la función al renderizar
<button onClick={handleClick()}>Click</button>

// ✅ Correcto — arrow function para pasar argumentos
<button onClick={() => handleClick(id)}>Click</button>
La regla:
  handleClick     → referencia → React la llamará cuando el usuario haga click
  handleClick()   → ejecución  → se llama AHORA, durante el render ❌
  () => handleClick(arg) → nueva función que envuelve el llamado con argumentos

🛡️ Prop de Función con Valor por Defecto

Si el callback es opcional, usá una función vacía como default para que el componente no rompa cuando no se pasa: 

// Sin default → rompe si no se pasa onPageChange
function Pagination({ onPageChange }) {
  onPageChange(3) // ❌ TypeError si no se proporcionó
}

// ✅ Con default → seguro aunque no se pase
function Pagination({ onPageChange = () => {} }) {
  onPageChange(3) // no hace nada, pero no rompe
}

🧩 Pagination con Callbacks

Actualizamos el componente para que notifique al padre con cada click:

// src/components/Pagination.jsx

function Pagination({ currentPage = 1, totalPages = 5, onPageChange = () => {} }) {
  const pages = Array.from({ length: totalPages }, (_, i) => i + 1)

  const handlePrevious = (e) => {
    e.preventDefault()
    if (currentPage > 1) onPageChange(currentPage - 1)
  }

  const handleNext = (e) => {
    e.preventDefault()
    if (currentPage < totalPages) onPageChange(currentPage + 1)
  }

  const handlePageClick = (e, page) => {
    e.preventDefault()
    onPageChange(page)
  }

  const styleLinkLeft = {
    opacity: currentPage === 1 ? 0.5 : 1,
    cursor: currentPage === 1 ? 'not-allowed' : 'pointer',
  }

  const styleLinkRight = {
    opacity: currentPage === totalPages ? 0.5 : 1,
    cursor: currentPage === totalPages ? 'not-allowed' : 'pointer',
  }

  return (
    <nav className="pagination">

      <a href="#" style={styleLinkLeft} onClick={handlePrevious}>
        <svg width="16" height="16" viewBox="0 0 24 24"
          fill="none" stroke="currentColor"
          strokeWidth="2" strokeLinecap="round" strokeLinejoin="round"
        >
          <path stroke="none" d="M0 0h24v24H0z" fill="none" />
          <path d="M15 6l-6 6l6 6" />
        </svg>
      </a>

      {pages.map((page) => (
        <a
          key={page}
          className={currentPage === page ? 'is-active' : ''}
          href="#"
          onClick={(e) => handlePageClick(e, page)}
        >
          {page}
        </a>
      ))}

      <a href="#" style={styleLinkRight} onClick={handleNext}>
        <svg width="16" height="16" viewBox="0 0 24 24"
          fill="none" stroke="currentColor"
          strokeWidth="1.5" strokeLinecap="round" strokeLinejoin="round"
        >
          <path stroke="none" d="M0 0h24v24H0z" fill="none" />
          <path d="M9 6l6 6l-6 6" />
        </svg>
      </a>

    </nav>
  )
}

export default Pagination

🔘 Ejemplo Adicional: Button Reutilizable

El mismo patrón funciona para cualquier componente interactivo:

// Componente Button reutilizable
function Button({ children, onClick, variant = 'primary' }) {
  return (
    <button className={`btn btn-${variant}`} onClick={onClick}>
      {children}
    </button>
  )
}

// Uso en App
function App() {
  const handleSave = () => console.log('Guardando...')
  const handleDelete = () => console.log('Eliminando...')

  return (
    <>
      <Button onClick={handleSave}>Guardar</Button>
      <Button onClick={handleDelete} variant="danger">Eliminar</Button>
    </>
  )
}

📦 Pasando Argumentos al Callback

Cuando necesitás enviar datos adicionales al padre, usás una arrow function:

// JobCard notifica al padre con el ID del trabajo
function JobCard({ job, onApply }) {
  return (
    <article>
      <h3>{job.title}</h3>
      {/* Arrow function para pasar job.id como argumento */}
      <button onClick={() => onApply(job.id)}>Aplicar</button>
    </article>
  )
}

// El padre recibe el ID y decide qué hacer
function App() {
  const handleApply = (jobId) => {
    console.log('Aplicando a trabajo:', jobId)
  }

  return <JobCard job={job} onApply={handleApply} />
}

✅ Resumen de la Clase

🔄 Callback        — función pasada como prop para notificar al padre
⬆️ Hijo → Padre    — el hijo llama al callback, el padre reacciona
📛 on / handle     — onEvento para props, handleEvento para funciones
⚡ Ref vs Ejecución — handleClick ✅  vs  handleClick() ❌
🛡️ Default vacío   — onPageChange = () => {} evita romper si no se pasa
📦 Con argumentos  — () => handleClick(id) para pasar datos al callback
