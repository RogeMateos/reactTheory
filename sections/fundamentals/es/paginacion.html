Paginación — Props y Comunicación Padre-Hijo

⬇️ Flujo de Datos: Padre → Hijo

El componente padre (App) controla los datos y los pasa al hijo (Pagination) como props. El hijo solo sabe renderizar lo que recibe — no maneja ni decide los datos. 

App (padre)
  │
  ├─ currentPage: 3
  ├─ totalPages: 5
  │
  ▼
Pagination (hijo)
  └─ Renderiza botones según estos valores

🎯 Props por Defecto

¿Qué pasa si el componente se usa sin pasarle props? Sin valores por defecto, se rompe.

// ❌ Forma antigua — verboso
function Pagination({ currentPage, totalPages }) {
  currentPage = currentPage || 1
  totalPages = totalPages || 5
  // ...
}

// ✅ Forma moderna — ES6, limpio y directo
function Pagination({ currentPage = 1, totalPages = 5 }) {
  // Si no se pasan props, usa los valores por defecto
}
// Sin props → usa valores por defecto
<Pagination />
// currentPage = 1, totalPages = 5

// Con una prop → combina pasada + defecto
<Pagination currentPage={3} />
// currentPage = 3, totalPages = 5

// Con ambas → usa las pasadas
<Pagination currentPage={3} totalPages={10} />
// currentPage = 3, totalPages = 10

🎨 Estilos Inline con Objetos

En React, los estilos inline se pasan como objetos JavaScript, no como strings. Las propiedades CSS usan camelCase. 

// Objeto de estilos — las propiedades son dinámicas
const styleLinkLeft = {
  opacity: currentPage === 1 ? 0.5 : 1,
  cursor: currentPage === 1 ? 'not-allowed' : 'pointer',
}

// Se aplica con la prop style={}
<a href="#" style={styleLinkLeft}>...</a>
CSS normal       →  React
background-color →  backgroundColor
font-size        →  fontSize
border-radius    →  borderRadius
z-index          →  zIndex

🚫 preventDefault()

Los <a href="#"> por defecto navegan al inicio de la página al hacer click. preventDefault() cancela ese comportamiento y ejecuta solo tu lógica. 

const handleNext = (e) => {
  e.preventDefault()  // ← cancela el scroll al inicio
  // ejecuta tu lógica
}

📛 Convención de Nombres para Event Handlers

React tiene una convención clara para nombrar funciones relacionadas con eventos:

Props que reciben funciones   →  on + Evento
  onPageChange, onClick, onSubmit

Funciones dentro del componente  →  handle + Evento
  handleNext, handlePrevious, handlePageClick
// En el padre: la función se llama handle...
const handlePageChange = (page) => {
  setCurrentPage(page)
}

// Al pasarla al hijo: la prop se llama on...
<Pagination onPageChange={handlePageChange} />

// En el hijo: recibe on... y la invoca
function Pagination({ onPageChange }) {
  return <button onClick={() => onPageChange(2)}>2</button>
}

🔢 Generando el Array de Páginas

Para renderizar los botones de página necesitamos un array de números:

const pages = Array.from({ length: totalPages }, (_, i) => i + 1)

// Si totalPages = 5 → [1, 2, 3, 4, 5]
// Si totalPages = 3 → [1, 2, 3]
¿Cómo funciona?

Array.from()             → crea un array nuevo
{ length: totalPages }   → define cuántos elementos
(_, i) => i + 1          → _ ignora el valor, i es el índice
                            índice 0 → página 1
                            índice 1 → página 2
                            índice n → página n+1

🔄 Renderizando la Lista con .map()

{pages.map((page) => (
  <a
    key={page}
    className={currentPage === page ? 'is-active' : ''}
    href="#"
  >
    {page}
  </a>
))}
key={page}  → obligatorio en listas; ayuda a React a identificar cada elemento
className   → ternaria para aplicar 'is-active' solo a la página actual

💡 Nota: Profundizaremos en renderizado de listas con .map() en la próxima clase. Por ahora entendé que transforma un array de datos en un array de elementos JSX.

🧩 Componente Pagination Completo

// src/components/Pagination.jsx

function Pagination({ currentPage = 1, totalPages = 5 }) {
  const pages = Array.from({ length: totalPages }, (_, i) => i + 1)

  const styleLinkLeft = {
    opacity: currentPage === 1 ? 0.5 : 1,
    cursor: currentPage === 1 ? 'not-allowed' : 'pointer',
  }

  const styleLinkRight = {
    opacity: currentPage === totalPages ? 0.5 : 1,
    cursor: currentPage === totalPages ? 'not-allowed' : 'pointer',
  }

  return (
    <nav className="pagination">

      {/* Botón anterior */}
      <a href="#" style={styleLinkLeft}>
        <svg width="16" height="16" viewBox="0 0 24 24"
          fill="none" stroke="currentColor"
          strokeWidth="2" strokeLinecap="round" strokeLinejoin="round"
        >
          <path stroke="none" d="M0 0h24v24H0z" fill="none" />
          <path d="M15 6l-6 6l6 6" />
        </svg>
      </a>

      {/* Botones de página */}
      {pages.map((page) => (
        <a
          key={page}
          className={currentPage === page ? 'is-active' : ''}
          href="#"
        >
          {page}
        </a>
      ))}

      {/* Botón siguiente */}
      <a href="#" style={styleLinkRight}>
        <svg width="16" height="16" viewBox="0 0 24 24"
          fill="none" stroke="currentColor"
          strokeWidth="1.5" strokeLinecap="round" strokeLinejoin="round"
        >
          <path stroke="none" d="M0 0h24v24H0z" fill="none" />
          <path d="M9 6l6 6l-6 6" />
        </svg>
      </a>

    </nav>
  )
}

export default Pagination

💪 Ejercicio: Pagination con Callbacks

El componente actual solo muestra la paginación — los clicks no hacen nada todavía. Tu objetivo: agregar callbacks para que el hijo le comunique al padre qué página se seleccionó.

Paso 1 — Recibir la función como prop

// Agregá onPageChange a las props del componente
function Pagination({ currentPage = 1, totalPages = 5, onPageChange }) {
  // ...
}

Paso 2 — Llamarla en los event handlers

const handlePrevious = (e) => {
  e.preventDefault()
  if (currentPage > 1) onPageChange(currentPage - 1)
}

const handleNext = (e) => {
  e.preventDefault()
  if (currentPage < totalPages) onPageChange(currentPage + 1)
}

const handlePageClick = (e, page) => {
  e.preventDefault()
  onPageChange(page)
}

Paso 3 — Conectar en App con useState

import { useState } from 'react'
import Pagination from './components/Pagination'

function App() {
  const [currentPage, setCurrentPage] = useState(1)
  const totalPages = 5

  const handlePageChange = (page) => {
    setCurrentPage(page)
    console.log('Página seleccionada:', page)
  }

  return (
    <main>
      <p>Estás en la página {currentPage}</p>
      <Pagination
        currentPage={currentPage}
        totalPages={totalPages}
        onPageChange={handlePageChange}
      />
    </main>
  )
}
Flujo completo:

App                        Pagination
 │                              │
 ├─ currentPage: 1              │
 ├─ handlePageChange ──────────→│  prop: onPageChange
 │                              │
 │                              │  click → onPageChange(2)
 │                              │
 │←─────────────────────────────┤
 │  setCurrentPage(2)           │
 │  re-render                   │
 └─ currentPage: 2 ────────────→│  nueva prop, re-render
