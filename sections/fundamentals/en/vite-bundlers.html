🚀 Vite & Modern Bundlers

🧠 What is a Bundler?

A bundler is a tool that takes multiple JavaScript files, CSS, images, and other assets, and combines them into a few optimized files for the browser.

❌ Problems without a bundler

  • ❌ 100+ separate files - Each import is an additional HTTP request
  • ❌ Slow in development - The browser must download and process each file
  • ❌ No optimization - No minification, no tree-shaking
  • ❌ Order issues - Scripts must load in the correct order
  • ❌ No modern transformations - Can't use JSX, TypeScript, etc.

🚀 What is Vite?

Vite (French word meaning "fast") is a modern development tool created by Evan You (creator of Vue.js) in 2020. It's the replacement for Create React App and Webpack for modern projects.

🔥 Key features

  • ⚡ Instant startup - Dev server starts in milliseconds
  • 🔥 Ultra-fast HMR - Changes reflect instantly without page reload
  • 📦 Optimized build - Uses Rollup for production (tree-shaking, code splitting)
  • 🔌 Plugins - Extensible ecosystem (React, TypeScript, PWA, etc.)
  • 🎯 Zero config - Works out-of-the-box for React, Vue, Svelte

⚔️ Bundler Comparison

Bundler Speed Configuration Technology Year
Webpack 🐢 Slow 😫 Complex JavaScript 2012
esbuild 🚀 Ultra fast 😊 Simple Go 2020
Vite ⚡ Instant 😍 Zero config esbuild + Rollup 2020
Turbopack 🚀 Very fast 😊 Simple Rust 2022

💡 Why Vite for React?

1. ⚡ Instant setup

# With Create React App (old)
npx create-react-app my-app
# ⏰ Takes 2-5 minutes, installs 1000+ dependencies

# With Vite (modern)
npm create vite@latest my-app -- --template react
# ⚡ Takes 10 seconds, installs ~50 dependencies

2. 🔥 Fast Refresh (HMR)

When you edit a component, Vite updates ONLY that component without reloading the entire page. React state remains intact. 

      You edit Counter.jsx
            ⬇️
      Vite detects the change
            ⬇️
      Sends new code to browser
            ⬇️
      React updates Counter without losing state
      ✅ Counter stays at "5"

3. 📦 Natural imports

// Vite supports all types of imports without configuration
import React from 'react'
import './App.css'                    // CSS
import logo from './logo.svg'         // Images
import data from './data.json'        // JSON
import Button from './Button.jsx'     // Components

4. 🏗️ Optimized production build

npm run build
# Generates dist/ folder with:
# ✅ Minified code
# ✅ Tree-shaking (removes unused code)
# ✅ Code splitting (divides into chunks)
# ✅ Optimized assets

🏗️ How Vite Works

🔍 Architecture in DEVELOPMENT

      Browser requests: /src/App.jsx
            ⬇️
      Vite intercepts the request
            ⬇️
      Transforms JSX → JavaScript using esbuild
            ⬇️
      Sends transformed file to browser
            ⬇️
      Browser executes the code

      ⚡ KEY: Does NOT bundle in dev, uses native ES Modules

📦 Architecture in PRODUCTION

      npm run build
            ⬇️
      Rollup analyzes entire project
            ⬇️
      Tree-shaking (removes unused code)
      Code splitting (divides into chunks)
      Minification (reduces size)
            ⬇️
      Generates dist/ folder
      ├── assets/
      │   ├── index-abc123.js  (optimized)
      │   └── index-def456.css (compressed)
      └── index.html

🔌 ES Modules - The technology behind Vite

ES Modules are JavaScript's native module system (introduced in ES6/2015). Vite leverages this to NOT bundle in development.

ES Module example

<!-- index.html -->
<script type="module" src="/src/main.jsx"></script>

<!-- Browser understands "type=module" and handles imports -->
// src/main.jsx
import React from 'react'  // Browser requests this file
import App from './App'     // Browser requests this file

// Each import is a separate HTTP request
// But Vite serves them instantly from memory

🧠 Why this is FAST

  • ✅ No bundling in dev - Vite does NOT combine files, just transforms them
  • ✅ Browser caching - Unchanged files are reused
  • ✅ On-demand transformation - Only processes files you actually use
  • ✅ esbuild is ultra fast - Written in Go, 10-100x faster than Babel

🔌 Plugins and Configuration

vite.config.js - Configuration file

// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],           // Official React plugin
  server: {
    port: 3000,                 // Development port
    open: true                  // Open browser automatically
  },
  build: {
    outDir: 'dist',             // Output folder
    sourcemap: true             // Generate sourcemaps for debugging
  }
})

🔌 Popular plugins

  • @vitejs/plugin-react - React support with Fast Refresh
  • vite-plugin-pwa - Progressive Web App
  • vite-plugin-checker - TypeScript and ESLint in dev server
  • vite-plugin-compression - gzip/brotli compression

📝 Interview Summary

  • 🚀 Vite is the modern bundler for React - Replaces CRA and Webpack
  • ⚡ Uses ES Modules in dev - That's why it's instant (no bundling)
  • 🔥 Ultra-fast HMR - Only updates the modified component
  • 📦 Uses Rollup in production - Tree-shaking, code splitting, minification
  • 🔧 Zero config - Works without configuration for React
  • 🎯 esbuild - Transforms JSX 10-100x faster than Babel (written in Go)

💡 Tip: In interviews, mention that Vite does NOT bundle in development (uses native ES Modules), but DOES bundle in production with Rollup. This shows you understand the architecture.

5️⃣ Fast Refresh in action

// BEFORE editing (counter is at 5)
function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  )
}

// AFTER editing (you change the text)
function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Counter: {count}</p>  {/* ← Changed this */}
      <button onClick={() => setCount(count + 1)}>
        Increment  {/* ← And this */}
      </button>
    </div>
  )
}

// ✅ Component updates WITHOUT reloading the page
// ✅ State remains 5 (not lost)

6️⃣ Custom configuration

// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 3000,              // Change port
    open: true,              // Open browser automatically
    host: true               // Expose on local network
  },
  resolve: {
    alias: {
      '@': '/src'            // Alias for imports (@/components/Button)
    }
  },
  build: {
    outDir: 'build',         // Change output folder
    sourcemap: true,         // Generate sourcemaps
    minify: 'esbuild'        // Use esbuild for minification
  }
})

7️⃣ Tree Shaking - Unused code elimination

// utils.js - we export 3 functions
export function add(a, b) { return a + b }
export function subtract(a, b) { return a - b }
export function multiply(a, b) { return a * b }

// App.jsx - we only use 1 function
import { add } from './utils'

function App() {
  const result = add(5, 3)
  return <div>{result}</div>
}

// ✅ In production (npm run build):
// - Only add() is included in the bundle
// - subtract() and multiply() are removed (tree-shaking)

🔟 Import assets dynamically

// Import image dynamically
function Avatar({ userId }) {
  // URL resolves correctly in build
  const avatarUrl = new URL(`./avatars/${userId}.png`, import.meta.url).href

  return <img src={avatarUrl} alt="Avatar" />
}

// Alternative: use static import
import avatar1 from './avatars/1.png'
import avatar2 from './avatars/2.png'

const avatars = { 1: avatar1, 2: avatar2 }

function Avatar({ userId }) {
  return <img src={avatars[userId]} alt="Avatar" />
}

⚡ Exercise 4: Compare with CDN approach

Goal:

Understand why Vite is better than importing React from CDN.

Steps:

  1. Create a file test-cdn.html:
<!DOCTYPE html>
<html>
<head>
  <script crossorigin src="https://unpkg.com/react@18/umd/react.development.js"></script>
  <script crossorigin src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
  <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
</head>
<body>
  <div id="root"></div>
  <script type="text/babel">
    function App() {
      return <h1>Hello from CDN</h1>
    }
    ReactDOM.render(<App />, document.getElementById('root'))
  </script>
</body>
</html>
  1. Open browser and go to DevTools → Network
  2. Reload page and observe:
    • ❌ 3+ HTTP requests for CDN libraries
    • ❌ React in development mode (unoptimized, ~1 MB)

  1. In your Vite project, go to Network
  2. Observe:
    • ✅ 1 optimized JS file (~150 KB compressed)
    • ✅ React in production mode (minified)
    • ✅ JSX already transformed (not done in browser)
