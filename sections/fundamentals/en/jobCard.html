Creating JobCard: Our First Real Component

Until now we've learned:

  • What JSX is
  • What components are
  • The difference between components and normal functions

Now we're going to apply all of that by creating a real component for our DevJobs application.

This will be the first component that represents a real piece of UI.

1️⃣ The Goal: A Job Card

We want to create something like this:

-------------------------------
Frontend Developer
Tech Solutions | Remote

We are looking for a developer...
[ Apply ]
-------------------------------

Each card will have different data, but the structure will be the same.

This is where React shines.

We create a reusable component.

2️⃣ The JobCard Component

This is our first real component.

function JobCard({ title, company, location, description }) {
  return (
    <article className="job-listing-card">
      <h3>{title}</h3>
      <small>
        {company} | {location}
      </small>
      <p>{description}</p>
      <button className="button-apply-job">
        Apply
      </button>
    </article>
  )
}

This component:

  • Receives 4 props
  • Returns JSX
  • Represents a job card

3️⃣ What are Props?

Props are data passed to a component.

Example:

<JobCard
  title="Frontend Developer"
  company="Tech Solutions"
  location="Remote"
  description="We are looking for a developer..."
/>

Inside the component we can use them:

{title}
{company}
{location}
{description}

React replaces those values automatically.

4️⃣ Props Destructuring

Notice this line:

function JobCard({ title, company, location, description })

This is called destructuring.

Instead of doing this:

function JobCard(props) {
  return (
    <h3>{props.title}</h3>
  )
}

We can do this:

function JobCard({ title }) {
  return <h3>{title}</h3>
}

Advantages:

  • Cleaner code
  • Easier to read
  • Very common in React

5️⃣ The App Component

Now we have a main component that uses JobCard.

function App() {
  return (
    <section>
      <h2>Available Jobs</h2>
      <div className="jobs-listings">
        <JobCard
          title="Frontend Developer"
          company="Tech Solutions"
          location="Remote"
          description="We are looking..."
        />
        <JobCard
          title="Senior Software Engineer"
          company="Data Driven Co."
          location="Mexico City"
          description="Looking for an engineer..."
        />
      </div>
    </section>
  )
}

Here we see component composition.

6️⃣ What is Component Composition?

React works like LEGO.

You build interfaces using small components.

Example:

App
 ├── Header
 ├── JobList
 │    ├── JobCard
 │    ├── JobCard
 │    └── JobCard
 └── Footer

Each component has its own responsibility.

7️⃣ Comparison with Vanilla JavaScript

Before with plain JavaScript we did this:

jobs.forEach((job) => {
  const article = document.createElement('article')
  article.innerHTML = `
    <h3>${job.title}</h3>
    <small>${job.company}</small>
  `
  container.appendChild(article)
})

Problems:

  • ❌ Lots of repeated code
  • ❌ Manual DOM manipulation
  • ❌ HTML inside strings
  • ❌ Hard to maintain

8️⃣ With React

We write the component once:

function JobCard({ title, company, location, description }) {
  return (
    <article className="job-listing-card">
      <h3>{title}</h3>
      <p>{description}</p>
    </article>
  )
}

And we use it as many times as we want.

Advantages:

  • ✅ Declarative code
  • ✅ Reusable components
  • ✅ React manages the DOM
  • ✅ Easier to maintain

1️⃣1️⃣ Best Practices with Components

1. One component = One responsibility

Correct:

function JobTitle({ children }) {
  return <h3>{children}</h3>
}

Incorrect:

function EverythingComponent() {
  // does too many things
}

2. Clear names

Correct:

UserProfileCard
JobListingItem
NavigationMenu

Incorrect:

Item
Thing
Comp1

3. Descriptive props

Correct:

function Button({ text, onClick, disabled })

Incorrect:

function Button({ t, oc, d })

Exercise 1: Create a ProductCard Component

Challenge: Create a component similar to JobCard but for products.

It should display:

  • Product name
  • Price
  • Description
  • An "Add to Cart" button

Solution:


function ProductCard({ name, price, description }) {
  return (
    <article className="product-card">

      <p className="price">${price}</p>
      <p>{description}</p>
      <button className="button-add-cart">
        Add to Cart
      </button>
    </article>
  )
}

function JobCard({

Exercise 3: Create a BlogPostCard Component

Challenge: Create a component for displaying blog posts.

It should have:

  • Title
  • Author
  • Date published
  • Excerpt
  • A "Read More" button

Solution:


function BlogPostCard({ title, author, date, excerpt }) {
  return (
    <article className="blog-post-card">
      <h2>{title}</h2>
      <div className="post-meta">
        <small>By {author} | {date}</small>
      </div>
      <p>{excerpt}</p>
      <button className="button-read-more">
        Read More
      </button>
    </article>
  )
}

function UserProfileCard({ avatar, username, bio, location }) {
