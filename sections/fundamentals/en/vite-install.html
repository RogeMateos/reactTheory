⚡ Installing Vite

🎯 From HTML to Professional Project

Up until now we've been working with React directly in an HTML file, loading React from a CDN and using Babel in the browser. This is fine for learning, but it's not what's done in real projects.

In this lesson we're going to create a professional React project using Vite, the modern development tool that companies use.

🧠 What are Node.js and npm?

Before installing Vite, we need to understand two fundamental concepts.

📦 Node.js

Node.js is a JavaScript runtime environment that allows you to execute JavaScript outside of the browser, directly on your computer. 

# You can run JavaScript in the terminal
node script.js

Why do we need it?

  • 🔧 Development tools (like Vite) are written in JavaScript
  • 📦 We need to run these tools on our computer
  • ⚡ Node.js allows Vite to compile and transform your code

📚 npm (Node Package Manager)

npm is Node.js's package manager. It's like an "app store" for JavaScript code.

What is it used for?

  • 📦 Install libraries - React, React Router, etc.
  • 🔧 Install tools - Vite, ESLint, etc.
  • 📝 Manage dependencies - What version of each library you use
  • 🚀 Run scripts - Commands for development and production

💿 Installing Node.js

If you don't have Node.js installed:

  1. Go to nodejs.org
  2. Download the LTS (Long Term Support) version
  3. Install following the instructions
  4. Verify the installation:
node --version
# Should show something like: v22.x.x

npm --version
# Should show something like: 11.x.x

🚀 Creating a project with Vite

Step 1: Create the project

Open your terminal and run:

npm create vite@latest

This command:

  • ✅ Downloads the latest version of Vite
  • ✅ Asks you some questions
  • ✅ Creates a new project with the necessary structure

Step 2: Answer the questions

1. Project name:

? Project name: › devjobs-react

Type your project name. For example: devjobs-react

2. Select a framework:

? Select a framework: ›
  Vanilla
  Vue
❯ React
  Preact
  Lit
  Svelte
  Solid
  Qwik
  Others

Use the arrows to select React and press Enter.

3. Select a variant:

? Select a variant: ›
  TypeScript
❯ TypeScript + SWC
  JavaScript
  JavaScript + SWC

Select JavaScript + SWC and press Enter.

⚡ Why SWC?

SWC (Speedy Web Compiler) is an ultra-fast compiler written in Rust.

What does SWC do?

  • Transforms JSX to JavaScript
  • Transforms modern code to compatible code
  • Is 20-70x faster than Babel

SWC vs Babel:

Babel:        ████████████████ (100s)
SWC:          █ (5s)

Why do we use it?

  • ⚡ Speed - Nearly instantaneous compilation
  • 🎯 Same result - Does the same as Babel but faster
  • 🔥 Faster Hot Reload - Changes are seen instantly

Step 3: Install dependencies

Once the project is created, navigate to the folder:

cd devjobs-react

And install the dependencies:

npm install

This installs:

  • ✅ React and React DOM
  • ✅ Vite
  • ✅ SWC
  • ✅ Other necessary tools

What's happening?

npm reads the package.json file that Vite created, downloads all the listed libraries, and saves them in a folder called node_modules/.

Step 4: Start the development server

npm run dev

You'll see something like:

  VITE v7.0.0  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help

Open your browser at http://localhost:5173/ and you'll see your React application running.

📁 Project structure

Vite creates this folder structure:

devjobs-react/
├── node_modules/       # Installed dependencies (don't touch)
├── public/             # Static files (images, etc.)
│   └── vite.svg
├── src/                # Your source code
│   ├── assets/         # Resources (images, CSS, etc.)
│   │   └── react.svg
│   ├── App.css         # App component styles
│   ├── App.jsx         # Main component
│   ├── index.css       # Global styles
│   └── main.jsx        # Application entry point
├── .gitignore          # Files Git should ignore
├── index.html          # Main HTML
├── package.json        # Project configuration and dependencies
├── package-lock.json   # Exact dependency versions
└── vite.config.js      # Vite configuration

📝 The .jsx extension

One important thing to understand is why we use the .jsx extension instead of just .js.

What does .jsx mean?

Files with the .jsx extension indicate they contain JSX code that needs to be transformed:

// App.jsx - Vite knows it has JSX
function App() {
  return <h1>Hello</h1> // ← This is JSX
}

Why is it important?

  • Vite automatically detects which files need compilation
  • Better for editors - Your IDE knows it may have JSX
  • Clear convention - Other developers know what to expect
  • Optimization - Vite can optimize these files better

.js vs .jsx

App.js   ← Pure JavaScript (no JSX)
App.jsx  ← JavaScript with JSX

Can you use .js with JSX?

Technically yes, Vite will detect it, but it's better to use .jsx to be explicit:

// ✅ Recommended: .jsx for files with JSX
// src/App.jsx
export default function App() {
  return <h1>Hello</h1>
}

// ⚠️ Works but not convention: .js with JSX
// src/App.js
export default function App() {
  return <h1>Hello</h1>
}

📊 Key differences from direct HTML

❌ Before (HTML + CDN)

<script src="https://esm.sh/react"></script>
<script src="https://unpkg.com/@babel/standalone"></script>
<script type="text/babel">
  const [count, setCount] = React.useState(0)
  // Everything in one file
</script>

Problems:

  • ❌ Babel in the browser (slow)
  • ❌ Single file (hard to maintain)
  • ❌ No Hot Reload
  • ❌ Not optimized

✅ Now (Vite + npm)

// src/App.jsx
import { useState } from 'react'

function App() {
  const [count, setCount] = useState(0)
  // ...
}

Advantages:

  • ✅ SWC compiles beforehand (fast)
  • ✅ Multiple files (organized)
  • ✅ Instant Hot Reload
  • ✅ Optimized for production

📄 Important files

Let's look at the most important files and what each one does.

1. index.html - The main HTML

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vite + React</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.jsx"></script>
  </body>
</html>

What does it do?

  • Defines the <div id="root"> where React will render the application
  • Loads the main.jsx file as a module
  • It's the HTML entry point of your application

Differences from what we did before:

<!-- Before: we loaded React from CDN -->
<script src="https://esm.sh/react"></script>

<!-- Now: Vite manages it for us -->
<script type="module" src="/src/main.jsx"></script>

2. main.jsx - React entry point

import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'
import './index.css'

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
)

What does it do?

  • Imports React - From node_modules/ (not from CDN)
  • Imports ReactDOM - To render in the browser
  • Imports App - Your main component
  • Imports styles - Global CSS
  • Renders the application - In the <div id="root">

React.StrictMode:

<React.StrictMode>
  <App />
</React.StrictMode>

It's a special component that:

  • Detects potential problems in your code
  • Only works in development (doesn't affect production)
  • Makes components render twice to find bugs

💡 Note about .jsx files: Files with the .jsx extension tell Vite that they contain JSX code that needs to be compiled. Vite automatically detects these files and transforms them using SWC.

3. App.jsx - Main component

import { useState } from 'react'
import './App.css'

function App() {
  const [count, setCount] = useState(0)

  return (
    <div className="App">
      <h1>Vite + React</h1>
      <button onClick={() => setCount((count) => count + 1)}>
        count is {count}
      </button>
    </div>
  )
}

export default App

What does it do?

  • Defines the main component of your application
  • Uses useState (no longer need React.useState)
  • Exports the component so main.jsx can import it

Important differences:

// Before (in HTML)
const [count, setCount] = React.useState(0)

// Now (with modules)
import { useState } from 'react'
const [count, setCount] = useState(0)

4. package.json - Project configuration

{
  "name": "devjobs-react",
  "private": true,
  "version": "0.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "react": "^19.2.0",
    "react-dom": "^19.2.0"
  },
  "devDependencies": {
    "@vitejs/plugin-react-swc": "^4.1.0",
    "vite": "^7.0.0"
  }
}

Important sections:

scripts - Commands you can run:

  • npm run dev - Starts the development server
  • npm run build - Creates the production version
  • npm run preview - Preview of the production build

dependencies - Libraries your app needs:

  • react - The React library
  • react-dom - To render in the browser

devDependencies - Development tools:

  • vite - The build tool
  • @vitejs/plugin-react-swc - Plugin to use SWC with React

5. vite.config.js - Vite configuration

import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react-swc'

export default defineConfig({
  plugins: [react()],
})

What does it do?

  • Configures Vite to use the React plugin with SWC
  • For now you don't need to modify this file

🛠️ Useful commands

Development

npm run dev

Starts the development server at http://localhost:5173/

Features:

  • 🔥 Hot Module Replacement - Changes are seen instantly
  • ⚡ Fast - Starts in less than 1 second
  • 🐛 Clear error messages - In the browser and console

Build for production

npm run build

Creates an optimized version in the dist/ folder:

  • 📦 Minified code - Smaller and faster
  • 🗜️ Optimized CSS - Compressed and without duplicates
  • 🎯 Optimized assets - Images, fonts, etc.
  • 🔢 Code splitting - Divides the code into chunks

Production preview

npm run preview

Serves the production build locally to test it before deploying.

Verify Node.js installation

node --version
npm --version

Install dependencies

# Install all dependencies from package.json
npm install

# Install a new dependency
npm install package-name

# Install a development dependency
npm install --save-dev package-name

    cd my-first-vite

  4. Start the development server: 
    npm run dev
  5. Open http://localhost:5173/ in your browser

  7. Create a production build: 
    npm run build

    npm run preview
