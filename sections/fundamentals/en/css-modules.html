CSS Modules

😰 The Problem: Global CSS Breaks on Its Own

When you write CSS globally, all styles share the same space. That means if two different files use the same class name, one will override the other.

It's not that someone made a mistake — it's that as the app grows, names repeat. .button, .card, .active, .title... are names any component would want to use. 

/* styles.css */
.button { background: blue; }

/* another-component.css */
.button { background: red; }  ← overrides the first one ❌

The result is unpredictable CSS: you change one style and something breaks elsewhere.

💡 The Solution: Locally Scoped Styles

CSS Modules solves this elegantly: each styles file has its own local scope. The classes you write only exist inside the component that uses them — they don't leak into the rest of the application.

The idea is simple: instead of naming the file Pagination.css, you name it Pagination.module.css. With that small change, React (via Vite or Webpack) transforms class names to make them globally unique. 

Pagination.css        → global CSS (anyone can use it)
Pagination.module.css → CSS Module (only Pagination sees it)

🔒 Encapsulation: Each Component is an Island

With CSS Modules you can use the same class name in different components without them interfering with each other: 

Button.module.css   → .button  (the blue button)
Card.module.css     → .button  (the red button inside Card)
Modal.module.css    → .button  (the green button inside Modal)

All three use .button, but since each lives in its own module, they don't clash. React transforms them internally to unique names like Button_button__a1b2c.

🗂️ Convention: One Module per Component

CSS Module files go next to the component that uses them, with the same name: 

src/components/
  Pagination.jsx
  Pagination.module.css   ← same name, same folder

  JobCard.jsx
  JobCard.module.css

  Button.jsx
  Button.module.css

This has a practical benefit: if you delete the component, you know exactly which CSS to delete with it.

⚖️ CSS Modules vs Global CSS — When to Use Each

Use CSS Modules for:
  ✅ Component-specific styles
  ✅ Reusable components (Button, Card, Modal)
  ✅ Medium or large projects

Use global CSS for:
  ✅ Reset / Normalize
  ✅ CSS variables (:root)
  ✅ body and html styles
  ✅ Generic utilities (.container, .text-center)
  ✅ Base fonts and typography

It's not an either/or choice — they can coexist in the same project.

📁 Create the .module.css File

/* src/components/Pagination.module.css */

.pagination {
  display: flex;
  justify-content: center;
  gap: 0.5rem;
  margin-block: 2rem;

  a {
    display: flex;
    align-items: center;
    justify-content: center;
    width: 2.5rem;
    height: 2.5rem;
    text-decoration: none;
    color: var(--text-muted);
    border-radius: 0.375rem;
    transition: all 0.3s;

    &:hover,
    &:focus {
      background-color: #fff;
    }

    &:active {
      transform: scale(0.9);
    }
  }
}

.isActive {
  background-color: var(--primary-light);
  color: white;
  pointer-events: none;
}

💡 Note: CSS nesting (a inside .pagination) and &:hover work with Vite and modern tools. Use camelCase for class names: .isActive instead of .is-active.

📥 Import and Use in the Component

// src/components/Pagination.jsx
import styles from './Pagination.module.css'
//      ↑
//  "styles" is an object with the file's class names

function Pagination({ currentPage = 1, totalPages = 5, onPageChange = () => {} }) {
  const pages = Array.from({ length: totalPages }, (_, i) => i + 1)

  // ...handlers...

  return (
    <nav className={styles.pagination}>
    {/*            ↑ styles.pagination → "Pagination_pagination__a1b2c" */}

      {pages.map((page) => (
        <a
          key={page}
          className={currentPage === page ? styles.isActive : ''}
          {/*                               ↑ only applied to the active page */}
          href="#"
          onClick={(e) => handlePageClick(e, page)}
        >
          {page}
        </a>
      ))}

    </nav>
  )
}

🔍 What Happens Under the Hood

When you import a CSS Module, you receive a JavaScript object where each key is your class name and the value is the generated unique name: 

// What you get when importing
styles = {
  pagination: 'Pagination_pagination__a1b2c',
  isActive:   'Pagination_isActive__d3e4f',
}

// What React renders in the DOM
<nav class="Pagination_pagination__a1b2c">
  <a class="Pagination_isActive__d3e4f">1</a>
</nav>
Generated name pattern:

[Component]_[class]__[hash]

.pagination → Pagination_pagination__a1b2c
.isActive   → Pagination_isActive__d3e4f

That hash is what guarantees they never clash with other classes.

🎨 Multiple Classes

// Template literal — most common way
<a className={`${styles.link} ${styles.active}`}>Link</a>

// Conditional
<a className={currentPage === page ? styles.isActive : ''}>{page}</a>

// Mix module + global class
<nav className={`${styles.pagination} container`}>
{/*             ↑ CSS Module           ↑ global CSS */}
</nav>

// Array + filter — cleaner with many conditions
<a className={[styles.link, isActive && styles.active].filter(Boolean).join(' ')}>
  Link
</a>

🌐 Escaping to Global Scope: :global()

Inside a CSS Module you can mark a class as global when you need to style elements from external libraries that inject their own HTML: 

/* Pagination.module.css */

.pagination {
  display: flex;  /* local ✅ */
}

:global(.container) {
  max-width: 1200px;  /* global — affects the whole app */
}

/* Style classes from an external library */
:global(.external-tooltip) {
  background: #333;
  color: #fff;
}

🐛 Common Errors

1. Forgetting .module in the filename

Pagination.css        ❌ global CSS — no local scope
Pagination.module.css ✅ CSS Module — local scope guaranteed

2. Using hyphens in class names

/* ❌ With hyphens — awkward to access */
.is-active { color: white; }

// In the component you need brackets
<a className={styles['is-active']}>Link</a>

/* ✅ With camelCase — clean access */
.isActive { color: white; }

<a className={styles.isActive}>Link</a>

3. Trying to use the class as a plain string

// ❌ The real class is "Pagination_pagination__a1b2c", not "pagination"
<nav className="pagination">

// ✅ Always through the styles object
<nav className={styles.pagination}>
