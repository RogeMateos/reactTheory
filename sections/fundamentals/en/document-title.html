Managing <title> in React

🐛 The common approach (and its problems)

It's common to see code that updates the page title using useEffect: 

// Works, but it's not the cleanest way
useEffect(() => {
  document.title = 'My React App'
}, [])

Manually manipulating the DOM from an effect breaks React's declarative nature. There's a more elegant alternative.

✅ The solution: use <title> directly in JSX

React allows you to use SEO tags like <title> or <meta> directly in JSX, without manually touching the DOM: 

function App() {
  return (
    <>
      <title>My React App</title>
      <div>{/* Rest of your app */}</div>
    </>
  )
}
Advantage Description
✅ Declarative The title is part of the render, not a side effect
✅ Less code No extra hook needed
✅ More React Everything is in JSX, where it belongs
✅ Automatic Updates with every render, no extra code

⚠️ The hidden problem: duplicate <title>

When you implement this solution, if you inspect the DOM in DevTools you'll see two <title> tags in the <head>.

Why? Because:

  • React renders its own <title> from the component
  • But index.html already has a default <title>

Official React warning: You should only render a single <title> at a time. If more than one component renders a <title> simultaneously, React places all of them in the <head> and the behavior of browsers and search engines is undefined.

🔧 Solution: remove <title> from index.html

Remove the <title> tag from the static HTML and let React control the title entirely from your components: 

<!-- ❌ Before (index.html) -->
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Vite + React</title>  <!-- remove this line -->
</head>

<!-- ✅ After (index.html) -->
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <!-- no title here — React manages it -->
</head>

🔄 Dynamic title based on state

The big advantage of managing the title from React is that it can depend on application state. The tab title updates in real time: 

function App() {
  const [jobs, setJobs]     = useState([])
  const [loading, setLoading] = useState(true)
  const [error, setError]   = useState(null)

  return (
    <>
      <title>
        {loading
          ? 'Loading...'
          : error
          ? 'Error loading'
          : `${jobs.length} jobs found`}
      </title>
      <div>{/* Rest of the app */}</div>
    </>
  )
}

Important about interpolation: Children of <title> must be a single string. Use template literals (`${jobs.length} jobs`) to produce one string. If you used {jobs.length} jobs without backticks, you'd be passing an array of two elements (number + text), which causes an error.

📋 Dynamic title use cases

// The tab title shows in real time:

"Loading…"           ← while data is being fetched
"Error loading"      ← if something goes wrong
"25 jobs found"      ← when data arrives
"No results found"   ← if the search returns nothing

🏗️ Full example with a getTitle function

function JobsApp() {
  const [jobs, setJobs]     = useState([])
  const [loading, setLoading] = useState(true)
  const [error, setError]   = useState(null)

  useEffect(() => {
    fetch('https://jscamp-api.vercel.app/api/jobs')
      .then((res) => res.json())
      .then((data) => {
        setJobs(data.data)
        setLoading(false)
      })
      .catch((err) => {
        setError(err)
        setLoading(false)
      })
  }, [])

  const getTitle = () => {
    if (loading)           return 'Loading jobs...'
    if (error)             return 'Error loading jobs'
    if (jobs.length === 0) return 'No jobs found'
    return `${jobs.length} jobs available - JSCamp`
  }

  return (
    <>
      <title>{getTitle()}</title>
      <main>
        {loading && <p>Loading...</p>}
        {error   && <p>Error: {error.message}</p>}
        {!loading && !error && <JobsList jobs={jobs} />}
      </main>
    </>
  )
}

⚠️ Cases where <title> does NOT go to <head>

According to the official React docs, there are two exceptions where <title> is not placed in the document <head>:

1. Inside an <svg> component

In this context, <title> doesn't represent the document title — it's an accessibility annotation for that SVG graphic: 

// ✅ Correct — SVG accessibility, does not affect <head>
<svg>
  <title>Sales chart</title>
  {/* SVG content */}
</svg>

2. With the itemProp prop

If <title> has an itemProp attribute, it also has no special behavior. It represents schema.org structured metadata about part of the page, not the document title: 

// ✅ Correct — structured metadata, does not affect <head>
<div itemScope itemType="https://schema.org/Movie">
  <title itemProp="name">The Godfather</title>
  {/* More structured metadata */}
</div>

📚 Libraries for larger projects

In larger apps you can use dedicated libraries. The principle is the same: manage SEO declaratively from React. 

// react-helmet
import { Helmet } from 'react-helmet'

function Page() {
  return (
    <>
      <Helmet>
        <title>My page - JSCamp</title>
        <meta name="description" content="..." />
      </Helmet>
      {/* Content */}
    </>
  )
}

// Next.js
import Head from 'next/head'

function Page() {
  return (
    <>
      <Head>
        <title>My page - JSCamp</title>
      </Head>
      {/* Content */}
    </>
  )
}

✅ Best practices and ❌ what to avoid

✅ Always do:

  • Use <title> in JSX instead of document.title in an effect
  • Remove the <title> from index.html to avoid duplicates
  • Use template literals to interpolate values in the title
  • Render only one <title> at a time
  • Reflect app state in the title (loading, error, results)

❌ Never do:

  • Keep a <title> in index.html and also render one from React
  • Use document.title directly when you can use JSX
  • Pass a node array as children of <title> ({count} jobs without template literal)
  • Render multiple <title> tags from different components at the same time
