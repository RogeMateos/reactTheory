Events in React: Making Components Respond to User Actions

Until now, our React components only display information. They are static.

But real applications need interaction:

  • Click on buttons
  • Type in inputs
  • Submit forms
  • Move the mouse
  • Press keys

All of that is handled with events.

In this lesson, you'll learn to listen to events in React and execute code when they occur.

1️⃣ What are Events?

An event is an action that occurs in the browser.

Common examples:

  • 👆 Click → user clicks
  • ⌨️ Input → user types in a field
  • 📝 Change → input value changes
  • 📤 Submit → form is submitted
  • 🖱️ Hover → mouse passes over an element

React allows you to listen to those events and react to them.

2️⃣ Our Example: "Apply" Button

Let's update our JobCard component so the Apply button responds to clicks.

function JobCard({ title, company, location, description }) {

  const handleApply = () => {
    console.log('Click on the button!')
    console.log('Applying to:', title, '-', company)

    alert(`You have applied to: ${title} at ${company}`)
  }

  return (
    <article className="job-listing-card">
      <div>
        <h3>{title}</h3>
        <small>
          {company} | {location}
        </small>
        <p>{description}</p>
      </div>

      <button
        className="button-apply-job"
        onClick={handleApply}
      >
        Apply
      </button>

    </article>
  )
}

Now the button executes a function when we click it.

3️⃣ Event Handler Functions

First, we define a function that will execute when the event occurs.

const handleApply = () => {
  console.log('Click on the button!')
  alert(`You have applied to: ${title} at ${company}`)
}

This is called:

  • Event Handler
  • or Event Handler Function

By convention, they usually start with handle.

Examples:

  • handleClick
  • handleSubmit
  • handleChange
  • handleApply

This makes the code clearer.

4️⃣ Connect the Function to the Event

Then we connect the function to the button.

<button onClick={handleApply}>
  Apply
</button>

When the user clicks:

  • React → detects the click
  • React → executes handleApply()

5️⃣ Event Syntax in React

Events in React have two important rules.

1. Use camelCase

❌ Incorrect:

onclick
onchange

✅ Correct:

onClick
onChange
onSubmit
onMouseEnter

2. Pass a Reference to the Function

✅ Correct:

<button onClick={handleApply}>

❌ Incorrect:

<button onClick={handleApply()}>

Why?

Because this would execute the function immediately when rendering, not when we click.

6️⃣ Alternative: Inline Functions

You can also write the function directly.

<button onClick={() => alert('Click!')}>
  Apply
</button>

This is called an inline function.

When to use each option?

Separate function (recommended)

const handleClick = () => {
  console.log('Click!')
  alert(title)
}

<button onClick={handleClick}>Apply</button>

Better when:

  • There are multiple lines
  • You reuse the function
  • You want to test it

Inline function

<button onClick={() => alert('Click!')}>
  Apply
</button>

Better when:

  • It's a single line
  • It's very simple

7️⃣ Accessing Props Inside the Event

Event handlers can use all the component's props.

function JobCard({ title, company, location }) {

  const handleApply = () => {
    console.log(title)
    console.log(company)
    console.log(location)

    alert(`Applying to ${title}`)
  }

  return (
    <button onClick={handleApply}>
      Apply
    </button>
  )
}

This works because of a fundamental JavaScript concept:

Closures

The function remembers the variables from the context where it was created.

8️⃣ Comparison with Vanilla JavaScript

Before (traditional JavaScript):

const jobsListingSection = document.querySelector('.jobs-listings')

jobsListingSection.addEventListener('click', function (event) {

  const element = event.target

  if (element.classList.contains('button-apply-job')) {

    element.textContent = 'Applied!'
    element.classList.add('is-applied')
    element.disabled = true

  }

})

Problems:

  • ❌ Search for elements with querySelector
  • ❌ Event delegation
  • ❌ Manual DOM manipulation
  • ❌ Hard to maintain code

In React, everything is much simpler:

const handleApply = () => {
  alert(`Applying to ${title}`)
}

<button onClick={handleApply}>
  Apply
</button>

Advantages:

  • ✅ Cleaner code
  • ✅ Each component manages its own logic
  • ✅ No manual DOM manipulation
  • ✅ Direct access to props

9️⃣ Common Events in React

Click

<button onClick={() => console.log('Click!')}>
  Click
</button>

Change (inputs)

<input
  onChange={(e) => console.log(e.target.value)}
/>

Submit (forms)

<form
  onSubmit={(e) => {
    e.preventDefault()
    console.log('Form submitted')
  }}
>
  <button type="submit">Send</button>
</form>

Hover

<div
  onMouseEnter={() => console.log('Entering')}
  onMouseLeave={() => console.log('Leaving')}
>
  Hover over me
</div>

Keyboard

<input
  onKeyDown={(e) => console.log(e.key)}
/>

🔟 The Event Object

Events receive an event object.

const handleClick = (event) => {
  console.log(event)
}

Useful properties:

  • event.target - the element that triggered the event
  • event.currentTarget - the element with the event listener
  • event.type - type of event
  • event.preventDefault() - prevent default behavior
  • event.stopPropagation() - stop event bubbling

1️⃣1️⃣ preventDefault Example

When we submit a form, the browser reloads the page.

We can prevent this.

function Form() {

  const handleSubmit = (event) => {
    event.preventDefault()

    console.log('Form submitted')
  }

  return (
    <form onSubmit={handleSubmit}>
      <input type="text" />
      <button type="submit">Send</button>
    </form>
  )
}

1️⃣2️⃣ Passing Arguments to Events

Sometimes you need to pass extra data.

Example:

function JobCard({ id }) {

  const handleApply = (jobId) => {
    console.log(jobId)
  }

  return (
    <button onClick={() => handleApply(id)}>
      Apply
    </button>
  )
}

1️⃣3️⃣ Multiple Events in One Component

A component can have several handlers.

function JobCard({ title }) {

  const handleApply = () => {
    alert(`Applying to ${title}`)
  }

  const handleSave = () => {
    alert(`Saving ${title}`)
  }

  const handleShare = () => {
    alert(`Sharing ${title}`)
  }

  return (
    <article>

      <h3>{title}</h3>

      <button onClick={handleApply}>
        Apply
      </button>

      <button onClick={handleSave}>
        Save
      </button>

      <button onClick={handleShare}>
        Share
      </button>

    </article>
  )
}

1️⃣4️⃣ Current Limitation

When we click Apply:

  • ✔ The function executes
  • ✔ An alert appears
  • ✔ We see messages in console

But...

  • ❌ The button doesn't change
  • ❌ It doesn't remember that you applied

Why?

Because we haven't used state yet.

1️⃣5️⃣ Common Beginner Mistakes

Mistake 1: Executing the function instead of passing it

❌ Incorrect:

<button onClick={handleClick()}>

✅ Correct:

<button onClick={handleClick}>

Mistake 2: Using onclick instead of onClick

❌ Incorrect:

onclick

✅ Correct:

onClick

Mistake 3: Forgetting preventDefault in forms

This causes page reload.

Mistake 4: Using huge inline functions

This makes maintenance difficult.

Exercise 2: Form Validation

Challenge: Create a form that prevents submission if email is invalid.

Solution:


function EmailForm() {
  const handleSubmit = (event) => {
    event.preventDefault()

    const email = event.target.email.value

    if (!email.includes('@')) {
      alert('Invalid email!')
      return
    }

    alert('Email is valid!')
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="email"
        name="email"
        placeholder="Enter email"
      />
      <button type="submit">Submit</button>
    </form>
  )
}

Exercise 3: Key Detection

Challenge: Create an input that logs when specific keys are pressed.

Solution:


function KeyDetector() {
  const handleKeyPress = (event) => {
    if (event.key === 'Enter') {
      console.log('Enter was pressed')
    } else if (event.key === 'Escape') {
      console.log('Escape was pressed')
    } else {
      console.log('Key:', event.key)
    }
  }

  return (
    <input
      type="text"
      onKeyDown={handleKeyPress}
      placeholder="Press any key"
    />
  )
}
