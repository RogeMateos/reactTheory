Protected Routes with React Router

📌 Quick Summary

This lesson explains how to protect routes in React Router using global state (Zustand, Context, or similar).

The core idea is the exact pattern used in real-world applications:

    ▼

    ▼

✅ Block 1 — ProtectedRoute

The implementation is simple and reusable:

export function ProtectedRoute({ children }) {
  const isLoggedIn = useAuthStore((state) => state.isLoggedIn)

  if (!isLoggedIn) {
    return <Navigate to="/login" replace />
  }

  return children
}

Small improvement: direct return expression

export function ProtectedRoute({ children }) {
  const isLoggedIn = useAuthStore(state => state.isLoggedIn)

  return isLoggedIn
    ? children
    : <Navigate to="/login" replace />
}

It's just one expression:


Is the user logged in?

Yes              No
 ↓               ↓
children       Navigate

Fewer lines and much easier to read.

✅ Block 2 — Subscribing to Global State Correctly

const isLoggedIn = useAuthStore(state => state.isLoggedIn)

Only subscribes to one property:


Store
{
  user,
  theme,
  jobs,
  favorites,
  isLoggedIn   ◀──── ProtectedRoute listens to this only
}

If favorites changes → ProtectedRoute does NOT re-render.

❌ What NOT to do:

const store = useAuthStore()
// Subscribes to the ENTIRE store
// Any change → re-render

✅ Block 4 — Better UX: Remember Where the User Was

Current behavior:


User → /profile → not logged in → /login → logs in → goes to /profile? ❌

We want to remember where they came from so after login they return to the original destination.

import { Navigate, useLocation } from 'react-router-dom'

export function ProtectedRoute({ children }) {
  const isLoggedIn = useAuthStore(state => state.isLoggedIn)
  const location = useLocation()

  if (!isLoggedIn) {
    return (
      <Navigate
        to="/login"
        replace
        state={{ from: location }}
      />
    )
  }

  return children
}

Then in the login page, after successful login:

const location = useLocation()
const from = location.state?.from?.pathname || '/profile'

navigate(from)

The flow:


User

    ▼
ProtectedRoute
    │
    ▼
/login
(state: "/profile")
    │
    ▼
Login successful
    │
    ▼
Returns automatically to /profile

A much more natural experience.

✅ Block 5 — Automatic Logout Redirect

If the user logs out while on /profile, they get redirected automatically — with no extra code needed.


ProfilePage
  │
  │ uses
  ▼
ProtectedRoute
  │
  │ listens to
  ▼
isLoggedIn = true

When logout() is called:
isLoggedIn → false

ProtectedRoute re-renders immediately

isLoggedIn = false
  ↓
Navigate("/login")

This happens automatically thanks to React's reactivity and Zustand's subscription model.

📋 Summary

  • ✅ ProtectedRoute reads isLoggedIn from the global store
  • ✅ Subscribe selectively — only to isLoggedIn, not the entire store
  • ✅ Wrap protected routes in <ProtectedRoute> inside your <Route>
  • ✅ Pass state={{ from: location }} on redirect to preserve navigation intent
  • ✅ Logout redirect happens automatically — no extra code needed
  • ✅ Centralize auth logic in a dedicated auth/ folder

📁 auth/authStore.js

import { create } from 'zustand'

const initialState = {
  isLoggedIn: false,
  user: null,
}

export const useAuthStore = create((set) => ({
  ...initialState,

  login: (userData) =>
    set({ isLoggedIn: true, user: userData }),

  logout: () => set(initialState),
}))

import { Navigate, useLocation } from 'react-router-dom'
import { useAuthStore } from './authStore'

export function ProtectedRoute({ children }) {
  const isLoggedIn = useAuthStore(state => state.isLoggedIn)
  const location = useLocation()

    return (
      <Navigate
        to="/login"
        replace

📁 pages/LoginPage.jsx — With redirect back

import { useNavigate, useLocation } from 'react-router-dom'
import { useAuthStore } from '../auth/authStore'

export function LoginPage() {
  const login = useAuthStore(state => state.login)
  const navigate = useNavigate()
  const location = useLocation()

  const from = location.state?.from?.pathname || '/'

  const handleLogin = () => {
    // Your auth logic here (API call, etc.)
    login({ name: 'Alice', id: 1 })
    navigate(from, { replace: true })
  }

  return (
    <div>
      <h1>Login</h1>
      <button onClick={handleLogin}>Sign In</button>
    </div>
  )
}

📁 App.jsx — Route configuration

import { BrowserRouter, Routes, Route } from 'react-router-dom'
import { ProtectedRoute } from './auth/ProtectedRoute'
import { LoginPage } from './pages/LoginPage'
import { ProfilePage } from './pages/ProfilePage'
import { Dashboard } from './pages/Dashboard'

export function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/login" element={<LoginPage />} />

        <Route
          path="/profile"
          element={
            <ProtectedRoute>
              <ProfilePage />
            </ProtectedRoute>
          }
        />

        <Route
          path="/dashboard"
          element={
            <ProtectedRoute>
              <Dashboard />
            </ProtectedRoute>
          }
        />
      </Routes>
    </BrowserRouter>
  )
}

📁 pages/ProfilePage.jsx — With logout

import { useAuthStore } from '../auth/authStore'

export function ProfilePage() {
  const user = useAuthStore(state => state.user)
  const logout = useAuthStore(state => state.logout)

  return (
    <div>
      <h1>Profile: {user?.name}</h1>
      <button onClick={logout}>Logout</button>
      {/* After logout, ProtectedRoute auto-redirects to /login */}
    </div>
  )
}

🎬 Scenario: User Tries to Visit a Protected Page

Initial state


Store: { isLoggedIn: false, user: null }
User navigates to: /profile

Step 1: React Router renders ProtectedRoute

// ProtectedRoute runs:
const isLoggedIn = useAuthStore(state => state.isLoggedIn)
// isLoggedIn = false

// So it returns:
return <Navigate to="/login" replace state={{ from: location }} />
// location.pathname = "/profile"

Step 2: User is on /login — state.from is preserved


URL: /login
location.state = { from: { pathname: "/profile" } }

Step 3: User fills in credentials and clicks Sign In

const from = location.state?.from?.pathname || '/'
// from = "/profile"

login({ name: 'Alice', id: 1 })
navigate(from, { replace: true })
// Navigates to "/profile"

Step 4: Store updates, ProtectedRoute re-renders


Store: { isLoggedIn: true, user: { name: 'Alice' } }

ProtectedRoute re-renders:
  isLoggedIn = true → returns children → ProfilePage renders

Final result


URL: /profile
User sees: ProfilePage with Alice's data

🔁 Scenario: Automatic Logout Redirect

User is on /profile, logged in


Store: { isLoggedIn: true }
URL: /profile
ProfilePage is rendering

User clicks Logout

// Inside ProfilePage:
const logout = useAuthStore(state => state.logout)
logout()
// Store resets to initialState: { isLoggedIn: false, user: null }

React reacts to the store change


ProtectedRoute was subscribed to isLoggedIn
isLoggedIn changed: true → false

ProtectedRoute re-renders:
  isLoggedIn = false → returns Navigate("/login")

URL changes to: /login

Zero extra code. The subscription model does all the work.

🗂️ Recommended Folder Structure


src/
  auth/
    ProtectedRoute.jsx   ← route guard component
    authStore.js         ← Zustand auth store
    useAuth.js           ← optional custom hook

  pages/
    LoginPage.jsx        ← handles state.from redirect
    ProfilePage.jsx      ← protected content
    Dashboard.jsx        ← protected content

  App.jsx                ← route definitions

All auth logic in one folder. Future changes (roles, refresh tokens, OAuth) stay contained.

💡 Key Takeaways

  • ProtectedRoute is just a component that reads auth state and decides what to render
  • Selective subscription: listen to isLoggedIn only — not the whole store
  • state.from: pass the original location so users return to their destination after login
  • Automatic redirect on logout: no manual code — Zustand's reactivity handles it
  • Centralize auth: one folder, all logic in one place
