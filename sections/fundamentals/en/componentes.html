React Components: A Clear Guide to Understanding Them

Before React, when we built interfaces with plain JavaScript, the code became a mess:

  • HTML + JS mixed everywhere
  • Many querySelector calls
  • Many appendChild calls
  • Repeated code

React solves this with one key idea:

👉 Components

A component is a reusable piece of the interface.

Think of them as LEGO blocks for building interfaces.

1️⃣ What is a Component?

A component is simply a JavaScript function that returns JSX. 

function Greeting() {
  return <h1>Hello from a component!</h1>
}

React will convert that JSX into HTML.

Using the Component

Once created, you use it like an HTML tag:

<Greeting />

This is one of React's most important ideas.

Instead of writing repeated HTML:

<h1>Hello</h1>
<h1>Hello</h1>
<h1>Hello</h1>

You can use:

<Greeting />
<Greeting />
<Greeting />

2️⃣ Element vs Component (Very Important for Interviews)

This question appears a lot in React interviews.

Element

An element is simple JSX.

<h1>Hello</h1>

React renders it directly to the DOM.

Component

A component is a function that returns elements.

function Greet() {
  return <h1>Hello</h1>
}

Then you use it like this:

<Greet />

🎯 Key rule for interviews:
A component is a function that returns JSX elements.

3️⃣ Your First Complete Component

Let's see a real example.

HTML Base

<div id="root"></div>

React always renders inside a container.

React Component

function Greet() {
  return <h1>Hello from a component!</h1>
}

Rendering it

const rootEl = document.querySelector('#root')
const root = ReactDOMClient.createRoot(rootEl)

root.render(<Greet />)

What happens internally

<Greet />
  ↓
executes the function
  ↓
return <h1>
  ↓
converts to real DOM

4️⃣ Props: Passing Data to Components

Components can receive data.

It's like passing arguments to a function.

function Greet(props) {
  return <h1>Hello {props.name}!</h1>
}

Usage

<Greet name="Miguel" />

Result

Hello Miguel

Simple Analogy

A component is like a coffee machine.

  • Component = machine
  • Props = ingredients
  • UI = final coffee

Example:

<Greet name="Miguel" />
<Greet name="Ana" />

Same machine, different ingredients.

5️⃣ Props Destructuring

In React, you'll almost always see this:

function Greet({ name }) {
  return <h1>Hello {name}</h1>
}

This is equivalent to:

props.name

But cleaner.

6️⃣ Multiple Props

You can pass several pieces of data.

function Greet({ name, age, city }) {
  return (
    <div>
      <h1>Hello {name}</h1>
      <p>You are {age} years old and live in {city}</p>
    </div>
  )
}

Usage

<Greet name="Miguel" age={25} city="Madrid" />

Important Rule

Strings:

name="Miguel"

JavaScript values:

age={25}

The curly braces mean:

👉 "This is JavaScript"

7️⃣ Styles in JSX

In normal HTML we'd do:

<h1 style="color:red; font-size:32px">

But in JSX, styles are JavaScript objects.

<h1 style={{ color: 'red', fontSize: '32px' }}>

Complete Example

function Greet({ name, color }) {

  const styles = {
    color: color,
    fontSize: '32px',
    fontWeight: 'bold',
    textAlign: 'center'
  }

  return <h1 style={styles}>Hello {name}</h1>
}

Usage

<Greet name="Miguel" color="blue" />
<Greet name="Ana" color="green" />

8️⃣ CamelCase in Styles

In CSS we use:

font-size
background-color
text-align

In JSX we use:

fontSize
backgroundColor
textAlign

Because JSX is JavaScript, not HTML.

9️⃣ Inline Styles with Props

We can write it shorter:

function Greet({ name, color }) {
  return (
    <h1 style={{ color, fontSize: '32px' }}>
      Hello {name}
    </h1>
  )
}

This works because in JavaScript:

color: color

Can be written as:

color

🔟 Component Reusability

Here's the true power of React.

<Greet name="Miguel" color="blue" />
<Greet name="Ana" color="green" />
<Greet name="Carlos" color="red" />

One component generates different interfaces.

1️⃣1️⃣ Component Composition

Components can be combined.

Example:

function App() {
  return (
    <div>
      <Greet name="Miguel" />
      <Greet name="Ana" />
    </div>
  )
}

This is like building with LEGO blocks.

1️⃣2️⃣ Advantages of Components

  • Reusability: Write once, use many times.
  • Organization: Each component has its logic.
  • Maintainability: Easier to change things.
  • Scalability: Large apps are built with many small components.

1️⃣3️⃣ Mental Model (Very Important)

Think of React like this:

App
 ├── Header
 ├── Sidebar
 ├── ProductCard
 ├── ProductCard
 ├── ProductCard
 └── Footer

Each part of the UI is a component.

1️⃣4️⃣ Final Summary

Components are:

Functions
  ↓
Receive props
  ↓
Return JSX
  ↓
React converts to DOM

What You Must Remember

  • ✔ Components = functions that return JSX
  • ✔ Props = data the component receives
  • ✔ JSX = JavaScript + HTML
  • ✔ Styles = JavaScript objects
  • ✔ Components = reusability

Example 5: Reusable Component (LEGO blocks)


function Button({ text, color, onClick }) {
  const buttonStyles = {
    padding: '10px 20px',
    backgroundColor: color,
    color: 'white',
    border: 'none',
    borderRadius: '4px',
    cursor: 'pointer',
    fontSize: '16px'
  }

  return (
    <button style={buttonStyles} onClick={onClick}>
      {text}
    </button>
  )
}

// Usage
function App() {
  return (
    <div>
      <Button text="Click me!" color="blue" onClick={() => alert('Clicked!')} />
      <Button text="Save" color="green" onClick={() => alert('Saved!')} />
      <Button text="Delete" color="red" onClick={() => alert('Deleted!')} />
    </div>
  )
}

Example 7: Complex Component with Multiple Props


function ProductCard({ title, price, description, image }) {
  const cardStyles = {
    border: '1px solid #ddd',
    borderRadius: '8px',
    overflow: 'hidden',
    maxWidth: '300px',
    boxShadow: '0 4px 6px rgba(0,0,0,0.1)'
  }

  const imageStyles = {
    width: '100%',
    height: '200px',
    objectFit: 'cover'
  }

  const contentStyles = {
    padding: '16px'
  }

  const titleStyles = {
    fontSize: '18px',
    fontWeight: 'bold',
    marginBottom: '8px'
  }

  const priceStyles = {
    fontSize: '20px',
    color: '#28a745',
    fontWeight: 'bold',
    marginBottom: '8px'
  }

  return (
    <div style={cardStyles}>
      <img src={image} alt={title} style={imageStyles} />
      <div style={contentStyles}>
        <h3 style={titleStyles}>{title}</h3>
        <p style={priceStyles}>${price}</p>
        <p>{description}</p>
      </div>
    </div>
  )
}

// Usage
<ProductCard
  title="Laptop"
  price={999}
  description="Powerful and portable"
  image="laptop.jpg"
/>

Practice Exercises

Exercise 1: Simple Greeting Component

Challenge: Create a Greeting component that:

  • Accepts a prop called time (morning, afternoon, evening)
  • Returns different greetings based on the time:
    • Morning: "Good morning!"
    • Afternoon: "Good afternoon!"
    • Evening: "Good evening!"

Expected usage:

<Greeting time="morning" />  // "Good morning!"

Solution:


function Greeting({ time }) {
  const greetings = {
    morning: 'Good morning!',
    afternoon: 'Good afternoon!',
    evening: 'Good evening!'
  }

  return <h1>{greetings[time]}</h1>
}

Exercise 2: User Profile Component

Challenge: Create a UserProfile component that:

  • Accepts props: name, role, email, avatar
  • Displays the user information in a styled card
  • Includes an avatar image

Expected usage:


<UserProfile
  name="John Doe"
  role="Developer"
  email="john@example.com"
  avatar="john.jpg"
/>

Solution:


function UserProfile({ name, role, email, avatar }) {
  const profileStyles = {
    textAlign: 'center',
    padding: '20px',
    border: '1px solid #ddd',
    borderRadius: '8px'
  }

  const avatarStyles = {
    width: '100px',
    height: '100px',
    borderRadius: '50%',
    objectFit: 'cover',
    marginBottom: '10px'
  }

  return (
    <div style={profileStyles}>
      <img src={avatar} alt={name} style={avatarStyles} />
      <h2>{name}</h2>
      <p><strong>{role}</strong></p>
      <p>{email}</p>
    </div>
  )
}

Exercise 3: Alert Component

Challenge: Create an Alert component that:

  • Accepts props: type (success, error, warning, info), message
  • Shows different colors for different types:
    • Success: green
    • Error: red
    • Warning: orange
    • Info: blue
  • Displays the message

Expected usage:


<Alert type="success" message="Data saved successfully!" />
<Alert type="error" message="Something went wrong!" />

Solution:


function Alert({ type, message }) {
  const colors = {
    success: '#28a745',
    error: '#dc3545',
    warning: '#ffc107',
    info: '#17a2b8'
  }

  const alertStyles = {
    padding: '12px 16px',
    borderRadius: '4px',
    backgroundColor: colors[type],
    color: type === 'warning' ? '#000' : '#fff',
    marginBottom: '10px'
  }

  return <div style={alertStyles}>{message}</div>
}

Exercise 4: Multiple Components Composition

Challenge: Create a complete app with:

  • A Header component
  • A Card component
  • An App component that uses both

Solution:


function Header() {
  return <header style={{ background: '#333', color: '#fff', padding: '20px' }}>
    <h1>My Website</h1>
  </header>
}

function Card({ title, content }) {
  const cardStyle = {
    border: '1px solid #ddd',
    padding: '16px',
    margin: '10px',
    borderRadius: '8px'
  }

  return (
    <div style={cardStyle}>
      <h3>{title}</h3>
      <p>{content}</p>
    </div>
  )
}

function App() {
  return (
    <div>
      <Header />
      <Card title="Card 1" content="This is the first card" />
      <Card title="Card 2" content="This is the second card" />
    </div>
  )
}

export default App;

Exercise 5: Interactive Challenge

Challenge: Create a Badge component that:

  • Shows a number badge
  • Has configurable color
  • Can be sized (small, medium, large)

Expected usage:


<Badge number={5} color="red" size="small" />
<Badge number={12} color="blue" size="large" />

Solution:


function Badge({ number, color = 'blue', size = 'medium' }) {
  const sizes = {
    small: { width: '24px', height: '24px', fontSize: '12px' },
    medium: { width: '32px', height: '32px', fontSize: '16px' },
    large: { width: '40px', height: '40px', fontSize: '20px' }
  }

  const badgeStyles = {
    ...sizes[size],
    borderRadius: '50%',
    backgroundColor: color,
    color: '#fff',
    display: 'flex',
    alignItems: 'center',
    justifyContent: 'center',
    fontWeight: 'bold'
  }

  return <div style={badgeStyles}>{number}</div>
}
