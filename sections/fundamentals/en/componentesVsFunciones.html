Components vs Normal Functions: Understanding the Difference

When we start with React, it's very common to think:

"If components are functions... then is any function a component?"

The answer is no.

Although components are JavaScript functions, not all functions are components.

Understanding this difference is key to writing React correctly.

1️⃣ What is a Component in React?

A React component is a special function that React uses to render the user interface (UI).

A component:

  • Returns something that React can display on screen
  • Is written in PascalCase
  • Is used as a JSX tag

Example

function Greeting() {
  return <h1>Hello from React</h1>
}

Using the component:

<Greeting />

React will execute the function and render:

<h1>Hello from React</h1>

Simple Analogy: The Printer

Imagine React as a UI printer.

Components are like templates.

  • Component → UI template
  • React → printer
  • Result → rendered HTML

Mental example:

<Greeting />
  ↓
function Greeting()
  ↓
<h1>Hello</h1>

Another Analogy: The Traffic Light

Think of Components and Normal Functions like a traffic light system:

  • 🟢 Components (Green light) → They control what you SEE on the screen
    • They render UI
    • They are in PascalCase
    • They are the "visual" part
  • 🟡 Normal Functions (Yellow light) → They help with the logic BEHIND the scenes
    • They do calculations
    • They are in camelCase
    • They support components

The traffic light tells you:

🟢 Green = "This is UI" (Component)
🟡 Yellow = "This is logic" (Normal Function)

2️⃣ What is a Normal Function?

A normal JavaScript function is simply a function that executes logic.

It's not designed to render UI.

Example

function calculateTotal(price, tax) {
  return price + price * tax
}

const total = calculateTotal(100, 0.21)

This function:

  • Does not return JSX
  • Is not used as a tag
  • Only executes logic

Key Difference

Type Purpose
Component Render UI
Normal Function Execute logic

3️⃣ How They are Invoked

This is one of the most important differences.

Components

They are used as JSX tags.

function Button() {
  return <button>Click me</button>
}

<Button />

React internally does something like:

Button()

But it manages the rendering lifecycle.

Normal Functions

They are called with parentheses.

function greet(name) {
  return `Hello ${name}`
}

greet("Miguel")

4️⃣ What They Return

Components

Return things that React can render.

function Title() {
  return <h1>My title</h1>
}

React converts it to HTML.

Normal Functions

Can return anything.

function add(a, b) {
  return a + b
}

function getUser() {
  return { name: "Miguel", age: 25 }
}

5️⃣ Naming Convention

React uses a very important rule.

Components → PascalCase

function MyComponent() {}
function PrimaryButton() {}
function JobCard() {}

Normal Functions → camelCase

function calculatePrice() {}
function validateEmail() {}
function getDate() {}

6️⃣ Why is PascalCase Mandatory?

React needs to distinguish between:

1️⃣ Native HTML

<div>
<button>
<h1>

2️⃣ Custom Components

<MyButton>
<App>
<Product>

Example

<MyButton />

React understands: → It's a custom component

If you use lowercase:

<mybutton />

React thinks: → It's an HTML tag

And that doesn't work correctly.

7️⃣ What Can React Render

A component can return several types of values.

JSX

function App() {
  return <h1>Hello world</h1>
}

Strings

function Greeting() {
  return "Hello"
}

Numbers

function Counter() {
  return 42
}

null or undefined

They don't render anything.

function Invisible() {
  return null
}

This is used a lot for conditional rendering.

8️⃣ Mixing Components and Functions

In real projects, they are always mixed.

Normal functions help components.

Example

function formatPrice(price) {
  return `${price}€`
}

function Product({ name, price }) {
  const formattedPrice = formatPrice(price)

  return (
    <div>
      <h3>{name}</h3>
      <p>{formattedPrice}</p>
    </div>
  )
}

Here we have:

  • Product → component
  • formatPrice → helper function

9️⃣ When to Use Each One

Use Components when

You need UI.

Examples:

  • Cards
  • Buttons
  • Layouts
  • Lists
  • Forms
function Card({ title, description }) {
  return (
    <div className="card">
      <h2>{title}</h2>
      <p>{description}</p>
    </div>
  )
}

Use Normal Functions when

You need logic.

Examples:

  • Format price
  • Validate email
  • Calculate totals
  • Transform data
function validateEmail(email) {
  return email.includes("@")
}

🔟 Common Beginner Mistakes

Error 1: Calling a component as a function

❌ Incorrect:

function Greeting() {
  return <h1>Hello</h1>
}

Greeting()

✅ Correct:

<Greeting />

Error 2: Using lowercase

❌ Incorrect:

function greeting() {
  return <h1>Hello</h1>
}

✅ Correct:

function Greeting() {
  return <h1>Hello</h1>
}

Error 3: Trying to render objects

❌ Incorrect:

function User() {
  return { name: "Miguel" }
}

React cannot render objects.

✅ Correct:

function User() {
  const user = { name: "Miguel", age: 25 }
  return (
    <div>
      {user.name} - {user.age}
    </div>
  )
}

💡 Key Idea You Must Remember

Components → build the interface
Functions → help with logic

React is built around composing small components.

Example 1: Component vs Function Side by Side


// COMPONENT - renders UI
function Welcome() {
  return <h1>Welcome!</h1>
}

// NORMAL FUNCTION - executes logic
function calculateDiscount(price, percentage) {
  return price * (1 - percentage / 100)
}

// Usage
export default function App() {
  const discountedPrice = calculateDiscount(100, 20)

  return (
    <div>
      <Welcome />
      <p>Price: ${discountedPrice}</p>
    </div>
  )
}

Example 2: E-commerce Product with Helper Function


// Helper function (normal function)
function formatCurrency(value) {
  return new Intl.NumberFormat('en-US', {
    style: 'currency',
    currency: 'USD'
  }).format(value)
}

function applyTax(price, taxRate) {
  return price + (price * taxRate)
}

// Component
function ProductCard({ name, basePrice, taxRate }) {
  const totalPrice = applyTax(basePrice, taxRate)
  const formattedPrice = formatCurrency(totalPrice)

  return (
    <div style={{
      border: '1px solid #ddd',
      padding: '16px',
      borderRadius: '8px'
    }}>
      <h3>{name}</h3>
      <p style={{ fontSize: '20px', fontWeight: 'bold' }}>
        {formattedPrice}
      </p>
      <button>Add to Cart</button>
    </div>
  )
}

// Usage
<ProductCard name="Laptop" basePrice={999} taxRate={0.21} />

Example 3: Form Validation


// Helper functions (normal functions)
function validateEmail(email) {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)
}

function validatePassword(password) {
  return password.length >= 8
}

// Component
function LoginForm() {
  const [email, setEmail] = React.useState('')
  const [password, setPassword] = React.useState('')
  const [errors, setErrors] = React.useState({})

  const handleSubmit = (e) => {
    e.preventDefault()
    const newErrors = {}

    if (!validateEmail(email)) {
      newErrors.email = 'Invalid email'
    }

    if (!validatePassword(password)) {
      newErrors.password = 'Password must be at least 8 characters'
    }

    if (Object.keys(newErrors).length > 0) {
      setErrors(newErrors)
    } else {
      console.log('Form submitted!')
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <input
          type="email"
          value={email}
          onChange={(e) => setEmail(e.target.value)}
          placeholder="Email"
        />
        {errors.email && <p style={{ color: 'red' }}>{errors.email}</p>}
      </div>

      <div>
        <input
          type="password"
          value={password}
          onChange={(e) => setPassword(e.target.value)}
          placeholder="Password"
        />
        {errors.password && <p style={{ color: 'red' }}>{errors.password}</p>}
      </div>

      <button type="submit">Login</button>
    </form>
  )
}

Example 4: Data Transformation


// Helper functions for data transformation
function filterByPrice(products, maxPrice) {
  return products.filter(p => p.price <= maxPrice)
}

function sortByName(products) {
  return [...products].sort((a, b) => a.name.localeCompare(b.name))
}

// Component
function ProductList({ products, maxPrice = Infinity }) {
  const filtered = filterByPrice(products, maxPrice)
  const sorted = sortByName(filtered)

  return (
    <div>
      <h2>Products</h2>
      <ul>
        {sorted.map(product => (
          <li key={product.id}>
            {product.name} - ${product.price}
          </li>
        ))}
      </ul>
    </div>
  )
}

// Usage
const products = [
  { id: 1, name: 'Laptop', price: 1000 },
  { id: 2, name: 'Mouse', price: 25 },
  { id: 3, name: 'Keyboard', price: 75 }
]

<ProductList products={products} maxPrice={500} />

Practice Exercises

Exercise 1: Identify Components vs Functions

Challenge: Look at the following code and identify which are components and which are normal functions. Explain why.


// 1
function calculateAge(birthYear) {
  return new Date().getFullYear() - birthYear
}

// 2
function UserProfile({ name, birthYear }) {
  const age = calculateAge(birthYear)
  return (
    <div>
      <h3>{name}</h3>
      <p>Age: {age}</p>
    </div>
  )
}

// 3
function getInitials(name) {
  return name
    .split(' ')
    .map(word => word[0])
    .join('')
}

// 4
function Avatar({ initials }) {
  return <div className="avatar">{initials}</div>
}

Solution:

  • calculateAge → Normal function (calculates logic)
  • UserProfile → Component (renders JSX)
  • getInitials → Normal function (transforms data)
  • Avatar → Component (renders JSX)

Exercise 2: Create a Component with Helper Functions

Challenge: Create a component called StudentCard that:

  • Receives props: name, grade (0-100)
  • Uses a helper function to convert grade to letter (A, B, C, D, F)
  • Uses another helper to determine if the student passed (grade >= 60)
  • Displays the information in a styled card

Solution:


function gradeToLetter(grade) {
  if (grade >= 90) return 'A'
  if (grade >= 80) return 'B'
  if (grade >= 70) return 'C'
  if (grade >= 60) return 'D'
  return 'F'
}

function hasPassed(grade) {
  return grade >= 60
}

function StudentCard({ name, grade }) {
  const letter = gradeToLetter(grade)
  const passed = hasPassed(grade)

  return (
    <div style={{
      border: '1px solid #ddd',
      padding: '16px',
      borderRadius: '8px',
      backgroundColor: passed ? '#e8f5e9' : '#ffebee'
    }}>
      <h3>{name}</h3>
      <p>Grade: {grade} ({letter})</p>
      <p>Status: {passed ? '✅ Passed' : '❌ Failed'}</p>
    </div>
  )
}

// Usage
<StudentCard name="Maria" grade={85} />
<StudentCard name="John" grade={55} />

Exercise 3: Build a Complete App

Challenge: Create an app with:

  • A normal function to calculate shipping cost
  • A normal function to calculate total with tax
  • A component to display the order summary

Solution:


// Helper functions
function calculateShipping(weight) {
  return weight * 2.5 // $2.50 per pound
}

function calculateTotal(subtotal, taxRate = 0.08) {
  return subtotal + (subtotal * taxRate)
}

// Component
function OrderSummary({ items, weight }) {
  const subtotal = items.reduce((sum, item) => sum + item.price, 0)
  const shipping = calculateShipping(weight)
  const total = calculateTotal(subtotal + shipping)

  return (
    <div style={{
      border: '1px solid #ddd',
      padding: '20px',
      maxWidth: '400px'
    }}>
      <h2>Order Summary</h2>

      <h3>Items:</h3>
      <ul>
        {items.map((item, idx) => (
          <li key={idx}>{item.name} - ${item.price}</li>
        ))}
      </ul>

      <hr />
      <p>Subtotal: ${subtotal.toFixed(2)}</p>
      <p>Shipping: ${shipping.toFixed(2)}</p>
      <p style={{ fontSize: '18px', fontWeight: 'bold' }}>
        Total: ${total.toFixed(2)}
      </p>
    </div>
  )
}

// Usage
const items = [
  { name: 'Laptop', price: 999 },
  { name: 'Mouse', price: 25 }
]

<OrderSummary items={items} weight={5} />
