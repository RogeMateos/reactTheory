Form Management & Filters

😰 The problem: where do form values live?

When App renders JobList and SearchForm as sibling components, SearchForm can't directly communicate its values to JobList — they simply don't know each other.

The solution: lift state to the common parent (App). The parent holds the filters and passes them down as props to whichever child needs them. 

                    App
                   /   \
          SearchForm   JobList
          (produces)   (consumes)

Data flows up via callbacks → App stores it → App passes it down as props

📋 FormData: read a form all at once

Instead of creating a useState for every field (location, level, etc.), the native FormData API lets you read all fields at once when the form is submitted.

The key is the name attribute on each <select> or <input>: that's exactly the key FormData uses to identify each field. 

<select name="location">   ← the name is the key
<select name="level">      ← the name is the key

const data = new FormData(e.target)
data.get("location")  → "remote"
data.get("level")     → "senior"

Why avoid a useState per field? With FormData you don't need to sync state on every keystroke. You only read the values when the user hits submit. Fewer re-renders, cleaner code.

🔗 Callback: the bridge between child and parent

The pattern is always the same: the parent creates the function (handleSearch) and passes it as a prop to the child. The child calls it with the data when the form is submitted. The parent updates its state, and React re-renders with the new props. 

// App.jsx
const handleSearch = (filters) => setFilters(filters)
<SearchForm onSearch={handleSearch} />

// SearchForm.jsx
props.onSearch({ location: "remote", level: "senior" })
// ↑ sends the data up to the parent

The child never stores filter state: it only notifies. The parent is the single source of truth.

🔍 Cascading filters: order matters

The key insight: we don't filter the original array multiple times independently. We filter in sequence: first by dropdowns, then by text search on that result, then paginate that result. 

jobs (original list)
  ↓ filter by selects (location, level)
jobsFilteredByDropdowns
  ↓ filter by text search
jobsWithTextFilter
  ↓ paginate
paginatedResults   ← this goes to JobList

If you filter in parallel over the original array, the filters ignore each other. In cascade, each filter works on the previous result — and that's exactly how the user expects it to behave.

⌨️ Real-time text filter (onChange)

Unlike the form submit (which fires once), the text search uses onChange and fires on every keystroke. It updates the textToFilter state in App, which immediately re-filters and re-renders JobList.

Why reset currentPage to 1? If you're on page 3 and filter down to 5 total results, page 3 no longer exists. Without the reset, the pagination component would end up in an inconsistent state.

🔧 SearchForm with FormData and useId

The SearchForm component uses useId to generate accessible IDs and FormData to read all fields on submit. 

import { useId } from 'react'

export function SearchForm({ onSearch }) {
  const locationId = useId()
  const experienceId = useId()

  const handleSubmit = (e) => {
    e.preventDefault()
    const data = new FormData(e.target)
    const filters = {
      location: data.get('location'),
      experienceLevel: data.get('experience-level'),
    }
    onSearch(filters)
  }

  return (
    <form className="search-form" onSubmit={handleSubmit}>
      <div className="form-group">
        <label htmlFor={locationId}>Location</label>
        <select id={locationId} name="location">
          <option value="">All</option>
          <option value="remote">Remote</option>
          <option value="cdmx">CDMX</option>
          <option value="barcelona">Barcelona</option>
        </select>
      </div>

      <div className="form-group">
        <label htmlFor={experienceId}>Experience level</label>
        <select id={experienceId} name="experience-level">
          <option value="">All</option>
          <option value="junior">Junior</option>
          <option value="mid">Mid-level</option>
          <option value="senior">Senior</option>
        </select>
      </div>

      <button type="submit">Search</button>
    </form>
  )
}

🏠 App.jsx: centralized state

App is the single owner of state. It centralizes the three states and applies the filters in cascade before passing the results to JobList. 

import { useState } from 'react'
import { JobList } from './components/JobList'
import { SearchForm } from './components/SearchForm'
import { Pagination } from './components/Pagination'
import { JOBS } from './data/jobs'

const JOBS_PER_PAGE = 5

export function App() {
  const [filters, setFilters] = useState({
    location: '',
    experienceLevel: '',
  })
  const [textToFilter, setTextToFilter] = useState('')
  const [currentPage, setCurrentPage] = useState(1)

  const handleSearch = (newFilters) => {
    setFilters(newFilters)
    setCurrentPage(1)
  }

  const handleTextFilter = (e) => {
    setTextToFilter(e.target.value)
    setCurrentPage(1)
  }

  // 1️⃣ Filter by dropdowns
  const jobsFilteredByDropdowns = JOBS.filter((job) => {
    if (filters.location && job.location !== filters.location) return false
    if (filters.experienceLevel && job.experienceLevel !== filters.experienceLevel) return false
    return true
  })

  // 2️⃣ Filter by free text
  const jobsWithTextFilter = jobsFilteredByDropdowns.filter((job) => {
    if (!textToFilter) return true
    const search = textToFilter.toLowerCase()
    return (
      job.title.toLowerCase().includes(search) ||
      job.company.toLowerCase().includes(search)
    )
  })

  // 3️⃣ Paginate
  const totalPages = Math.ceil(jobsWithTextFilter.length / JOBS_PER_PAGE)
  const paginatedResults = jobsWithTextFilter.slice(
    (currentPage - 1) * JOBS_PER_PAGE,
    currentPage * JOBS_PER_PAGE
  )

  return (
    <main>
      <SearchForm onSearch={handleSearch} />

      <input
        type="search"
        placeholder="Search by title or company..."
        value={textToFilter}
        onChange={handleTextFilter}
      />

      <JobList jobs={paginatedResults} />

      <Pagination
        currentPage={currentPage}
        totalPages={totalPages}
        onPageChange={setCurrentPage}
      />
    </main>
  )
}

🔎 How the text filter works

const jobsWithTextFilter = jobsFilteredByDropdowns.filter((job) => {
  if (!textToFilter) return true   // no text → everything passes
  const search = textToFilter.toLowerCase()
  return (
    job.title.toLowerCase().includes(search) ||
    job.company.toLowerCase().includes(search)
  )
})

.toLowerCase() on both sides so "REACT" matches "React". .includes() is a partial match — "rea" already matches "React Developer".

🐛 Common mistakes

Mistake 1: Forgetting the name attribute on selects

// ❌ FormData can't read this
<select id={locationId}>

// ✅ With name, FormData reads it correctly
<select id={locationId} name="location">

Without the name attribute, data.get('location') returns null. The id is for accessibility (links the label), the name is for FormData.

Mistake 2: Always filtering from jobs instead of cascading

// ❌ Each filter works on the original — they ignore each other
const filterByLocation = jobs.filter(...)
const filterByText = jobs.filter(...)  // ← ignores filterByLocation

// ✅ Each filter works on the previous result
const filterByLocation = jobs.filter(...)
const filterByText = filterByLocation.filter(...)  // ← correct cascade

Mistake 3: Not resetting currentPage when filtering

// ❌ If you're on page 3 and filter to 5 results → page 3 doesn't exist
const handleSearch = (filters) => {
  setFilters(filters)
  // setCurrentPage(1)  ← forgotten
}

// ✅ Always go back to page 1 when filters change
const handleSearch = (filters) => {
  setFilters(filters)
  setCurrentPage(1)  // ✅
}

✅ Class summary

📋 FormData          — reads all form fields at once with data.get('name')
🔗 Lifting state     — parent stores, child notifies via callbacks
🎯 onSearch prop     — callback that sends filters up to the parent
🔍 Cascading filters — dropdowns → text → pagination (order matters)
⌨️ onChange          — real-time text, fires on every keystroke
🔄 Reset page        — always go back to page 1 when filters change
🆔 useId + name      — useId for accessibility, name for FormData
