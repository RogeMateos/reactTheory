Building a Single Page Application (SPA) from Scratch with React

What is a Single Page Application (SPA)?

A Single Page Application (SPA) is a web application that doesn't reload the page when navigating between different routes. Instead of requesting new pages from the server, the SPA loads all necessary code at startup and then dynamically renders content based on the URL.

In this lesson, you'll learn to create your own SPA without using React Router or other libraries, implementing the entire navigation system from scratch. This will help you understand how SPAs work internally before using more advanced tools.

Why Build an SPA?

Traditional Navigation (Multi-Page Application)

In a traditional web app, every time you click a link:

1. A request is made to the server
2. The server returns a complete HTML page
3. The browser downloads CSS, JavaScript, images, fonts again...
4. Application state is lost
5. There's a "flicker" while the new page loads

SPA Navigation

In an SPA, when you click:

1. Page reload is prevented
2. Browser URL changes
3. React re-renders the corresponding content
4. Resources are NOT downloaded again
5. Instant and smooth navigation

Advantages:

  • ⚡ Instant navigation: no resource reloading
  • 💾 State preservation: variables and data persist
  • 🎨 Better UX: smooth transitions, no flickers
  • 📦 Less data: only new content is downloaded

The Problem: Links Reload the Page

By default, when you click an <a> tag:

function Header() {
  return (
    <header>
      <a href="/">Home</a>
      <a href="/about">About</a>
      <a href="/contact">Contact</a>
    </header>
  )
}

Each click reloads the entire page → downloads HTML, CSS, JS, images again…

Solution: Create a Custom Link Component

We'll create our own <Link> component that intercepts clicks and prevents reloading:

function Link({ href, children, ...props }) {
  const handleClick = (event) => {
    // 1. Prevent default behavior (no reload)
    event.preventDefault()

    // 2. Change URL without reloading the page
    window.history.pushState({}, '', href)

    // 3. Emit an event to notify the change
    const navigationEvent = new PopStateEvent('popstate')
    window.dispatchEvent(navigationEvent)
  }

  return (
    <a href={href} onClick={handleClick} {...props}>
      {children}
    </a>
  )
}

What Does Each Part Do?

1. preventDefault()

Prevents the browser from doing its default behavior, which is navigating to the link.

event.preventDefault()

Without this, the browser would follow the link traditionally.

2. pushState()

Changes the browser's URL without reloading the page:

window.history.pushState({}, '', href)
  • First parameter: state object (we leave it empty)
  • Second parameter: title (obsolete, we leave it empty)
  • Third parameter: the new URL

After this, the address bar shows the new URL, but the page doesn't reload.

3. Emit popstate Event

For React to notice the URL change, we manually emit a popstate event:

const navigationEvent = new PopStateEvent('popstate')
window.dispatchEvent(navigationEvent)

This event fires automatically when the user uses the browser's back/forward buttons, but since we're changing the URL manually with pushState, we must emit it ourselves.

Replace Traditional Links

Now we replace all <a> tags with our <Link> component:

function Header() {
  return (
    <header>
      <Link href="/">Home</Link>
      <Link href="/about">About</Link>
      <Link href="/contact">Contact</Link>
    </header>
  )
}

Great! Now when clicking:

  • ✅ The URL changes
  • ✅ The page doesn't reload
  • ❌ But… the content doesn't change

The Problem: React Doesn't React to URL Changes

Although the URL changes, no component is re-rendering because React doesn't know the URL has changed.

We need to:

  1. Listen for URL changes
  2. Update React state with the new route
  3. Render the corresponding content based on that route

Solution: Keep the Route in State

We'll create a state that contains the current route:

import { useState, useEffect } from 'react'

function App() {
  const [currentPath, setCurrentPath] = useState(window.location.pathname)

  return (
    <div>
      <Header />
      <main>
        <h1>Current route: {currentPath}</h1>
      </main>
    </div>
  )
}

Now we have the route in state, but it still doesn't update when we navigate.

Listen for URL Changes with useEffect

We need to listen for the popstate event to detect when the URL changes:

import { useState, useEffect } from 'react'

function App() {
  const [currentPath, setCurrentPath] = useState(window.location.pathname)

  useEffect(() => {
    // Function that updates state with the new route
    const handleLocationChange = () => {
      setCurrentPath(window.location.pathname)
    }

    // Listen for the popstate event
    window.addEventListener('popstate', handleLocationChange)

    // Clean up the event listener on unmount
    return () => {
      window.removeEventListener('popstate', handleLocationChange)
    }
  }, [])

  return (
    <div>
      <Header />
      <main>
        <h1>Current route: {currentPath}</h1>
      </main>
    </div>
  )
}

Important: We use the same function reference (handleLocationChange) to add and remove the listener. If we used different anonymous functions, it wouldn't clean up correctly.

Why the Empty Array []?

useEffect(() => {
  // ...
}, [])

We want the listener to be added only once when the component mounts, not on every render. That's why we use an empty array as a dependency.

Render Content Based on Route

Now that we have currentPath updated, we can render different content based on the route:

function App() {
  const [currentPath, setCurrentPath] = useState(window.location.pathname)

  useEffect(() => {
    const handleLocationChange = () => {
      setCurrentPath(window.location.pathname)
    }

    window.addEventListener('popstate', handleLocationChange)

    return () => {
      window.removeEventListener('popstate', handleLocationChange)
    }
  }, [])

  return (
    <div>
      <Header />
      <main>
        {currentPath === '/' && <HomePage />}
        {currentPath === '/search' && <SearchPage />}
      </main>
    </div>
  )
}

We now have a functional SPA!

The Complete Flow

When the user clicks a <Link>:

1. Link captures the click → event.preventDefault()
2. Changes the URL → window.history.pushState({}, '', href)
3. Emits the event → window.dispatchEvent(new PopStateEvent('popstate'))
4. useEffect detects the event → handleLocationChange()
5. Updates state → setCurrentPath(window.location.pathname)
6. React re-renders → displays the corresponding component

function useNavigate() {
  return (path) => {
    window.history.pushState({}, '', path)

    window.dispatchEvent(navigationEvent)
  }
}

function LoginPage() {
  const navigate = useNavigate()

    event.preventDefault()

    // Navigate programmatically

  }

  return (
    <form onSubmit={handleLogin}>

      <button type="submit">Login</button>
    </form>
  )

Example 6: Router with Dynamic Routes

function Router({ routes, currentPath }) {
  for (const route of routes) {
    // Convert /user/:id to regular expression
    const pattern = route.path.replace(/:(\w+)/g, '([^/]+)')
    const regex = new RegExp(`^${pattern}$`)
    const match = currentPath.match(regex)

    if (match) {
      // Extract parameters
      const paramNames = [...route.path.matchAll(/:(\w+)/g)].map(m => m[1])
      const params = {}

      paramNames.forEach((name, index) => {
        params[name] = match[index + 1]
      })

      // Render with params
      return <route.component params={params} />
    }
  }

  return <NotFoundPage />
}

// Usage:
const routes = [
  { path: '/', component: HomePage },
  { path: '/user/:id', component: UserPage },
  { path: '/post/:slug', component: PostPage }
]

function UserPage({ params }) {
  return <h1>User: {params.id}</h1>
}
