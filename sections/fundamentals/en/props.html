Props — Component Communication

🎁 What are Props?

Props (short for properties) are arguments you pass to a React component, the same way you pass arguments to a function. 

// Regular function
function greet(name) {
  return `Hello ${name}`
}
greet('Miguel')  // "Hello Miguel"

// React component — same idea
function Greeting({ name }) {
  return <h1>Hello {name}</h1>
}
<Greeting name="Miguel" />  // <h1>Hello Miguel</h1>

Props have 4 key characteristics:

📦 They are objects with the data you pass to the component
⬇️ They flow from parent to child (never the other way)
🔒 They are immutable (cannot be modified inside the component)
🔄 They make components reusable

⬇️ One-Way Data Flow

Props always flow from parent to child. The flow is unidirectional:

App (parent)
  │
  ├─ passes: title="Frontend Developer"
  ├─ passes: company="TechCorp"
  │
  ▼
JobCard (child)
  └─ uses: {title} and {company} to render

⚠️ Fundamental rule: A child component can never pass props to its parent directly. For that, you use callbacks — which we'll cover later.

🔓 Destructuring Props

There are two ways to receive props. The second one is recommended:

// ❌ Without destructuring — repetitive and verbose
function JobCard(props) {
  return (
    <article>
      <h3>{props.title}</h3>
      <p>{props.company}</p>
    </article>
  )
}

// ✅ With destructuring — clean and readable
function JobCard({ title, company }) {
  return (
    <article>
      <h3>{title}</h3>
      <p>{company}</p>
    </article>
  )
}

Benefits of destructuring:

✅ Cleaner and more readable code
✅ You immediately see which props the component uses
✅ Less code repetition

🎯 Default Props

You can assign a default value when a prop is not provided:

function JobCard({ title, company, location = 'Remote' }) {
  return (
    <article>
      <h3>{title}</h3>
      <p>{company}</p>
      <p>{location}</p>  {/* "Remote" if not passed */}
    </article>
  )
}

// Without location → uses "Remote"
<JobCard title="Frontend Dev" company="TechCorp" />

// With location → uses that value
<JobCard title="Frontend Dev" company="TechCorp" location="Madrid" />

❓ Conditional Rendering

You can use props to show or hide content. Two approaches:

&& operator — renders if true, renders nothing if false:

{isRemote && <span>🏠 Remote</span>}
{isFeatured && <span>⭐ Featured</span>}

⚠️ Classic trap with &&: if the value is not a boolean but a number, React renders the number 0 on screen.

{results.length && <span>Results</span>}
If results.length is 0, React renders the number 0 in the UI.

Ternary — the safest and most expressive option:

{isRemote ? <span>🏠 Remote</span> : null}
{results.length > 0 ? <span>Results</span> : null}

The ternary always works correctly because the result is explicit.

🧩 Fragments — Group Without Polluting the DOM

React has a rule: a component can only return one root element. The traditional solution is wrapping in a <div>, but that adds unnecessary nodes to the DOM. 

// ❌ Error — multiple elements without a wrapper
function JobActions() {
  return (
    <button>Apply</button>
    <button>Save</button>
  )
}

// ⚠️ Works, but adds an unnecessary <div>
function JobActions() {
  return (
    <div>
      <button>Apply</button>
      <button>Save</button>
    </div>
  )
}

// ✅ Fragment — groups without adding DOM nodes
function JobActions() {
  return (
    <>
      <button>Apply</button>
      <button>Save</button>
    </>
  )
}

Simple rule to choose:

Need styles or event handlers on the wrapper  → <div>
Just grouping elements, no wrapper needed     → <></>

📦 Types of Values You Can Pass as Props

// String
<JobCard title="Frontend Developer" location="Madrid" />

// Number — always in curly braces, never in quotes
<JobCard salary={45000} experience={3} />

// Boolean
<JobCard isRemote={true} isFeatured={false} />
<JobCard isRemote />          {/* shorthand for true */}

// Array
<JobCard tags={['React', 'TypeScript', 'CSS']} />

// Object
<JobCard
  job={{
    title: 'Frontend Developer',
    company: 'TechCorp',
    salary: 45000,
  }}
/>

// Function (callback)
<JobCard onApply={() => console.log('Applying...')} />

🏗️ Nested Destructuring

When you pass an object as a prop, you can destructure its content in two ways:

// Option 1: destructure inside the component (more readable)
function JobCard({ job }) {
  const { title, company, location } = job

  return (
    <article>
      <h3>{title}</h3>
      <p>{company}</p>
      <p>{location}</p>
    </article>
  )
}

// Option 2: destructure in the parameters (more compact)
function JobCard({ job: { title, company, location } }) {
  return (
    <article>
      <h3>{title}</h3>
      <p>{company}</p>
      <p>{location}</p>
    </article>
  )
}

💡 Recommendation: Option 1 is more readable, especially when the object has many properties.

🧩 Component Composition

Props let you build complex components from simple ones. This is one of the most powerful patterns in React: 

// Simple, reusable component
function Badge({ children, type = 'default' }) {
  return <span className={`badge badge-${type}`}>{children}</span>
}

// Component that composes Badge
function JobCard({ title, isRemote, isFeatured }) {
  return (
    <article>
      <h3>{title}</h3>
      {isRemote && <Badge type="info">🏠 Remote</Badge>}
      {isFeatured && <Badge type="success">⭐ Featured</Badge>}
    </article>
  )
}
Benefits of composition:

♻️ Badge is reusable anywhere in the app
🧩 Easy to maintain — changing Badge updates everywhere
📖 More readable and declarative code

❓ Conditional Rendering with Fragments

function JobCard({ job, showDetails }) {
  return (
    <article>
      <h3>{job.title}</h3>

      {showDetails && (
        <>
          <p>{job.description}</p>
          <p>{job.requirements}</p>
          <p>{job.benefits}</p>
        </>
      )}
    </article>
  )
}

💪 Exercise: Full JobCard with Props

Now we put it all together. You'll build a JobCard component that receives different types of props and conditionally renders badges and details.

JobCard.jsx

function JobCard({ job, isRemote, isFeatured, isNew }) {
  const { title, company, location, salary, description, tags } = job

  return (
    <article className={`job-card ${isFeatured ? 'featured' : ''}`}>

      <header className="job-card-header">
        <h3 className="job-title">{title}</h3>
        <p className="job-company">{company}</p>

        <div className="badges">
          {isNew && <span className="badge new">🆕 New</span>}
          {isFeatured && <span className="badge featured">⭐ Featured</span>}
          {isRemote && <span className="badge remote">🏠 Remote</span>}
        </div>
      </header>

      <div className="job-card-body">
        <p className="job-location">📍 {location}</p>
        <p className="job-salary">💰 {salary}</p>
        <p className="job-description">{description}</p>
      </div>

      <footer className="job-card-footer">
        <span className="job-tags">{tags.join(', ')}</span>
        <button className="btn-apply">Apply</button>
      </footer>

    </article>
  )
}

export default JobCard

✅ Class Summary

🎁 Props         — arguments you pass to components
⬇️ Flow          — always parent to child, never the reverse
🔓 Destructure   — { title, company } in the parameters
🎯 Defaults      — location = 'Remote' if not passed
❓ Conditional   — ternary is safer than &&
🧩 Fragments     — <></> to group without polluting the DOM
