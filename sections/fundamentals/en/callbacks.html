Callbacks — Passing Functions as Props

🔄 What is a Callback?

A callback is a function passed as an argument to another function, to be executed later — hence the name "call back".

In React, callbacks are how child components communicate back to their parent: 

Parent defines a function      →  handlePageChange
Parent passes it as a prop     →  onPageChange={handlePageChange}
Child receives it              →  function Pagination({ onPageChange })
Child calls it when needed     →  onPageChange(3)

⬆️ Child → Parent Communication

Props flow from parent to child. Callbacks reverse that communication: the child notifies the parent that something happened. 

Without callback — child can't communicate back:

  App (parent)
    │
    └─ currentPage, totalPages ──→ Pagination (child)
                                       │
                                     click  ← user
                                       │
                                    now what? 🤷
With callback — child notifies the parent:

  App (parent)
    │
    ├─ currentPage, totalPages ──→ Pagination (child)
    │                                  │
    └─ handlePageChange ────────→ onPageChange
                                       │
                                     click ← user
                                       │
                                  onPageChange(3)
                                       │
    handlePageChange(3) ←─────────────┘
    └─ updates data in App

📛 on / handle Convention

Props that receive functions     →  on + Event
  onPageChange, onClick, onSubmit, onDelete

Functions inside the component  →  handle + Event
  handlePageChange, handleClick, handleSubmit
// Parent — defines handle..., passes on...
const handlePageChange = (page) => { ... }
<Pagination onPageChange={handlePageChange} />

// Child — receives on..., defines internal handle...
function Pagination({ onPageChange }) {
  const handleNext = () => {
    onPageChange(currentPage + 1)  // calls the parent's callback
  }
}

⚡ Reference vs Immediate Execution

This is one of the most common confusions in React — pay close attention:

// ✅ Correct — passes the REFERENCE to the function
<button onClick={handleClick}>Click</button>

// ❌ Wrong — EXECUTES the function during render
<button onClick={handleClick()}>Click</button>

// ✅ Correct — arrow function to pass arguments
<button onClick={() => handleClick(id)}>Click</button>
The rule:
  handleClick     → reference → React calls it when the user clicks
  handleClick()   → execution → called RIGHT NOW, during render ❌
  () => handleClick(arg) → new function wrapping the call with arguments

🛡️ Function Prop with Default Value

If the callback is optional, use an empty function as default so the component doesn't break when it's not provided: 

// No default → breaks if onPageChange isn't passed
function Pagination({ onPageChange }) {
  onPageChange(3) // ❌ TypeError if not provided
}

// ✅ With default → safe even if not passed
function Pagination({ onPageChange = () => {} }) {
  onPageChange(3) // does nothing, but doesn't break
}

🧩 Pagination with Callbacks

We update the component to notify the parent on each click:

// src/components/Pagination.jsx

function Pagination({ currentPage = 1, totalPages = 5, onPageChange = () => {} }) {
  const pages = Array.from({ length: totalPages }, (_, i) => i + 1)

  const handlePrevious = (e) => {
    e.preventDefault()
    if (currentPage > 1) onPageChange(currentPage - 1)
  }

  const handleNext = (e) => {
    e.preventDefault()
    if (currentPage < totalPages) onPageChange(currentPage + 1)
  }

  const handlePageClick = (e, page) => {
    e.preventDefault()
    onPageChange(page)
  }

  const styleLinkLeft = {
    opacity: currentPage === 1 ? 0.5 : 1,
    cursor: currentPage === 1 ? 'not-allowed' : 'pointer',
  }

  const styleLinkRight = {
    opacity: currentPage === totalPages ? 0.5 : 1,
    cursor: currentPage === totalPages ? 'not-allowed' : 'pointer',
  }

  return (
    <nav className="pagination">

      <a href="#" style={styleLinkLeft} onClick={handlePrevious}>
        <svg width="16" height="16" viewBox="0 0 24 24"
          fill="none" stroke="currentColor"
          strokeWidth="2" strokeLinecap="round" strokeLinejoin="round"
        >
          <path stroke="none" d="M0 0h24v24H0z" fill="none" />
          <path d="M15 6l-6 6l6 6" />
        </svg>
      </a>

      {pages.map((page) => (
        <a
          key={page}
          className={currentPage === page ? 'is-active' : ''}
          href="#"
          onClick={(e) => handlePageClick(e, page)}
        >
          {page}
        </a>
      ))}

      <a href="#" style={styleLinkRight} onClick={handleNext}>
        <svg width="16" height="16" viewBox="0 0 24 24"
          fill="none" stroke="currentColor"
          strokeWidth="1.5" strokeLinecap="round" strokeLinejoin="round"
        >
          <path stroke="none" d="M0 0h24v24H0z" fill="none" />
          <path d="M9 6l6 6l-6 6" />
        </svg>
      </a>

    </nav>
  )
}

export default Pagination

🔘 Additional Example: Reusable Button

The same pattern works for any interactive component:

// Reusable Button component
function Button({ children, onClick, variant = 'primary' }) {
  return (
    <button className={`btn btn-${variant}`} onClick={onClick}>
      {children}
    </button>
  )
}

// Usage in App
function App() {
  const handleSave = () => console.log('Saving...')
  const handleDelete = () => console.log('Deleting...')

  return (
    <>
      <Button onClick={handleSave}>Save</Button>
      <Button onClick={handleDelete} variant="danger">Delete</Button>
    </>
  )
}

📦 Passing Arguments to the Callback

When you need to send additional data to the parent, use an arrow function:

// JobCard notifies the parent with the job ID
function JobCard({ job, onApply }) {
  return (
    <article>
      <h3>{job.title}</h3>
      {/* Arrow function to pass job.id as argument */}
      <button onClick={() => onApply(job.id)}>Apply</button>
    </article>
  )
}

// Parent receives the ID and decides what to do
function App() {
  const handleApply = (jobId) => {
    console.log('Applying to job:', jobId)
  }

  return <JobCard job={job} onApply={handleApply} />
}

✅ Class Summary

🔄 Callback        — function passed as prop to notify the parent
⬆️ Child → Parent  — child calls the callback, parent reacts
📛 on / handle     — onEvent for props, handleEvent for functions
⚡ Ref vs Execution — handleClick ✅  vs  handleClick() ❌
🛡️ Empty default   — onPageChange = () => {} avoids breaking if not passed
📦 With arguments  — () => handleClick(id) to pass data to the callback
