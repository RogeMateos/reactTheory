Pagination — Props and Parent-Child Communication

⬇️ Data Flow: Parent → Child

The parent component (App) owns the data and passes it down to the child (Pagination) via props. The child only knows how to render what it receives — it doesn't manage or decide the data. 

App (parent)
  │
  ├─ currentPage: 3
  ├─ totalPages: 5
  │
  ▼
Pagination (child)
  └─ Renders buttons based on these values

🎯 Default Props

What happens if the component is used without passing props? Without defaults, it breaks.

// ❌ Old way — verbose
function Pagination({ currentPage, totalPages }) {
  currentPage = currentPage || 1
  totalPages = totalPages || 5
  // ...
}

// ✅ Modern way — ES6, clean and direct
function Pagination({ currentPage = 1, totalPages = 5 }) {
  // If no props are passed, uses the defaults
}
// No props → uses defaults
<Pagination />
// currentPage = 1, totalPages = 5

// One prop → combines passed + default
<Pagination currentPage={3} />
// currentPage = 3, totalPages = 5

// Both props → uses the passed values
<Pagination currentPage={3} totalPages={10} />
// currentPage = 3, totalPages = 10

🎨 Inline Styles with Objects

In React, inline styles are passed as JavaScript objects, not strings. CSS properties use camelCase. 

// Style object — properties are dynamic
const styleLinkLeft = {
  opacity: currentPage === 1 ? 0.5 : 1,
  cursor: currentPage === 1 ? 'not-allowed' : 'pointer',
}

// Applied with the style prop
<a href="#" style={styleLinkLeft}>...</a>
CSS           →  React
background-color →  backgroundColor
font-size        →  fontSize
border-radius    →  borderRadius
z-index          →  zIndex

🚫 preventDefault()

<a href="#"> elements scroll to the top of the page by default when clicked. preventDefault() cancels that behavior and only runs your custom logic. 

const handleNext = (e) => {
  e.preventDefault()  // ← cancels the scroll-to-top
  // run your logic
}

📛 Event Handler Naming Convention

React has a clear convention for naming event-related functions:

Props that receive functions     →  on + Event
  onPageChange, onClick, onSubmit

Functions inside the component  →  handle + Event
  handleNext, handlePrevious, handlePageClick
// In the parent: function is named handle...
const handlePageChange = (page) => {
  setCurrentPage(page)
}

// When passing to the child: prop is named on...
<Pagination onPageChange={handlePageChange} />

// In the child: receives on... and calls it
function Pagination({ onPageChange }) {
  return <button onClick={() => onPageChange(2)}>2</button>
}

🔢 Generating the Pages Array

To render the page buttons, we need an array of numbers:

const pages = Array.from({ length: totalPages }, (_, i) => i + 1)

// If totalPages = 5 → [1, 2, 3, 4, 5]
// If totalPages = 3 → [1, 2, 3]
How it works:

Array.from()             → creates a new array
{ length: totalPages }   → defines how many elements
(_, i) => i + 1          → _ ignores the value, i is the index
                            index 0 → page 1
                            index 1 → page 2
                            index n → page n+1

🔄 Rendering the List with .map()

{pages.map((page) => (
  <a
    key={page}
    className={currentPage === page ? 'is-active' : ''}
    href="#"
  >
    {page}
  </a>
))}
key={page}  → required in lists; helps React identify each element
className   → ternary to apply 'is-active' only to the current page

💡 Note: We'll go deeper into list rendering with .map() in the next class. For now, understand that it transforms an array of data into an array of JSX elements.

🧩 Full Pagination Component

// src/components/Pagination.jsx

function Pagination({ currentPage = 1, totalPages = 5 }) {
  const pages = Array.from({ length: totalPages }, (_, i) => i + 1)

  const styleLinkLeft = {
    opacity: currentPage === 1 ? 0.5 : 1,
    cursor: currentPage === 1 ? 'not-allowed' : 'pointer',
  }

  const styleLinkRight = {
    opacity: currentPage === totalPages ? 0.5 : 1,
    cursor: currentPage === totalPages ? 'not-allowed' : 'pointer',
  }

  return (
    <nav className="pagination">

      {/* Previous button */}
      <a href="#" style={styleLinkLeft}>
        <svg width="16" height="16" viewBox="0 0 24 24"
          fill="none" stroke="currentColor"
          strokeWidth="2" strokeLinecap="round" strokeLinejoin="round"
        >
          <path stroke="none" d="M0 0h24v24H0z" fill="none" />
          <path d="M15 6l-6 6l6 6" />
        </svg>
      </a>

      {/* Page buttons */}
      {pages.map((page) => (
        <a
          key={page}
          className={currentPage === page ? 'is-active' : ''}
          href="#"
        >
          {page}
        </a>
      ))}

      {/* Next button */}
      <a href="#" style={styleLinkRight}>
        <svg width="16" height="16" viewBox="0 0 24 24"
          fill="none" stroke="currentColor"
          strokeWidth="1.5" strokeLinecap="round" strokeLinejoin="round"
        >
          <path stroke="none" d="M0 0h24v24H0z" fill="none" />
          <path d="M9 6l6 6l-6 6" />
        </svg>
      </a>

    </nav>
  )
}

export default Pagination

💪 Exercise: Pagination with Callbacks

The current component only displays the pagination — clicks don't do anything yet. Your goal: add callbacks so the child can tell the parent which page was selected.

Step 1 — Receive the function as a prop

// Add onPageChange to the component's props
function Pagination({ currentPage = 1, totalPages = 5, onPageChange }) {
  // ...
}

Step 2 — Call it in event handlers

const handlePrevious = (e) => {
  e.preventDefault()
  if (currentPage > 1) onPageChange(currentPage - 1)
}

const handleNext = (e) => {
  e.preventDefault()
  if (currentPage < totalPages) onPageChange(currentPage + 1)
}

const handlePageClick = (e, page) => {
  e.preventDefault()
  onPageChange(page)
}

Step 3 — Connect in App with useState

import { useState } from 'react'
import Pagination from './components/Pagination'

function App() {
  const [currentPage, setCurrentPage] = useState(1)
  const totalPages = 5

  const handlePageChange = (page) => {
    setCurrentPage(page)
    console.log('Selected page:', page)
  }

  return (
    <main>
      <p>You are on page {currentPage}</p>
      <Pagination
        currentPage={currentPage}
        totalPages={totalPages}
        onPageChange={handlePageChange}
      />
    </main>
  )
}

App                        Pagination

 │                              │
 │                              │  click → onPageChange(2)
 │                              │
 │←─────────────────────────────┤
 │  setCurrentPage(2)           │
