JavaScript Modules — Import and Export

😱 The Problem: Everything in One File

As your application grows, having everything in a single file becomes unmanageable: 

// App.jsx — 500 lines of code 😱

function Header() { return <header>...</header> }
function Footer() { return <footer>...</footer> }
function JobCard({ job }) { return <article>...</article> }
function JobList({ jobs }) { return <div>...</div> }
function SearchForm() { return <form>...</form> }

function App() {
  return (
    <div>
      <Header />
      <SearchForm />
      <JobList />
      <Footer />
    </div>
  )
}

export default App
📜 Hard to read      — 500+ lines in one file
🔍 Hard to find      — Where is the component I need?
🐛 Hard to debug     — Everything mixed together
👥 Hard to collaborate — Git conflicts
♻️ Not reusable      — Can't use these components in other files

✅ The Solution: Modules

We split each component into its own file and use import / export:

src/
├── components/
│   ├── Header.jsx
│   ├── Footer.jsx
│   ├── JobCard.jsx
│   ├── JobList.jsx
│   └── SearchForm.jsx
└── App.jsx

Step 1 — Create the component file

// src/components/Header.jsx

function Header() {
  return (
    <header className="header">
      <h1>DevJobs</h1>
      <nav>
        <a href="#jobs">Jobs</a>
        <a href="#about">About</a>
      </nav>
    </header>
  )
}

export default Header

Step 2 — Import it in App.jsx

// src/App.jsx
import Header from './components/Header'

function App() {
  return (
    <div className="app">
      <Header />
    </div>
  )
}

export default App

📤 Export Default vs Export Named

Export Default — one main export per file

// Header.jsx
function Header() {
  return <header>...</header>
}

export default Header
// When importing, you can use ANY name
import Header from './components/Header'
import MyHeader from './components/Header'  // ✅ also works
import TopBar from './components/Header'    // ✅ also works

⚠️ Problem: If each importer picks a different name for the same component, the codebase becomes inconsistent and hard to follow.

Export Named — multiple exports with specific names

// Header.jsx
export function Header() {
  return <header>...</header>
}

export function HeaderLogo() {
  return <img src="logo.png" alt="Logo" />
}

export function HeaderNav() {
  return <nav>...</nav>
}
// When importing, you MUST use the exact name with curly braces {}
import { Header, HeaderLogo, HeaderNav } from './components/Header'

🏆 Why We Prefer Named Exports

Consistency across the app

// ❌ Default export — every file can choose a different name
import Header from './components/Header'    // App.jsx
import MyHeader from './components/Header'  // OtherPage.jsx
import TopBar from './components/Header'    // AnotherPage.jsx
// 😱 Three different names for the same component

// ✅ Named export — always the same name
import { Header } from './components/Header'  // App.jsx
import { Header } from './components/Header'  // OtherPage.jsx
import { Header } from './components/Header'  // AnotherPage.jsx
// 🎉 One name everywhere

Safe refactoring

With named exports, if you rename a component, your IDE can update all references automatically. With default exports, each file may have a different name and the IDE doesn't know which ones to update.

Autocomplete

// Named: your IDE suggests the exact name as you type
import { Header } from './components/Header'
//       ^ Autocomplete of the correct name ✅

// Default: you have to remember or guess the name
import Header from './components/Header'
//     ^ No IDE guidance ⚠️

✏️ Two Ways to Write Named Exports

// Form 1: inline export (recommended)
export function Header() {
  return <header>...</header>
}

export function HeaderLogo() {
  return <img src="logo.png" />
}
// Form 2: export at the end of the file
function Header() {
  return <header>...</header>
}

function HeaderLogo() {
  return <img src="logo.png" />
}

export { Header, HeaderLogo }

Both are equivalent. Form 1 is more common in modern React.

🔄 Renaming Imports with as

Sometimes two different files export something with the same name. Use as to resolve the collision:

// ❌ Error — both are called "Button"
import { Button } from './components/Button'
import { Button } from './ui/Button'
// ✅ Rename with as
import { Button } from './components/Button'
import { Button as UIButton } from './ui/Button'

function App() {
  return (
    <>
      <Button>Component Button</Button>
      <UIButton>UI Button</UIButton>
    </>
  )
}
// Real example with two "Card" components
import { Card as JobCard } from './components/JobCard'
import { Card as UserCard } from './components/UserCard'

function Dashboard() {
  return (
    <>
      <JobCard job={job} />
      <UserCard user={user} />
    </>
  )
}

📂 Import Paths

// Same directory
import { Header } from './Header'

// Subdirectory
import { Header } from './components/Header'

// Parent directory
import { Header } from '../Header'

// Two levels up
import { Header } from '../../Header'

📥 Importing Multiple Items

// Multiple named exports at once
import { Header, Footer, Nav } from './components'

// Default + named together
import App, { config, API_URL } from './App'

// Everything as a namespace (use sparingly)
import * as Utils from './utils'
Utils.formatDate()
Utils.formatCurrency()

🛠️ Full Example: Splitting the App

// src/components/JobCard.jsx
export function JobCard({ job, isRemote, isFeatured }) {
  const { title, company, location } = job

  return (
    <article className={`job-card ${isFeatured ? 'featured' : ''}`}>
      <h3>{title}</h3>
      <p>{company}</p>
      <p>📍 {location}</p>
      {isRemote && <span className="badge">🏠 Remote</span>}
    </article>
  )
}
// src/App.jsx
import { JobCard } from './components/JobCard'

function App() {
  return (
    <main>
      <JobCard
        job={{ title: 'Frontend Dev', company: 'TechCorp', location: 'Madrid' }}
        isRemote={true}
        isFeatured={true}
      />
    </main>
  )
}

export default App

🐛 Common Errors — and How to Avoid Them

1. Forgetting the export

// ❌ The component exists but can't be imported
function Header() {
  return <header>...</header>
}

// ✅ Correct
export function Header() {
  return <header>...</header>
}

2. Mixing up default and named

// Header.jsx — NAMED export
export function Header() {
  return <header>...</header>
}

// ❌ Error — Header is not a default export
import Header from './components/Header'

// ✅ Correct — named export requires curly braces
import { Header } from './components/Header'

3. Missing ./ in the path

// ❌ Error — bundler looks in node_modules
import { Header } from 'components/Header'

// ✅ Correct — relative paths start with ./
import { Header } from './components/Header'

4. Extension in Vite

// Vite resolves the extension automatically
import { Header } from './components/Header'      // ✅
import { Header } from './components/Header.jsx'  // ✅ also works

💪 Exercise: Split the App into Modules

Take this monolithic code and split it into individual files:

// App.jsx — all together ❌

function Badge({ children, type }) {
  return <span className={`badge badge-${type}`}>{children}</span>
}

function JobCard({ job, isRemote }) {
  return (
    <article>
      <h3>{job.title}</h3>
      <p>{job.company}</p>
      {isRemote && <Badge type="info">🏠 Remote</Badge>}
    </article>
  )
}

function App() {
  return (
    <main>
      <JobCard
        job={{ title: 'Frontend Dev', company: 'TechCorp' }}
        isRemote={true}
      />
    </main>
  )
}

export default App
Target structure:

src/
├── components/
│   ├── Badge.jsx      → export function Badge
│   └── JobCard.jsx    → export function JobCard
└── App.jsx            → import { Badge } from './components/Badge'
                          import { JobCard } from './components/JobCard'

✅ Class Summary

📦 Modules        — split code into multiple files
📤 export default  — one main export per file
📤 export named   — multiple exports with exact names
📥 import         — bring code from other files
🔄 as             — rename imports to avoid collisions
🏆 Named exports  — preferred: consistent and with autocomplete
🗂️ Structure      — one component per file, inside /components
