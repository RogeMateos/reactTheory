Integrating Navigation with a Search Form

🎯 The goal

Connect a search form with an SPA router. When the user submits the form, the app should navigate to /search with the entered text as a query param — without reloading the page.

😤 The initial problem

We have a form with an input and a search button on the Home page, but it does nothing. We need to:

  • Capture the submit event
  • Extract the input value
  • Build the URL with query params
  • Navigate using our router, without reloading
// Current state (non-functional)
function Home() {
  return (
    <form>
      <input type="text" placeholder="Search..." />
      <button disabled>🔍</button>
    </form>
  )
}

⚠️ SVG attributes in JSX — camelCase

In HTML, SVG attributes use kebab-case, but in JSX they must be camelCase: 

// ❌ Wrong (HTML)
<path stroke-width="2" stroke-linecap="round" />

// ✅ Correct (JSX)
<path strokeWidth="2" strokeLinecap="round" />

Changes to apply:

  • stroke-width → strokeWidth
  • stroke-linecap → strokeLinecap
  • stroke-linejoin → strokeLinejoin
  • Remove the disabled attribute from the button

🛑 Why event.preventDefault()?

In a traditional web app, when you submit a form, the browser:

  • Reloads the full page
  • Loses all JavaScript state
  • Makes a request to the server

In an SPA we want the opposite:

  • ✅ Keep the page without reloading
  • ✅ Preserve state
  • ✅ Handle navigation with JavaScript
const handleSearch = (event) => {
  event.preventDefault() // ← Prevents the reload
  // Now we control navigation ourselves
}

📦 FormData — extract values without controlled state

The FormData API lets you read form values directly, without needing useState for each field. 

const handleSearch = (event) => {
  event.preventDefault()

  const formData = new FormData(event.target)
  const searchTerm = formData.get('search')

  console.log('Searching:', searchTerm)
}

Important: the input needs a name attribute so FormData can identify it: <input name="search" />

⚖️ FormData vs useState — when to use each?

Use FormData Use useState
You only need the value on submit You need real-time validation
No validation while typing You want to show the value in other components
Multiple fields and less code You need to disable the button based on input

🔗 encodeURIComponent — safe characters in the URL

If the user types "React de Dev", the URL would be broken: 

/search?text=React de Dev  ← invalid! spaces are not allowed

You need to encode it:

const encodedTerm = encodeURIComponent(searchTerm)
targetUrl += `?text=${encodedTerm}`

Examples:

  • "React de Dev" → /search?text=React%20de%20Dev ✅
  • "C++" → /search?text=C%2B%2B ✅
  • "¿Qué es React?" → /search?text=%C2%BFQu%C3%A9%20es%20React%3F ✅
Character Encoded Reason
space%20Not valid in URL
+%2BSpecial meaning
?%3FDelimits query params
&%26Separates query params
=%3DAssigns values
#%23URL fragment

🧩 Reusing navigateTo from the useRouter custom hook

If we used window.history.pushState directly:

// ❌ Problem: only changes the URL, doesn't re-render the view
window.history.pushState({}, '', targetUrl)

Problems:

  • ❌ URL changes, but the view doesn't update
  • ❌ Router listeners don't fire
  • ❌ Component state doesn't change
  • ❌ Duplicated router logic

The solution: use the useRouter custom hook that already has navigateTo. The hook updates the URL and the state → automatic re-render.

Advantage: separation of concerns. The component only cares about UI; the router logic lives in one place.

Step 1 — Initial submit handler

function Home() {
  const handleSearch = (event) => {
    event.preventDefault()
    // Logic here
  }

  return (
    <form onSubmit={handleSearch}>
      <input type="text" name="search" placeholder="Search courses..." />
      <button type="submit">🔍</button>
    </form>
  )
}

Step 2 — Extract the value with FormData

const handleSearch = (event) => {

Step 3 — Build the URL with encodeURIComponent

const handleSearch = (event) => {
  event.preventDefault()

  const formData = new FormData(event.target)
  const searchTerm = formData.get('search')

  let targetUrl = '/search'

  if (searchTerm) {
    const encodedTerm = encodeURIComponent(searchTerm)
    targetUrl += `?text=${encodedTerm}`
  }

  console.log('Navigate to:', targetUrl)
}

Step 4 — The useRouter custom hook

// hooks/useRouter.js
import { useState, useEffect } from 'react'

export function useRouter() {
  const [currentPath, setCurrentPath] = useState(window.location.pathname)

  useEffect(() => {
    const handleLocationChange = () => {
      setCurrentPath(window.location.pathname)
    }

    window.addEventListener('popstate', handleLocationChange)
    return () => window.removeEventListener('popstate', handleLocationChange)
  }, [])

  const navigateTo = (path) => {
    window.history.pushState({}, '', path)
    setCurrentPath(path)
  }

  return { currentPath, navigateTo }
}

Step 5 — Full integration in Home

import { useRouter } from './hooks/useRouter'

function Home() {
  const { navigateTo } = useRouter()

  const handleSearch = (event) => {
    event.preventDefault()

    const formData = new FormData(event.target)
    const searchTerm = formData.get('search')

    let targetUrl = '/search'

    if (searchTerm) {
      const encodedTerm = encodeURIComponent(searchTerm)
      targetUrl += `?text=${encodedTerm}`
    }

    navigateTo(targetUrl)
  }

  return (
    <div className="home">
      <h1>Find the best course for you</h1>

      <form onSubmit={handleSearch}>
        <input
          type="text"
          name="search"
          placeholder="Search courses..."
          className="search-input"
        />
        <button type="submit" className="search-button">
          <svg>...</svg>
        </button>
      </form>
    </div>
  )
}

export default Home
