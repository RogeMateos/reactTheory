API Pagination: limit and offset

🔎 The initial problem

Filters are already working with the API, but pagination isn't integrated. The state exists on the frontend but the request never uses it: 

// Current state:
✅ Filters are correctly sent to the API
✅ Pagination exists in the frontend
❌ Pagination does NOT work with the API — page changes don't trigger a new request

📐 How API pagination works

Most APIs don't use a page parameter. Instead they use two fundamental parameters: limit and offset.

Parameter Description Example
limit How many results to return per page 10
offset How many results to skip from the start 0, 10, 20… 
// Page 1 → ?limit=10&offset=0   → results 0–9
// Page 2 → ?limit=10&offset=10  → results 10–19
// Page 3 → ?limit=10&offset=20  → results 20–29

🧮 The offset formula

To calculate the offset from a page number:

offset = (currentPage - 1) * RESULTS_PER_PAGE
Current page Results per page Offset Calculation
1 10 0 (1-1) × 10 = 0
2 10 10 (2-1) × 10 = 10
3 10 20 (3-1) × 10 = 20
5 20 80 (5-1) × 20 = 80

🐛 Critical bug: wrong total pages

There's a very common mistake when calculating total pages:

// ❌ jobs.length is only the current page's results (always 10 or less)
const totalPages = Math.ceil(jobs.length / RESULTS_PER_PAGE)
// → Math.ceil(10 / 10) = 1 ... always 1 page

// ✅ Correct: use the real total from the API
const totalPages = Math.ceil(total / RESULTS_PER_PAGE)
// → Math.ceil(143 / 10) = 15 pages

Rule: jobs.length is the number of results on the current page. The total the API returns is the actual record count in the database. They are completely different things.

🔄 Reset pagination when filters change

If the user is on page 5 of "JavaScript" and switches to "Python" (which only has 2 pages), staying on page 5 makes no sense. Go back to page 1 whenever any filter changes: 

// ✅ Second useEffect whose only job is to reset the page
useEffect(() => {
  setCurrentPage(1)
}, [textToFilter, filters.technology, filters.location, filters.experienceLevel])

🏗️ useEffect with pagination integrated

We add limit, offset, and currentPage as a dependency: 

const RESULTS_PER_PAGE = 10

useEffect(() => {
  async function fetchJobs() {
    try {
      setLoading(true)

      const params = new URLSearchParams()

      // Filters
      if (textToFilter)              params.append('text',       textToFilter)
      if (filters.technology)        params.append('technology', filters.technology)
      if (filters.location)          params.append('type',       filters.location)
      if (filters.experienceLevel)   params.append('level',      filters.experienceLevel)

      // Pagination
      const offset = (currentPage - 1) * RESULTS_PER_PAGE
      params.append('limit',  RESULTS_PER_PAGE)
      params.append('offset', offset)

      const res = await fetch(
        `https://jscamp-api.vercel.app/api/jobs?${params.toString()}`
      )
      const json = await res.json()

      setJobs(json.data)
      setTotal(json.total)
    } catch (error) {
      console.error('Error fetching jobs:', error)
    } finally {
      setLoading(false)
    }
  }

  fetchJobs()
}, [textToFilter, filters.technology, filters.location, filters.experienceLevel, currentPage])
//                                                                                 ↑ key

🔄 Full component with two useEffects

function JobSearch() {
  const [jobs, setJobs]               = useState([])
  const [loading, setLoading]         = useState(true)
  const [total, setTotal]             = useState(0)
  const [currentPage, setCurrentPage] = useState(1)
  const [textToFilter, setTextToFilter] = useState('')
  const [filters, setFilters] = useState({
    technology: '',
    location: '',
    experienceLevel: '',
  })

  // Total pages — uses API total, NOT jobs.length
  const totalPages = Math.ceil(total / RESULTS_PER_PAGE)

  // useEffect 1: reset page when a filter changes
  useEffect(() => {
    setCurrentPage(1)
  }, [textToFilter, filters.technology, filters.location, filters.experienceLevel])

  // useEffect 2: fetch whenever page or filters change
  useEffect(() => {
    async function fetchJobs() {
      try {
        setLoading(true)

        const params = new URLSearchParams()
        if (textToFilter)              params.append('text',       textToFilter)
        if (filters.technology)        params.append('technology', filters.technology)
        if (filters.location)          params.append('type',       filters.location)
        if (filters.experienceLevel)   params.append('level',      filters.experienceLevel)

        const offset = (currentPage - 1) * RESULTS_PER_PAGE
        params.append('limit',  RESULTS_PER_PAGE)
        params.append('offset', offset)

        const res = await fetch(
          `https://jscamp-api.vercel.app/api/jobs?${params.toString()}`
        )
        const json = await res.json()

        setJobs(json.data)
        setTotal(json.total)
      } catch (error) {
        console.error('Error fetching jobs:', error)
      } finally {
        setLoading(false)
      }
    }

    fetchJobs()
  }, [textToFilter, filters.technology, filters.location, filters.experienceLevel, currentPage])

  const nextPage = () => { if (currentPage < totalPages) setCurrentPage(p => p + 1) }
  const prevPage = () => { if (currentPage > 1) setCurrentPage(p => p - 1) }

  return (
    <div>
      <SearchForm ... />
      {loading ? (
        <div>Loading jobs...</div>
      ) : (
        <>
          <JobListing jobs={jobs} />
          <Pagination
            currentPage={currentPage}
            totalPages={totalPages}
            onNext={nextPage}
            onPrev={prevPage}
          />
        </>
      )}
    </div>
  )
}

📋 Full flow step by step

1. User types "JavaScript"
   └─ textToFilter changes
   └─ useEffect 1 runs: setCurrentPage(1)
   └─ currentPage changes to 1
   └─ useEffect 2 runs: fetch page 1 + text=javascript

2. User navigates to page 3
   └─ setCurrentPage(3)
   └─ useEffect 2 runs: fetch page 3 + text=javascript
   └─ URL → ?text=javascript&limit=10&offset=20

3. User changes filter to "Python"
   └─ filters.technology changes
   └─ useEffect 1 runs: setCurrentPage(1)   ← reset
   └─ useEffect 2 runs: fetch page 1 + technology=python

🧩 Reusable Pagination component

function Pagination({ currentPage, totalPages, onNext, onPrev, onGoToPage }) {
  return (
    <div className="pagination">
      <button onClick={onPrev} disabled={currentPage === 1}>
        Previous
      </button>

      <span>Page {currentPage} of {totalPages}</span>

      <button onClick={onNext} disabled={currentPage === totalPages}>
        Next
      </button>

      {/* Optional: page numbers */}
      {Array.from({ length: totalPages }, (_, i) => i + 1).map((page) => (
        <button
          key={page}
          onClick={() => onGoToPage(page)}
          className={page === currentPage ? 'active' : ''}
        >
          {page}
        </button>
      ))}
    </div>
  )
}

🔍 Verifying in DevTools → Network

When changing pages you should see requests like these:

Page 1, no filters:  /api/jobs?limit=10&offset=0
Page 2, no filters:  /api/jobs?limit=10&offset=10
Page 3, no filters:  /api/jobs?limit=10&offset=20

With active filters:
/api/jobs?text=javascript&technology=react&limit=10&offset=20

And the API response should include:

{
  "total": 143,   // ← what we use to calculate totalPages
  "limit": 10,
  "offset": 20,
  "results": 10,
  "data": [ /* jobs for this page */ ]
}

✅ Best practices and ❌ what to avoid

✅ Always do:

  • Use total from the API to calculate totalPages, never jobs.length
  • Include currentPage in the fetch useEffect dependencies
  • Have a separate useEffect that resets the page when filters change
  • Disable Previous/Next buttons when appropriate (disabled)
  • Verify in DevTools that the offset changes correctly between pages

❌ Never do:

  • Use jobs.length to calculate total pages
  • Pass the full filters object as a dependency
  • Omit currentPage from the dependency array
  • Forget to reset the page when a filter changes
