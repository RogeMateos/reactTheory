Integrating Filters with the API

🔎 The initial problem

The app already makes an HTTP call to the API and the filters update in state correctly. But there's a critical problem: the request doesn't send those filters.

Filters are passed to the form and update when the user interacts with them, but that information never reaches the request URL. The API always receives the same query with no filters. 

// ❌ The request completely ignores the user's filters
useEffect(() => {
  fetch('https://jscamp-api.vercel.app/api/jobs')
    .then((res) => res.json())
    .then((data) => setJobs(data.data))
}, []) // ← never re-runs, never sends filters

🎯 Goal

When the user types "JavaScript" or changes any filter (technology, type, experience level), the app should re-query the API sending those filters as query params.

🔧 How to send filters to the API

The API accepts filters in the URL via query params: 

GET https://jscamp-api.vercel.app/api/jobs?text=react&limit=10&offset=0
Parameter Description
text Free text search
technology Filter by technology (javascript, python…)
type Filter by modality (remote, onsite, hybrid)
level Experience level (junior, mid, senior)

🛠️ URLSearchParams — the web platform standard

To build query params dynamically, we use URLSearchParams, a native browser API. No packages needed. 

const params = new URLSearchParams()

// Only appended if they have a value
if (textToFilter)            params.append('text',       textToFilter)
if (filters.technology)      params.append('technology', filters.technology)
if (filters.location)        params.append('type',       filters.location)
if (filters.experienceLevel) params.append('level',      filters.experienceLevel)

const url = `https://jscamp-api.vercel.app/api/jobs?${params.toString()}`

Key advantage: if no filters are active, params.toString() returns an empty string and the URL ends with a bare ?, which works perfectly.

🐛 First attempt — why it still doesn't work

You build the URL with filters but selecting "JavaScript" does nothing. Why? Because useEffect has an empty dependency array: 

useEffect(() => {
  // Even if you build the URL with filters here...
  fetch(url)
    .then(...)
}, []) // ❌ Only runs ONCE on mount — never again

The linter will also warn you: "React Hook useEffect has missing dependencies". Don't ignore it — that warning is telling you exactly what's wrong.

✅ The solution: correct dependencies

The effect needs to re-run every time a filter changes. For that, list each filter as a dependency: 

useEffect(() => {
  // ...fetch logic
}, [textToFilter, filters.technology, filters.location, filters.experienceLevel])
//  ↑ Now the effect re-runs when any of these change

🏗️ Full implementation with filters

The useEffect builds the URL dynamically with active filters and re-runs every time any of them changes: 

useEffect(() => {
  const params = new URLSearchParams()

  if (textToFilter) {
    params.append('text', textToFilter)
  }
  if (filters.technology) {
    params.append('technology', filters.technology)
  }
  if (filters.location) {
    params.append('type', filters.location)
  }
  if (filters.experienceLevel) {
    params.append('level', filters.experienceLevel)
  }

  const queryParams = params.toString()

  fetch(`https://jscamp-api.vercel.app/api/jobs?${queryParams}`)
    .then((res) => res.json())
    .then((data) => setJobs(data.data))
}, [textToFilter, filters.technology, filters.location, filters.experienceLevel])

🔄 What happens when the user changes a filter?

// Flow with active filters:

1. User selects technology: "JavaScript"
   └─ filters.technology changes → "javascript"

2. React detects the dependency change
   └─ useEffect re-runs

3. URLSearchParams builds the URL:
   └─ ?technology=javascript

4. fetch() makes the new request
   └─ https://jscamp-api.vercel.app/api/jobs?technology=javascript

5. API returns filtered jobs
   └─ setJobs(data.data) → re-render with new results

// With multiple active filters:
// ?text=javascript&type=remote&level=junior

🚫 Handling the no-results case

When the filter combination finds nothing, you need to give feedback to the user. Without this, the screen stays blank with no explanation: 

// In the component JSX:
{jobs.length === 0 ? (
  <p
    style={{
      padding: '2rem',
      textWrap: 'balance',
    }}
  >
    No jobs found matching the search criteria.
  </p>
) : (
  <JobsList jobs={jobs} />
)}

📋 Verification checklist

// Tests that should work:

✅ Text: "Python"       → API receives ?text=python
✅ Text: "JavaScript"   → API receives ?text=javascript
✅ Technology: React    → API receives ?technology=react
✅ Type: Remote         → API receives ?type=remote

// Combinations:
✅ JavaScript + Remote  → ?technology=javascript&type=remote
✅ JS + Remote + Junior → ?technology=javascript&type=remote&level=junior

📦 Full component with everything integrated

function JobSearch({ filters, textToFilter }) {
  const [jobs, setJobs] = useState([])
  const [loading, setLoading] = useState(true)
  const [total, setTotal] = useState(0)

  useEffect(() => {
    async function fetchJobs() {
      try {
        setLoading(true)

        const params = new URLSearchParams()
        if (textToFilter)              params.append('text',       textToFilter)
        if (filters.technology)        params.append('technology', filters.technology)
        if (filters.location)          params.append('type',       filters.location)
        if (filters.experienceLevel)   params.append('level',      filters.experienceLevel)

        const res = await fetch(
          `https://jscamp-api.vercel.app/api/jobs?${params.toString()}`
        )
        const json = await res.json()

        setJobs(json.data)
        setTotal(json.total)
      } catch (error) {
        console.error('Error loading jobs:', error)
      } finally {
        setLoading(false)
      }
    }

    fetchJobs()
  }, [textToFilter, filters.technology, filters.location, filters.experienceLevel])

  if (loading) return <div>Loading jobs...</div>

  return (
    <>
      {jobs.length === 0 ? (
        <p style={{ padding: '2rem', textWrap: 'balance' }}>
          No jobs found matching the search criteria.
        </p>
      ) : (
        <JobsList jobs={jobs} />
      )}
    </>
  )
}

🐛 Classic bug: filter selected but nothing changes

The symptom: user selects a filter, the list doesn't update. The cause is almost always an empty or incomplete dependency array. 

// ❌ Cause
useEffect(() => {
  fetch(`...?technology=${filters.technology}`)  // reads the filter
    .then(...)
}, []) // but never re-runs when it changes

// ✅ Fix
useEffect(() => {
  fetch(`...?technology=${filters.technology}`)
    .then(...)
}, [filters.technology]) // now it re-runs on change

🐛 Bug: linter warning about missing dependencies

If you see "React Hook useEffect has missing dependencies: 'filters.technology'", don't silence it with // eslint-disable-next-line. The linter is right — that variable should be in the dependencies.

Rule: If a variable is used inside useEffect, it must be in the dependency array. No exceptions.

🐛 Bug: infinite loop when adding an object as dependency

If you put the whole filters object as a dependency instead of its properties, you can trigger an infinite loop: 

// ❌ Dangerous — if filters is recreated on each render, infinite loop
useEffect(() => {
  // ...
}, [filters]) // ← full object

// ✅ Safe — primitive values
useEffect(() => {
  // ...
}, [filters.technology, filters.location, filters.experienceLevel])

Primitives (string, number, boolean) are compared by value. Objects are compared by reference — if the object is recreated on every render, React always sees a "new dependency" and the effect runs infinitely.

✅ Best practices and ❌ what to avoid

✅ Always do:

  • List all variables used inside the effect as dependencies
  • Use URLSearchParams to build query strings — not manual string concatenation
  • Add dependencies granularly (primitives, not full objects)
  • Show feedback when there are no results
  • Respect linter warnings about missing dependencies

❌ Never do:

  • Silence linter warnings with // eslint-disable
  • Pass the full filters object as a dependency if it's recreated on every render
  • Build URLs with manual string concatenation ('?text=' + text + '&type=' + type)
  • Leave the dependency array empty if the effect uses state or props values
