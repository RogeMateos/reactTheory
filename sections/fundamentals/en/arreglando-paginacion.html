Fixing Pagination: URL as State

🧠 What problem does this solve?

Before, the /search route always looked the same no matter what the user did: 

page 1
page 2
search React
search Vue

👉 The URL NEVER changed.

So:

Situation Result
Reload the page ❌ Lose all state
Share the URL ❌ Doesn't work
Browser back button ❌ Weird behavior

✅ What we want to achieve

The URL should represent the state of the app. For example: 

/search?page=3&text=react

That URL says it all:

Parameter Meaning
page=3 You are on page 3
text=react You are searching for "React"

The URL now "tells the story" of the application.

🚀 The REAL benefit

With this change, the app:

✅ Remembers state
✅ Can be reloaded without losing anything
✅ Can be shared with anyone
✅ Works properly with browser history
✅ Has professional URLs

📌 The MOST important idea

The URL becomes THE SOURCE OF TRUTH. Not just React state. The URL now also stores the application state.

🔥 What does URLSearchParams do?

It helps you read and modify URL parameters easily, without breaking the existing ones. 

const searchParams = new URLSearchParams(window.location.search)

// If the URL is: /search?text=react
searchParams.set('page', '2')
// Now it is:    /search?text=react&page=2

👉 It keeps what was there and only adds or changes what's needed.

🎯 What does buildPageURL achieve?

Building pagination links correctly so each button has a real URL: 

// BEFORE — all buttons pointed to "#"
<a href="#">1</a>
<a href="#">2</a>
<a href="#">3</a>

// AFTER — each button has its correct URL
/search?page=1&text=react
/search?page=2&text=react
/search?page=3&text=react

💻 Implementation

function buildPageURL(search, page) {
  // Grab the current URL parameters
  const searchParams = new URLSearchParams(search)

  // Only modify the "page" parameter
  searchParams.set('page', page)

  // Return the full query string
  return `?${searchParams.toString()}`
}

// Usage in the pagination component:
function Pagination({ search, currentPage, totalPages }) {
  return (
    <nav>
      {Array.from({ length: totalPages }, (_, i) => i + 1).map(page => (
        <a
          key={page}
          href={buildPageURL(search, page)}
          className={page === currentPage ? 'active' : ''}
        >
          {page}
        </a>
      ))}
    </nav>
  )
}

⚠️ Common mistakes

Mistake Problem Solution
Using href="#" URL never changes Use buildPageURL()
Building URL manually Overwrites other params Use URLSearchParams
useState only Doesn't survive reload Read from useSearchParams
Not resetting page on search Page 5 on a brand new search When text changes, reset page=1

🔧 Block 1: simplifying buildPageURL

Problem detected: the code works, but using window.location.href inside the component makes it less testable and more coupled to the browser.

// ❌ Over-coupled version — takes more than needed
const url = new URL(window.location.href)

// ✅ Improved version — only works with what it needs
const buildPageURL = (pageNumber) => {
  const searchParams = new URLSearchParams(window.location.search)
  searchParams.set('page', String(pageNumber))
  return `${window.location.pathname}?${searchParams.toString()}`
}

The improvement is using window.location.search and window.location.pathname instead of building a full URL object. You only work with what you need.

🔧 Block 2: previous and next pages

Problem detected: without guards, you can generate invalid URLs:

// ❌ No limits — generates broken URLs
previousPage = 0            → /search?page=0
nextPage = totalPages + 1   → /search?page=11

// ✅ With Math.max / Math.min — always valid
const previousPage = Math.max(currentPage - 1, 1)
const nextPage = Math.min(currentPage + 1, totalPages)

Even if you block the click with preventDefault, the href was still pointing to an incorrect URL. With this, it's always valid.

🔧 Block 3: readability with isFirstPage / isLastPage

Problem detected: aria-disabled only helps accessibility but doesn't disable the link. Also, repeating currentPage === 1 in multiple places clutters the code.

// ✅ Descriptive variables for better readability
const isFirstPage = currentPage === 1
const isLastPage = currentPage === totalPages

<a
  href={buildPageURL(previousPage)}
  aria-disabled={isFirstPage}
  onClick={(event) => {
    if (isFirstPage) event.preventDefault()
  }}
>
  Previous
</a>

✅ Complete final improved version

function Pagination({ currentPage, totalPages }) {
  const buildPageURL = (pageNumber) => {
    const searchParams = new URLSearchParams(window.location.search)
    searchParams.set('page', String(pageNumber))
    return `${window.location.pathname}?${searchParams.toString()}`
  }

  const isFirstPage = currentPage === 1
  const isLastPage  = currentPage === totalPages

  const previousPage = Math.max(currentPage - 1, 1)
  const nextPage     = Math.min(currentPage + 1, totalPages)

  return (
    <nav aria-label="Pagination">
      <ul>
        <li>
          <a
            href={buildPageURL(previousPage)}
            aria-disabled={isFirstPage}
            onClick={(event) => { if (isFirstPage) event.preventDefault() }}
          >
            Previous
          </a>
        </li>

        {Array.from({ length: totalPages }, (_, index) => {
          const page = index + 1
          return (
            <li key={page}>
              <a
                href={buildPageURL(page)}
                aria-current={page === currentPage ? 'page' : undefined}
              >
                {page}
              </a>
            </li>
          )
        })}

        <li>
          <a
            href={buildPageURL(nextPage)}
            aria-disabled={isLastPage}
            onClick={(event) => { if (isLastPage) event.preventDefault() }}
          >
            Next
          </a>
        </li>
      </ul>
    </nav>
  )
}

📌 Summary of improvements

Problem Solution
new URL(window.location.href) Use location.search + location.pathname
Invalid pages (0 or totalPages+1) Math.max / Math.min as guards
Condition repeated in multiple places isFirstPage / isLastPage as variables
aria-disabled doesn't block the click onClick with preventDefault
