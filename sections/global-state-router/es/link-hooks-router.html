Link y hooks de React Router: useNavigate y useLocation

🔗 Modernizando nuestro componente Link

Vamos a modernizar completamente el routing de la app. El objetivo: reemplazar implementaciones manuales por herramientas nativas de React Router, manteniendo la misma API externa que ya usan nuestros componentes. 

✅ Mantenemos nuestro propio componente Link, pero cambiamos su implementación interna
✅ Integramos useNavigate y useLocation
✅ Aplicamos el patrón de abstracción para encapsular React Router
✅ No hace falta cambiar el resto de la app

El Link que teníamos antes

Nuestro Link anterior hacía todo manualmente:

export function Link({ to, children, ...props }) {
  const handleClick = (e) => {
    e.preventDefault()
    window.history.pushState({}, '', to)
    // Disparar un evento custom para avisar al sistema de routing...
  }

  return (
    <a href={to} onClick={handleClick} {...props}>
      {children}
    </a>
  )
}

Todo esto era manual. Y ya no lo necesitamos, porque React Router ya trae un componente Link.

Qué hacemos ahora

Mantenemos nuestro Link, pero cambiamos su interior:

import { Link as RRLink } from 'react-router'

export function Link(props) {
  return <RRLink {...props} />
}

Así conseguimos esto:

  • Toda la app sigue usando nuestro <Link>
  • Si mañana queremos cambiar de librería de routing, tocamos un solo archivo
  • No hay que recorrer componente por componente haciendo cambios

🏗️ Patrón de abstracción

Esta técnica se conoce como patrón de abstracción o patrón de abstracción de dependencias.

Ventaja Qué significa
Sin acoplamiento directo a React Router Tus componentes no necesitan saber que React Router existe
Un solo lugar para cambiar Un archivo en vez de decenas si cambiás de librería
API pública estable Quien consume tu componente no tiene que cambiar nada 
El principio: exponé tu propia API estable,
ocultá los detalles de implementación de la librería,
y centralizá los imports de esa librería en un solo lugar.

🧭 Reemplazando navigateTo con useNavigate

Teníamos un router custom con un método navigateTo que navegaba manualmente. React Router ofrece su propio hook: useNavigate.

El problema de nuestra implementación manual

function useRouter() {
  const navigateTo = (path) => {
    window.history.pushState({}, '', path)
    // Disparar eventos custom
    // Actualizar estado
    // ... mucho código manual
  }

  return { navigateTo }
}

Esto funciona, pero:

  • Lo tenemos que mantener nosotros
  • Puede tener bugs
  • Estamos reinventando la rueda

La solución con useNavigate

import { useNavigate } from 'react-router'

export function useRouter() {
  const navigate = useNavigate()

  const navigateTo = (path) => {
    navigate(path)
  }

  return { navigateTo }
}

Qué ganamos

✅ Ya no implementamos pushState a mano: React Router lo hace mejor
✅ Los componentes siguen llamando a navigateTo: la API externa no cambia
✅ Cambiamos la dependencia sin romper el resto de la app

📍 Reemplazando currentPath manual con useLocation

Antes teníamos un estado currentPath, un useEffect escuchando eventos del historial y lógica propia para detectar cambios en la URL. React Router incluye useLocation, que nos da:

Propiedad Qué contiene
pathname La ruta actual: /search
search La query string: ?q=react
hash El hash: #section
state Estado pasado durante la navegación

El código manual que teníamos antes

function useRouter() {
  const [currentPath, setCurrentPath] = useState(window.location.pathname)

  useEffect(() => {
    const handleLocationChange = () => {
      setCurrentPath(window.location.pathname)
    }

    window.addEventListener('popstate', handleLocationChange)
    window.addEventListener('pushstate', handleLocationChange)

    return () => {
      window.removeEventListener('popstate', handleLocationChange)
      window.removeEventListener('pushstate', handleLocationChange)
    }
  }, [])

  return { currentPath }
}

Todo este useEffect y los listeners de eventos ya no hacen falta.

La solución con useLocation

import { useLocation } from 'react-router'

export function useRouter() {
  const location = useLocation()

  const currentPath = location.pathname

  return {
    currentPath,
    location,
  }
}

Mucho más limpio. Sin efectos secundarios para mantener.

🔧 El hook useRouter completo

Juntando todo:

import { useNavigate, useLocation } from 'react-router'

export function useRouter() {
  const navigate = useNavigate()
  const location = useLocation()

  function navigateTo(path) {
    navigate(path)
  }

  const currentPath = location.pathname

  return {
    currentPath,
    navigateTo,
  }
}
✅ Cero código manual: sin pushState, sin eventos custom, sin useEffect
✅ React Router hace el trabajo pesado, y lo hace mejor que nosotros
✅ Nuestra API queda igual: los componentes que usaban useRouter no cambian

📊 Antes vs Después

// ❌ Antes — Link.jsx (manual)
export function Link({ to, children, ...props }) {
  const handleClick = (e) => {
    e.preventDefault()
    window.history.pushState({}, '', to)
    window.dispatchEvent(new Event('pushstate'))
  }
  return <a href={to} onClick={handleClick} {...props}>{children}</a>
}

// ✅ Después — Link.jsx (React Router)
import { Link as RRLink } from 'react-router'
export function Link(props) {
  return <RRLink {...props} />
}
// ❌ Antes — useRouter.js (manual)
export function useRouter() {
  const [currentPath, setCurrentPath] = useState(window.location.pathname)
  useEffect(() => {
    const handler = () => setCurrentPath(window.location.pathname)
    window.addEventListener('popstate', handler)
    window.addEventListener('pushstate', handler)
    return () => {
      window.removeEventListener('popstate', handler)
      window.removeEventListener('pushstate', handler)
    }
  }, [])
  const navigateTo = (path) => {
    window.history.pushState({}, '', path)
    window.dispatchEvent(new Event('pushstate'))
  }
  return { currentPath, navigateTo }
}

// ✅ Después — useRouter.js (React Router)
import { useNavigate, useLocation } from 'react-router'
export function useRouter() {
  const navigate = useNavigate()
  const location = useLocation()
  function navigateTo(path) { navigate(path) }
  return { currentPath: location.pathname, navigateTo }
}

🚀 Extendiendo el patrón

Una vez que tenés esta capa de abstracción, extenderla es trivial:

export function useRouter() {
  const navigate = useNavigate()
  const location = useLocation()

  function navigateTo(path) {
    navigate(path)
  }

  function goBack() {
    navigate(-1)
  }

  return {
    currentPath: location.pathname,
    navigateTo,
    goBack,
    location,
  }
}

Todos los componentes reciben la nueva funcionalidad automáticamente, sin cambiar ni una línea en ellos:

const { goBack } = useRouter()

<button onClick={goBack}>← Volver</button>
