Navegando al Detalle

Lo que tenemos vs lo que queremos

Ahora mismo cada tarjeta de oferta es solo un elemento de display — no se puede clicar para ir a ningún lado.

Antes:

Listado de ofertas
┌─────────────────────┐
│ Frontend Developer  │   ❌ No se puede clicar
│ Acme Corp           │
│ London              │
└─────────────────────┘
Después:

Listado de ofertas
┌─────────────────────┐
│ Frontend Developer  │   ✅ Click
│ Acme Corp           │      ↓
│ London              │   /jobs/123
└─────────────────────┘

La mejor solución: usar <Link>

Cuando una tarjeta lleva a otra página, debe ser un enlace, no un onClick. Es HTML semántico y hace diferencia en accesibilidad.

import { Link } from 'react-router'
import styles from './JobCard.module.css'

export function JobCard({ job }) {
  return (
    <Link
      to={`/jobs/${job.id}`}
      className={styles.cardLink}
      aria-label={`Ver detalles de ${job.title} en ${job.company}`}
    >
      <article className={styles.card}>
        <h3>{job.title}</h3>
        <p>{job.company}</p>
        <p>{job.location}</p>
      </article>
    </Link>
  )
}

Visualmente, toda la tarjeta se convierte en zona clickeable:

<Link to="/jobs/123">
  ┌─────────────────────┐
  │ Frontend Developer  │
  │ Acme Corp           │
  │ London              │
  └─────────────────────┘
</Link>

<Link> es para navegación normal

Usalo cuando el usuario quiere ir a otra página — es así de simple.

<Link to={`/jobs/${job.id}`}>
  Ver detalles
</Link>
Ventaja Por qué importa
Click normalFunciona como se espera
Click derecho → abrir en nueva pestañaLos usuarios lo esperan de los enlaces
Cmd/Ctrl + clickFeature de power user, gratis con <Link>
Accesible con tecladoTab + Enter funciona
Amigable con screen readersSe anuncia correctamente

useNavigate es para navegación después de una acción

Usalo cuando primero pasa algo y después navegás.

const navigate = useNavigate()

function handleSubmit() {
  saveData()           // primero la acción
  navigate('/success') // después la navegación
}
Usuario envía formulario
        ↓
Guardás datos
        ↓
Navegás a /success

No uses useNavigate para una tarjeta simple que solo es un enlace. Es la herramienta incorrecta y perdés todos los comportamientos nativos del browser listados arriba.

Árbol de decisión

Situación Usá
Tarjeta que va a una página de detalle<Link>
Ítem de menú de navegación<Link>
Botón "Ver más"<Link>
Redirigir después de enviar un formulariouseNavigate()
Redirigir después de loginuseNavigate()
Navegar programáticamente según una condiciónuseNavigate()

⚠️ No metas un botón dentro de un <Link>

Si la tarjeta tiene una acción secundaria (como "guardar en favoritos"), no anides un botón dentro del Link:

// ❌ Problema de HTML y accesibilidad
<Link to={`/jobs/${job.id}`}>
  <article>
    <h3>{job.title}</h3>
    <button>Guardar</button>  ← elemento interactivo anidado
  </article>
</Link>

// ✅ Correcto
<article className={styles.card}>
  <Link to={`/jobs/${job.id}`} className={styles.cardContent}>
    <h3>{job.title}</h3>
    <p>{job.company}</p>
  </Link>

  <button onClick={handleSaveFavorite}>
    Guardar
  </button>
</article>

Versión completa recomendada

import { Link } from 'react-router'
import styles from './JobCard.module.css'

export function JobCard({ job }) {
  return (
    <Link
      to={`/jobs/${job.id}`}
      className={styles.cardLink}
      aria-label={`Ver detalles de ${job.title} en ${job.company}`}
    >
      <article className={styles.card}>
        <header className={styles.header}>
          <h3 className={styles.title}>{job.title}</h3>
          {job.isNew && <span className={styles.badge}>Nuevo</span>}
        </header>

        <div className={styles.meta}>
          <p>🏢 {job.company}</p>
          <p>📍 {job.location}</p>
        </div>

        {job.tags?.length > 0 && (
          <div className={styles.tags}>
            {job.tags.map((tag) => (
              <span key={tag} className={styles.tag}>
                {tag}
              </span>
            ))}
          </div>
        )}

        <footer className={styles.footer}>
          <span>{job.salary || 'Salario a convenir'}</span>
          <span>Ver más →</span>
        </footer>
      </article>
    </Link>
  )
}

Referencia rápida

Pregunta Respuesta
¿Es un enlace a otra página?Usá <Link>
¿Primero hago una acción y luego navego?Usá useNavigate()
¿Quiero que toda la tarjeta sea clickeable?Envolvé con <Link>
¿Quiero un botón de favorito dentro?No lo metas dentro del Link

La solución más profesional

<Link to={`/jobs/${job.id}`}>
  <article>...</article>
</Link>
Usar <Link> te da click derecho, Ctrl+click, navegación por teclado y soporte para screen readers de forma gratuita. Un onClick no te daría nada de eso.
