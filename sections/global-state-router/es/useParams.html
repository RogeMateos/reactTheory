Parámetros de Ruta con useParams

Qué le falta a nuestra app

Tenemos el listado de ofertas funcionando con React Router, pero hay algo clave que falta: no podemos entrar al detalle de una oferta concreta.

✅ Vemos un listado de trabajos
✅ Podemos navegar por algunas rutas
❌ No existe una página de detalle para cada oferta individual

Tenemos el id de cada trabajo, pero no lo estamos usando en la URL para cargar la información específica de esa oferta.

En esta clase vamos a:

  • Crear una página de detalle para una oferta
  • Definir una ruta dinámica que reciba un parámetro (la id)
  • Usar el hook useParams de React Router para leer esa id desde la URL

Qué son los parámetros de ruta

En una URL como /jobs/1234:

  • /jobs es la ruta base
  • 1234 es un parámetro dinámico: la id de la oferta

React Router nos permite definir rutas que no son fijas, sino que incluyen segmentos variables que luego podemos leer:

/jobs/:id
URL visitada Valor de id
/jobs/abc "abc"
/jobs/42 "42"
/jobs/frontend-dev "frontend-dev"

El objetivo es capturar ese valor para poder:

  • Pedir los detalles a la API
  • Mostrar el contenido específico de esa oferta

Por qué esto importa

Imaginá que encontrás una oferta que te interesa. Si la URL es /jobs/1234, podés:

  • Copiarla y enviarla por WhatsApp
  • Guardarla en favoritos
  • Recargar la página sin perder contexto

Si la información estuviera solo en el estado de React (sin estar en la URL), al recargar perderías todo. Esto es lo que significa el deep linking — y es una característica clave de cualquier app web profesional.

Creando la página de detalle

Primero creamos un nuevo componente para el detalle de la oferta.

// src/pages/JobDetail.jsx

export function JobDetail() {
  return <h1>Job detail</h1>
}

De momento solo muestra un texto genérico. Luego lo mejoramos con la id y los datos reales.

Definiendo la ruta dinámica con :id

Ahora creamos una ruta que reciba una id dinámica. En el archivo donde definimos las rutas (App.jsx o router.jsx):

import { Route, Routes } from 'react-router'
import { JobDetail } from './pages/JobDetail'

export function App() {
  return (
    <Routes>
      {/* Otras rutas */}
      <Route path="/jobs/:id" element={<JobDetail />} />
    </Routes>
  )
}
Punto clave Explicación
path="/jobs/:id" :id indica que esa parte de la URL es un parámetro dinámico
element={<JobDetail />} Cuando la URL coincida con /jobs/algo, se renderiza JobDetail

Leyendo los parámetros con useParams

React Router nos da el hook useParams para leer la id desde la URL:

// src/pages/JobDetail.jsx
import { useParams } from 'react-router'

export function JobDetail() {
  const params = useParams()
  console.log(params)

  return (
    <>
      <h1>Job detail</h1>
      <pre>{JSON.stringify(params, null, 2)}</pre>
    </>
  )
}

Si visitás /jobs/jajajata, verás en consola:

{ "id": "jajajata" }

Desestructurando el parámetro

En la práctica siempre desestructuramos directamente:

import { useParams } from 'react-router'

export function JobDetail() {
  const { id } = useParams()

  return (
    <>
      <h1>Job detail</h1>
      <h2>La ID es: {id}</h2>
    </>
  )
}

⚠️ El nombre del parámetro es el que VOS definís en la ruta

Este es un error muy típico. El nombre del parámetro NO es siempre id — depende de lo que pongas en el path de la ruta.

// Ruta definida con :jobId
<Route path="/jobs/:jobId" element={<JobDetail />} />
// El componente DEBE usar jobId, no id
export function JobDetail() {
  const { jobId } = useParams()  // ✅ coincide con la ruta
  // const { id } = useParams()  // ❌ sería undefined
}

El nombre del parámetro tiene que coincidir exactamente entre el path y useParams().

Usando la id para pedir datos reales

El paso natural siguiente es usar la id para pedir datos reales de una API:

import { useEffect, useState } from 'react'
import { useParams } from 'react-router'

export function JobDetail() {
  const { id } = useParams()
  const [job, setJob] = useState(null)

  useEffect(() => {
    if (!id) return

    fetch(`/api/jobs/${id}`)
      .then((res) => res.json())
      .then((data) => setJob(data))
      .catch((error) => console.error('Error cargando el job', error))
  }, [id])

  if (!job) return <p>Cargando oferta...</p>

  return (
    <section>
      <h1>{job.title}</h1>
      <p>{job.description}</p>
      <p>Ubicación: {job.location}</p>
    </section>
  )
}
Concepto clave: La id viene de la URL, no de props. Esto hace que el componente sea independiente — si el usuario recarga la página, la id sigue estando en la URL.

Múltiples parámetros en la URL

React Router también permite tener más de un parámetro en la ruta:

<Route path="/jobs/:id/page/:page" element={<JobDetail />} />
import { useParams } from 'react-router'

export function JobDetail() {
  const { id, page } = useParams()

  return (
    <>
      <h1>Job detail</h1>
      <p>La ID es: {id}</p>
      <p>Página: {page}</p>
    </>
  )
}

Visitando /jobs/1234/page/2 obtenemos:

{ "id": "1234", "page": "2" }

Parámetros de ruta vs Query strings

Usá parámetros de ruta cuando… Usá query strings cuando…
El valor identifica un recurso específico Son filtros opcionales
La página no tiene sentido sin él La página funciona sin ellos
Querés URLs limpias y semánticas Tenés muchos parámetros opcionales
/jobs/1234 /jobs?keyword=react&page=2

Convirtiendo la id a número

useParams() siempre devuelve strings. Si necesitás un número:

export function JobDetail() {
  const { id } = useParams()
  const numericId = Number(id)
  const jobId = parseInt(id, 10)  // alternativa

  console.log(typeof id)        // "string"
  console.log(typeof numericId) // "number"
}

Validando parámetros

Es buena práctica validar que el parámetro tenga el formato esperado:

import { useParams, Navigate } from 'react-router'

export function JobDetail() {
  const { id } = useParams()

  // Si la id no es un número, redirigimos a 404
  if (isNaN(Number(id))) {
    return <Navigate to="/404" replace />
  }

  // Resto del componente...
}

O mostrar un mensaje de error inline:

export function JobDetail() {
  const { id } = useParams()

  if (!id || isNaN(Number(id))) {
    return (
      <div>
        <h1>Error</h1>
        <p>ID de oferta inválida</p>
      </div>
    )
  }

  // Resto del componente...
}

Lo que vimos

Concepto Punto clave
Rutas dinámicas Usá :nombreParam en el path para definir un segmento variable
useParams() Devuelve un objeto con todos los parámetros de la URL actual
Nombres de parámetros Tienen que coincidir exactamente entre el path y useParams()
Los valores siempre son strings Convertí con Number() o parseInt() si necesitás un número
Deep linking Los params en la URL hacen la app compartible y resistente a recargas
Múltiples params /jobs/:id/page/:page → { id, page }

El modelo mental

Ruta definida como:   /jobs/:id
URL visitada:         /jobs/1234
useParams() devuelve: { id: "1234" }

Ruta definida como:   /jobs/:jobId
URL visitada:         /jobs/1234
useParams() devuelve: { jobId: "1234" }
Con esto ya tenés las bases para construir páginas de detalle, perfiles de usuario, posts de blog con slug, y cualquier patrón de navegación basado en recursos con React Router.
