Cómo Crear Rutas con React Router

🚀 Introducción

Vamos a migrar nuestro router casero hacia una solución profesional y robusta. Hasta este punto, nuestro router propio era funcional pero tenía limitaciones importantes: 

❌ No es un estándar de la industria
❌ No incluye todas las funcionalidades que necesitaremos en producción
❌ Está menos optimizado
❌ Reinventamos la rueda en lugar de usar una solución probada

React Router nos ofrece:

✅ Navegación declarativa entre páginas
✅ Gestión automática del historial del navegador
✅ Hooks especializados para parámetros, localización y navegación programática
✅ Manejo robusto de errores y rutas no encontradas
✅ Soporte para múltiples modos de enrutado

📦 Instalación

npm install react-router

Paso 1: envolver la app con <BrowserRouter>

El primer paso es envolver toda la aplicación con BrowserRouter. Este componente proporciona el contexto de enrutado necesario para que React Router funcione en toda la app.

En main.jsx:

import { BrowserRouter } from 'react-router-dom'

createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
)

🔀 Otros tipos de Router

Router Cuándo usarlo
BrowserRouter El más común. URLs limpias: /search
HashRouter Usa # en la URL: example.com/#/about. Útil cuando no podés configurar el servidor
MemoryRouter Historial en memoria, no modifica la URL. Ideal para testing y React Native

Paso 2: definir rutas con <Routes> y <Route>

En App.jsx, importamos los componentes y definimos las rutas:

import { Routes, Route } from 'react-router-dom'

<Routes>
  <Route path="/" element={<HomePage />} />
  <Route path="/search" element={<SearchPage />} />
  <Route path="*" element={<NotFoundPage />} />
</Routes>

🧩 Anatomía de una ruta

Cada <Route> tiene dos props fundamentales:

Prop Qué hace
path El patrón de URL que debe coincidir
element El componente React (en JSX) que se renderiza cuando la ruta coincide

React Router usa element (recibe JSX) en lugar de component (solo referencia). Esto nos da más flexibilidad para pasar props directamente. 

// ❌ Sintaxis antigua (React Router v5)
<Route path="/" component={HomePage} />

// ✅ Sintaxis actual (React Router v6+)
<Route path="/" element={<HomePage />} />

🔍 Cómo funciona el matching de rutas

React Router evalúa las rutas de arriba hacia abajo dentro de <Routes>. Cuando encuentra una coincidencia, renderiza ese componente y deja de buscar. 

Usuario navega a /search
        ↓
React Router compara con path="/"     → no coincide
        ↓
React Router compara con path="/search" → ✅ coincide
        ↓
Renderiza <SearchPage />
        ↓
Las rutas siguientes NO se evalúan

Resultado con nuestras rutas

URL visitada Componente renderizado
/ <HomePage />
/search <SearchPage />
/about, /xyz, cualquier otra <NotFoundPage />

⚠️ El comodín * como página 404

El path="*" coincide con cualquier ruta que no haya sido capturada antes. Por eso siempre debe ir al final: 

// ❌ Mal — * captura todo, las otras rutas nunca se alcanzan
<Routes>
  <Route path="*" element={<NotFoundPage />} />
  <Route path="/" element={<HomePage />} />
  <Route path="/search" element={<SearchPage />} />
</Routes>

// ✅ Bien — específicas primero, comodín al final
<Routes>
  <Route path="/" element={<HomePage />} />
  <Route path="/search" element={<SearchPage />} />
  <Route path="*" element={<NotFoundPage />} />
</Routes>

El orden importa: rutas específicas primero, comodín al final.

🚫 Errores comunes al configurar rutas

1. Olvidar envolver con <BrowserRouter>

Sin el wrapper en main.jsx, React Router lanza un error indicando que sus componentes deben estar dentro de un Router.

// ❌ Sin BrowserRouter — falla
createRoot(document.getElementById('root')).render(<App />)

// ✅ Con BrowserRouter
createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
)

2. Usar component en lugar de element

// ❌ Sintaxis antigua
<Route path="/" component={HomePage} />

// ✅ Sintaxis correcta (v6+)
<Route path="/" element={<HomePage />} />

3. Olvidar el comodín para el 404

Sin path="*", las URLs no encontradas simplemente no renderizan nada — página en blanco.

// ✅ Siempre incluir al final
<Route path="*" element={<NotFoundPage />} />

4. Paths sin /

// ❌ Sin slash inicial
<Route path="search" element={<SearchPage />} />

// ✅ Con slash inicial
<Route path="/search" element={<SearchPage />} />
