Lazy Load en las Rutas

El problema que tenemos

Imaginá que tu aplicación tiene 4 páginas:

🏠 Home
🔍 Search
📄 Detail
❌ NotFound

Cuando el usuario entra en www.miapp.com solo ve Home. Pero React está descargando esto:

⬇️ Home
⬇️ Search
⬇️ Detail
⬇️ NotFound
⬇️ Pagination
⬇️ CSS de Search
⬇️ CSS de Detail

Aunque el usuario NO los necesita todavía.

Qué queremos conseguir

Entrada inicial
      ↓
🏠 Home   →  Solo descarga: Home

Cuando pulsa Buscar
      ↓
🔍 Search →  Descarga: Search + Pagination + CSS

Cuando entra en un empleo
      ↓
📄 Detail →  Descarga: Detail + Markdown + CSS

Antes vs Después

❌ Antes — todo de golpe ✅ Después — solo lo necesario 
APP
├── Home
├── Search
├── Detail
├── NotFound
├── Pagination
└── CSS
 
APP
└── Home

(el resto se descarga
 cuando el usuario navega)
🐢 Más JS, más descarga, inicio lento⚡ Menos JS, menos descarga, inicio rápido

La regla de oro: piensá en Netflix

Cuando abrís Netflix:

NO descarga todas las películas

Descarga solamente:
  La pantalla principal

Y cuando elegís una película:
  Descarga esa película

React Lazy funciona exactamente igual.

Cómo funciona React.lazy

Normalmente hacés:

import { HomePage } from './pages/Home'

Eso significa: descárgalo YA.

Con lazy:

const HomePage = lazy(() => import('./pages/Home'))

Eso significa:

📦 No lo descargues todavía
      ↓
⏳ Espera
      ↓
👤 ¿El usuario lo necesita?

  Sí → descárgalo ahora
  No → no hagas nada

Import normal vs Import lazy

// ❌ Import normal
import SearchPage from './SearchPage'

// React:
// Inicio App → Descarga SearchPage (siempre)
// ✅ Import lazy
const SearchPage = lazy(() => import('./SearchPage'))

// React:
// Inicio App → NO descarga SearchPage
//           → Usuario entra en /search
//           → Ahora sí descarga SearchPage

⚠️ El error raro: export default

React.lazy espera un export default. Si usás named export, puede fallar:

// ❌ Puede dar problemas con lazy
export function HomePage() {
  // ...
}
// React recibe: { HomePage: function }
// No es lo que espera
// ✅ Funciona perfecto con lazy
function HomePage() {
  // ...
}

export default HomePage
// React recibe: HomePage ← justo lo que espera

Si ves algo parecido a:

Cannot convert an object to a primitive value

Fijate si el componente tiene export default.

¿Qué hace Suspense?

Imaginate una conexión lenta.

Sin Suspense:

Click Search
     ↓
🤔 (silencio total)
🤔
🤔
     ↓
Aparece Search

El usuario piensa: ¿Habrá funcionado?

Con Suspense:

Click Search
     ↓
⏳ Cargando...
     ↓
Search

Mucho mejor UX.

Cómo se usa

import { lazy, Suspense } from 'react'
import { Routes, Route } from 'react-router'

const HomePage   = lazy(() => import('./pages/Home'))
const SearchPage = lazy(() => import('./pages/Search'))
const DetailPage = lazy(() => import('./pages/Detail'))

function App() {
  return (
    <Suspense fallback={<p>Cargando...</p>}>
      <Routes>
        <Route path="/"        element={<HomePage />}   />
        <Route path="/search"  element={<SearchPage />} />
        <Route path="/jobs/:id" element={<DetailPage />} />
      </Routes>
    </Suspense>
  )
}

Flujo completo con Suspense

Usuario entra
      ↓
🏠 Home (descargada)

Usuario pulsa Buscar
      ↓
⏳ Cargando...
      ↓
🔍 Search (descargada)

Usuario pulsa una oferta
      ↓
⏳ Cargando...
      ↓
📄 Detail (descargada)
Prop Qué hace
fallbackLo que se muestra mientras carga el componente lazy

Lo más importante para entrevistas

Pregunta Respuesta
¿Qué problema resuelve React.lazy?Evita descargar código que el usuario todavía no necesita
¿Qué hace React.lazy?Divide el bundle en trozos y carga cada página bajo demanda
¿Para qué sirve Suspense?Mostrar un fallback mientras se descarga un componente lazy

Referencia rápida

import { lazy, Suspense } from 'react'

// 1. Definir lazy imports
const SearchPage = lazy(() => import('./pages/Search'))

// 2. Envolver con Suspense
<Suspense fallback={<p>Cargando...</p>}>
  <Routes>
    <Route path="/search" element={<SearchPage />} />
  </Routes>
</Suspense>
React.lazy convierte tus rutas en capítulos de Netflix. No descargás toda la serie al abrir la app — descargás el episodio cuando realmente lo vas a ver.
