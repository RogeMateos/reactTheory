Navigating to Detail Pages

What we have vs what we want

Right now each job card is just a display element — you can't click it to go anywhere.

Before:

Listado de ofertas
┌─────────────────────┐
│ Frontend Developer  │   ❌ Not clickable
│ Acme Corp           │
│ London              │
└─────────────────────┘
After:

Listado de ofertas
┌─────────────────────┐
│ Frontend Developer  │   ✅ Click
│ Acme Corp           │      ↓
│ London              │   /jobs/123
└─────────────────────┘

The best solution: use <Link>

When a card leads to another page, it should be a link, not an onClick. This is semantic HTML and it matters for accessibility.

import { Link } from 'react-router'
import styles from './JobCard.module.css'

export function JobCard({ job }) {
  return (
    <Link
      to={`/jobs/${job.id}`}
      className={styles.cardLink}
      aria-label={`See details for ${job.title} at ${job.company}`}
    >
      <article className={styles.card}>
        <h3>{job.title}</h3>
        <p>{job.company}</p>
        <p>{job.location}</p>
      </article>
    </Link>
  )
}

Visually, the entire card becomes the clickable zone:

<Link to="/jobs/123">
  ┌─────────────────────┐
  │ Frontend Developer  │
  │ Acme Corp           │
  │ London              │
  └─────────────────────┘
</Link>

<Link> is for normal navigation

Use it when the user wants to go to another page — that's it.

<Link to={`/jobs/${job.id}`}>
  See details
</Link>
Advantage Why it matters
Normal clickWorks as expected
Right click → open in new tabUsers expect this from links
Cmd/Ctrl + clickPower user feature, free with <Link>
Keyboard accessibleTab + Enter works
Screen reader friendlyAnnounced correctly

useNavigate is for navigation after an action

Use it when something happens first and then you navigate.

const navigate = useNavigate()

function handleSubmit() {
  saveData()       // action first
  navigate('/success')  // then navigate
}
User submits form
        ↓
Save data
        ↓
Navigate to /success

Don't use useNavigate for a simple card that's just a link. It's the wrong tool and you lose all the browser link behaviors listed above.

Decision tree

Situation Use
Card that goes to a detail page<Link>
Nav menu item<Link>
"See more" button<Link>
Redirect after form submituseNavigate()
Redirect after loginuseNavigate()
Navigate programmatically on conditionuseNavigate()

⚠️ Don't put a button inside a <Link>

If the card has a secondary action (like "save to favourites"), don't nest a button inside the Link:

// ❌ HTML/accessibility issue
<Link to={`/jobs/${job.id}`}>
  <article>
    <h3>{job.title}</h3>
    <button>Save</button>  ← nested interactive element
  </article>
</Link>

// ✅ Correct
<article className={styles.card}>
  <Link to={`/jobs/${job.id}`} className={styles.cardContent}>
    <h3>{job.title}</h3>
    <p>{job.company}</p>
  </Link>

  <button onClick={handleSaveFavourite}>
    Save
  </button>
</article>

Full recommended version

import { Link } from 'react-router'
import styles from './JobCard.module.css'

export function JobCard({ job }) {
  return (
    <Link
      to={`/jobs/${job.id}`}
      className={styles.cardLink}
      aria-label={`See details for ${job.title} at ${job.company}`}
    >
      <article className={styles.card}>
        <header className={styles.header}>
          <h3 className={styles.title}>{job.title}</h3>
          {job.isNew && <span className={styles.badge}>New</span>}
        </header>

        <div className={styles.meta}>
          <p>🏢 {job.company}</p>
          <p>📍 {job.location}</p>
        </div>

        {job.tags?.length > 0 && (
          <div className={styles.tags}>
            {job.tags.map((tag) => (
              <span key={tag} className={styles.tag}>
                {tag}
              </span>
            ))}
          </div>
        )}

        <footer className={styles.footer}>
          <span>{job.salary || 'Salary on request'}</span>
          <span>See more →</span>
        </footer>
      </article>
    </Link>
  )
}

<Link to={`/jobs/${job.id}`}>
  <article>...</article>
</Link>
Using <Link> gives you right-click, Ctrl+click, keyboard navigation, and screen reader support for free. An onClick would give you none of that.
