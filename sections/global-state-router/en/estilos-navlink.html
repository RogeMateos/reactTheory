Active Styles with NavLink

What problem does NavLink solve?

Imagine you're on a job board website:

🏠 Home     💼 Jobs     📞 Contact

You go to Jobs. But the menu still looks the same:

🏠 Home     💼 Jobs     📞 Contact
❌ No visual indication of where you are
❌ You can keep clicking "Jobs" over and over
❌ Bad user experience

The solution: NavLink

NavLink is a smart Link. In addition to navigating, it knows: "Am I currently on this page?"

Regular Link NavLink

🏠 Home
💼 Jobs   ← ACTIVE
📞 Contact

Now the user knows exactly where they are.

How does NavLink know it's active?

React Router gives you a variable called isActive:

// When NOT on /jobs
isActive = false
// <a class="header-nav-link">Jobs</a>

// When ON /jobs
isActive = true
// <a class="header-nav-link navLinkActive">Jobs</a>

NavLink adds an extra class navLinkActive when the route matches.

The code

<NavLink
  to="/jobs"
  className={({ isActive }) =>
    isActive
      ? "header-nav-link navLinkActive"
      : "header-nav-link"
  }
>
  Jobs
</NavLink>

Examples: 🏠 Home, 💼 Jobs, 📞 Contact, ⚙️ Settings

CSS for the active state

.header-nav-link.navLinkActive {
  color: yellow;
  font-weight: bold;
  border-bottom: 2px solid yellow;
}
Without active styles:
🏠 Home
💼 Jobs
📞 Contact

With active styles (at /jobs):
🏠 Home
💼 JOBS
───────
📞 Contact

Disable the current link

If you're already on Jobs, why let the user click it again? You can block the click:

.header-nav-link.navLinkActive {
  pointer-events: none;
}
Before:
💼 Jobs
(click)
(click)  ← redundant

After:
💼 Jobs (current)
🚫 Not clickable

Hidden bonus: Accessibility

When the link is active, React Router automatically adds:

<a
  class="navLinkActive"
  aria-current="page"
>
  Jobs
</a>

This helps:

✅ Screen readers
✅ Keyboard navigation
✅ Accessibility

You don't need to do anything.
NavLink does it for you.
