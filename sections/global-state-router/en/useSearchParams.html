useSearchParams

The hook that replaces everything manual

Before, you did all this by hand:

window.location.search
new URLSearchParams(...)
navigate(`?${params.toString()}`)

Now React Router gives you a dedicated hook:

const [searchParams, setSearchParams] = useSearchParams()

The complete flow

Input / filters
      ↓
React state
      ↓
useSearchParams
      ↓
URL updated
      ↓
?text=react&technology=javascript

Reading values from the URL

Given this URL:

/jobs?text=react&technology=javascript&remote=true

You can read each value like this:

searchParams.get('text')        // "react"
searchParams.get('technology')  // "javascript"
searchParams.get('remote')      // "true"

It's as if the URL were an object:

{
  text: 'react',
  technology: 'javascript',
  remote: 'true'
}
useSearchParams is like useState, but for URL query parameters.

Lazy initialization explained simply

This works, but the expression evaluates on every render:

// ⚠️ Evaluates on every render
const [textToFilter, setTextToFilter] = useState(
  searchParams.get('text') ?? ''
)

Better with a function:

// ✅ Only evaluates once
const [textToFilter, setTextToFilter] = useState(() => {
  return searchParams.get('text') ?? ''
})
Without function:
render 1 → reads URL
render 2 → reads URL again
render 3 → reads URL again

With function:
render 1 → reads URL
render 2 → does NOT re-initialize
render 3 → does NOT re-initialize

Updating filters in the URL

const updateFiltersInURL = ({ text, technology, remote }) => {
  setSearchParams((prevParams) => {
    const params = new URLSearchParams(prevParams)

    if (text) params.set('text', text)
    else params.delete('text')

    if (technology) params.set('technology', technology)
    else params.delete('technology')

    if (remote) params.set('remote', 'true')
    else params.delete('remote')

    return params
  })
}
The logic:

Has value    →  params.set()
No value     →  params.delete()
text = "react"
technology = "javascript"
remote = true

Result:
/jobs?text=react&technology=javascript&remote=true

If you clear text:
/jobs?technology=javascript&remote=true

⚠️ Important mistake: append vs set

Don't use append for regular filters:

// ❌ Creates duplicates
params.append('text', text)
// ?text=react&text=react&text=react

// ✅ Replaces the previous value
params.set('text', text)
// ?text=react
Method When to use it
params.set()Regular filters — replaces the value
params.append()Multiple values with the same key
params.delete()Remove empty filters from the URL

What we achieve with useSearchParams

useSearchParams lets you sync your app's filters with the URL without writing manual code using window.location.search or navigate.

It makes things like this work correctly:

/jobs?text=react&technology=javascript

And when you reload the page, the filters stay active.

Benefit Result
CleanerNo manual window.location.search
Easier to maintainOne hook instead of scattered logic
More professionalURL as the source of truth for filters
Less repetitiveNo URLSearchParams boilerplate

Quick reference

import { useSearchParams } from 'react-router'

const [searchParams, setSearchParams] = useSearchParams()

// Read
const text = searchParams.get('text') ?? ''

// Write
setSearchParams((prev) => {
  const params = new URLSearchParams(prev)
  if (text) params.set('text', text)
  else params.delete('text')
  return params
})
