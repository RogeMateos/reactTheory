Link & React Router Hooks: useNavigate and useLocation

🔗 Modernizing our Link component

We're going to modernize our app's routing completely. The goal: replace manual implementations with React Router's native tools while keeping the same external API our components already use. 

✅ Keep our own Link component — but change its internals
✅ Integrate useNavigate and useLocation
✅ Apply the abstraction pattern to encapsulate React Router
✅ Zero changes needed in the rest of the app

The Link we had before

Our old Link was doing everything manually:

export function Link({ to, children, ...props }) {
  const handleClick = (e) => {
    e.preventDefault()
    window.history.pushState({}, '', to)
    // Dispatch custom event to notify routing system...
  }

  return (
    <a href={to} onClick={handleClick} {...props}>
      {children}
    </a>
  )
}

All of this manually… none of this is needed anymore because React Router already has a Link component.

What we do now

We keep our Link, but change its internals:

import { Link as RRLink } from 'react-router'

export function Link(props) {
  return <RRLink {...props} />
}

This way:

  • The whole app keeps using our <Link>
  • If tomorrow we want to use another routing library, we only change this one file
  • No need to go through every component one by one

🏗️ Abstraction Pattern

This technique is called the abstraction pattern or dependency abstraction pattern.

Advantage What it means
No coupling to React Router Your components don't know React Router exists
Single place to change One file instead of dozens if you switch libraries
Stable public API Consumers never need to change 
The principle: expose your own stable API,
hide the library's implementation details,
centralize all imports of that library in one place.

🧭 Replacing navigateTo with useNavigate

We had a custom router with a navigateTo method doing navigation manually. React Router offers its own hook: useNavigate.

The problem with our manual implementation

function useRouter() {
  const navigateTo = (path) => {
    window.history.pushState({}, '', path)
    // Dispatch custom events
    // Update state
    // ... a lot of manual code
  }

  return { navigateTo }
}

This works, but:

  • We have to maintain it ourselves
  • It can have bugs
  • We're reinventing the wheel

The solution with useNavigate

import { useNavigate } from 'react-router'

export function useRouter() {
  const navigate = useNavigate()

  const navigateTo = (path) => {
    navigate(path)
  }

  return { navigateTo }
}

What we gain

✅ No need to manually implement pushState — React Router does it better
✅ Components keep calling navigateTo — external API doesn't change
✅ We changed the dependency without breaking anything

📍 Replacing manual currentPath with useLocation

Before, we had a currentPath state, a useEffect listening to history events, and our own logic to detect URL changes. React Router includes useLocation, which gives us:

Property What it contains
pathname The current path: /search
search Query string: ?q=react
hash The hash: #section
state State passed during navigation

The manual code we had before

function useRouter() {
  const [currentPath, setCurrentPath] = useState(window.location.pathname)

  useEffect(() => {
    const handleLocationChange = () => {
      setCurrentPath(window.location.pathname)
    }

    window.addEventListener('popstate', handleLocationChange)
    window.addEventListener('pushstate', handleLocationChange)

    return () => {
      window.removeEventListener('popstate', handleLocationChange)
      window.removeEventListener('pushstate', handleLocationChange)
    }
  }, [])

  return { currentPath }
}

All this useEffect and event listeners — no longer needed.

The solution with useLocation

import { useLocation } from 'react-router'

export function useRouter() {
  const location = useLocation()

  const currentPath = location.pathname

  return {
    currentPath,
    location,
  }
}

Much cleaner. No side effects to manage.

🔧 The complete useRouter hook

Putting it all together:

import { useNavigate, useLocation } from 'react-router'

export function useRouter() {
  const navigate = useNavigate()
  const location = useLocation()

  function navigateTo(path) {
    navigate(path)
  }

  const currentPath = location.pathname

  return {
    currentPath,
    navigateTo,
  }
}
✅ Zero manual code — no pushState, no custom events, no useEffect
✅ React Router does the heavy lifting — and does it better than us
✅ Our API stays the same — components that used useRouter don't change

📊 Before vs After

// ❌ Before — Link.jsx (manual)
export function Link({ to, children, ...props }) {
  const handleClick = (e) => {
    e.preventDefault()
    window.history.pushState({}, '', to)
    window.dispatchEvent(new Event('pushstate'))
  }
  return <a href={to} onClick={handleClick} {...props}>{children}</a>
}

// ✅ After — Link.jsx (React Router)
import { Link as RRLink } from 'react-router'
export function Link(props) {
  return <RRLink {...props} />
}
// ❌ Before — useRouter.js (manual)
export function useRouter() {
  const [currentPath, setCurrentPath] = useState(window.location.pathname)
  useEffect(() => {
    const handler = () => setCurrentPath(window.location.pathname)
    window.addEventListener('popstate', handler)
    window.addEventListener('pushstate', handler)
    return () => {
      window.removeEventListener('popstate', handler)
      window.removeEventListener('pushstate', handler)
    }
  }, [])
  const navigateTo = (path) => {
    window.history.pushState({}, '', path)
    window.dispatchEvent(new Event('pushstate'))
  }
  return { currentPath, navigateTo }
}

// ✅ After — useRouter.js (React Router)
import { useNavigate, useLocation } from 'react-router'
export function useRouter() {
  const navigate = useNavigate()
  const location = useLocation()
  function navigateTo(path) { navigate(path) }
  return { currentPath: location.pathname, navigateTo }
}

🚀 Extending the pattern

Once you have this abstraction layer, extending it is trivial:

export function useRouter() {
  const navigate = useNavigate()
  const location = useLocation()

  function navigateTo(path) {
    navigate(path)
  }

  function goBack() {
    navigate(-1)
  }

  return {
    currentPath: location.pathname,
    navigateTo,
    goBack,
    location,
  }
}

All components get the new functionality automatically — without changing a single line in them:

const { goBack } = useRouter()

<button onClick={goBack}>← Back</button>
