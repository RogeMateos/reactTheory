What is React Router?

🏠 Starting point: our homemade router

So far we've built something really powerful:

✅ The URL synced with all application filters
✅ Our own router in App.jsx with very few lines

This is great for two reasons:

  1. It forces us to understand how routing works under the hood.
  2. It gives us confidence so we don't see React Router as black magic.

But it also has one important drawback:

neither you nor I are going to build a better router than the team
that has been maintaining React Router for years.

⚠️ Limitations of the router we built

Our homemade router works, but there are many things missing that you'll need in a real app:

Limitation What it means
404 page If the user types a wrong route, the experience is not ideal
Protected routes All redirect and token logic has to be wired up manually
Code splitting Loading routes on demand requires manually combining lazy + Suspense
Data loading Loading states, error handling and when to fetch are not solved

None of this is solved in our current router — and these are exactly the kinds of problems React Router has been working on for years.

📦 What is React Router

React Router is the standard routing library in the React ecosystem.

Stat Value
npm downloads Billions
GitHub stars Tens of thousands
Contributors Hundreds
Projects depending on it Millions

This is not a random library. It's a critical piece of the React ecosystem.

🤔 Why use a library instead of building our own

✅ Already solved most problems you don't even know exist yet
✅ Designed to scale with your application
✅ Updated to support new browser and React features
✅ Huge community, documentation, examples, tutorials
Our homemade router → great as an exercise
React Router        → built for real projects

⚙️ Usage modes of React Router

React Router is not just a BrowserRouter with a couple of routes. It has several modes depending on what your app needs:

1. Framework Mode

Gives you a full framework on top of React. Includes:

✅ Advanced routing
✅ Rendering strategies
✅ Data loading (loaders)
✅ Actions for mutations
✅ Enhanced navigation and forms

So large you could do an entire course just on this.

2. Data Mode

Focuses on:

✅ Route-associated data loading
✅ Error and loading state handling

A middle ground between the full framework mode and the simpler router.

3. Declarative Mode (the one we use)

The simplest and most direct part:

✅ Declare routes
✅ Navigate between them

Ideal for:

  • Understanding the core of React Router properly
  • Not mixing too many concepts at once
  • Improving our homemade router with a solid API

Even staying with declarative mode, we'll already gain a huge number of utilities and improvements over what we have now.

🎯 What we'll achieve with React Router

Over the following lessons we'll progressively migrate from our homemade router to React Router:

✅ Keep the URL sync we already have
✅ Define routes declaratively and more readably
✅ Set up:
   → 404 page
   → Protected routes
   → Lazy loading of components
   → Better loading state and error handling

All of it without reinventing the wheel.
