Exercise 2.1 — Breadcrumbs (optional but recommended)

Let the user know which part of the app they're in:

<nav aria-label="Navigation path">
  <ol>
    <li>{/* Link to jobs list */}</li>
    <li aria-current="page">{/* Current job title */}</li>
  </ol>
</nav>
Key points:

✅ Shows the navigation hierarchy
✅ Use aria-label and aria-current for accessibility
✅ The last element should NOT be a link

Exercise 2.2 — Mark the active link

Use the URL information to mark the corresponding menu link as "active".

You need:

1. Get the current route
   (think about router hooks you've already used)

2. Compare the current route with the link's route

3. Apply a different CSS class when they match

4. Add aria-current="page" for accessibility
// A common pattern is a NavLink component
// that wraps your Link and adds this logic

<NavLink
  to="/jobs"
  className={({ isActive }) =>
    isActive ? "nav-link active" : "nav-link"
  }
>
  Jobs
</NavLink>

Exercise 4 — Initialize all filters from the URL

Filters "work, but when you refresh, they're not initializing correctly." We've already seen how to do it with the text search filter. Now apply the same logic to the rest.

For each filter (mode, contract, salary...):

When the filter changes in the UI:
  → Update React state
  → Update the URL with the new value

On load or refresh:
  → Read the URL parameter
  → Initialize the filter state with that value

// Function to update a filter
const updateFilter = (name, value) => {
  const params = new URLSearchParams(window.location.search)

  if (value) params.set(name, value)
  else params.delete(name)

  // update URL
}
// Example in a component
function Filters() {
  const { mode, updateMode } = useModeFilter()

  return (
    <select value={mode} onChange={(e) => updateMode(e.target.value)}>
      <option value="">Any</option>
      <option value="remote">Remote</option>
      <option value="hybrid">Hybrid</option>
      <option value="onsite">On-site</option>
    </select>
  )
}
