The concept of "prose styles"

When you render rich content (markdown, long text, articles), you need a specific style layer for that type of content. This is called prose styles.

Why do we need this?

Because content generated by markdown has:

Multiple heading levels (h1, h2, h3)
Long paragraphs
Bullet lists
Possible links
Inline code

And all these elements need:

Clear visual hierarchy
Consistent spacing
Readable typography
Appropriate line width

Adding prose styles to index.css

At the end of your index.css, add a style block to handle all prose content:

/* Prose styles for detail content */
.prose {
  max-width: 70ch;
  line-height: 1.7;
  font-size: 1rem;
  color: #111827;
}

.prose h1,
.prose h2,
.prose h3 {
  font-weight: 700;
  margin-bottom: 0.75rem;
  margin-top: 1.5rem;
  color: #111827;
}

.prose h2 {
  font-size: 1.5rem;
}

.prose h3 {
  font-size: 1.25rem;
}

.prose p {
  margin-bottom: 1rem;
}

.prose ul,
.prose ol {
  margin-left: 1.25rem;
  margin-bottom: 1rem;
}

.prose ul {
  list-style: disc;
}

.prose ol {
  list-style: decimal;
}

.prose li {
  margin-bottom: 0.5rem;
}

.prose a {
  color: #0066cc;
  text-decoration: underline;
}

.prose a:hover {
  color: #0052a3;
}

.prose code {
  background: #f3f4f6;
  padding: 0.125rem 0.25rem;
  border-radius: 4px;
  font-size: 0.875em;
  font-family: 'Courier New', monospace;
}

We put it at the end so these styles override more generic ones. CSS works in cascade — order matters.

Applying styles in the component

On the container where you render the markdown content, add the prose class:

export function JobDetail() {
  const { id } = useParams()
  const [job, setJob] = useState(null)

  // ... fetch logic

  return (
    <div className={styles.container}>
      {/* Header, breadcrumb, etc. */}

      <article className="prose">
        <div dangerouslySetInnerHTML={{ __html: htmlContent }} />
      </article>
    </div>
  )
}
Key point Detail
Where prose goesOn the container of the markdown-generated content
What inherits itEverything that comes out of the markdown (p, h2, h3, ul, etc.)
What you don't need to touchThe generated HTML — styles apply automatically

Small details that make a big difference

1. Vertical spacing

.prose h2 {
  margin-top: 2rem;       /* More space above */
  margin-bottom: 0.75rem; /* Less space below */
}
Spacing checklist:

✅ Enough space between paragraphs
✅ Headings have more margin above than below
✅ Content breathes and doesn't feel cramped

2. Line width

Lines that are too long are hard to read.

.prose {
  max-width: 70ch; /* 70 characters is the standard */
}

The ch unit represents the width of the "0" character. It's perfect for measuring line length because it adapts to font size.

3. Lists with icons

If you want lists with icons instead of bullets:

.prose ul.check {
  list-style: none;
  padding-left: 0;
}

.prose ul.check li {
  position: relative;
  padding-left: 1.5rem;
}

.prose ul.check li::before {
  content: '✓';
  position: absolute;
  left: 0;
  color: #16a34a;
  font-weight: bold;
}

4. Contrast and readability

.prose {
  color: #111827;   /* Almost black, good contrast */
  line-height: 1.7; /* Comfortable spacing */
  font-size: 1rem;  /* 16px, readable size */
}

Bonus: Improved link hover

.prose a {
  color: #0066cc;
  text-decoration: underline;
  transition: color 0.2s ease;
}

.prose a:hover {
  color: #0052a3;
  text-decoration: none;
}

Bonus: Responsive typography

@media (max-width: 640px) {
  .prose {
    font-size: 0.9375rem; /* 15px on mobile */
  }

  .prose h2 {
    font-size: 1.25rem;
  }

  .prose h3 {
    font-size: 1.125rem;
  }
}

Bonus: Dark mode

@media (prefers-color-scheme: dark) {
  .prose {
    color: #e5e7eb;
  }

  .prose h1,
  .prose h2,
  .prose h3 {
    color: #f9fafb;
  }

  .prose a {
    color: #60a5fa;
  }

  .prose a:hover {
    color: #93c5fd;
  }

  }
}

Your goal

Make the detail page look exactly like the reference design. Not "close enough". Not "similar". Exactly the same.

Comparison method

Step 1: Open two windows side by side
  ┌─────────────────┬─────────────────┐
  │ Reference       │ Your app        │
  │ design          │                 │
  └─────────────────┴─────────────────┘

Step 2: Spot the differences
  Do the headings have the same size?
  Do the lists have the same bullet style?
  Does the text have the same spacing?
  Are the colors identical?

Step 3: Iterate
  Tweak one detail → reload → compare → repeat

Typical differences you'll find

Problem Solution
Headings smaller than they should beAdjust font-size
Text too close to the edgeAdd padding to the container
Different bullet styleChange list-style
Colors not exactUse a color picker
Inconsistent spacingReview margin and padding
