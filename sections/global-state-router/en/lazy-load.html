Lazy Loading Routes

The problem we have

Imagine your app has 4 pages:

🏠 Home
🔍 Search
📄 Detail
❌ NotFound

When the user visits www.myapp.com they only see Home. But React is downloading all of this:

⬇️ Home
⬇️ Search
⬇️ Detail
⬇️ NotFound
⬇️ Pagination
⬇️ Search CSS
⬇️ Detail CSS

Even though the user doesn't need any of it yet.

What we want

Initial load
      ↓
🏠 Home   →  Only downloads: Home

User clicks Search
      ↓
🔍 Search →  Downloads: Search + Pagination + CSS

User clicks a job
      ↓
📄 Detail →  Downloads: Detail + Markdown + CSS

Before vs After

❌ Before — everything at once ✅ After — only what's needed 
APP
├── Home
├── Search
├── Detail
├── NotFound
├── Pagination
└── CSS
 
APP
└── Home

(the rest downloads
 when the user navigates)
🐢 More JS, more download, slow start⚡ Less JS, less download, fast start

The golden rule: think Netflix

When you open Netflix:

It does NOT download all movies

It only downloads:
  The home screen

When you pick a movie:
  It downloads that movie

React Lazy works exactly the same way.

How React.lazy works

Normally you do:

import { HomePage } from './pages/Home'

That means: download it NOW.

With lazy:

const HomePage = lazy(() => import('./pages/Home'))

That means:

📦 Don't download it yet
      ↓
⏳ Wait
      ↓
👤 Does the user need it?

  Yes → download it now
  No  → do nothing

Normal import vs lazy import

// ❌ Normal import
import SearchPage from './SearchPage'

// React:
// App starts → downloads SearchPage (always)
// ✅ Lazy import
const SearchPage = lazy(() => import('./SearchPage'))

// React:
// App starts → does NOT download SearchPage
//           → user visits /search
//           → NOW it downloads SearchPage

⚠️ The weird error: export default

React.lazy expects an export default. Using a named export can cause it to fail:

// ❌ Can break with lazy
export function HomePage() {
  // ...
}
// React receives: { HomePage: function }
// Not what it expects
// ✅ Works perfectly with lazy
function HomePage() {
  // ...
}

export default HomePage
// React receives: HomePage ← exactly what it expects

If you see something like:

Cannot convert an object to a primitive value

Check whether the component has export default.

What does Suspense do?

Imagine a slow connection.

Without Suspense:

Click Search
     ↓
🤔 (total silence)
🤔
🤔
     ↓
Search appears

User thinks: Did it work?

With Suspense:

Click Search
     ↓
⏳ Loading...
     ↓
Search

Much better UX.

How to use it

import { lazy, Suspense } from 'react'
import { Routes, Route } from 'react-router'

const HomePage   = lazy(() => import('./pages/Home'))
const SearchPage = lazy(() => import('./pages/Search'))
const DetailPage = lazy(() => import('./pages/Detail'))

function App() {
  return (
    <Suspense fallback={<p>Loading...</p>}>
      <Routes>
        <Route path="/"         element={<HomePage />}   />
        <Route path="/search"   element={<SearchPage />} />
        <Route path="/jobs/:id" element={<DetailPage />} />
      </Routes>
    </Suspense>
  )
}

User arrives
      ↓
🏠 Home (downloaded)

User clicks Search
      ↓
⏳ Loading...
      ↓
🔍 Search (downloaded)

import { lazy, Suspense } from 'react'

// 1. Define lazy imports

// 2. Wrap with Suspense

    <Route path="/search" element={<SearchPage />} />
  </Routes>
</Suspense>
React.lazy turns your routes into Netflix episodes. You don't download the whole series when you open the app — you download the episode when you're actually going to watch it.
