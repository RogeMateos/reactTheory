How to Create Routes with React Router

🚀 Introduction

We're going to migrate our homemade router to a professional, robust solution. Up to this point, our custom router worked but had important limitations: 

❌ Not an industry standard
❌ Doesn't include all the features we'll need in production
❌ Less optimized
❌ Reinventing the wheel instead of using a proven solution

React Router gives us:

✅ Declarative navigation between pages
✅ Automatic browser history management
✅ Specialized hooks for params, location and programmatic navigation
✅ Robust error handling and not-found routes
✅ Support for multiple routing modes

📦 Installation

npm install react-router

Step 1: wrap the app with <BrowserRouter>

The first step is to wrap the entire application with BrowserRouter. This component provides the routing context that React Router needs to work throughout the app.

In main.jsx:

import { BrowserRouter } from 'react-router-dom'

createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
)

🔀 Other Router types

Router When to use it
BrowserRouter The most common. Clean URLs: /search
HashRouter Uses # in the URL: example.com/#/about. Useful when you can't configure the server
MemoryRouter History in memory, doesn't modify the URL. Ideal for testing and React Native

Step 2: define routes with <Routes> and <Route>

In App.jsx, import the components and define your routes:

import { Routes, Route } from 'react-router-dom'

<Routes>
  <Route path="/" element={<HomePage />} />
  <Route path="/search" element={<SearchPage />} />
  <Route path="*" element={<NotFoundPage />} />
</Routes>

🧩 Anatomy of a route

Each <Route> has two fundamental props:

Prop What it does
path The URL pattern that must match
element The React component (as JSX) rendered when the route matches

React Router uses element (receives JSX) instead of component (just a reference). This gives more flexibility to pass props directly. 

// ❌ Old syntax (React Router v5)
<Route path="/" component={HomePage} />

// ✅ Current syntax (React Router v6+)
<Route path="/" element={<HomePage />} />

🔍 How route matching works

React Router evaluates routes top to bottom inside <Routes>. When it finds a match, it renders that component and stops searching. 

User navigates to /search
        ↓
React Router compares with path="/"       → no match
        ↓
React Router compares with path="/search" → ✅ match
        ↓
Renders <SearchPage />
        ↓
Remaining routes are NOT evaluated

Result with our routes

URL visited Component rendered
/ <HomePage />
/search <SearchPage />
/about, /xyz, anything else <NotFoundPage />

⚠️ The * wildcard as a 404 page

path="*" matches any route not captured before. That's why it must always go last: 

// ❌ Wrong — * catches everything, other routes are never reached
<Routes>
  <Route path="*" element={<NotFoundPage />} />
  <Route path="/" element={<HomePage />} />
  <Route path="/search" element={<SearchPage />} />
</Routes>

// ✅ Correct — specific routes first, wildcard at the end
<Routes>
  <Route path="/" element={<HomePage />} />
  <Route path="/search" element={<SearchPage />} />
  <Route path="*" element={<NotFoundPage />} />
</Routes>

Order matters: specific routes first, wildcard at the end.

🚫 Common mistakes when setting up routes

1. Forgetting to wrap with <BrowserRouter>

Without the wrapper in main.jsx, React Router throws an error saying its components must be inside a Router.

// ❌ Without BrowserRouter — fails
createRoot(document.getElementById('root')).render(<App />)

// ✅ With BrowserRouter
createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
)

2. Using component instead of element

// ❌ Old syntax
<Route path="/" component={HomePage} />

// ✅ Correct syntax (v6+)
<Route path="/" element={<HomePage />} />

3. Forgetting the wildcard for 404

Without path="*", unmatched URLs render nothing — a blank page.

// ✅ Always include at the end
<Route path="*" element={<NotFoundPage />} />

4. Paths without /

// ❌ Missing leading slash
<Route path="search" element={<SearchPage />} />

// ✅ With leading slash
<Route path="/search" element={<SearchPage />} />
