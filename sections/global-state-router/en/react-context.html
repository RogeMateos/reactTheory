React Context

Prop Drilling: the real problem

Imagine you have a logged-in user:

const [isLogin, setIsLogin] = useState(false)

And the button that needs it is deep in the tree:

App
 └── Header
      └── Navbar
           └── UserMenu
                └── UserButton  ← needs isLogin

But the state lives in App. Without Context you have to do this:

<App isLogin={isLogin}>
  ↓
<Header isLogin={isLogin}>
  ↓
<Navbar isLogin={isLogin}>
  ↓
<UserMenu isLogin={isLogin}>
  ↓
<UserButton isLogin={isLogin}>
Header    ❌ doesn't use it
Navbar    ❌ doesn't use it
UserMenu  ❌ doesn't use it
UserButton ✅ uses it

All of them receive it anyway.

This is called Prop Drilling — you're "drilling" props through components that don't need them.

React Context to the rescue

Context creates a kind of global WiFi.

App                            📡 AuthContext
 ↓                          /        |        \
Header                  Header     Home     Detail
 ↓
Navbar                  Everyone accesses directly.
 ↓
UserMenu
 ↓
UserButton

Think of the difference between an ethernet cable and WiFi in your home.

createContext()

const AuthContext = createContext()

It doesn't store data. It has no state. It just creates a channel.

createContext()
      ↓

Channel created

It's like buying a router.
There's no internet yet — we need the Provider.

The Provider

The Provider is what broadcasts the data. Everything inside it can access it:

function AuthProvider({ children }) {
  const [isLogin, setIsLogin] = useState(false)

  function login()  { setIsLogin(true)  }
  function logout() { setIsLogin(false) }

  const value = { isLogin, login, logout }

  return (
    <AuthContext value={value}>
      {children}
    </AuthContext>
  )
}
┌───────────────────────┐
│ AuthProvider          │
│                       │
│ isLogin = false       │
│ login()               │
│ logout()              │
└───────────┬───────────┘
            │
            ▼
          value
            │
            ▼
    Available throughout the app

Wrapping the application

Typically in main.jsx:

<AuthProvider>
  <App />
</AuthProvider>
AuthProvider
      │
      ▼
     App
   /  |  \
Header Home Detail

All have access to the context.

useContext()

Now any component can connect directly:

const { isLogin } = useContext(AuthContext)
Before:

<App isLogin={isLogin} login={login} logout={logout} />
  ↓
<Header isLogin={isLogin} login={login} logout={logout} />
  ↓
<Nav isLogin={isLogin} login={login} logout={logout} />
  ↓
<UserButton isLogin={isLogin} login={login} logout={logout} />

After:

<App />
<Header />
<Nav />
<UserButton />

// Inside UserButton:
const { isLogin } = useContext(AuthContext)

⚠️ Important optimisation

When the Context changes, React re-renders every component that consumes it. So where you call useContext matters:

// ❌ Bad — Header re-renders on every login change
function Header() {
  const { isLogin } = useContext(AuthContext)
  return (...)
}

// ✅ Better — only UserButton re-renders
function Header() {
  return (
    <>
      <Logo />
      <Nav />
      <UserButton />
    </>
  )
}

function UserButton() {
  const { isLogin } = useContext(AuthContext)
}
On login change:

❌ Bad:               ✅ Better:
Header    🔄          Logo       ⚪
                      Nav        ⚪
                      Header     ⚪
                      UserButton 🔄

The three steps

Step What it does Code
1Create the channelcreateContext()
2Broadcast the data<AuthContext value={value}>
3Consume the datauseContext(AuthContext)

Quick reference

// 1. Create
const AuthContext = createContext()

// 2. Provider
function AuthProvider({ children }) {
  const [isLogin, setIsLogin] = useState(false)
  const value = {
    isLogin,
    login:  () => setIsLogin(true),
    logout: () => setIsLogin(false),
  }
  return (
    <AuthContext value={value}>{children}</AuthContext>
  )
}

// 3. Consume
function UserButton() {
  const { isLogin, login, logout } = useContext(AuthContext)
}
React Context is a way to create global state so that any component can access it without having to pass props through the entire application.
